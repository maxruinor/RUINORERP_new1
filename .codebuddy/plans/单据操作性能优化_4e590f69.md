## Product Overview

用户报告在单据编辑窗体中执行保存、提交、审核、反审等操作时响应较慢。经过代码分析，发现多处重复调用、串行阻塞和未缓存反射等性能问题，需要系统性优化。

## Core Features

- 消除保存/提交/审核操作中的重复锁定检查和重复状态验证
- 将可并行的串行await改为并行执行
- 修复UnifiedStateManager中未使用缓存反射的问题
- 优化审核/反审后的UI更新和日志记录逻辑

## 技术栈

- C# (.NET Framework 4.8)
- WinForms + Krypton UI
- SqlSugar ORM
- Autofac DI

## 实现方案

### 问题分析总结

经过深入代码分析，发现以下性能问题：

**P0 - 高影响（用户感知明显）：**

1. **保存操作 - 锁定检查重复调用**：DoButtonClick(MenuItemEnums.保存) 在 line 3042 调用 `CheckLockStatusAndUpdateUI`，然后在 `Save(T entity)` 的 line 7306 又再次调用。同一次保存操作，锁定状态被检查了两次。虽然有频率控制和缓存，但缓存过期时意味着额外的网络请求。

2. **Save方法内三个串行操作可并行**：`BaseSaveOrUpdateWithChild`（数据库写入）完成后，`PostSaveLockManagement`（网络锁定请求）和 `CreateAuditLog`（审计日志写入）完全串行。PostSaveLockManagement 对于新增单据会调用 `LockBillAsync`，这是一个网络请求，可与审计日志并行。

**P1 - 中等影响：**

3. **提交操作 - 状态验证被调用三次**：

- DoButtonClick 提交 case (line 3156): `CanExecuteActionWithMessage`
- Submit() (line 8199): `ValidateActionTransitionAsync`
- Submit() (line 8206): `CanExecuteActionWithMessage`
每次都调用 `GetStatusType` + `GetBusinessStatus` + 反射验证。

4. **审核操作 - 串行await链**：Review() 中审核成功后的操作（SyncTodoStatusAsync、LockBill、自动审核应收款单等）全部串行执行。

5. **审核操作 - DoButtonClick中审核也会调UpdateAllUIStates**：审核方法内部(line 4205)已调用 `UpdateAllUIStates`，而DoButtonClick审核case(line 3341)又调用一次，导致重复UI刷新。

**P2 - 低影响：**

6. **CanExecuteStateTransitionAction 未使用缓存反射** (UnifiedStateManager.cs line 885-886)：每次直接 `GetMethod` + `MakeGenericMethod`，未使用已有的 `GetCachedGenericMethod`。

7. **UnitOfWorkManage 日志字符串开销**：事务管理中大量字符串格式化日志，即使LogDebug级别未启用也会创建字符串。

### 实现策略

采用"消除重复 + 并行化"的优化策略，不改变业务逻辑，只调整调用顺序和消除冗余调用：

- **消除重复**：移除Save方法中多余的锁定检查，移除Submit中重复的状态验证，移除审核后DoButtonClick中重复的UI更新
- **并行化**：将Save中后置操作（锁定管理+审计日志）改为 `Task.WhenAll` 并行
- **缓存修复**：修复CanExecuteStateTransitionAction使用缓存反射

## 实现注意事项

- 所有修改不改变业务逻辑和最终结果，只优化调用路径
- 并行化需要注意异常处理，使用 `Task.WhenAll` 时确保一个任务的失败不影响另一个
- DoButtonClick中移除的重复UI更新，需要确认对应方法内部已经调用了
- 修改后需要验证保存、提交、审核、反审四个核心操作的完整性

## 关键代码结构

### Save方法优化后的调用模式

```
// 当前（串行）:
await ctr.BaseSaveOrUpdateWithChild<T>(entity);
await PostSaveLockManagement(entity, pkid);
await MainForm.Instance.AuditLogHelper.CreateAuditLog<T>(...);

// 优化后（并行）:
await ctr.BaseSaveOrUpdateWithChild<T>(entity);
// 以下两个互不依赖的操作并行执行
var lockTask = PostSaveLockManagement(entity, pkid);
var logTask = MainForm.Instance.AuditLogHelper.CreateAuditLog<T>(...);
await Task.WhenAll(lockTask, logTask);
```
