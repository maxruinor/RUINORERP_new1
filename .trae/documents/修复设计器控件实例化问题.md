## 修复设计器控件实例化问题

### 问题分析
通过检查代码，发现以下两个控件在设计器文件中缺少实例化代码，导致NullReferenceException：

1. **UCBasicDataImport.designer.cs** 中的 `kbtnManageConfigs` 控件：
   - 已声明（第956行）：`private Krypton.Toolkit.KryptonButton kbtnManageConfigs;`
   - 已添加到容器（第607行）：`this.kryptonPanel5.Controls.Add(this.kbtnManageConfigs);`
   - 已设置属性（第704-708行），但**缺少实例化代码**

2. **frmColumnMappingConfig.designer.cs** 中的 `kbtnSetForeignKey` 控件：
   - 已声明（第208行）：`private Krypton.Toolkit.KryptonButton kbtnSetForeignKey;`
   - 已添加事件处理（第95行）：`this.kbtnSetForeignKey.Click += new System.EventHandler(this.kbtnSetForeignKey_Click);`
   - 已设置属性（第90-94行），但**缺少实例化代码**

### 修复方案

#### 1. 修复 UCBasicDataImport.designer.cs
- 在 InitializeComponent 方法的控件实例化部分（第80-90行附近）添加 `kbtnManageConfigs` 的实例化代码：
  ```csharp
  this.kbtnManageConfigs = new Krypton.Toolkit.KryptonButton();
  ```

#### 2. 修复 frmColumnMappingConfig.designer.cs
- 在 InitializeComponent 方法的控件实例化部分（第40-45行附近）添加 `kbtnSetForeignKey` 的实例化代码：
  ```csharp
  this.kbtnSetForeignKey = new Krypton.Toolkit.KryptonButton();
  ```

### 修复步骤
1. 打开 UCBasicDataImport.designer.cs 文件
2. 在控件实例化区域添加 kbtnManageConfigs 的实例化代码
3. 打开 frmColumnMappingConfig.designer.cs 文件
4. 在控件实例化区域添加 kbtnSetForeignKey 的实例化代码
5. 保存文件并验证修复效果

### 技术要点
- Windows Forms 设计器生成的代码要求控件必须先实例化，然后才能设置属性和添加到容器中
- 实例化代码通常位于 InitializeComponent 方法的开头部分
- 修复后应确保所有控件都有正确的实例化、属性设置和事件绑定顺序
