# 单一业务单据业务审核流程优化方案（轻量级）

## 一、需求分析

### 1.1 当前问题
- 单一业务单据（借出单、归还单、调拨单等）的"审核"按钮可能执行了业务逻辑
- 缺少"领导审批通过后仓库方可执行出库"的业务流程
- 需要将审核和结案分离，审核仅记录状态，结案时才执行库存操作

### 1.2 业务需求
- **审核**：仅记录单据状态，不执行库存操作，由领导层审批
- **结案**：由仓库执行，结案时才执行库存操作
- 最小化代码改动，保持架构稳定

## 二、优化方案

### 2.1 流程调整

#### 2.1.1 当前流程
```
草稿 → 提交 → 审核（可能执行库存操作）→ 完结
```

#### 2.1.2 优化后流程
```
草稿 → 提交 → 审核（仅记录状态，不执行库存操作）→ 结案（执行库存操作）→ 完结
```

### 2.2 核心改动点

#### 2.2.1 UI层改动
1. **调整按钮位置**：将"审核"按钮移到"结案"按钮下方
2. **修改审核逻辑**：审核时仅更新单据状态，不执行库存操作
3. **修改结案逻辑**：结案时执行库存操作

#### 2.2.2 业务层改动
1. **分离审核逻辑**：将审核方法中的库存操作逻辑移除
2. **增强结案逻辑**：在结案方法中添加库存操作逻辑

#### 2.2.3 枚举和数据库
- **无需改动**：使用现有的`DataStatus`和`ApprovalStatus`枚举
- **无需新增表**：使用现有的表结构

```csharp
// 新增方法：判断是否为单一业务单据
private bool IsSingleBusinessDocument()
{
    var bizType = EditEntity?.GetPropertyValue("BizType");
    if (bizType == null) return false;

    int bizTypeValue = 0;
    if (bizType is int)
    {
        bizTypeValue = (int)bizType;
    }
    else if (bizType is BizType)
    {
        bizTypeValue = (int)(BizType)bizType;
    }

    // 判断是否为单一业务单据类型
    return bizTypeValue == (int)BizType.借出单 ||
           bizTypeValue == (int)BizType.归还单 ||
           bizTypeValue == (int)BizType.调拨单 ||
           bizTypeValue == (int)BizType.其他入库单 ||
           bizTypeValue == (int)BizType.其他出库单;
}

// 新增方法：单一业务单据审核（仅记录状态）
protected virtual async Task<bool> SimpleBusinessApproval()
{
    if (EditEntity == null)
    {
        MainForm.Instance.uclog.AddLog("请先加载数据", Color.Red);
        return false;
    }

    try
    {
        // 1. 显示审核对话框
        var approvalResult = ShowApprovalDialog();
        if (approvalResult == null || !approvalResult.HasValue)
        {
            return false;
        }

        // 2. 调用Controller的审核方法（仅记录状态）
        var controller = GetController();
        var result = await controller.SimpleBusinessApprovalAsync(EditEntity, approvalResult.Value);

        // 3. 更新UI状态
        if (result.Succeeded)
        {
            MainForm.Instance.uclog.AddLog("审核成功", Color.Green);
            await RefreshData();
        }
        else
        {
            MainForm.Instance.uclog.AddLog($"审核失败：{result.ErrorMsg}", Color.Red);
        }

        return result.Succeeded;
    }
    catch (Exception ex)
    {
        logger?.LogError(ex, "单一业务单据审核失败");
        MainForm.Instance.uclog.AddLog($"审核失败：{ex.Message}", Color.Red);
        return false;
    }
}
```

```csharp
// 新增方法：单一业务单据结案（执行库存操作）
protected virtual async Task<bool> SingleBusinessCloseCase()
{
    if (EditEntity == null)
    {
        MainForm.Instance.uclog.AddLog("请先加载数据", Color.Red);
        return false;
    }

    try
    {
        // 1. 验证单据状态（必须已审核）
        var approvalStatus = EditEntity?.GetPropertyValue("ApprovalStatus");
        if (approvalStatus == null || (int)approvalStatus != (int)ApprovalStatus.审核通过)
        {
            MainForm.Instance.uclog.AddLog("单据未审核，无法结案", Color.Red);
            return false;
        }

        // 2. 显示结案对话框
        var closeCaseResult = ShowCloseCaseDialog();
        if (closeCaseResult == null || string.IsNullOrEmpty(closeCaseResult))
        {
            return false;
        }

        // 3. 调用Controller的结案方法（执行库存操作）
        var controller = GetController();
        var result = await controller.SingleBusinessCloseCaseAsync(EditEntity, closeCaseResult);

        // 4. 更新UI状态
        if (result.Succeeded)
        {
            MainForm.Instance.uclog.AddLog("结案成功", Color.Green);
            await RefreshData();
        }
        else
        {
            MainForm.Instance.uclog.AddLog($"结案失败：{result.ErrorMsg}", Color.Red);
        }

        return result.Succeeded;
    }
    catch (Exception ex)
    {
        logger?.LogError(ex, "单一业务单据结案失败");
        MainForm.Instance.uclog.AddLog($"结案失败：{ex.Message}", Color.Red);
        return false;
    }
}
```

### 3.2 业务层改动

#### 3.2.1 Controller层扩展
**文件位置**：为每个单一业务单据的Controller添加Partial类

**示例**：`RUINORERP.Business\tb_ProdBorrowingControllerPartial.cs`

```csharp
public partial class tb_ProdBorrowingController<T>
{
    /// <summary>
    /// 单一业务单据审核（仅记录状态，不执行库存操作）
    /// </summary>
    /// <param name="entity">单据实体</param>
    /// <param name="approval">审核结果</param>
    /// <returns>操作结果</returns>
    public async Task<ReturnResults<tb_ProdBorrowing>> SimpleBusinessApprovalAsync(tb_ProdBorrowing entity, ApprovalEntity approval)
    {
        ReturnResults<tb_ProdBorrowing> rr = new ReturnResults<tb_ProdBorrowing>();

        try
        {
            // 1. 开启事务
            _unitOfWorkManage.BeginTran();

            // 2. 更新审核状态（仅记录状态，不执行库存操作）
            entity.ApprovalStatus = approval.ApprovalResults ? (int)ApprovalStatus.审核通过 : (int)ApprovalStatus.审核驳回;
            entity.ApprovalOpinions = approval.ApprovalOpinions;
            entity.Approver_by = approval.Approver_by;
            entity.Approver_at = approval.Approver_at;

            // 3. 更新单据状态
            if (approval.ApprovalResults)
            {
                entity.DataStatus = (int)DataStatus.确认;
            }
            else
            {
                entity.DataStatus = (int)DataStatus.草稿;
            }

            // 4. 保存到数据库
            bool rs = await _tb_ProdBorrowingServices.Update(entity);

            // 5. 提交事务
            _unitOfWorkManage.CommitTran();

            rr.Succeeded = rs;
            rr.ReturnObject = entity;

            // 6. 更新缓存
            if (rs)
            {
                _eventDrivenCacheManager.UpdateEntity<tb_ProdBorrowing>(entity);
            }
        }
        catch (Exception ex)
        {
            _unitOfWorkManage.RollbackTran();
            rr.Succeeded = false;
            rr.ErrorMsg = ex.Message;
            _logger.LogError(ex, "单一业务单据审核失败");
        }

        return rr;
    }

    /// <summary>
    /// 单一业务单据结案（执行库存操作）
    /// </summary>
    /// <param name="entity">单据实体</param>
    /// <param name="closeCaseOpinions">结案意见</param>
    /// <returns>操作结果</returns>
    public async Task<ReturnResults<tb_ProdBorrowing>> SingleBusinessCloseCaseAsync(tb_ProdBorrowing entity, string closeCaseOpinions)
    {
        ReturnResults<tb_ProdBorrowing> rr = new ReturnResults<tb_ProdBorrowing>();

        try
        {
            // 1. 开启事务
            _unitOfWorkManage.BeginTran();

            // 2. 执行库存操作
            // TODO: 调用库存服务执行库存操作
            // var inventoryService = _appContext.GetRequiredService<IInventoryService>();
            // await inventoryService.ProcessBorrowingInventory(entity);

            // 3. 更新单据状态为完结
            entity.DataStatus = (int)DataStatus.完结;
            entity.CloseCaseOpinions = closeCaseOpinions;

            // 4. 保存到数据库
            bool rs = await _tb_ProdBorrowingServices.Update(entity);

            // 5. 提交事务
            _unitOfWorkManage.CommitTran();

            rr.Succeeded = rs;
            rr.ReturnObject = entity;

            // 6. 更新缓存
            if (rs)
            {
                _eventDrivenCacheManager.UpdateEntity<tb_ProdBorrowing>(entity);
            }
        }
        catch (Exception ex)
        {
            _unitOfWorkManage.RollbackTran();
            rr.Succeeded = false;
            rr.ErrorMsg = ex.Message;
            _logger.LogError(ex, "单一业务单据结案失败");
        }

        return rr;
    }
}
```

## 四、兼容性处理

### 4.1 数据兼容性
- **无需数据迁移**：使用现有的表结构和数据
- **状态映射**：现有的`DataStatus`和`ApprovalStatus`枚举无需修改

### 4.2 代码兼容性
- **保留原有审核方法**：在Controller中保留原有的`Review`方法，用于订单类业务
- **新增方法**：添加`SimpleBusinessApprovalAsync`和`SingleBusinessCloseCaseAsync`方法，用于单一业务单据
- **UI层路由**：在`Item_Click`方法中，根据单据类型路由到不同的处理方法

### 4.3 向后兼容
- **订单类业务**：保持原有流程不变
- **单一业务单据**：使用新的审核和结案流程
- **功能开关**：可以通过配置控制是否启用新的流程

## 七、总结

本方案通过调整审核和结案的职责分工，实现了"领导审批通过后仓库方可执行出库"的业务需求，具有以下特点：

1. **改动最小化**：仅调整UI层按钮逻辑和业务层方法，不涉及架构改动
2. **无需新增表和枚举**：使用现有的表结构和枚举
3. **向后兼容**：订单类业务保持原有流程不变
4. **易于实施**：实施步骤清晰，工作量可控
5. **风险可控**：改动范围小，测试和回滚相对容易

通过本方案的实施，将有效提升单一业务单据的审批规范性，满足企业管理需求，同时保持系统的稳定性和可维护性。
