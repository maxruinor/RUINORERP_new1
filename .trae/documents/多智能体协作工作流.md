# ERP系统多智能体协作工作流

## 概述

本文档定义了 ERP 系统代码重构、功能完善及代码审查的多智能体协作流程。当您（用户）提出复杂的代码任务时，AI 将自动按照此工作流协调多个"虚拟 Agent"来完成工作。

## 智能体角色定义

### 1. 任务规划Agent (Task Planner)

**职责**：
- 接收用户的任务需求
- 分析任务范围和复杂度
- 将大任务分解为可执行的子任务
- 确定任务优先级和依赖关系

**触发条件**：
- 用户提出任何代码重构、功能实现、bug修复等任务

**工作内容**：
1. 分析任务需求
2. 制定执行计划
3. 确定需要协作的模块
4. 输出任务分解清单

### 2. 代码实现Agent (Code Implementation)

**职责**：
- 执行具体的代码编写、重构、修复工作
- 遵循项目代码规范
- 确保代码与现有系统兼容

**触发条件**：
- 任务规划完成后，需要进行代码实现时

**工作内容**：
1. 搜索现有相关代码
2. 分析现有实现方式
3. 执行代码修改
4. 确保符合项目规范

### 3. 代码审查Agent (Code Review)

**职责**：
- 检查代码质量
- 识别潜在问题
- 提供改进建议

**触发条件**：
- 代码实现完成后

**工作内容**：
1. 检查代码规范
2. 识别安全风险
3. 发现潜在bug
4. 评估代码质量

### 4. 验证测试Agent (Validation & Testing)

**职责**：
- 验证功能实现的完整性
- 执行测试检查bug
- 验证编译通过性

**触发条件**：
- 代码审查通过后

**工作内容**：
1. 功能完整性检查
2. 回归测试
3. 编译验证

## 协作流程

### 标准流程

```
用户提出任务
    ↓
任务规划Agent → 分析需求 → 分解任务 → 制定计划
    ↓
代码实现Agent → 搜索代码 → 分析现有 → 执行修改
    ↓
代码审查Agent → 代码检查 → 问题识别 → 建议改进
    ↓
验证测试Agent → 功能验证 → Bug检测 → 编译验证
    ↓
完成 → 向用户报告结果
```

### 异常处理流程

```
如果代码审查发现问题 → 返回代码实现Agent进行修订
    ↓
修订完成后 → 再次进行代码审查
    ↓
直到审查通过 → 进入验证测试

如果验证测试失败 → 返回相关Agent进行修复
    ↓
修复后重新验证
```

## 任务示例

### 示例1：修复Bug

**用户输入**：
"修复销售订单保存时出现的重复提交问题"

**协作流程**：

1. **任务规划Agent**：
   - 分析：这是一个bug修复任务
   - 分解：搜索相关代码 → 分析问题原因 → 修复代码 → 验证修复

2. **代码实现Agent**：
   - 搜索"销售订单"、"保存"、"重复提交"相关代码
   - 分析现有的防重复机制
   - 定位问题代码并修复

3. **代码审查Agent**：
   - 检查修复代码的质量
   - 确保没有引入新问题

4. **验证测试Agent**：
   - 验证修复是否有效
   - 确保编译通过

### 示例2：功能实现

**用户输入**：
"实现产品多图片上传功能"

**协作流程**：

1. **任务规划Agent**：
   - 分析：这是一个功能实现任务
   - 分解：需求分析 → 方案设计 → 代码实现 → 测试验证

2. **代码实现Agent**：
   - 搜索现有的图片上传实现
   - 设计多图片存储方案
   - 实现前端和后端代码

3. **代码审查Agent**：
   - 检查代码规范
   - 评估性能和安全性

4. **验证测试Agent**：
   - 验证功能完整性

### 示例3：代码重构

**用户输入**：
"重构缓存管理模块，提升性能"

**协作流程**：

1. **任务规划Agent**：
   - 分析：这是一个重构任务
   - 分解：代码分析 → 重构方案 → 执行重构 → 验证兼容性

2. **代码实现Agent**：
   - 分析现有缓存实现
   - 识别性能瓶颈
   - 实施优化

3. **代码审查Agent**：
   - 确保重构后代码质量
   - 检查是否破坏现有功能

4. **验证测试Agent**：
   - 验证性能提升
   - 确保功能正常

## 使用方法

### 基本语法

在会话中，您可以直接提出任务：

```
[任务类型关键词]
- 修复xxx问题
- 实现xxx功能
- 重构xxx模块
- 优化xxx性能
- 代码审查xxx
```

### 强制多智能体协作

如果需要强制启动多智能体协作流程，可以使用以下命令：

```
/multi-agent [任务描述]
例如：/multi-agent 修复登录页面崩溃的问题
```

### 指定特定Agent

如果只需要特定Agent参与：

```
@task-planner [任务]
@code-implementation [任务]
@code-review [任务]
@validation [任务]
```

## 项目规范集成

本工作流严格遵循以下规范：

1. **代码规范**：使用 project_rules.md 中定义的规范
2. **Windows Forms规范**：遵循VS IDE的窗体设计规范
3. **WPF规范**：遵循WPF的XAML和代码分离规范
4. **类创建规则**：创建新类前必须搜索现有类

## 注意事项

1. 每次任务完成后，会向您展示完整的执行报告
2. 如果某个环节失败，会自动回退到相关Agent进行修复
3. 所有代码修改都会在您的确认下进行
4. 不自动提交代码，由您人工决定是否提交
