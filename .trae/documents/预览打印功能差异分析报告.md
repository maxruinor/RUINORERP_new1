# 预览打印功能差异分析报告

## 一、执行流程分析

### 1.1 打印配置入口预览流程（RptPrintConfig.cs 正确）

```
RptPrintConfig.btnPreView_Click 
    → PrintReport(RptMode.PREVIEW) 
    → 直接使用 this.PrintDataSources（外部传入的已处理数据）
    → FastReport.RegisterData(PrintDataSources, "rd")
    → RptPreviewForm 显示
```

**关键特点：**
- 数据来源：`PrintDataSources` 属性（由外部传入）
- 无需再次查询数据库
- 数据已经是完整的打印对象

### 1.2 单据编辑UI入口预览流程（BaseBillEditGeneric.cs 错误）

```
BaseBillEditGeneric.Preview() 
    → PrintHelper<T>.Print(list, RptMode.PREVIEW, PrintConfig)
    → ExecutePrintAsync() 
    → ctr.GetPrintDataSource(keyid) 重新从数据库查询
    → RptPreviewForm 显示
```

**关键特点：**
- 数据来源：EditEntity（当前编辑界面上的实体）
- 需要通过Controller.GetPrintDataSource()从数据库重新查询
- 可能存在数据不一致问题

---

## 二、根本原因分析

### 2.1 核心差异：数据获取方式不同

| 环节 | 打印配置入口 | 单据编辑UI入口 |
|------|-------------|----------------|
| 数据来源 | 直接传入的PrintDataSources | EditEntity + 重新查询 |
| 查询方式 | 无需查询 | 通过Controller.GetPrintDataSource(keyid) |
| 数据完整性 | 已处理完成的打印对象 | 依赖具体Controller实现 |

### 2.2 问题根源代码位置

**问题代码 - PrintHelper.cs 第198行：**
```csharp
var PrintData = await ctr.GetPrintDataSource(keyid).ConfigureAwait(false) as List<M>;
```

**对比 - RptPrintConfig.cs 第269行：**
```csharp
TargetReport.RegisterData(PrintDataSources, "rd");  // 直接使用传入数据
```

### 2.3 可能导致错误的具体原因

1. **数据关联问题**：GetPrintDataSource() 可能只返回主表数据，未包含明细数据或关联数据
2. **视图/实体映射问题**：重新查询后，数据可能映射到视图而非实体，导致某些字段缺失
3. **权限控制差异**：从数据库查询可能受到数据权限过滤，而直接传入的数据可能包含更多内容
4. **上下文状态问题**：EditEntity 可能处于编辑状态，其数据可能未完全提交到数据库

## 三、详细差异对比

### 3.1 数据准备环节

| 项目 | 打印配置入口 | 单据编辑UI入口 |
|------|-------------|----------------|
| 数据获取 | 直接使用传入数据 | 通过Controller.GetPrintDataSource重新查询 |
| 数据完整性 | 依赖传入者的处理 | 依赖各Controller的实现 |
| 明细数据 | 完整保留 | 可能缺失 |

### 3.2 参数传递方式

- **RptPrintConfig**：使用 `PrintDataSources` 属性接收完整打印数据
- **BaseBillEditGeneric**：使用 `EditEntity` 属性，然后重新查询

### 3.3 模板处理

两处均使用相同的模板选择逻辑：
```csharp
var printTemplate = printConfig.tb_PrintTemplates.Where(t => t.IsDefaultTemplate == true).FirstOrDefault();
```

## 四、解决方案设计

### 4.1 统一预览打印底层实现方案

#### 方案A：修改BaseBillEditGeneric的Preview方法（推荐）

**修改目标：** BaseBillEditGeneric.cs 第8706-8722行

**修改前：**
```csharp
public async Task Preview()
{
    bool rs = false;
    try
    {
        if (EditEntity == null)
        {
            MessageBox.Show("请提供正确的打印预览数据！");
```

**修改后：**
```csharp
public async Task Preview()
{
    bool rs = false;
    try
    {
        if (EditEntity == null)
        {
            MessageBox.Show("请提供正确的打印预览数据！");
            return;
        }

        // 检查是否已有完整的打印数据，如果有则直接使用，避免重新查询导致数据不一致
        if (this.PrintDataSources != null && this.PrintDataSources.Count > 0)
        {
            // 使用统一入口的打印配置
            if (PrintConfig == null || PrintConfig.tb_PrintTemplates == null)
            {
                List<T> list = new List<T>();
                list.Add(EditEntity);
                PrintConfig = PrintHelper<T>.GetPrintConfig(list);
            }
            rs = await PrintHelper<object>.PrintDirect(this.PrintDataSources, RptMode.PREVIEW, PrintConfig);
        }
        else
        {
            // 兼容旧逻辑：如果没有预准备的数据，则使用原有方式
            List<T> list = new List<T>();
            list.Add(EditEntity);
            if (PrintConfig == null || PrintConfig.tb_PrintTemplates == null)
            {
                PrintConfig = PrintHelper<T>.GetPrintConfig(list);
            }
            rs = await PrintHelper<T>.Print(list, RptMode.PREVIEW, PrintConfig);
        }
    }
    catch (Exception ex)
    {
        MainForm.Instance.logger.Error("打印配置加载异常", ex);
    }
}
```

#### 方案B：在PrintHelper中添加统一的直接打印方法

在PrintHelper.cs中添加新的静态方法：

```csharp
/// <summary>
/// 直接打印预览（不重新查询数据）
/// 用于已经准备好打印数据的场景
/// </summary>
public static async Task<bool> PrintDirect(List<object> printDataSources, RptMode rptMode, tb_PrintConfig printConfig)
{
    bool rs = false;
    try
    {
        rs = await Task.Run(async () =>
        {
            return await ExecutePrintDirectAsync(printDataSources, rptMode, printConfig);
        }).ConfigureAwait(false);
    }
    catch (Exception ex)
    {
        MainForm.Instance.logger.LogError(ex, "直接打印任务启动失败");
        MessageBox.Show($"打印失败: {ex.Message}", "错误", MessageBoxButtons.OK, MessageBoxIcon.Error);
    }
    return rs;
}

/// <summary>
/// 执行直接打印操作（不重新查询数据）
/// </summary>
private static async Task<bool> ExecutePrintDirectAsync(List<object> printDataSources, RptMode rptMode, tb_PrintConfig printConfig)
{
    bool rs = false;
    List<CurrentUserInfo> currUserInfos = new List<CurrentUserInfo>();
    currUserInfos.Add(MainForm.Instance.AppContext.CurUserInfo);

    List<tb_Company> companyInfos = new List<tb_Company>();
    companyInfos.Add(MainForm.Instance.AppContext.CompanyInfo);

    FastReport.Report FReport = new FastReport.Report();

    try
    {
        // 直接注册传入的数据源（与RptPrintConfig一致）
        FReport.RegisterData(printDataSources, "rd");
        FReport.RegisterData(currUserInfos, "currUserInfo");
        FReport.RegisterData(companyInfos, "companyInfo");

        if (printConfig.tb_PrintTemplates == null)
        {
            MessageBox.Show("请先配置正确的打印模板！", "提示", MessageBoxButtons.OK, MessageBoxIcon.Warning);
            return rs;
        }

        var printTemplate = printConfig.tb_PrintTemplates.Where(t => t.IsDefaultTemplate == true).FirstOrDefault();
        if (printTemplate == null && printConfig.tb_PrintTemplates.Count > 0)
        {
            printTemplate = printConfig.tb_PrintTemplates[0];
        }

        if (printTemplate != null && printTemplate.TemplateFileStream != null)
        {
            byte[] ReportBytes = (byte[])printTemplate.TemplateFileStream;
            using (System.IO.MemoryStream Stream = new System.IO.MemoryStream(ReportBytes))
            {
                FReport.Load(Stream);
            }
        }
        else
        {
            MessageBox.Show("请先配置正确的打印模板！", "提示", MessageBoxButtons.OK, MessageBoxIcon.Warning);
            return rs;
        }

        // 准备报表
        FReport.Preplay();

        if (rptMode == RptMode.PREVIEW)
        {
            MainForm.Instance.Invoke(new Action(() =>
            {
                RptPreviewForm frm = new RptPreviewForm();
                frm.Text = printConfig.BizName + "打印预览";
                frm.MyReport = FReport;
                frm.ShowDialog();
            }));
        }
        else if (rptMode == RptMode.PRINT)
        {
            string printerName = GetPrinterWithPriority(printConfig, printConfig?.BizName);
            if (!string.IsNullOrEmpty(printerName))
            {
                FReport.PrintSettings.ShowDialog = false;
                FReport.PrintSettings.Printer = printerName;
            }
            FReport.PrintPrepared();
        }

        rs = true;
    }
    catch (Exception ex)
    {
        MainForm.Instance.logger.LogError("直接打印报表异常。", ex);
        MessageBox.Show($"打印过程发生异常: {ex.Message}", "错误", MessageBoxButtons.OK, MessageBoxIcon.Error);
    }
    finally
    {
        FReport?.Dispose();
    }

    return rs;
}
```

## 七、总结

**根本原因：** 从单据编辑UI执行预览时，PrintHelper会通过Controller.GetPrintDataSource()重新从数据库查询数据，这个过程可能导致数据丢失或不一致；而从打印配置执行预览时，直接使用传入的PrintDataSources，数据完整性得到保障。

**解决方案：** 在BaseBillEditGeneric中添加对预准备数据源的支持，使其与RptPrintConfig使用相同的底层打印逻辑，确保行为一致性。
