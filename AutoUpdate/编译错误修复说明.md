# 编译错误修复说明

## 问题描述

在实现异步文件复制优化后，出现了两个 CS7036 编译错误：

```
错误 CS7036: 未提供与"FrmUpdate.CopyFileAsync(string, string, string)"的所需参数"VerNo"对应的参数
- 位置 1: 第 1753 行（子目录递归调用）
- 位置 2: 第 3760 行（ApplyAppAsync 中的调用）
```

## 根本原因

`CopyFileAsync` 方法有两个不同的使用场景：

1. **带版本号的复制**：`CopyFileAsync(sourcePath, objPath, VerNo)` 
   - 用于 `LastCopyAsync()` 方法
   - 只复制特定版本的文件
   - 需要根据 `htUpdateFile` 和版本号过滤文件

2. **不带版本号的复制**：`CopyFileAsync(sourcePath, objPath)`
   - 用于 `ApplyAppAsync()` 方法  
   - 复制所有在更新列表中的文件
   - 不需要版本号过滤

最初只实现了带版本号的版本，导致以下问题：
- 在带版本号的方法内部递归调用子目录时，没有传递 `VerNo` 参数
- `ApplyAppAsync()` 调用了不存在的不带参数版本

## 解决方案

### 1. 修复子目录递归调用（第 1753 行）

**修改前：**
```csharp
var dirTask = CopyFileAsync(dirs[i], destSubDir);  // ❌ 缺少 VerNo 参数
```

**修改后：**
```csharp
var dirTask = CopyFileAsync(dirs[i], destSubDir, VerNo);  // ✅ 传递 VerNo 参数
```

### 3. 更新 ApplyAppAsync 调用（第 3938 行）

**修改前：**
```csharp

```

```csharp
await CopyFileAsync(tempUpdatePath, AppDomain.CurrentDomain.BaseDirectory);  // ✅ 异步方法
```

## 方法签名对比

### 带版本号的版本
```csharp
public async Task CopyFileAsync(string sourcePath, string objPath, string VerNo)
```
- **用途**：按版本复制文件
- **文件过滤**：检查文件名 AND 版本号
- **调用场景**：`LastCopyAsync()` 处理版本化更新
- **递归调用**：传递相同的 `VerNo` 给子目录

### 不带版本号的版本
```csharp
public async Task CopyFileAsync(string sourcePath, string objPath)
```
- **用途**：复制所有更新文件
- **文件过滤**：只检查文件名
- **调用场景**：`ApplyAppAsync()` 应用完整更新
- **递归调用**：不传递版本号
