## 关键技术点

### 1. 双进程更新机制

**为什么需要辅助进程？**
- AutoUpdate.exe 正在运行时，无法替换自身文件
- 需要一个独立的进程来完成自我更新
- AutoUpdateUpdater.exe 作为"清洁工"，在主进程退出后工作

**工作流程**：
```
AutoUpdate.exe（主进程）          AutoUpdateUpdater.exe（辅助进程）
     |                                    |
     |--- 复制其他文件 ------------------>|
     |--- 启动辅助进程 ------------------>|
     |--- 等待 2 秒                      |
     |--- 退出                           |
     |                                    |--- 等待主进程退出
     |                                    |--- 替换 AutoUpdate.exe
     |                                    |--- 启动 ERP.exe
     |                                    |--- 清理临时文件
     |                                    |--- 退出
```

### 2. 用户体验优化

**等待时间设计**：
```csharp
Thread.Sleep(2000);  // 2 秒等待
```

**为什么是 2 秒？**
- ✅ 足够让用户看到"更新成功"的提示
- ✅ 不会让用户等待太久
- ✅ 给辅助进程足够的启动时间
- ✅ 平衡用户体验和效率

**之前的优化**：
- 从 5 秒减少到 2 秒（提升响应速度）
- 添加清晰的阶段提示（减少焦虑）
- 使用 ✅ 符号（视觉反馈）

### 3. 错误处理

**如果辅助进程启动失败**：
```csharp
else
{
    AppendAllText("警告：自我更新辅助进程启动失败，使用传统文件复制方式");
    
    // 确保配置文件正确复制
    CopyConfigFileToRoot();
    
    // 直接启动 ERP 系统
    StartERPApplication();
}
```

**降级方案**：
- 不更新 AutoUpdate.exe 自身
- 但其他文件已更新
- 直接启动主程序
- 下次更新时会再次尝试更新 AutoUpdate.exe

## 用户常见问题

### Q1: 复制完成后需要我手动启动程序吗？
**A**: 不需要！复制完成后会**自动启动**主程序（ERP）。您只需要耐心等待即可。

### Q2: 为什么要等待 2 秒？
**A**: 
- 让您看到"更新成功"的提示
- 给辅助进程足够的启动时间
- 确保更新过程稳定可靠

### Q3: 如果程序没有自动启动怎么办？
**A**: 
1. 检查任务管理器，看是否有 ERP.exe 进程
2. 如果确实没有启动，可以手动启动 ERP 程序
3. 查看日志文件了解失败原因

### Q4: 更新过程中可以取消吗？
**A**: 
- 文件复制阶段：不建议取消，可能导致文件不完整
- 一旦开始复制，最好等待完成
- 如果真的需要取消，可以强制关闭更新窗口

### Q5: 为什么需要辅助进程？
**A**: 
- Windows 不允许替换正在运行的 exe 文件
- 辅助进程在主进程退出后完成自我更新
- 这是 Windows 平台的标准做法
