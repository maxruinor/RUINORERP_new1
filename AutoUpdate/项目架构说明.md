# AutoUpdate 与 AutoUpdateUpdater 项目架构说明

## 📋 项目关系

### 两个独立的项目

```
AutoUpdate/                    AutoUpdateUpdater/
├── 主更新程序                  ├── 辅助更新程序
├── UpdateSystemConstants.cs   ├── UpdaterConstants.cs
├── FrmUpdate.cs               ├── Program.cs
├── SmartSnapshotManager.cs    ├── UpdateConfig.cs
└── ...                        └── ...
```

### 职责划分

| 项目 | 职责 | 运行时机 |
|------|------|----------|
| **AutoUpdate** | 主更新程序，负责下载、复制文件、创建快照 | 用户手动触发或自动检测更新时 |
| **AutoUpdateUpdater** | 辅助进程，专门用于更新 AutoUpdate.exe 自身 | AutoUpdate 需要自我更新时启动 |

---

## 🔧 常量定义分离

### ❌ 错误做法（已修复）

之前 `AutoUpdateUpdater` 引用了 `AutoUpdate.UpdateSystemConstants`：

```csharp
// AutoUpdateUpdater/Program.cs - 错误的做法
private const int PROCESS_WAIT_TIMEOUT_MS = AutoUpdate.UpdateSystemConstants.ProcessExitTimeoutMs;
```

**问题**：
1. ❌ 循环依赖风险
2. ❌ 耦合过紧，两个独立项目不应该互相依赖
3. ❌ 部署问题，AutoUpdateUpdater 是独立进程

### ✅ 正确做法（当前实现）

每个项目有自己的常量定义文件：

```
AutoUpdate/UpdateSystemConstants.cs      → namespace AutoUpdate
AutoUpdateUpdater/UpdaterConstants.cs    → namespace AutoUpdateUpdater
```

**优势**：
1. ✅ 完全解耦，两个项目互不依赖
2. ✅ 可以独立编译和部署
3. ✅ 常量值可以不同（如果需要）
4. ✅ 符合单一职责原则

## 🎯 设计原则

### 1. 项目独立性

- AutoUpdate 和 AutoUpdateUpdater 是**完全独立**的两个项目
- 它们之间**没有代码依赖**
- 通过**进程间通信**协作（命令行参数、配置文件）

### 2. 常量隔离

- 每个项目维护自己的常量定义
- 常量值可以相同，但**定义位置必须独立**
- 避免跨项目引用导致的耦合

### 3. 职责清晰

| 功能 | AutoUpdate | AutoUpdateUpdater |
|------|-----------|-------------------|
| 下载更新文件 | ✅ | ❌ |
| 复制文件到目标目录 | ✅ | ✅ |
| 创建版本快照 | ✅ | ❌ |
| 启动主程序(ERP) | ✅ | ✅ |
| 更新 AutoUpdate.exe | ❌ | ✅ |
| 日志轮转 | ✅ | ✅ |

## 🔄 协作流程

```
用户触发更新
    ↓
AutoUpdate 检查更新
    ↓
需要更新 AutoUpdate.exe 自身？
    ├─ 否 → 直接更新其他文件
    └─ 是 → 启动 AutoUpdateUpdater
              ↓
         AutoUpdateUpdater 等待 AutoUpdate 退出
              ↓
         复制新的 AutoUpdate.exe
              ↓
         启动新的 AutoUpdate.exe
              ↓
         AutoUpdateUpdater 退出
```

## ⚠️ 注意事项

### 1. 不要跨项目引用

❌ **禁止**：
```csharp
// AutoUpdateUpdater 中
using AutoUpdate;  // 禁止！
var timeout = UpdateSystemConstants.ProcessExitTimeoutMs;  // 禁止！
```

✅ **正确**：
```csharp
// AutoUpdateUpdater 中
using AutoUpdateUpdater;  // 使用本项目的命名空间
var timeout = UpdaterConstants.ProcessExitTimeoutMs;  // 正确！
```

### 2. 常量同步

如果两个项目需要使用相同的常量值：
- **手动保持同步**（推荐）
- 或者在注释中说明"此值与 AutoUpdate 保持一致"

### 3. 项目特定常量

某些常量只属于特定项目：
- `AsyncSnapshotEnabled` - 只在 AutoUpdate 中
- `ErpStartupMaxRetries` - 只在 AutoUpdate 中
- `DefaultLogFileName` - 两个项目有不同的日志文件名
