## 🔴 P0 严重问题（必须修复）

### 1. 窗体加载性能

- [ ] **Load 事件是否异步？**
  - ❌ 发现同步阻塞操作
  - ✅ 使用 `async/await` 异步加载
  
- [ ] **是否分阶段加载？**
  - ❌ 所有数据一次性加载
  - ✅ 关键数据优先，非关键数据后台加载
  
- [ ] **是否显示加载提示？**
  - ❌ 白屏等待
  - ✅ 显示 BusyDialog 或进度条
  
- [ ] **是否并行加载独立数据？**
  - ❌ 串行加载多个独立数据源
  - ✅ 使用 `Task.WhenAll` 并行加载

**审查示例**:
```csharp
// ❌ 不合格
private void Form_Load(object sender, EventArgs e)
{
    LoadData();      // 阻塞 UI
    LoadPermissions(); // 阻塞 UI
    BindData();      // 阻塞 UI
}

// ✅ 合格
private async void Form_Load(object sender, EventArgs e)
{
    ShowBusyDialog("加载中...");
    
    await Task.Run(async () =>
    {
        var task1 = LoadDataAsync();
        var task2 = LoadPermissionsAsync();
        await Task.WhenAll(task1, task2);
    });
    
    BindData();
    HideBusyDialog();
}
```

### 2. 数据绑定性能

- [ ] **是否批量绑定？**
  - ❌ 逐行添加 `Rows.Add()`
  - ✅ 使用 `BindDataBatch()` 批量绑定
  
- [ ] **是否暂停重绘？**
  - ❌ 直接绑定，频繁重绘
  - ✅ `SuspendLayout()` / `ResumeLayout()`
  
- [ ] **是否启用双缓冲？**
  - ❌ 列表闪烁严重
  - ✅ `EnableDoubleBuffered(true)`
  
- [ ] **是否分页加载？**
  - ❌ 一次性加载所有数据
  - ✅ 分页查询，每页 50-100 条

**审查示例**:
```csharp
// ❌ 不合格
foreach (var item in dataList)
{
    dataGridView.Rows.Add(item);  // 逐行添加
}

// ✅ 合格
dataGridView.SuspendLayout();
dataGridView.EnableDoubleBuffered(true);
dataGridView.BindDataBatch(dataList);  // 批量绑定
dataGridView.ResumeLayout();
```

### 3. 反射优化

- [ ] **是否使用反射缓存？**
  - ❌ 每次调用都反射
  - ✅ 使用 `ConcurrentDictionary` 缓存
  
- [ ] **缓存是否静态？**
  - ❌ 实例级缓存，重复反射
  - ✅ 静态缓存，全局共享
  
- [ ] **是否避免重复获取特性？**
  - ❌ 多次调用 `GetCustomAttributes`
  - ✅ 一次性获取并缓存

**审查示例**:
```csharp
// ❌ 不合格
private void InitBaseValue()
{
    var props = typeof(T).GetProperties();  // 每次都反射
    foreach (var prop in props)
    {
        var attrs = prop.GetCustomAttributes(false);
    }
}

// ✅ 合格
private static readonly ConcurrentDictionary<Type, PropertyInfo[]> _cache 
    = new ConcurrentDictionary<Type, PropertyInfo[]>();

private PropertyInfo[] GetCachedProperties(Type type)
{
    return _cache.GetOrAdd(type, t => t.GetProperties());
}
```

### 4. 异步操作

- [ ] **I/O 操作是否异步？**
  - ❌ 使用同步文件/数据库操作
  - ✅ 使用 `ReadAsync` / `WriteAsync` / `ToListAsync`
  
- [ ] **网络请求是否异步？**
  - ❌ `HttpClient.GetString()` 同步调用
  - ✅ `HttpClient.GetStringAsync()` 异步调用
  
- [ ] **是否避免 async void？**
  - ❌ `async void` 方法（事件处理除外）
  - ✅ `async Task` 方法
  
- [ ] **是否使用 ConfigureAwait(false)？**
  - ❌ 不必要的上下文切换
  - ✅ `await xxx.ConfigureAwait(false)`

**审查示例**:
```csharp
// ❌ 不合格
public void LoadData()
{
    var data = dbContext.Query<T>().ToList();  // 同步阻塞
    File.WriteAllText("log.txt", data);        // 同步阻塞
}

// ✅ 合格
public async Task LoadDataAsync()
{
    var data = await dbContext.Query<T>().ToListAsync();  // 异步
    await File.WriteAllTextAsync("log.txt", data);         // 异步
}
```

### 5. 菜单响应

- [ ] **菜单点击是否异步打开窗体？**
  - ❌ 同步创建和加载窗体
  - ✅ 异步创建，后台加载数据
  
- [ ] **是否先显示窗体后加载数据？**
  - ❌ 数据加载完才显示窗体
  - ✅ 先显示空壳，后台加载数据

**审查示例**:
```csharp
// ❌ 不合格
private void menuItem_Click(object sender, EventArgs e)
{
    var form = new UCSaleOrder();
    form.LoadData();  // 阻塞
    form.Show();
}

// ✅ 合格
private async void menuItem_Click(object sender, EventArgs e)
{
    ShowLoadingIndicator();
    
    var form = await Task.Run(() => new UCSaleOrder());
    form.Show();  // 先显示
    
    _ = Task.Run(async () =>
    {
        await form.LoadDataAsync();  // 后台加载
        HideLoadingIndicator();
    });
}
```

## 🟡 P1 中等问题（计划修复）

### 6. 内存管理

- [ ] **事件是否正确注销？**
  - ❌ 只订阅不注销
  - ✅ `Dispose` 时注销所有事件
  
- [ ] **是否实现 IDisposable？**
  - ❌ 非托管资源未释放
  - ✅ 实现 `IDisposable` 模式
  
- [ ] **是否及时释放控件？**
  - ❌ 控件关闭后未释放
  - ✅ `form.Dispose()` 释放资源

**审查示例**:
```csharp
// ❌ 不合格
public UCSomeControl()
{
    someService.SomeEvent += OnSomeEvent;  // 未注销
}

// ✅ 合格
protected override void Dispose(bool disposing)
{
    if (disposing)
    {
        someService.SomeEvent -= OnSomeEvent;  // 注销事件
    }
    base.Dispose(disposing);
}
```

### 7. 图片加载

- [ ] **是否压缩/缩放图片？**
  - ❌ 直接加载原图（可能 10MB+）
  - ✅ 按需缩放后加载
  
- [ ] **是否异步加载图片？**
  - ❌ 同步加载阻塞 UI
  - ✅ `LoadAndResizeImageAsync` 异步加载
  
- [ ] **是否使用图片缓存？**
  - ❌ 重复加载相同图片
  - ✅ 使用缓存避免重复加载

**审查示例**:
```csharp
// ❌ 不合格
pictureBox.Image = Image.FromFile("large.jpg");  // 加载 10MB 原图

// ✅ 合格
await pictureBox.LoadAndResizeImageAsync(
    "large.jpg",
    maxWidth: 800,
    maxHeight: 600
);  // 加载缩放后的图片
```

### 8. 字符串操作

- [ ] **循环中是否使用 StringBuilder？**
  - ❌ `string +=` 拼接
  - ✅ `StringBuilder.Append`
  
- [ ] **SQL 拼接是否参数化？**
  - ❌ 字符串拼接 SQL
  - ✅ 使用参数化查询

**审查示例**:
```csharp
// ❌ 不合格
string sql = "SELECT * FROM t WHERE 1=1";
foreach (var id in ids)
{
    sql += " AND id=" + id;  // 字符串拼接
}

// ✅ 合格
var sb = new StringBuilder("SELECT * FROM t WHERE 1=1");
foreach (var id in ids)
{
    sb.Append(" AND id=").Append(id);
}
```

### 9. 数据库查询

- [ ] **是否使用分页？**
  - ❌ `ToList()` 加载所有数据
  - ✅ `Skip().Take()` 分页查询
  
- [ ] **是否只查询需要的字段？**
  - ❌ `SELECT *` 查询所有字段
  - ✅ 投影查询需要的字段
  
- [ ] **是否使用索引？**
  - ❌ WHERE 条件无索引
  - ✅ 为查询条件添加索引

**审查示例**:
```csharp
// ❌ 不合格
var all = context.Query<T>().ToList();  // 全表扫描

// ✅ 合格
var page = context.Query<T>()
    .Where(x => x.Status == 1)
    .OrderBy(x => x.CreateTime)
    .Skip((page - 1) * 50)
    .Take(50)
    .Select(x => new { x.Id, x.Name })  // 只查需要的字段
    .ToList();
```

### 10. 定时器使用

- [ ] **定时器频率是否合理？**
  - ❌ `Interval = 100` (100ms)
  - ✅ `Interval >= 5000` (5 秒)
  
- [ ] **是否按需刷新？**
  - ❌ 无条件刷新
  - ✅ 检查 `NeedRefresh()` 再刷新

**审查示例**:
```csharp
// ❌ 不合格
timer.Interval = 100;  // 100ms 一次
timer.Tick += (s, e) => RefreshData();  // 频繁刷新

// ✅ 合格
timer.Interval = 5000;  // 5 秒一次
timer.Tick += async (s, e) => 
{
    if (NeedRefresh())
    {
        await RefreshDataAsync();
    }
};
```
