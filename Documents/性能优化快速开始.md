### 步骤 3: 实施第二个优化（5 分钟）

**打开** `MenuHelper.cs`  
**添加** using:

```csharp
using System.Collections.Concurrent;
using System.Reflection;
```

**添加** 缓存字段（类开始处）:

```csharp
private static readonly ConcurrentDictionary<Type, PropertyInfo> _curMenuInfoCache 
    = new ConcurrentDictionary<Type, PropertyInfo>();
```

**修改** 反射代码（约第 448 行）:

```csharp
// 修改前
var curMenuInfoProperty = menu.GetType().GetProperty("CurMenuInfo");

// 修改后
var propertyInfo = _curMenuInfoCache.GetOrAdd(menu.GetType(), type => 
    type.GetProperty("CurMenuInfo"));

if (propertyInfo != null && propertyInfo.CanWrite)
{
    propertyInfo.SetValue(menu, pr);
}
```

**重新运行**,重复打开同一窗体，感受速度提升。

## ⚠️ 六、注意事项

### 6.1 测试要点

- ✅ 每步修改后都要测试
- ✅ 记录优化前后数据
- ✅ 发现异常立即回滚
- ✅ 不要一次性改动太多

### 6.2 回滚方案

```csharp
// 如果优化后有问题，直接改回原代码
// 例如：
// 修改前：bindingSourceSub.DataSource = lines;
// 修改后：this.dgvDetail.BindDataBatch(lines, true);

// 回滚：直接改回去即可
this.dgvDetail.DataSource = lines;
```

### 6.3 常见问题

**Q: 优化后数据不显示？**  
A: 检查 `autoGenerateColumns` 参数，设置为 `true`

**Q: 优化后无法编辑？**  
A: 确保使用 `BindingList<T>` 而不是 `List<T>`

**Q: 异步创建后功能异常？**  
A: 确保控件创建在 UI 线程执行
