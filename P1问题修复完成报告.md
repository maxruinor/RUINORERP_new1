## 二、详细修复内容

### 修复1: ConnectionManager状态更新时机调整

#### 问题描述
原代码在连接尝试**之前**就更新了 `_serverAddress` 和 `_serverPort`,导致:
- 如果连接失败,重连机制会使用新地址(可能与用户预期不符)
- 状态与实际连接情况不一致

#### 修复方案
将状态更新移到连接**成功后**执行:

**修改前**:
```csharp
// 在尝试连接之前更新服务器信息
_serverAddress = serverAddress;
_serverPort = serverPort;

bool connected = await _socketClient.ConnectAsync(serverAddress, serverPort, cancellationToken);

if (connected)
{
    // ...
}
else
{
    //_logger?.LogWarning("连接服务器失败...");  // 被注释
}
```

**修改后**:
```csharp
// 尝试连接
bool connected = await _socketClient.ConnectAsync(serverAddress, serverPort, cancellationToken);

if (connected)
{
    // 仅在连接成功后更新服务器信息
    // 这样可以确保重连机制使用的是已确认成功的地址
    _serverAddress = serverAddress;
    _serverPort = serverPort;
    
    _logger?.LogDebug("成功连接到服务器 {ServerAddress}:{ServerPort}", serverAddress, serverPort);
    // ...
}
else
{
    _logger?.LogWarning("连接服务器失败 {ServerAddress}:{ServerPort}", serverAddress, serverPort);
}
```

#### 改进效果
- ✅ **避免错误重连**: 连接失败时不会更新状态,重连机制不会使用错误地址
- ✅ **启用日志记录**: 取消注释,记录连接失败的详细信息
- ✅ **状态一致性**: 确保 `_serverAddress` 和 `_serverPort` 始终反映实际成功的连接

#### 影响场景示例

**修复前的风险场景**:
```
1. 用户已连接到 192.168.1.100:5000
2. 用户修改为 192.168.1.200:5000 并点击登录
3. ConnectAsync 被调用,_serverAddress 立即更新为 192.168.1.200
4. 连接失败(网络问题)
5. 自动重连启动,持续尝试连接 192.168.1.200
6. 用户想改回 192.168.1.100,但后台一直在重连 192.168.1.200 ❌
```

**修复后的行为**:
```
1. 用户已连接到 192.168.1.100:5000
2. 用户修改为 192.168.1.200:5000 并点击登录
3. ConnectAsync 被调用,但 _serverAddress 仍为 192.168.1.100
4. 连接失败(网络问题)
5. _serverAddress 保持为 192.168.1.100(未更新)
6. 用户可以重新输入其他地址,不会有后台重连干扰 ✅
```

### 修复2: ReconnectAndWelcomeAsync重构

#### 问题描述
原代码中 `ReconnectAndWelcomeAsync()` 调用了 `InitializeConnectionAndWelcomeFlowAsync()`,导致:
- 窗体加载时的初始化与用户触发的重连可能并发执行
- 两个流程同时操作 `_welcomeCompletionTcs`,产生竞态条件
- 方法职责不清晰,增加维护复杂度

#### 修复方案
在 `ReconnectAndWelcomeAsync()` 中直接实现完整的连接和欢迎流程,不再复用其他方法:

**修改前**:
```csharp
private async Task ReconnectAndWelcomeAsync()
{
    // ... 验证和断开连接 ...
    
    // 重置欢迎流程状态
    _welcomeCompletionTcs = new TaskCompletionSource<(bool, string)>();
    _welcomeCompleted = false;
    _welcomeAnnouncement = string.Empty;
    
    // 执行连接和欢迎流程 - 复用方法
    await InitializeConnectionAndWelcomeFlowAsync();  // ❌ 可能导致竞态
}
```

**修改后**:
```csharp
private async Task ReconnectAndWelcomeAsync()
{
    try
    {
        // ... 验证和断开连接 ...
        
        // 重置欢迎流程状态
        _welcomeCompletionTcs = new TaskCompletionSource<(bool, string)>();
        _welcomeCompleted = false;
        _welcomeAnnouncement = string.Empty;
        
        // 直接执行连接和欢迎流程，不重用 InitializeConnectionAndWelcomeFlowAsync
        // 避免与窗体加载时的初始化流程产生竞态条件
        MainForm.Instance?.ShowStatusText($"正在连接到服务器 {newServerIP}:{newServerPort}...");
        
        var connectTask = connectionManager.ConnectAsync(newServerIP, newServerPort);
        var completedTask = await Task.WhenAny(connectTask, Task.Delay(TimeSpan.FromSeconds(10)));
        
        bool connectResult = false;
        if (completedTask == connectTask)
        {
            connectResult = await connectTask;
        }
        else
        {
            MainForm.Instance?.logger?.LogWarning($"连接服务器 {newServerIP}:{newServerPort} 超时");
            MainForm.Instance?.ShowStatusText($"连接服务器 {newServerIP}:{newServerPort} 超时");
            return;
        }
        
        if (!connectResult)
        {
            MainForm.Instance?.ShowStatusText($"无法连接到服务器 {newServerIP}:{newServerPort}");
            return;
        }
        
        MainForm.Instance?.ShowStatusText($"服务器 {newServerIP}:{newServerPort} 连接成功");
        MainForm.Instance.PrintInfoLog("服务器连接成功,等待欢迎消息...");
        
        // 等待欢迎流程完成(等待最多15秒)
        var welcomeTimeout = TimeSpan.FromSeconds(15);
        var welcomeTask = _welcomeCompletionTcs.Task;
        completedTask = await Task.WhenAny(welcomeTask, Task.Delay(welcomeTimeout));
        
        if (completedTask == welcomeTask)
        {
            var (success, announcement) = await welcomeTask;
            _welcomeCompleted = success;
            
            // 处理欢迎结果...
        }
        else
        {
            MainForm.Instance?.logger?.LogWarning("欢迎流程验证超时,但连接已建立");
            MainForm.Instance.ShowStatusText("服务器连接成功,欢迎验证超时");
        }
    }
    catch (Exception ex)
    {
        MainForm.Instance?.logger?.LogError(ex, "重新连接和欢迎流程时发生异常");
        MainForm.Instance?.ShowStatusText($"重新连接时发生错误: {ex.Message}");
    }
}
```

#### 改进效果
- ✅ **消除竞态条件**: 不再有并发初始化流程
- ✅ **职责清晰**: 每个方法负责自己的完整流程
- ✅ **易于维护**: 逻辑独立,互不干扰
- ✅ **代码量增加但更清晰**: +73行,但可读性和可维护性显著提升

#### 影响场景示例

**修复前的风险场景**:
```
时间线:
T0: 窗体加载,启动 InitializeConnectionAndWelcomeFlowAsync (任务A)
T1: 任务A正在连接服务器...
T2: 用户修改IP,触发 TextChanged
T3: 1.5秒后,DebouncedReconnectAsync 执行
T4: ReconnectAndWelcomeAsync 创建新的 _welcomeCompletionTcs
T5: ReconnectAndWelcomeAsync 调用 InitializeConnectionAndWelcomeFlowAsync (任务B)
T6: 任务A和任务B同时运行,都尝试设置 _welcomeCompletionTcs
T7: 只有一个能成功,另一个的 TrySetResult 返回 false ❌
```

**修复后的行为**:
```
时间线:
T0: 窗体加载,启动 InitializeConnectionAndWelcomeFlowAsync (任务A)
T1: 任务A正在连接服务器...
T2: 用户修改IP,触发 TextChanged
T3: 1.5秒后,DebouncedReconnectAsync 执行
T4: ReconnectAndWelcomeAsync 创建新的 _welcomeCompletionTcs
T5: ReconnectAndWelcomeAsync 直接执行连接逻辑 (任务B)
T6: 任务A和任务B各自独立运行,互不干扰
T7: 每个任务都有自己的 _welcomeCompletionTcs,无冲突 ✅
```
