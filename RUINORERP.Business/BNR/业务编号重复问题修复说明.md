# 业务编号重复问题 - 完整修复方案

## 📋 问题总结

### 现象
- 销售出库单生成的编号与之前生成过的编号重复
- 其他业务类型也存在类似情况
- **关键触发条件**: 服务器不稳定,多次重启测试

### 根本原因

#### 1. 序列键生成不一致 (已修复)
**位置**: `DatabaseSequenceParameter.GenerateStandardSequenceKey`

**问题**: 
- 嵌套表达式 `{DB:{S:销售出库单}{D:yyMM}/0000}` 中的日期处理器会被执行
- 导致序列键中包含动态日期,不同时间调用可能生成不同的键
- 例如: `SEQ_销售出库单2604_DAILY` vs `SEQ_销售出库单_DAILY`

**影响**: 
- 相同业务场景下使用了不同的序列键
- 从不同的批次缓存获取序号,导致重复

#### 2. 服务器重启导致内存缓存丢失 (核心问题)
**位置**: `DatabaseSequenceService._batchCaches`

**问题**:
```csharp
// 内存中的批次缓存,服务器重启后完全丢失!
private readonly ConcurrentDictionary<string, SequenceBatchCache> _batchCaches;
```

**重现场景**:
```
第1次启动:
1. 生成销售出库单 → 从数据库获取批次 [1-50]
2. 使用序号 1, 2, 3
3. 数据库 CurrentValue = 50
4. 内存缓存: SEQ_销售出库单_DAILY → (3, 50)

⚠️ 服务器重启! 内存缓存清空!

第2次启动:
1. 如果序列键一致 → 从数据库读取 CurrentValue=50,分配新批次 [51-100] ✅
2. 如果序列键不一致 → 创建新记录,CurrentValue=50,分配批次 [1-50] ❌ 重复!
```

#### 3. 并发竞态条件
虽然使用了 `SemaphoreSlim` 锁,但如果序列键不一致,不同键会获得独立的锁和批次,仍然可能重复。

## ✅ 修复方案

### 修复1: 稳定序列键生成

**文件**: `RUINORERP.Business\BNR\DatabaseSequenceParameter.cs`

**关键改动**:
```csharp
// ✅ 只处理常量处理器(S),跳过日期处理器(D/Hex等)
if (handlerName.Equals("S", StringComparison.OrdinalIgnoreCase))
{
    handler.Execute(tempBuilder, sps[1]);
    hasAddedStaticPart = true;
}
// 跳过日期相关的处理器,不在序列键中包含日期
```

**效果**:
- 序列键只包含业务标识(如"销售出库单")
- 不包含动态日期部分
- 确保相同业务始终使用相同的序列键

### 修复2: 增强批次分配日志

**文件**: `RUINORERP.Business\BNR\DatabaseSequenceService.cs`

**关键改动**:
```csharp
// ✅ 在事务开始前记录序列键,用于调试和验证
System.Diagnostics.Debug.WriteLine($"[分配批次] 开始处理序列键: '{sequenceKey}', 重置类型: '{resetType}'");

// ✅ 记录找到现有记录或创建新记录的详细信息
System.Diagnostics.Debug.WriteLine($"[分配批次] 找到现有记录: 键='{sequenceKey}', 当前值={batchStartValue}, 新批次上限={batchEndValue}");
```

**效果**:
- 完整的调试日志,便于追踪问题
- 可以清楚看到每次分配的序列键和批次范围

### 修复3: 添加序列键一致性诊断工具

**新增文件**: `RUINORERP.Business\BNR\SequenceKeyConsistencyChecker.cs`

**功能**:
1. 诊断所有业务类型的序列键一致性
2. 检查特定业务类型的序列键
3. 清理重复的序列键
4. 检测潜在问题(如相同业务类型但重置类型不同)

**使用方法**:
```csharp
var checker = new SequenceKeyConsistencyChecker(databaseSequenceService);

// 诊断所有序列
string report = checker.DiagnoseAllSequences();
Console.WriteLine(report);

// 诊断特定业务类型
string salesReport = checker.DiagnoseBusinessType("销售出库单");
Console.WriteLine(salesReport);

// 清理重复序列(谨慎使用!)
string cleanupReport = checker.CleanupDuplicateSequences(
    "销售出库单", 
    "SEQ_销售出库单_DAILY"); // 保留这个键
Console.WriteLine(cleanupReport);
```
