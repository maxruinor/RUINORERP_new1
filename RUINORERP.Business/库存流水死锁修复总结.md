## 🔍 问题根源

### 原始问题

在多个Controller中（如`tb_SaleOutControllerPartial.cs`），存在以下代码模式：

```csharp
_unitOfWorkManage.BeginTran();  // 开始事务

// ... 大量的业务逻辑处理 ...

// 在事务内调用BatchRecordTransactions
await tranController.BatchRecordTransactions(transactionList);  // ⚠️ 问题点

// ... 更多业务逻辑 ...

_unitOfWorkManage.CommitTran();  // 提交事务
```

**问题分析**：

1. **事务持有时间长**：
   - 业务逻辑处理耗时（数秒）
   - 库存表锁被长时间持有
   - 并发请求容易死锁

2. **事务上下文混乱**：
   - `BatchRecordTransactions`直接使用`GetDbClient()`
   - 没有检查是否在事务中
   - 可能导致事务嵌套问题

3. **缺少错误处理**：
   - 没有死锁重试机制
   - 死锁后直接失败
