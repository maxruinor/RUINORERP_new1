# 库存流水死锁问题全面分析与修复方案

## 📋 问题概述

**问题描述**：在数据库监控中发现死锁现象，涉及`tb_InventoryTransaction`表的INSERT操作。

**捕获的SQL语句**：
```sql
INSERT INTO [tb_InventoryTransaction] 
([TranID],[ProdDetailID],[Location_ID],[BizType],[ReferenceId],[ReferenceNo],
 [QuantityChange],[AfterQuantity],[BatchNumber],[UnitCost],[TransactionTime],[OperatorId],[Notes])
VALUES
(@TranID,@ProdDetailID,@Location_ID,@BizType,@ReferenceId,@ReferenceNo,
 @QuantityChange,@AfterQuantity,@BatchNumber,@UnitCost,@TransactionTime,@OperatorId,@Notes)
```

## 🔍 死锁原因分析

### 1. 核心问题：事务内外数据库访问混合

**问题代码模式**（在多个Controller中存在）：

```csharp
// ❌ 问题模式：事务开始后，在事务内调用其他服务
_unitOfWorkManage.BeginTran();  // 开始事务

// ... 业务逻辑 ...

// 在事务内调用BatchRecordTransactions
tb_InventoryTransactionController<tb_InventoryTransaction> tranController = 
    _appContext.GetRequiredService<tb_InventoryTransactionController<tb_InventoryTransaction>>();
await tranController.BatchRecordTransactions(transactionList);  // ⚠️ 问题点

// ... 更多业务逻辑 ...

_unitOfWorkManage.CommitTran();  // 提交事务
```

### 4. 死锁场景分析

#### 场景1：并发审核单据

```
时间线：

T0: 事务A开始（销售出库单审核）
    _unitOfWorkManage.BeginTran()

T1: 事务A更新库存表
    UPDATE tb_Inventory SET Quantity = Quantity - 10 WHERE ...

T2: 事务B开始（调拨单审核）
    _unitOfWorkManage.BeginTran()

T3: 事务B更新库存表（同一产品）
    UPDATE tb_Inventory SET Quantity = Quantity + 5 WHERE ...
    ⚠️ 被事务A阻塞，等待锁

T4: 事务A插入库存流水
    INSERT INTO tb_InventoryTransaction ...
    ⚠️ 等待库存表的排他锁（但事务B持有）

T5: 死锁形成！
    - 事务A等待事务B释放锁
    - 事务B等待事务A释放锁
    - 数据库检测到死锁，回滚其中一个事务
```

#### 场景2：长事务持有锁

```csharp
// ❌ 问题代码示例（tb_SaleOutControllerPartial.cs:483-1007）
```

## 🔧 修复方案

### 方案1：修复BatchRecordTransactions方法（核心修复）

**目标**：确保库存流水插入在当前事务上下文中执行

```csharp
/// <summary>
/// 批量记录库存流水（修复版）
/// </summary>
public async Task<bool> BatchRecordTransactions(List<tb_InventoryTransaction> transactions)
{
    if (transactions == null || !transactions.Any())
        return false;

    // 过滤掉没有实际库存变化的记录
    var validTransactions = transactions.Where(t => t.QuantityChange != 0).ToList();
    if (!validTransactions.Any())
        return false;

    // 设置交易时间
    var now = DateTime.Now;
    foreach (var tran in validTransactions)
    {
        tran.TransactionTime = now;
    }

    // ✅ 修复：确保在当前事务上下文中执行
    var dbClient = _unitOfWorkManage.GetDbClient();
    
    try
    {
        // 使用AsTable指定表名，确保正确路由
        var rs = await dbClient
            .Insertable(validTransactions)
            .With(SqlWith.NoLock)  // 添加WITH(NOLOCK)提示（可选，谨慎使用）
            .ExecuteReturnSnowflakeIdListAsync();
        
        _logger?.LogDebug($"成功插入{rs.Count}条库存流水记录");
        return rs.Count > 0;
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, $"插入库存流水失败，记录数：{validTransactions.Count}");
        throw;
    }
}
```

### 方案2：添加事务上下文传递机制

**目标**：明确区分事务内和非事务场景

```csharp
/// <summary>
/// 批量记录库存流水（带事务参数版本）
/// </summary>
/// <param name="transactions">库存流水列表</param>
/// <param name="useTransaction">是否使用当前事务（默认true）</param>
public async Task<bool> BatchRecordTransactions(
    List<tb_InventoryTransaction> transactions, 
    bool useTransaction = true)
{
    if (transactions == null || !transactions.Any())
        return false;

    var validTransactions = transactions.Where(t => t.QuantityChange != 0).ToList();
    if (!validTransactions.Any())
        return false;

    var now = DateTime.Now;
    foreach (var tran in validTransactions)
    {
        tran.TransactionTime = now;
    }

    var dbClient = _unitOfWorkManage.GetDbClient();
    
    // 检查当前事务状态
    var transactionState = _unitOfWorkManage.GetTransactionState();
    bool inTransaction = transactionState.IsActive;
    
    // 如果要求使用事务但当前没有事务，则开启新事务
    if (useTransaction && !inTransaction)
    {
        _unitOfWorkManage.BeginTran();
        try
        {
            var rs = await dbClient
                .Insertable(validTransactions)
                .ExecuteReturnSnowflakeIdListAsync();
            _unitOfWorkManage.CommitTran();
            return rs.Count > 0;
        }
        catch
        {
            _unitOfWorkManage.RollbackTran();
            throw;
        }
    }
    else
    {
        // 在当前事务中执行，或直接执行（不使用事务）
        var rs = await dbClient
            .Insertable(validTransactions)
            .ExecuteReturnSnowflakeIdListAsync();
        return rs.Count > 0;
    }
}
```

### 方案4：添加死锁重试机制

**目标**：捕获死锁异常并自动重试

```csharp
/// <summary>
/// 带重试机制的库存流水记录
/// </summary>
public async Task<bool> BatchRecordTransactionsWithRetry(
    List<tb_InventoryTransaction> transactions,
    int maxRetries = 3,
    int retryDelayMs = 100)
{
    int retryCount = 0;
    
    while (retryCount < maxRetries)
    {
        try
        {
            return await BatchRecordTransactions(transactions);
        }
        catch (Exception ex)
        {
            retryCount++;
            
            // 检查是否为死锁错误
            bool isDeadlock = ex.Message.Contains("deadlock") || 
                              ex.Message.Contains("1205") ||
                              ex.Message.Contains("锁");
            
            if (!isDeadlock || retryCount >= maxRetries)
            {
                // 不是死锁或已达到最大重试次数，重新抛出异常
                _logger?.LogError(ex, 
                    $"插入库存流水失败，重试次数：{retryCount}/{maxRetries}");
                throw;
            }
            
            // 等待后重试
            _logger?.LogWarning(
                $"检测到死锁，第{retryCount}次重试，等待{retryDelayMs}ms...");
            await Task.Delay(retryDelayMs);
            
            // 指数退避
            retryDelayMs *= 2;
        }
    }
    
    return false;
}
```

### 方案5：优化索引和表结构

**目标**：减少锁争用，提高查询性能

```sql
-- 检查当前索引
EXEC sp_helpindex 'tb_InventoryTransaction';

-- 推荐的索引优化
-- 1. ProdDetailID + Location_ID 联合索引（最常用）
CREATE NONCLUSTERED INDEX IX_tb_InventoryTransaction_Prod_Location
ON tb_InventoryTransaction(ProdDetailID, Location_ID)
INCLUDE (TransactionTime, QuantityChange, AfterQuantity);

-- 2. TransactionTime 索引（按时间查询）
CREATE NONCLUSTERED INDEX IX_tb_InventoryTransaction_TransactionTime
ON tb_InventoryTransaction(TransactionTime)
INCLUDE (ProdDetailID, Location_ID, BizType);

-- 3. BizType + ReferenceId 联合索引（按业务单据查询）
CREATE NONCLUSTERED INDEX IX_tb_InventoryTransaction_BizType_ReferenceId
ON tb_InventoryTransaction(BizType, ReferenceId)
INCLUDE (TransactionTime, QuantityChange);

-- 4. ProdDetailID + TransactionTime 覆盖索引（库存查询优化）
CREATE NONCLUSTERED INDEX IX_tb_InventoryTransaction_Prod_Time
ON tb_InventoryTransaction(ProdDetailID, TransactionTime DESC)
INCLUDE (AfterQuantity, QuantityChange);
```
