RUINOR ERP 功能模块详细介绍

RUINOR ERP系统采用模块化设计，每个模块专注于特定的业务领域，通过模块间的协作实现企业资源的全面管理。

1. 生产制造模块

技术实现

核心控制器：BOMController, ManufacturingOrderController, ProductionPlanController

数据模型：tb_BOM_S, tb_BOM_SDetail, tb_ManufacturingOrder, tb_ProductionPlan

主要功能

配方管理 (BOM管理)

  • 标准物料清单(BOM)的定义和维护
  • BOM版本控制和历史记录
  • 物料替代关系的管理
  • BOM结构的多级展开和查询

生产计划管理

  • 主生产计划(MPS)制定
  • 物料需求计划(MRP)计算
  • 产能计划和负荷分析
  • 生产进度跟踪和调整

生产订单管理

  • 生产订单的创建和下达
  • 工单派工和进度跟踪
  • 生产过程的实时监控
  • 完工入库和质量检验

成本管理

  • 材料成本核算
  • 人工成本计算
  • 制造费用分摊
  • 成本差异分析

业务流程

销售订单 → MRP计算 → 生产计划 → 生产订单 → 领料 → 生产 → 检验 → 入库

2. 供应链管理模块

技术实现

核心控制器：PurOrderController, SaleOrderController, InventoryController, StockInOutController

数据模型：tb_PurOrder, tb_SaleOrder, tb_Inventory, tb_StockIn, tb_StockOut

主要功能

采购管理

  • 供应商管理和评估
  • 采购计划和预算控制
  • 采购订单的创建和跟踪
  • 到货验收和入库处理

销售管理

  • 客户信息管理
  • 销售报价和订单处理
  • 销售合同的签订和执行
  • 销售发货和发票管理

库存管理

  • 库存实时查询和预警
  • 安全库存设置和监控
  • 库存盘点和差异处理
  • 库存调拨和转移

出入库管理

  • 采购入库管理
  • 销售出库管理
  • 生产领料和退料
  • 其他出入库业务

业务流程

采购需求 → 采购订单 → 到货验收 → 采购入库 → 库存管理 → 销售订单 → 销售出库

3. 售后服务模块

技术实现

核心控制器：AfterSaleController, RepairOrderController, AssetDisposalController

数据模型：tb_AS_AfterSaleApply, tb_AS_RepairOrder, tb_AS_RepairMaterialPickup

主要功能

售后申请管理

  • 售后申请的受理和登记
  • 服务类型和优先级的设定
  • 服务人员的指派和调度
  • 服务进度的跟踪和反馈

维修订单管理

  • 维修订单的创建和分配
  • 维修过程和结果的记录
  • 维修费用的核算和结算
  • 维修质量的评估和改进

资产处置管理

  • 固定资产的报废申请
  • 资产处置流程的审批
  • 处置结果的记录和归档
  • 资产残值的核算

提前交付管理

  • 提前交付申请的审批
  • 交付计划的调整和优化
  • 交付过程的监控和协调
  • 交付结果的评估和反馈

业务流程

售后申请 → 服务指派 → 维修处理 → 费用结算 → 客户反馈 → 服务改进

4. 审批工作流模块

技术实现

核心控制器：ApprovalController, WorkflowController

数据模型：tb_Approval, tb_WorkflowDefinition, tb_WorkflowInstance

技术框架：WorkflowCore工作流引擎

主要功能

工作流定义

  • 业务流程的可视化设计
  • 审批节点的配置和管理
  • 条件分支和并行处理
  • 超时处理和异常处理

多级审批

  • 审批级别的灵活配置
  • 审批人的动态指定
  • 审批意见的记录和跟踪
  • 审批流程的实时监控

条件审批

  • 基于金额的审批条件
  • 基于业务类型的审批规则
  • 审批流程的动态调整
  • 审批结果的自动处理

流程监控

  • 流程进度的实时跟踪
  • 审批时效的统计分析
  • 流程异常的预警和处理
  • 流程效率的优化建议

业务流程

业务申请 → 工作流启动 → 审批节点 → 条件判断 → 审批处理 → 流程结束

5. 财务管理模块

技术实现

核心控制器：FinancialController, ReceivableController, PayableController, ExpenseController

数据模型：tb_Receivable, tb_Payable, tb_Expense, tb_FixedAssets

主要功能

应收应付管理

  • 应收账款的管理和催收
  • 应付账款的核对和支付
  • 账龄分析和风险评估
  • 坏账准备的计提和处理

费用管理

  • 费用预算的制定和控制
  • 费用报销的申请和审批
  • 费用分摊和核算
  • 费用分析和优化

固定资产管理

  • 固定资产的登记和分类
  • 折旧计算和计提
  • 资产盘点和清查
  • 资产处置和转移

发票管理

  • 发票的开具和登记
  • 发票的认证和抵扣
  • 发票的统计和分析
  • 发票的归档和查询

业务流程

业务发生 → 财务凭证 → 账务处理 → 财务报表 → 财务分析 → 决策支持

模块间协作关系

模块名称 主要协作模块 协作内容
生产制造模块 供应链管理、财务管理 物料需求传递、成本核算
供应链管理模块 生产制造、财务管理 采购计划、库存管理、应收应付
售后服务模块 供应链管理、审批工作流 维修领料、服务审批
审批工作流模块 所有业务模块 业务流程审批控制
财务管理模块 所有业务模块 财务核算和分析

模块扩展机制

插件化架构

RUINOR ERP采用插件化架构设计，支持以下扩展方式：

  • 功能模块扩展：通过标准化接口实现新功能模块的集成
  • 业务流程扩展：支持自定义业务流程的定义和执行
  • 报表扩展：提供报表模板和数据分析工具的扩展
  • 接口扩展：支持与第三方系统的接口集成

扩展开发指南

系统提供完善的扩展开发文档和API接口，支持开发者进行二次开发和定制化改造。
