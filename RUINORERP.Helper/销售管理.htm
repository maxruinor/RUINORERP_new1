销售管理

销售管理模块用于管理企业的销售业务流程，包括客户管理、销售订单、销售出库等。

功能概述

销售管理主要包括客户档案管理、销售报价、销售订单、销售出库、销售退货等业务。

客户管理

  • 维护客户基本信息
  • 管理客户联系人信息
  • 记录客户交易历史
  • 分析客户价值和信用状况

销售订单管理

  • 创建和维护销售订单
  • 跟踪订单执行状态
  • 处理订单变更
  • 订单审批流程

销售出库管理

  • 根据销售订单生成出库单
  • 记录实际发货信息
  • 处理部分发货和超量发货
  • 发货确认和物流跟踪

操作流程

销售订单流程

  1. 选择客户并创建销售订单
  2. 填写销售明细信息
  3. 提交订单审批
  4. 审批通过后订单生效
  5. 备货完成后创建销售出库单
  6. 货物发出后确认出库

销售订单审核流程

销售订单创建后需要经过审核流程才能生效：

  1. 销售订单创建完成后，状态为"未审核"
  2. 具有审核权限的用户可以对订单进行审核操作
  3. 审核时需要填写审核意见，选择"同意"或"拒绝"
  4. 审核同意后，订单状态更新为"已审核"，可以进行后续出库操作
  5. 审核拒绝后，订单状态保持"未审核"，需要修改后重新提交审核

销售订单反审核流程

对于已审核但尚未结案的销售订单，可以执行反审核操作：

  1. 选择需要反审核的已审核销售订单
  2. 点击"反审核"按钮，系统会提示确认操作
  3. 确认后，订单状态从"已审核"回退到"未审核"状态
  4. 反审核后的订单可以进行修改操作，修改后需要重新提交审核

销售订单结案流程

销售订单在完成所有出库操作后可以进行结案：

  1. 确保销售订单已审核且所有货物已出库
  2. 选择需要结案的销售订单
  3. 点击"结案"按钮，系统会提示确认操作
  4. 确认后，订单状态更新为"已结案"，不能再进行修改操作

销售退货流程

  1. 客户提出退货申请
  2. 创建销售退货单
  3. 提交退货申请审批
  4. 审批通过后执行退货
  5. 记录退货信息并更新库存

销售退货审核流程

销售退货单创建后需要经过审核流程才能生效：

  5. 审核拒绝后，退货单状态保持"未审核"，需要修改后重新提交审核

销售退货反审核流程

  4. 反审核后的退货单可以进行修改操作，修改后需要重新提交审核

关键字段说明

字段名 说明
销售订单号 销售订单的唯一标识号
客户 货物购买方的名称
销售日期 销售订单创建的日期
要求发货日期 客户要求发货的日期
销售数量 计划销售的货物数量
单价 货物的销售单价
金额 该项货物的总金额
已出库数量 已实际出库的货物数量
