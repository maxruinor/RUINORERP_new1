# 产品编号重复问题 - 根本原因与解决方案

## 📋 问题现象

用户在创建新产品时,系统提示**产品编号重复**,需要重新操作,体验很差。

## 🔍 根本原因分析

### 原因1: 服务器重启导致内存缓存丢失

**位置**: `DatabaseSequenceService._batchCaches`

```csharp
// 内存中的批次缓存,服务器重启后完全丢失!
private readonly ConcurrentDictionary<string, SequenceBatchCache> _batchCaches;
```

**重现场景**:
```
第1次启动:
1. 生成车载产品 → 从数据库获取批次 [1-50]
2. 使用序号 1, 2, 3 (生成 CZ000001, CZ000002, CZ000003)
3. 数据库 CurrentValue = 50
4. 内存缓存: SEQ_CZ → (3, 50)

⚠️ 服务器重启! 内存缓存清空!

第2次启动:
1. 再次生成车载产品
2. 如果序列键一致(SEQ_CZ) → 从数据库读取 CurrentValue=50,分配新批次 [51-100] ✅
3. 但如果序列键不一致(例如之前是 SEQ_CZ_NONE,现在是 SEQ_CZ) → 创建新记录 ❌
   - 新记录 CurrentValue = 50
   - 返回序号 1
   - 生成 CZ000001 ← **重复!**
```

### 原因2: 序列键可能不一致

虽然代码中使用了简单的类目代码(如"CZ"),理论上应该稳定,但可能存在以下情况:

1. **历史数据问题**: 数据库中可能已经存在多个不同格式的序列键
   - `SEQ_CZ`
   - `SEQ_CZ_NONE`
   - `SEQ_车载设备` (中文类目名)
   
2. **规则配置变化**: 如果曾经修改过编号规则,可能导致生成了不同的序列键

3. **并发竞争**: 虽然使用了锁,但如果两个请求同时发现记录不存在,可能都会尝试插入

### 原因3: 数据库唯一约束导致用户体验差

当发生重复时:
1. 前端调用 `GenerateProductRelatedCodeAsync` 生成编号
2. 后端返回编号(例如 CZ000001)
3. 用户填写其他信息后保存
4. **数据库唯一约束检查失败** → 抛出异常
5. 用户看到错误提示,需要重新操作

**问题**: 编号生成时没有实时检查唯一性,等到保存时才发现问题!

## ✅ 解决方案

### 方案1: 增强序列键稳定性(已实施)

**文件**: `RUINORERP.Business\BNR\DatabaseSequenceParameter.cs`

**修复内容**:
- 只提取静态标识符作为序列键
- 跳过日期处理器,避免动态日期影响键的稳定性
- 添加详细的调试日志

**效果**: 确保相同业务始终使用相同的序列键

### 方案2: 添加编号唯一性预检查

**新增方法**: 在生成编号后立即检查是否已存在

```csharp
/// <summary>
/// 生成产品编号并验证唯一性
/// </summary>
public async Task<string> GenerateUniqueProdNoAsync(tb_Prod prod)
{
    int maxRetries = 3;
    for (int i = 0; i < maxRetries; i++)
    {
        // 生成编号
        string prodNo = GenerateProdNoAsync(prod);
        
        // 检查是否已存在
        bool exists = await _db.Queryable<tb_Prod>()
            .AnyAsync(p => p.ProductNo == prodNo);
        
        if (!exists)
        {
            return prodNo; // 唯一,返回
        }
        
        // 如果重复,记录日志并重试
        _logger.LogWarning($"产品编号 {prodNo} 已存在,重试 ({i + 1}/{maxRetries})");
    }
    
    // 重试多次后仍然重复,使用时间戳确保唯一性
    string fallbackNo = $"PROD-{DateTime.Now:yyyyMMddHHmmss}-{new Random().Next(100, 999)}";
    _logger.LogError($"生成产品编号多次重试后仍重复,使用备用编号: {fallbackNo}");
    return fallbackNo;
}
```

### 方案3: 清理数据库中的重复序列键

**执行诊断脚本**: `产品编号重复诊断.sql`

**清理步骤**:
1. 运行诊断脚本,找出重复的序列键
2. 确定哪个是正确的序列键(通常是最近更新的那个)
3. 删除多余的序列键记录
4. 重置正确序列键的 CurrentValue 为最大值+1

**示例SQL**:
```sql
-- 假设发现 SEQ_CZ 和 SEQ_CZ_NONE 都存在
-- 保留 SEQ_CZ,删除 SEQ_CZ_NONE

-- 1. 备份
SELECT * INTO SequenceNumbers_Backup_20260429 
FROM SequenceNumbers 
WHERE SequenceKey IN ('SEQ_CZ', 'SEQ_CZ_NONE');

-- 2. 获取最大值
DECLARE @MaxValue BIGINT;
SELECT @MaxValue = MAX(CurrentValue) 
FROM SequenceNumbers 
WHERE SequenceKey IN ('SEQ_CZ', 'SEQ_CZ_NONE');

-- 3. 删除多余的键
DELETE FROM SequenceNumbers 
WHERE SequenceKey = 'SEQ_CZ_NONE';

-- 4. 更新保留的键
UPDATE SequenceNumbers 
SET CurrentValue = @MaxValue,
    LastUpdated = GETDATE()
WHERE SequenceKey = 'SEQ_CZ';
```

### 方案4: 优化批次大小配置(✅ 已实施)

**当前配置**: `BATCH_SIZE = 10` (从50优化为10)

**优化原因**:
- 同一业务类型同时操作的用户很少(不超过10个)
- 减小批次可以避免服务器重启时浪费序号
- 性能影响小,数据库访问频率仍然很低

**效果对比**:
| 批次大小 | 重启前使用3个 | 重启后浪费 | 数据库访问频率 |
|---------|-------------|-----------|--------------|
| 50      | 1,2,3       | 47个      | 低           |
| 10      | 1,2,3       | 7个       | 略高(但仍可接受) |

**示例**:
```
第1次启动: 获取批次 [1-10], 使用 1,2,3
服务器重启!
第2次启动: 从数据库读取 CurrentValue=10, 获取新批次 [11-20]
结果: 只浪费了 4-10 共7个序号 (之前会浪费47个)
```

### 方案5: 前端优化 - 实时显示生成的编号

**改进用户体验**:

### 步骤3: 添加唯一性预检查

```csharp
public async Task<string> GenerateProdNoAsync(tb_Prod prod)
{
    try
    {
        string categoryCode = GetCategoryCode(prod);
        string rule = $"{{DB:{categoryCode}/000000}}";
        string productSequence = _bnrFactory.Create(rule);
        string prodType = GetProdType(prod);
        string prodNo = $"{categoryCode}{productSequence}-{prodType}";
        
        // ✅ 新增: 检查唯一性
        bool exists = await _db.Queryable<tb_Prod>()
            .AnyAsync(p => p.ProductNo == prodNo);
        
        if (exists)
        {
            _logger.LogWarning($"产品编号 {prodNo} 已存在,生成备用编号");
            // 重试一次
            productSequence = _bnrFactory.Create(rule);
            prodNo = $"{categoryCode}{productSequence}-{prodType}";
        }
        
        return prodNo;
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "生成产品编号失败");
        return $"PROD-{DateTime.Now:yyyyMMddHHmmss}-{new Random().Next(100, 999)}";
    }
}
```
