### 1. ConfigExtensions.cs

**位置**: `RUINORERP.Server/Workflow/ConfigExtensions.cs`

**新增内容**:
```csharp
// 1. 配置模型
public enum WorkflowPersistenceType
{
    Memory,     // 内存持久化（默认）
    SqlServer,   // SQL Server 持久化
    SQLite      // SQLite 文件持久化
}

public class WorkflowCoreOptions
{
    public WorkflowPersistenceType PersistenceType { get; set; } = WorkflowPersistenceType.Memory;
    public string ConnectionString { get; set; }
    public bool AutoCreateSchema { get; set; } = true;
    public bool AutoMigrateSchema { get; set; } = true;
}

// 2. 主配置方法
public static void AddWorkflowCoreServices(this IServiceCollection services, IConfiguration configuration)
{
    var options = configuration.GetSection("WorkflowCore").Get<WorkflowCoreOptions>() ?? new WorkflowCoreOptions();
    
    ConfigureWorkflowPersistence(services, options);
    ConfigureWorkflowDSL(services);
    RegisterWorkflowSteps(services);
}

// 3. 持久化配置路由
private static void ConfigureWorkflowPersistence(IServiceCollection services, WorkflowCoreOptions options)
{
    try
    {
        switch (options.PersistenceType)
        {
            case WorkflowPersistenceType.SqlServer:
                ConfigureSqlServerPersistence(services, options);
                break;
            case WorkflowPersistenceType.SQLite:
                ConfigureSQLitePersistence(services, options);
                break;
            case WorkflowPersistenceType.Memory:
            default:
                ConfigureMemoryPersistenceCServices);
                break;
        }
    }
    catch (Exception ex)
    {
        try
        {
            var logger = services.BuildServiceProvider().GetService<ILogger<ConfigExtensions>>();
            logger?.LogError(ex, "配置工作流持久化失败，降级到内存模式。类型: {PersistenceType}", options.PersistenceType);
        }
        catch
        {
            System.Diagnostics.Debug.WriteLine($"配置工作流持久化失败：{ex.GetType().Name} - {ex.Message}");
        }
        
        ConfigureMemoryPersistence(services);
    }
}
```

**改进**:
- ✅ 移除了 MySQL 和 MongoDB 支持（简化为三种）
- ✅ 添加了完整的异常处理和降级机制
- ✅ 移除了不需要的参数（DatabaseName）
- ✅ 代码结构清晰，使用 Region 分组
- ✅ 保留了旧的兼容方法并标记为 Obsolete

## ✨ 核心功能

### 1. 三种持久化策略

| 策略 | 说明 | NuGet 包 | 适用场景 |
|------|------|----------|----------|
| **Memory** | 内存存储，快速启动 | 开发环境、单元测试 |
| **Sql Server** | SQL Server 存储，状态持久化 | 生产环境（推荐） |
| **SQLite** | SQLite 文件存储，轻量级 | 嵌入式系统、小型应用 |

### 2. 配置灵活性

```json
// 开发环境 - 无需数据库配置
{
  "WorkflowCore": {
    "PersistenceType": "Memory"
  }
}

// 生产环境 - SQL Server 持久化
{
  "WorkflowCore": {
    "PersistenceType": "SqlServer",
    "ConnectionString": "Server=.;Database=WorkflowCore;Trusted_Connection=True;",
    "AutoCreateSchema": true,
    "AutoMigrateSchema": true
  }
}

// 嵌入式系统 - SQLite 持久化
{
  "WorkflowCore": {
    "PersistenceType": "SQLite",
    "ConnectionString": "Data Source=workflow.db;",
    "AutoCreateSchema": true
  }
}
```

### 3. 容错降级机制

```
配置工作流持久化失败

    ↓

    ↓
确保应用正常启动
```

## 🔍 完整工作流设计理解

### 1. 外部事件驱动架构

```
┌─────────────┐     ┌─────────────┐
│   业务系统  │────▶│  事件总线  │────▶│ 工作流引擎 │
└─────────────┘     └─────────────┘     └─────────────┘
   │                                           │
   └───────────发布事件───────────┘
                    ▲
              通过事件总线解耦
```

**优势**:
- ✅ 工作流不直接依赖业务逻辑
- ✅ 可以独立测试工作流逻辑
- ✅ 支持跨系统的事件驱动

### 2. 工作流与业务解耦

**正确做法**:
- 工作流只负责流程编排
- 业务逻辑在独立的步骤类或服务中实现
- 通过依赖注入而不是静态访问

**错误做法（已修复）**:
- 工作流步骤直接访问全局 UI（`frmMainNew.Instance`）
- 工作流步骤直接操作业务数据（未通过服务抽象）

### 3. 类型安全的空值处理

**修复前**:
```csharp
// 可能导致 NullReferenceException
data.approvalEntity.BillID.ToString()
```

**修复后**:
```csharp
// 安全的空值处理
data.approvalEntity?.BillID.ToString() ?? ""
```

## 🧪 使用示例

### 场景1: 开发环境使用内存持久化

**配置**:
```json
{
  "WorkflowCore": {
    "PersistenceType": "Memory"
  }
}
```

**启动**:
```bash
dotnet run
```

**预期结果**:
- ✅ 应用快速启动
- ✅ 无需工作流数据库
- ✅ 工作流在内存中运行
- ⚠️ 服务重启后工作流状态丢失

### 场景2: 生产环境使用 SQL Server 持久化

**安装必要包**:
```bash
dotnet add package WorkflowCore.Persistence.SqlServer
```

**配置**:
```json
{
  "ConnectionStrings": {
    "WorkflowCore": "Server=prod-db;Database=WorkflowCore;User Id=sa;Password=xxx;"
  },
  
  "WorkflowCore": {
    "PersistenceType": "SqlServer",
    "ConnectionString": "Server=prod-db;Database=WorkflowCore;User Id=sa;Password=xxx;",
    "AutoCreateSchema": true,
    "AutoMigrateSchema": true
  }
}
```

**启动**:
```bash
dotnet run
```

**预期结果**:
- ✅ 应用正常启动
- ✅ SQL Server 中自动创建 `WorkflowCore` 数据库
- ✅ 包含 5 个工作流表
- ✅ 工作流状态持久化到数据库
- ✅ 服务重启后工作流继续执行

### 场景3: 切换持久化模式

**从内存切换到 SQL Server**:
1. 安装 SQL Server 持久化包
2. 修改 `appsettings.json` 配置
3. 重启应用

**从 SQL Server 切换到内存**:
1. 修改 `appsettings.json` 配置
2. 重启应用（工作流状态丢失是预期行为）
