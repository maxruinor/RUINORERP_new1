## ✨ 核心特性

### 1. 多种持久化策略支持

| 策略 | 说明 | 需配置 |
|------|------|----------|
| **Memory** | 内存存储，开发环境推荐 | ❌ |
| **SqlServer** | SQL Server 存储，生产环境推荐 | ✅ ConnectionString |
| **MySQL** | MySQL 存储，跨平台部署 | ✅ ConnectionString |
| **MongoDB** | MongoDB 存储，文档数据库 | ⚠️ ConnectionString + DatabaseName |
| **SQLite** | SQLite 存储，轻量级 | ⚠️ ConnectionString（文件路径） |

### 2. 容错降级机制

当持久化配置失败时：
- 自动记录错误日志
- 降级到内存持久化
- 确保应用正常启动

**日志示例**:
```
配置工作流持久化失败，降级到内存模式。类型: SqlServer
```

### 3. 向后兼容

- 旧方法标记为 `[Obsolete]` 但保留可用
- 现有代码无需立即修改
- 编译时会有警告，提醒升级

## 📊 配置参数验证

### 参数有效性检查

| 参数 | SQL Server | MySQL | MongoDB | SQLite | Memory |
|------|-----------|-------|---------|---------|---------|
| `PersistenceType` | ✅ 必需 | ✅ 必需 | ✅ 必需 | ✅ 必需 | ✅ 必需 |
| `ConnectionString` | ✅ 必需 | ✅ 必需 | ⚠️ 可选（有默认值）| ⚠️ 可选（有默认值）| ❌ 忽略 |
| `AutoCreateSchema` | ✅ 可选（默认 true）| ✅ 可选（默认 true）| ❌ 忽略 | ✅ 可选（默认 true）| ❌ 忽略 |
| `AutoMigrateSchema` | ✅ 可选（默认 true）| ✅ 可选（默认 true）| ❌ 忽略 | ❌ 忽略 | ❌ 忽略 |
| `DatabaseName` | ❌ 忽略 | ❌ 忽略 | ✅ 可选（默认 "workflow"）| ❌ 忽略 | ❌ 忽略 |

### 配置验证规则

1. **SQL Server / MySQL**:
   - `ConnectionString` 不能为空或空白
   - 空值会抛出 `InvalidOperationException`

2. **MongoDB**:
   - `ConnectionString` 默认为 `mongodb://localhost:27017`
   - `DatabaseName` 默认为 `workflow`

3. **SQLite**:
   - `ConnectionString` 默认为 `database.db`
   - 支持相对路径和绝对路径

4. **Memory**:
   - 无需任何额外配置
   - 所有参数被忽略

## 🔒 安全注意事项

### 1. 连接字符串安全

⚠️ **不要在配置文件中明文存储密码**

**推荐做法**:
- 使用 Windows 集成安全性（`Trusted_Connection=True`）
- 从环境变量读取敏感信息
- 使用加密的配置提供程序

### 2. 数据库用户权限

**最小权限原则**:
- 只授予必要的 CRUD 权限
- 不授予数据库管理权限
- 使用只读用户（如果适用）

### 3. 生产环境配置

**建议设置**:
```json
{
  "WorkflowCore": {
    "PersistenceType": "SqlServer",
    "ConnectionString": "...",
    "AutoCreateSchema": false,
    "AutoMigrateSchema": false
  }
}
```

**原因**:
- 生产环境禁用自动创建和迁移
- 由 DBA 手动管理数据库结构
- 避免意外的表结构变更
