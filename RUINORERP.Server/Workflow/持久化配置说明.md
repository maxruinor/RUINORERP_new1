# WorkflowCore 持久化配置说明

## 概述

本项目的工作流系统（WorkflowCore）支持三种持久化策略，可通过 `appsettings.json` 配置文件进行灵活切换。

**支持的持久化类型**：
- **Memory** - 内存持久化（开发环境推荐）
- **Sql Server** - SQL Server 持久化（生产环境推荐）
- **SQLite** - SQLite 文件持久化（轻量级应用）

---

## 配置文件位置

`RUINORERP.Server/appsettings.json`

---

## 配置结构

```json
{
  "ConnectionStrings": {
    "WorkflowCore": "Server=.;Database=WorkflowCore;Trusted_Connection=True;"
  },

  "WorkflowCore": {
    "PersistenceType": "Memory",
    "ConnectionString": "Server=.;Database=WorkflowCore;Trusted_Connection=True;",
    "AutoCreateSchema": true,
    "AutoMigrateSchema": true
  }
}
```

---

## 配置参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|----------|------|
| `PersistenceType` | 枚举 | `Memory` | 持久化类型：`Memory`、`SqlServer`、`SQLite` |
| `ConnectionString` | string | `null` | 数据库连接字符串（SQL Server 必需配置，SQLite 可选） |
| `AutoCreateSchema` | bool | `true` | 自动创建工作流表结构 |
| `AutoMigrateSchema` | bool | `true` | 自动迁移工作流表结构 |

---

## 配置示例

### 示例1：内存持久化（开发环境推荐）

**适用场景**：
- 开发环境快速启动
- 单元测试
- 工作流无需跨重启保存状态

```json
{
  "WorkflowCore": {
    "PersistenceType": "Memory"
  }
}
```

**特点**：
- ✅ 启动速度快
- ✅ 无需数据库配置
- ✅ 无需额外 NuGet 包
- ❌ 服务重启后所有工作流状态丢失
- ❌ 无法追踪历史执行记录

---

### 示例2：SQL Server 持久化（生产环境推荐）

**适用场景**：
- 生产环境
- 需要持久化工作流状态
- 服务重启后继续执行工作流

**安装必要包**：
```bash
dotnet add package WorkflowCore.Persistence.SqlServer
```

```json
{
  "ConnectionStrings": {
    "WorkflowCore": "Server=.;Database=WorkflowCore;Trusted_Connection=True;"
  },
  
  "WorkflowCore": {
    "PersistenceType": "SqlServer",
    "ConnectionString": "Server=.;Database=WorkflowCore;Trusted_Connection=True;",
    "AutoCreateSchema": true,
    "AutoMigrateSchema": true
  }
}
```

**连接字符串示例**：

```csharp
// Windows 认证（推荐）
Server=.;Database=WorkflowCore;Trusted_Connection=True;

// SQL Server 认证
Server=192.168.1.100;Database=WorkflowCore;User Id=sa;Password=YourPassword;

// 带其他参数
Server=.;Database=WorkflowCore;Trusted_Connection=True;Pooling=true;Min Pool Size=5;Max Pool Size=20;
```

**特点**：
- ✅ 工作流状态持久化
- ✅ 服务重启后继续执行
- ✅ 可追踪历史执行记录
- ✅ 支持高并发和事务
- ⚠️ 需要安装 `WorkflowCore.Persistence.SqlServer` 包

---

### 示例3：SQLite 持久化（轻量级应用）

**适用场景**：
- 单机部署
- 无需数据库服务器
- 轻量级应用
- 嵌入式系统

**安装必要包**：
```bash
dotnet add package WorkflowCore.Persistence.Sqlite
```

```json
{
  "WorkflowCore": {
    "PersistenceType": "SQLite",
    "ConnectionString": "Data Source=workflow.db;",
    "AutoCreateSchema": true
  }
}
```

**连接字符串示例**：

```csharp
// 当前目录（默认）
Data Source=workflow.db;

// 绝对路径
Data Source=C:\\Data\\workflow.db;

// 只读模式
Data Source=workflow.db;Read Only=true;

// 禁用连接池
Data Source=workflow.db;Pooling=false;
```

**特点**：
- ✅ 无需数据库服务器
- ✅ 单文件部署简单
- ✅ 跨平台支持
- ⚠️ 需要安装 `WorkflowCore.Persistence.Sqlite` 包
- ⚠️ 并发性能相对较低

---

## 数据库初始化

### SQL Server

当 `AutoCreateSchema: true` 时，首次运行会自动创建以下表：

- `WorkflowInstance` - 工作流实例
- `ExecutionPointer` - 执行指针
- `ExecutionError` - 执行错误
- `Event` - 事件
- `EventSubscription` - 事件订阅

### SQLite

会自动创建 `workflow.db` 文件并包含必要的表结构。

## 配置切换指南

### 从内存模式切换到 SQL Server

1. 安装必要的 NuGet 包：
   ```bash
   dotnet add package WorkflowCore.Persistence.SqlServer
   ```

2. 修改 `appsettings.json`：

```json
{
  "ConnectionStrings": {
    "WorkflowCore": "Server=.;Database=WorkflowCore;Trusted_Connection=True;"
  },
  
  "WorkflowCore": {
    "PersistenceType": "SqlServer",
    "ConnectionString": "Server=.;Database=WorkflowCore;Trusted_Connection=True;",
    "AutoCreateSchema": true,
    "AutoMigrateSchema": true
  }
}
```

3. 重启应用程序

4. 确认数据库中已创建表结构

### 从 SQL Server 切换到内存模式

```json
{
  "WorkflowCore": {
    "PersistenceType": "Memory"
  }
}
```

重启后所有工作流状态将丢失（这是预期行为）。

### 切换到 SQLite 模式

1. 安装必要的 NuGet 包：
   ```bash
   dotnet add package WorkflowCore.Persistence.Sqlite
   ```

2. 修改 `appsettings.json`：

```json
{
  "WorkflowCore": {
    "PersistenceType": "SQLite",
    "ConnectionString": "Data Source=workflow.db;",
    "AutoCreateSchema": true
  }
}
```

3. 重启应用程序

## 故障排查

### 问题1：SQL Server 连接失败

**错误信息**：
```
InvalidOperationException: SQL Server 持久化模式需要配置 ConnectionString
```

**解决方案**：
1. 检查 `ConnectionString` 配置是否正确
2. 确认 SQL Server 服务正在运行
3. 验证网络连接和防火墙设置
4. 检查用户名和密码是否正确
5. 确认是否安装了 `WorkflowCore.Persistence.SqlServer` 包

### 问题2：SQLite 文件访问失败

**错误信息**：
```
IOException: 无法访问数据库文件
```

**解决方案**：
1. 检查文件路径是否正确
2. 确认应用程序有文件读写权限
3. 检查磁盘空间是否充足
4. 确认没有其他进程锁定文件

### 问题3：表已存在但结构不匹配

**解决方案**：
```json
{
  "WorkflowCore": {
    "AutoMigrateSchema": true
  }
}
```
设置 `AutoMigrateSchema: true` 会自动迁移表结构。

### 问题4：降级到内存模式

**日志信息**：
```
配置工作流持久化失败，降级到内存模式。类型: SqlServer
```

**原因**：持久化配置失败后，系统会自动降级到内存模式以确保应用能正常启动。

**解决方案**：
1. 检查日志中的详细错误信息
2. 验证 `ConnectionString` 配置
3. 确认目标数据库服务器可访问
4. 检查是否安装了正确的持久化包

## 性能优化建议

### SQL Server

```json
{
  "WorkflowCore": {
    "PersistenceType": "SqlServer",
    "ConnectionString": "Server=.;Database=WorkflowCore;Trusted_Connection=True;Pooling=true;Min Pool Size=10;Max Pool Size=100;Connection Timeout=30;",
    "AutoCreateSchema": false
  }
}
```

**优化参数**：
- **Pooling**: 启用连接池
- **Min Pool Size**: 最小连接数 10
- **Max Pool Size**: 最大连接数 100
- **Connection Timeout**: 连接超时 30 秒
- **AutoCreateSchema**: 生产环境设置为 false

### SQLite

```json
{
  "WorkflowCore": {
    "PersistenceType": "SQLite",
    "ConnectionString": "Data Source=workflow.db;Pooling=false;Journal Mode=WAL;",
    "AutoCreateSchema": true
  }
}
```

**优化参数**：
- **Pooling=false**: SQLite 建议禁用连接池
- **Journal Mode=WAL**: 使用 WAL 模式提高并发性能
- **AutoCreateSchema**: 生产环境设置为 false

## 安全建议

### 1. 不要在配置文件中明文存储密码

使用 Windows 集成安全性（Trusted_Connection）：

```json
{
  "WorkflowCore": {
    "PersistenceType": "SqlServer",
    "ConnectionString": "Server=.;Database=WorkflowCore;Trusted_Connection=True;"
  }
}
```

### 2. 限制数据库用户权限

创建专门的工作流数据库用户，只授予必要的权限：
- 表的 CRUD 权限
- 不授予数据库管理权限
- SQL Server 使用最小权限原则

### 3. SQLite 文件权限

确保 SQLite 数据库文件：
- 应用程序有读写权限
- 文件不在用户可编辑目录（防止误删）
- 定期备份数据库文件

## 持久化方案选择建议

| 场景 | 推荐方案 | 原因 |
|------|----------||------|
| 开发环境 | **Memory** | 快速启动，无需数据库 |
| 测试环境 | **Memory** 或 **SQLite** | 测试隔离，易于清理 |
| 生产环境（小型） | **SQL Server** | 性能好，状态持久化 |
| 生产环境（嵌入式） | **SQLite** | 部署简单，跨平台 |
| 分布式部署 | **SQL Server** | 支持集群和高并发 |

## 常见问题 FAQ

**Q: 为什么不支持 MySQL 和 MongoDB？**

A: 基于项目实际需求分析：
   - 项目主要使用 SQL Server 作为主数据库
   - MySQL 和 MongoDB 增加了部署复杂度和维护成本
   - 三种持久化方式（Memory、SqlServer、SQLite）已能满足 99% 的使用场景
   - 如未来需要，可以扩展支持

**Q: 生产环境推荐使用哪种持久化？**

A: 
   - **SQL Server**（推荐）：性能最好，支持高并发，与现有数据库一致
   - **SQLite**（备选）：部署简单，适合小型单机应用

**Q: 如何从内存模式迁移到持久化模式？**

A: 
   1. 备份现有工作流数据（内存模式无法持久化）
   2. 配置新的持久化方式
   3. 重启应用
   4. 新启动的工作流将自动持久化
   5. 旧的工作流状态会丢失（这是正常行为）
