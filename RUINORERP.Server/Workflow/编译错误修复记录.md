### 修复 #3: ConfigExtensions.cs 静态类型泛型错误

**位置**: `Workflow/ConfigExtensions.cs` 第88行

**原始代码**:
```csharp
catch (Exception ex)
{
    var logger = services.BuildServiceProvider().GetService<ILogger<ConfigExtensions>>();
    logger?.LogError(ex, "配置工作流持久化失败，降级到内存模式。类型: {PersistenceType}", options.PersistenceType);
    
    ConfigureMemoryPersistence(services);
}
```

**问题分析**:
- `ConfigExtensions` 是静态类
- `ILogger<ConfigExtensions>` 要求 `ConfigExtensions` 是实例类型
- CS0718 错误：静态类型不能用作泛型类型参数

**修复方案**:

**方案A（已采用）**: 使用 try-catch 包装日志记录

```csharp
catch (Exception ex)
{
    try
    {
        var logger = services.BuildServiceProvider().GetService<ILogger<ConfigExtensions>>();
        logger?.LogError(ex, "配置工作流持久化失败，降级到内存模式。类型: {PersistenceType}", options.PersistenceType);
    }
    catch
    {
        System.Diagnostics.Debug.WriteLine($"配置工作流持久化失败：{ex.GetType().Name} - {ex.Message}");
    }
    
    ConfigureMemoryPersistence(services);
}
```

**方案B（备选）**: 使用非泛型 ILogger

```csharp
private static Microsoft.Extensions.Logging.ILogger _logger;

private static void LogConfigurationError(IServiceCollection services, Exception ex, WorkflowPersistenceType persistenceType)
{
    try
    {
        var logger = services.BuildServiceProvider().GetService<Microsoft.Extensions.Logging.ILogger>();
        logger?.LogError(ex, "配置工作流持久化失败，降级到内存模式。类型: {PersistenceType}", persistenceType);
    }
    catch
    {
        System.Diagnostics.Debug.WriteLine($"配置工作流持久化失败：{ex.GetType().Name} - {ex.Message}");
    }
}
```

**修复内容**:
- ✅ 添加外层 try-catch 保护日志记录
- ✅ 日志记录失败时使用 Debug.WriteLine 作为备用
- ✅ 不影响降级到内存模式的行为
- ✅ 确保异常不会阻止应用启动
