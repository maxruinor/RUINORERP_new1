#### 2. 新增HandleSaveAsync方法

```csharp
/// <summary>
/// 异步处理保存操作
/// 正确等待保存完成后再更新UI状态，避免按钮状态错误恢复
/// </summary>
private async Task HandleSaveAsync()
{
    // 防止重复点击：立即禁用保存按钮
    var saveButton = toolStripButtonSave;
    if (saveButton != null)
    {
        saveButton.Enabled = false;
    }

    try
    {
        // ✅ 等待保存操作完成
        await Save(true);

        // 保存成功后，UI状态由Save方法内部的UpdateAllUIStates调用负责更新
        // 不需要在这里手动恢复按钮状态
    }
    catch (Exception ex)
    {
        logger?.LogError(ex, "保存操作失败");
        MainForm.Instance.uclog?.AddLog($"保存失败：{ex.Message}", UILogType.错误);

        // 保存失败时，根据当前实体状态恢复按钮状态
        if (saveButton != null && BoundEntity != null)
        {
            // 如果实体有未保存的更改，保持按钮启用
            saveButton.Enabled = BoundEntity.HasChanged;
        }
        else if (saveButton != null)
        {
            // 其他情况，禁用按钮
            saveButton.Enabled = false;
        }
    }
}
```

## 技术要点

### 1. 异步操作的正确处理

**错误做法**：
```csharp
_ = SomeAsyncMethod();  // 火即忘，无法获取完成状态
finally { /* 立即执行 */ }
```

**正确做法**：
```csharp
await SomeAsyncMethod();  // 等待完成后再继续
```

### 2. UI状态管理的职责分离

- **HandleSaveAsync**: 负责按钮的禁用和异常恢复
- **Save方法**: 负责实际保存逻辑和成功后的UI更新
- **UpdateAllUIStates**: 负责根据状态管理系统统一更新所有UI控件状态

### 状态流转

```
草稿 (新建/修改) → 保存成功 → 草稿 (已保存，HasChanged=false)
    ↓
    状态管理器判断：
    - HasChanged = false → 禁用保存按钮
    - 当前状态 = 草稿 → 启用提交、修改、删除等按钮
```

## 注意事项

1. **不要手动恢复保存按钮状态**
   - 保存成功后的按钮状态应由 `UpdateAllUIStates` 根据状态管理系统统一管理
   - 只有在异常情况下才需要手动恢复

2. **保持职责分离**
   - UI事件处理（DoButtonClick）只负责启动异步操作
   - 实际业务逻辑在具体的async方法中处理
   - UI状态更新由统一的UpdateAllUIStates负责

3. **异步操作的异常处理**
   - 使用try-catch捕获异常
   - 根据异常类型和业务需求决定是否恢复按钮状态
