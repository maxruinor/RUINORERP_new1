# RUINORERP 企业级报表中心

## 📊 项目简介

RUINORERP 企业级报表中心是一个基于现有 ChartFramework 架构重构的高性能、可扩展的数据可视化平台。旨在为运营团队及高层管理人员提供多维度的业务数据可视化支持 (如销售分析、库存监控、生产进度等),以辅助科学决策。

### 核心特性

- ✅ **分层架构**: Service 层 + Adapter 层分离，高内聚低耦合
- ✅ **动态查询**: 支持任意维度 + 指标组合的动态 SQL 构建
- ✅ **智能缓存**: 自动缓存热点数据，TTL 可配置
- ✅ **插件化设计**: 报表容器支持动态注册新报表类型
- ✅ **交互增强**: 钻取 (Drill-down)、联动过滤、右键菜单
- ✅ **性能优化**: 分层加载、虚拟滚动、GPU 加速渲染
- ✅ **中文友好**: 完整的中文字体支持和文档

### 使用示例

#### 方式 1: 使用统计服务 (简单场景)

```csharp
var db = MainForm.Instance.AppContext.Db;
var cache = new MemoryCacheService();
var service = new AggregationService(db, cache);

var query = new StatisticsQuery
{
    EntityType = "tb_SaleOrder",
    TimeField = "Created_at",
    StartTime = DateTime.Now.AddMonths(-1),
    EndTime = DateTime.Now,
    Dimensions = new List<string> { "Employee_ID" },
    Metrics = new List<string> { "TotalAmount", "OrderID" }
};

// 执行查询 (自动缓存)
var chartData = await service.ExecuteQueryAsync(query);

// 渲染图表
DisplayChart(chartData);
```

#### 方式 2: 使用数据适配器 (复杂场景)

```csharp
var db = MainForm.Instance.AppContext.Db;
var adapter = new OrderAnalysisAdapter(db);

var request = new DataRequest
{
    TimeField = "Created_at",
    StartTime = DateTime.Now.AddMonths(-1),
    EndTime = DateTime.Now,
    ChartType = ChartType.Column,
    Dimensions = new List<DimensionConfig>
    {
        new DimensionConfig("Created_at", "订单日期", DimensionType.DateTime)
    },
    Metrics = new List<MetricConfig>
    {
        new MetricConfig("OrderID", "订单数", MetricType.Count, MetricUnit.笔),
        new MetricConfig("TotalAmount", "金额", MetricType.Sum, MetricUnit.元)
    }
};

// 获取数据
var chartData = await adapter.GetDataAsync(request);

// 使用 ChartBuilderFactory 渲染
var builder = ChartBuilderFactory.CreateBuilder(request, adapter);
var chartControl = await builder.BuildChart(request);
```

## 🎯 核心组件说明

### 1. IStatisticsService (统计服务接口)

**用途**: 提供通用的统计查询能力，支持动态维度 + 指标组合。

**适用场景**:
- ✅ 简单的 SUM/COUNT/AVG聚合
- ✅ 需要缓存加速的查询
- ✅ 跨多个实体的通用统计

**示例代码**: 见上方"使用示例 - 方式 1"

### 2. BaseChartAdapter (数据适配器基类)

**用途**: 为特定业务场景提供数据查询和转换逻辑。

**与 IStatisticsService 的区别**:
- Adapter 更贴近业务，可编写复杂 SQL 关联
- Adapter 支持自定义数据转换规则
- Service 更通用，适合标准化统计

**开发步骤**:
1. 继承 `BaseChartAdapter`
2. 实现 `PrimaryTableName` 属性
3. 实现 `GetDimensions()` 和 `GetMetrics()`
4. 实现 `TransformToChartData()`

### 3. QueryPanelGenerator (查询面板生成器)

**用途**: 根据参数对象自动生成查询界面。

**支持的控件类型**:
- DatePicker (日期选择)
- ComboBox (下拉选择)
- NumericUpDown (数值输入)
- TextBox (文本输入)

**使用示例**:
```csharp
public class SalesQueryParams
{
    [QueryParameter(DisplayName = "业务员", Order = 1)]
    public long? Employee_ID { get; set; }
    
    [QueryParameter(DisplayName = "开始日期", Order = 2)]
    public DateTime StartDate { get; set; }
}

var panel = QueryPanelGenerator.Generate(new SalesQueryParams());
```

### 4. DrillDownService (钻取服务)

**用途**: 点击图表数据点时显示详细信息。

**工作原理**:
1. 捕获用户点击事件
2. 构建钻取查询 (添加过滤条件)
3. 显示详情窗口

**使用示例**:
```csharp
var drillDownService = new DrillDownService(adapter);
await drillDownService.ShowDrillDownAsync(dataPoint, currentRequest);
```

### 5. ProgressiveLoader (分层加载器)

**用途**: 处理大数据量场景，避免 UI 阻塞。

**优化策略**:
- 分批加载 (默认每批 100 条)
- 增量更新图表
- 进度报告

**使用示例**:
```csharp
var loader = new ProgressiveLoader();
await loader.LoadAndRenderProgressivelyAsync(
    adapter, 
    request, 
    onDataReceived: (chunk) => UpdateChart(chunk)
);
```

<!-- App.config -->
<configuration>

    <add key="Cache.TTL.Minutes" value="30"/>

确保以下字段有索引以优化性能:
- 时间字段 (如 `Created_at`)
- 分组字段 (如 `Employee_ID`, `Customer_ID`)
- 外键字段 (如 `OrderID`, `ProdID`)

## 📈 性能基准

### 测试结果 (待完善)

| 场景 | 数据量 | 响应时间 (无缓存) | 响应时间 (有缓存) |
|------|--------|------------------|------------------|
| 简单聚合 | 1 万条 | < 500ms | < 100ms |
| 多维度分组 | 10 万条 | < 2s | < 300ms |
| 跨表关联 | 50 万条 | < 3s | < 500ms |
| 大数据集 | 100 万条 | < 5s | < 1s |

### 优化建议

1. **数据库层**
   - 使用 EXPLAIN 分析查询计划
   - 避免 SELECT *
   - 合理使用索引

2. **应用层**
   - 启用缓存
   - 限制单次查询最多 1000 条
   - 使用异步查询 (async/await)

3. **UI 层**
   - 大数据集使用虚拟滚动
   - 禁用不必要的动画
   - 分批加载数据

## ❓ 常见问题

### Q1: 什么时候使用 StatisticsService vs Adapter?

**A**: 
- 如果只需要简单的 GROUP BY + 聚合，用 **Service**
- 如果需要 JOIN 多个表或复杂计算，用 **Adapter**

### Q2: 如何支持实时刷新？

**A**: 可以使用 SignalR 推送数据变更，然后调用 `chartControl.UpdateData(newData)`。

### Q3: 大数据量导致内存溢出怎么办？

**A**: 使用 `ProgressiveLoader` 进行分批加载，并启用虚拟滚动。

### Q4: 如何集成 Redis 缓存？

**A**: 实现 `IReportCacheService` 接口，替换 `MemoryCacheService` 即可。
