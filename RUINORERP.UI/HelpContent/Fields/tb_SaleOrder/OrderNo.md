# 订单编号

## 说明
销售订单的唯一标识编号,由系统自动生成,不可修改。

## 编号规则
订单编号采用编码规则自动生成,格式: `SO + 年月 + 4位流水号`

例如: `SO2024010001`

### 示例
| 编号 | 说明 |
|------|------|
| SO2024010001 | 2024年1月的第1个订单 |
| SO2024010256 | 2024年1月的第256个订单 |
| SO2024020001 | 2024年2月的第1个订单 |

## 使用方法
1. **自动生成**: 保存订单时自动生成
2. **手动修改**: 不支持手动修改
3. **查询引用**: 在其他单据中通过订单编号引用本订单

## 注意事项
1. **唯一性**: 订单编号在系统中唯一
2. **不可修改**: 生成后不允许修改
3. **自动补号**: 删除订单后编号不会重复使用
4. **编码规则**: 可在系统配置中修改编码规则
