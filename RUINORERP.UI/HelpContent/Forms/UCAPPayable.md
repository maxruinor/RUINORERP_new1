# 应付账款窗体

## 窗体概述

应付账款窗体用于管理采购业务产生的应付账款，是财务管理的核心功能之一。该窗体记录了应支付给供应商的货款，与采购入库单紧密关联。当采购入库后，系统会自动生成应付账款记录，用户可以通过本窗体进行付款、核销、查询等操作。

应付账款管理是企业资金流管理的关键环节。通过该窗体，用户可以跟踪每笔应付款项的付款进度，管理供应商关系，合理安排资金使用。同时，窗体提供了丰富的统计分析功能，帮助财务人员了解应付款项的结构和变化趋势。

## 窗体布局

### 主信息区
- **供应商信息**：供应商名称、应付余额、账期
- **单据信息**：应付单号、应付日期、到期日期
- **金额信息**：应付金额、已付金额、应付余额
- **状态信息**：付款状态、账龄、付款计划

### 明细区（表格）
- 来源单据（采购入库单）
- 入库日期、入库单号
- 应付金额、已付金额、应付余额
- 到期日期、账龄

### 汇总信息区
- 总应付金额
- 总已付金额
- 总应付余额

## 基本操作流程

### 查看应付账款

1. 进入应付账款查询界面
2. 可以按供应商、日期范围、状态等条件查询
3. 查看每笔应付账款的详细信息

### 付款处理

1. 在应付款列表中选择要付款的记录
2. 点击"付款"按钮
3. 填写付款金额、付款日期、付款账户
4. 系统自动进行核销
5. 审核付款单

### 应付核销

1. 选择要核销的应付账款
2. 点击"核销"按钮
3. 选择要核销的付款单
4. 确认核销金额
5. 完成核销处理

### 供应商对账

1. 选择要对账的供应商
2. 生成对账单
3. 与供应商确认
4. 确认后形成对账记录

## 重要业务场景

### 场景1：供应商付款

**业务场景**：向供应商支付货款

**操作步骤**：
1. 进入付款管理，创建付款单
2. 选择付款供应商
3. 选择或输入付款金额
4. 选择付款账户
5. 选择要核销的应付单据
6. 审核付款单
7. 系统自动核销应付账款

### 场景2：部分付款

**业务场景**：部分支付供应商货款

**操作步骤**：
1. 付款金额小于应付金额
2. 系统自动核销部分金额
3. 剩余金额形成新的应付账款
4. 继续安排后续付款

### 场景3：预付冲应付

**业务场景**：之前预付的款项冲减应付账款

**操作步骤**：
1. 供应商有预付款记录
2. 创建付款单选择预付冲销
3. 系统自动用预付款核销应付
4. 核销后更新应付账款状态

### 场景4：现金折扣

**业务场景**：供应商提供现金折扣

**操作步骤**：
1. 在付款期限内核销可享受折扣
2. 系统自动计算折扣金额
3. 按折扣后金额付款
4. 记录折扣收益

## 关键字段说明

### 金额信息字段

| 字段 | 说明 | 计算方式 |
|-----|------|---------|
| 应付金额 | 原始应付金额 | 采购入库单金额 |
| 已付金额 | 已付款项 | 付款核销金额合计 |
| 应付余额 | 未付金额 | 应付金额 - 已付金额 |
| 账龄 | 账期 | 到期日期 - 当前日期 |

### 状态字段

| 状态 | 说明 |
|-----|------|
| 未付款 | 未支付任何款项 |
| 部分付款 | 已付部分款项 |
| 已付清 | 全部付款完成 |
| 逾期 | 已超过到期日期 |

- 到期自动提醒

## 常见业务问题

### Q1：应付账款与采购入库金额不一致？

A：可能存在退货、折让等情况，需要检查对应的采购入库单和调整单据。

### Q2：付款后应付账款余额不为零？

A：检查核销是否完全，可能是部分付款或核销金额不匹配。

### Q3：如何查看某供应商的应付账款历史？

A：在供应商档案中可以查看该供应商的所有应收应付记录。

### Q4：如何管理供应商账期？

A：在供应商档案中设置付款账期，系统会自动计算到期日期并提醒。

### Q5：如何合理安排付款？

A：可以查看付款计划报表，根据到期顺序和资金情况合理安排付款。

- 供应商名称
- 应付金额
- 已付金额

- 账龄区间

- 占比

### 资金预测表

- 时间段
- 预计付款金额
- 可用资金

## 注意事项

1. **付款及时性**：及时付款可以维持良好的供应商关系
2. **账期管理**：注意供应商的付款账期，合理安排资金
3. **凭证管理**：付款后及时生成记账凭证
4. **对账定期**：定期与供应商对账，确认往来
