# 应收账款窗体

## 窗体概述

应收账款窗体用于管理销售业务产生的应收账款，是财务管理的核心功能之一。该窗体记录了向客户收取货款的权利，与销售出库单紧密关联。当销售出库后，系统会自动生成应收账款记录，用户可以通过本窗体进行收款、核销、查询等操作。

应收账款管理是企业资金流管理的关键环节。通过该窗体，用户可以跟踪每笔应收账款的催收进度，管理客户信用，及时发现坏账风险。同时，窗体提供了丰富的统计分析功能，帮助财务人员了解应收账款的结构和变化趋势。

## 窗体布局

### 主信息区
- **客户信息**：客户名称、信用额度、账期
- **单据信息**：应收单号、应收日期、到期日期
- **金额信息**：应收金额、已收金额、应收余额
- **状态信息**：收款状态、账龄、催收状态

### 明细区（表格）
- 来源单据（销售出库单）
- 出库日期、出库单号
- 应收金额、已收金额、应收余额
- 到期日期、账龄

### 汇总信息区
- 总应收金额
- 总已收金额
- 总应收余额

## 基本操作流程

### 查看应收账款

1. 进入应收账款查询界面
2. 可以按客户、日期范围、状态等条件查询
3. 查看每笔应收账款的详细信息

### 收款处理

1. 在应收账款列表中选择要收款的记录
2. 点击"收款"按钮
3. 填写收款金额、收款日期、收款账户
4. 系统自动进行核销
5. 审核收款单

### 应收核销

1. 选择要核销的应收账款
2. 点击"核销"按钮
3. 选择要核销的收款单
4. 确认核销金额
5. 完成核销处理

### 坏账处理

1. 确认某笔应收账款无法收回
2. 点击"坏账"按钮
3. 填写坏账原因和金额
4. 审核后计入坏账准备

## 重要业务场景

### 场景1：客户收款

**业务场景**：收到客户支付的货款

**操作步骤**：
1. 进入收款管理，创建收款单
2. 选择收款客户
3. 选择或输入收款金额
4. 选择收款账户
5. 选择要核销的应收单据
6. 审核收款单
7. 系统自动核销应收账款

### 场景2：部分收款

**业务场景**：客户部分支付货款

**操作步骤**：
1. 收款金额小于应收金额
2. 系统自动核销部分金额
3. 剩余金额形成新的应收账款
4. 继续跟踪催收

### 场景3：预收冲应收

**业务场景**：客户之前预付的款项冲减应收账款

**操作步骤**：
1. 客户有预收款记录
2. 创建收款单选择预收冲销
3. 系统自动用预收款核销应收
4. 核销后更新应收账款状态

### 场景4：客户对账

**业务场景**：与客户核对往来账目

**操作步骤**：
1. 选择要对账的客户
2. 生成对账单
3. 与客户确认
4. 确认后形成对账记录

## 关键字段说明

### 金额信息字段

| 字段 | 说明 | 计算方式 |
|-----|------|---------|
| 应收金额 | 原始应收金额 | 销售出库单金额 |
| 已收金额 | 已收款项 | 收款核销金额合计 |
| 应收余额 | 未收金额 | 应收金额 - 已收金额 |
| 账龄 | 逾期天数 | 到期日期 - 当前日期 |

### 状态字段

| 状态 | 说明 |
|-----|------|
| 未收款 | 未收到任何款项 |
| 部分收款 | 已收部分款项 |
| 已收清 | 全部收款完成 |
| 逾期 | 已超过到期日期 |
| 坏账 | 确认无法收回 |

## 账龄分析

系统自动计算应收账款账龄：
- **30天以内**：正常
- **31-60天**：逾期较轻
- **61-90天**：逾期严重
- **90天以上**：高风险

账龄分析报表可以帮助：
- 识别高风险客户
- 制定催收策略
- 计提坏账准备

## 常见业务问题

### Q1：应收账款与销售出库金额不一致？

A：可能存在折让、退货等情况，需要检查对应的销售出库单和调整单据。

### Q2：收款后应收账款余额不为零？

A：检查核销是否完全，可能是部分收款或核销金额不匹配。

### Q3：如何查看某客户的应收账款历史？

A：在客户档案中可以查看该客户的所有应收应付记录。

### Q4：坏账如何处理？

A：需要先确认坏账，经过审批后计入坏账准备，同时冲减应收账款。

### Q5：信用额度如何控制？

A：可以在系统参数中设置信用额度控制规则，超额时系统会自动提醒或阻止。

## 注意事项

1. **收款及时性**：及时收款可以改善企业现金流
2. **信用控制**：注意客户的信用额度和账期
3. **凭证管理**：收款后及时生成记账凭证
4. **对账定期**：定期与客户对账，确认往来
