# BOM物料清单窗体

## 窗体概述

BOM物料清单窗体用于创建和管理产品的物料清单结构。通过本窗体，可以定义产品与原材料、半成品之间的组成关系，包括每个子项的数量、损耗率、供应商等信息。BOM是生产管理、采购计划和成本核算的基础数据。

## 窗体布局

### 基本信息区
- **产品信息**：产品编码、产品名称、产品分类
- **版本信息**：版本号、生效日期、状态
- **基本信息**：BOM类型、工艺路线

### 明细信息区（表格）
- 子项物料编码、名称
- 用量、数量、单位
- 损耗率
- 供应商
- 备注

### 汇总信息区
- 子项总数
- 预计成本

## 基本操作流程

### 新建BOM

1. 点击"新建"按钮
2. 选择产品（父项）
3. 设置版本号和生效日期
4. 添加子项物料
5. 填写用量和损耗率
6. 保存BOM

### 编辑BOM

1. 查询要修改的BOM
2. 打开BOM详情
3. 修改子项或用量
4. 保存修改

### 审核BOM

1. BOM创建或修改后
2. 提交审核
3. 审核通过后生效

## 重要业务场景

### 场景1：创建产品BOM

**业务场景**：为产品建立物料清单

**操作步骤**：
1. 选择父项产品
2. 添加子项物料
3. 填写用量（生产1个父项需要的子项数量）
4. 设置损耗率（可选）
5. 指定供应商（可选）
6. 保存并审核

### 场景2：BOM变更

**业务场景**：产品配方或材料调整

**操作步骤**：
1. 找到要变更的BOM
2. 修改版本号
3. 调整子项或用量
4. 设置新的生效日期
5. 保存并审核

### 场景3：BOM复制

**业务场景**：快速创建类似产品的BOM

**操作步骤**：
1. 选择要复制的源BOM
2. 点击"复制"按钮
3. 修改父项产品
4. 根据需要调整子项
5. 保存新BOM

## 关键字段说明

### 基本信息字段

| 字段 | 控件 | 业务意义 | 使用场景 |
|-----|------|---------|---------|
| 父项产品 | `cmbParentProd_ID` | 要建立BOM的产品 | 必选 |
| 版本号 | `txtVersion` | BOM版本标识 | 必填，如V1.0 |
| 生效日期 | `dtpEffectiveDate` | 版本生效时间 | 必选 |
| BOM类型 | `cmbBOMType` | BOM分类 | 标准、可选 |

### 明细字段

| 字段 | 控件 | 业务意义 | 使用场景 |
|-----|------|---------|---------|
| 子项物料 | `cmbChildProd_ID` | 组成父项的物料 | 必选 |
| 用量 | `txtQuantity` | 需要的数量 | 必填 |
| 单位 | `cmbUnit` | 用量单位 | 必选 |
| 损耗率 | `txtLossRate` | 损耗百分比 | 可选 |
| 供应商 | `cmbSupplier_ID` | 供应商 | 可选 |

### 用量计算

- **理论用量**：基础用量
- **实际用量**：理论用量 × (1 + 损耗率)
- **MRP计算**：使用实际用量计算需求

## 常见问题

### Q1：一个产品可以有多个BOM吗？

A：可以。通过版本号区分不同版本的BOM。

### Q2：子项可以是半成品吗？

A：可以。半成品本身也可以有BOM，系统会自动展开。

### Q3：损耗率如何设置？

A：根据生产工艺和产品特性设置，一般在1%-10%之间。

### Q4：BOM如何快速导入？

A：支持Excel导入功能，可以批量导入BOM明细。

### Q5：未审核的BOM可以用于生产吗？

A：不能。只有审核生效后的BOM才能被MRP和生单使用。
