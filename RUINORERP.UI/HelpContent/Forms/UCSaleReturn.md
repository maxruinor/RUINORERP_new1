# 销售退货单窗体

## 窗体概述

销售退货单窗体用于处理客户退货业务，是销售业务的逆向流程。当客户因质量问题或其他原因需要退货时，需要通过本窗体创建退货单，办理退货入库并处理退款或冲减应收账款。

销售退货单与销售出库单紧密关联，系统可以根据出库单生成退货单。退货处理完成后，系统会自动更新库存数量、冲减应收账款，并根据退货原因生成相应的财务调整凭证。

## 窗体布局

### 主信息区
- **来源信息**：原销售出库单、退货单号、退货日期
- **客户信息**：客户、客户联系人、联系电话
- **仓库信息**：退货仓库、库位
- **退款信息**：退款方式、退款金额、退款账户
- **退货原因**：退货类型、详细原因说明

### 明细区（表格）
- 产品明细列表
- 原出库数量、退货数量
- 单价、金额
- 退货原因

### 汇总信息区
- 退货总数量
- 退货总金额
- 退款金额

## 基本操作流程

### 从销售出库单退货

1. 在销售退货界面点击"新建"
2. 选择要退货的销售出库单
3. 系统自动带出原出库明细
4. 填写退货数量（小于或等于原出库数量）
5. 选择退货原因
6. 保存并审核退货单
7. 系统自动处理库存和财务

### 直接创建退货单

1. 点击"新建"按钮
2. 不选择原出库单，直接创建
3. 选择客户
4. 手动添加产品明细
5. 填写退货数量和原因
6. 审核退货单

### 退货检验

1. 退货单创建后进入检验流程
2. 检验人员检查退货商品
3. 根据检验结果处理：
   - 合格：确认入库
   - 不合格：与客户协商处理

## 重要业务场景

### 场景1：全部退货

**业务场景**：客户将整批货物全部退回

**操作步骤**：
1. 选择原销售出库单
2. 退货数量等于原出库数量
3. 选择退货原因（质量、错发等）
4. 审核退货单
5. 系统自动：
   - 库存增加
   - 应收账款冲减
   - 如已收款则退款

### 场景2：部分退货

**业务场景**：客户只退回部分货物

**操作步骤**：
1. 选择原销售出库单
2. 退货数量小于原出库数量
3. 填写退货原因
4. 审核退货单
5. 系统自动：
   - 按比例增加库存
   - 按比例冲减应收账款
   - 部分退款（如已收款）

### 场景3：换货处理

**业务场景**：客户退货后换货

**操作步骤**：
1. 先创建退货单处理退货
2. 再创建新的销售订单处理换货
3. 换货可以享受原订单价格
4. 整体处理不影响库存和财务

### 场景4：退货退款

**业务场景**：收到退货后退款给客户

**操作步骤**：
1. 退货检验合格后入库
2. 创建退款单
3. 选择退款方式和退款账户
4. 审核退款单
5. 系统自动：
   - 减少银行存款
   - 冲减应收账款

## 关键字段说明

### 来源信息字段

| 字段 | 控件 | 业务意义 | 使用场景 |
|-----|------|---------|---------|
| 原出库单 | `cmbSaleOut_ID` | 来源销售出库单 | 必选，确定退货来源 |
| 退货单号 | `txtSaleReturnNo` | 退货单唯一标识 | 系统自动生成 |
| 退货日期 | `dtpReturnDate` | 办理退货的日期 | 默认当天 |

### 客户信息字段

| 字段 | 控件 | 业务意义 | 使用场景 |
|-----|------|---------|---------|
| 客户 | `cmbCustomerVendor_ID` | 退货客户 | 从原出库单带出 |
| 联系人 | `txtContactPerson` | 客户联系人 | 退货对接人 |
| 联系电话 | `txtContactPhone` | 客户电话 | 紧急联系 |

### 退货原因字段

| 字段 | 控件 | 业务意义 | 使用场景 |
|-----|------|---------|---------|
| 退货类型 | `cmbReturnType` | 退货原因分类 | 质量、错发、滞销等 |
| 退货原因 | `txtReturnReason` | 详细说明 | 必填，特殊情况说明 |

### 退款信息字段

| 字段 | 控件 | 业务意义 | 使用场景 |
|-----|------|---------|---------|
| 退款方式 | `cmbRefundType` | 退款途径 | 退款到账户、原路退回等 |
| 退款金额 | `txtRefundAmount` | 退款金额 | 系统计算或手动输入 |
| 退款账户 | `cmbAccount_ID` | 退款账户 | 退款使用的银行账户 |

## 退货明细表格

### 退货数量

- **默认值**：原出库数量
- **可修改**：可以小于或等于原出库数量
- **限制**：不能超过原出库数量

### 退货金额

- **计算方式**：退货数量 × 单价
- **含税**：包含税额
- **退款依据**：作为退款计算的基础

### 退货原因

- 每个明细行可以设置退货原因
- 方便统计分析退货原因

## 状态流转

### 退货单状态

| 状态 | 说明 | 可执行操作 |
|-----|------|-----------|
| 新建 | 退货单刚创建，未审核 | 编辑、审核、删除 |
| 检验中 | 等待质量检验 | 查看、检验操作 |
| 已检验 | 检验完成，等待入库 | 确认入库 |
| 已审核 | 已入库，库存已更新 | 取消审核、打印 |
| 已退款 | 已完成退款 | 查询 |
| 已取消 | 退货单已取消 | 查询（不可编辑） |

### 退货类型

| 类型 | 说明 |
|-----|------|
| 质量问题 | 商品存在质量缺陷 |
| 错发商品 | 收到的商品与订单不符 |
| 滞销商品 | 客户不再需要 |
| 物流损坏 | 运输过程中损坏 |
| 其他 | 其他原因 |

## 常见业务问题

### Q1：退货数量可以超过原出库数量吗？

A：不能。退货数量不能超过原销售出库的数量。如果需要退更多，需要检查是否有其他出库单。

### Q2：退货后需要退款吗？

A：不一定。如果客户尚未付款，只需冲减应收账款；如果已付款，则需要退款。

### Q3：如何处理部分退款？

A：创建退货单后，根据实际退款金额创建退款单，系统会自动处理部分退款的情况。

### Q4：退货商品如何处理？

A：根据检验结果：合格品入库继续销售；不合格品与客户协商退换或索赔。

### Q5：退货影响销售业绩吗？

A：退货会冲减销售收入和利润，在报表中会单独列示退货数据。

## 注意事项

1. **退货检验**：退货必须经过检验才能入库
2. **数量控制**：退货数量不能超过原出库数量
3. **退款审批**：大额退款可能需要审批
4. **凭证处理**：退货后需要生成红字凭证冲减收入
