# BOM物料清单模块

## 模块概述

BOM（物料清单）模块是生产管理的核心基础，定义了产品与原材料、半成品之间的结构关系。BOM记录了制造一个成品需要哪些物料、各需要多少数量，是MRP运算、生产领料、成本计算的重要依据。

BOM模块与生产管理、采购管理、库存管理紧密关联。通过BOM，系统可以计算出生产一定数量的成品需要哪些原材料及数量，进而进行物料需求运算生成采购建议。BOM也是计算产品成本的重要依据，材料的成本直接构成产品的成本。

## 核心功能

### 1. BOM结构管理

```
成品 → 一级物料 → 二级物料 → ... → 原材料
```

**结构说明**：

- **父项**：被组合的产品（成品或半成品）
- **子项**：组成父项的物料
- **用量**：生产一个父项需要的子项数量
- **损耗率**：生产过程中的损耗比例

### 2. BOM版本管理

BOM可能因为设计变更、工艺调整等原因需要修改，系统支持BOM版本管理：

- **版本号**：标识BOM的版本
- **生效日期**：版本生效时间
- **替代关系**：新版本替代旧版本

### 3. BOM查询

支持多种方式查询BOM：

- **按产品查询**：查看某产品的BOM结构
- **按物料查询**：查看某物料被哪些产品使用（反查）
- **层级展开**：查看完整的BOM层级结构

## 常见业务场景

### 场景1：创建产品BOM

**适用情况**：新产品需要定义BOM结构

**操作步骤**：
1. 选择要创建BOM的产品（父项）
2. 添加子项物料
3. 填写用量和损耗率
4. 设置用量单位
5. 保存BOM

### 场景2：BOM变更

**适用情况**：产品配方或工艺调整

**操作步骤**：
1. 找到要变更的BOM
2. 创建新版本
3. 修改子项或用量
4. 设置生效日期
5. 审核发布

### 场景3：BOM反查

**适用情况**：某物料有哪些产品在使用

**操作步骤**：
1. 输入物料编码
2. 执行反查
3. 查看使用该物料的所有产品
4. 可以查看各层级的用量

### 场景4：BOM展开

**适用情况**：查看产品的完整物料需求

**操作步骤**：
1. 选择产品
2. 输入要生产的数量
3. 执行BOM展开
4. 查看所有层级的物料需求
5. 可以导出或传递到采购

## 关键概念

### 物料类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 原材料 | 最底层的物料 | 钢材、塑料 |
| 半成品 | 加工过的物料 | 部件、组件 |
| 成品 | 最终产品 | 整机、产品 |

### 用量类型

| 类型 | 说明 | 计算方式 |
|------|------|----------|
| 固定用量 | 不随数量变化 | 一个产品用一个 |
| 变动用量 | 随数量成比例 | 1个产品用2个 |

### 损耗率

- **定义**：生产过程中的物料损耗比例
- **计算**：实际用量 = 理论用量 × (1 + 损耗率)
- **应用**：影响MRP运算和成本计算

## 数据报表

### BOM清单

列出所有产品的BOM结构：
- 产品编码、名称
- 层级数
- 子项数量
- 版本状态

### 物料需求表

根据BOM展开计算物料需求：
- 物料编码、名称
- 需求数量
- 现有库存
- 采购建议

### 成本明细

根据BOM计算产品成本：
- 物料成本
- 人工成本
- 制造费用
- 总成本

## 常见问题

### Q1：BOM如何设置多层级的？

A：在子项中可以选择半成品，半成品本身也有BOM，系统会自动展开到最底层。

### Q2：BOM版本有什么作用？

A：可以保留历史版本的BOM记录，便于追溯和对比。新版本可以设置生效日期后再使用。

### Q3：BOM与工艺是什么关系？

A：BOM定义需要什么物料，工艺定义如何加工。两者配合使用进行生产管理。

### Q4：如何快速创建类似产品的BOM？

A：可以使用BOM复制功能，复制现有产品的BOM然后修改。

### Q5：BOM准确性如何保证？

A：需要及时维护BOM变更，生产中发现差异及时反馈和修正。

## 系统集成

### 与MRP集成

- BOM是MRP运算的基础
- MRP根据BOM计算物料需求

### 与生产集成

- 生产领料根据BOM生成
- 验证物料齐套性

### 与成本集成

- BOM构成产品成本
- 材料成本来自BOM
