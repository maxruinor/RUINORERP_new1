# 采购管理模块

## 模块概述

采购管理模块是企业供应链管理的核心组成部分，负责处理从采购需求生成、采购订单下达、供应商供货到采购入库的完整采购业务流程。该模块支持多种采购场景，包括标准采购、外协加工、采购退货、受托代销等，能够满足不同业务模式的采购需求。

采购管理与销售管理、库存管理、财务管理模块紧密关联。采购需求可能来源于销售订单的生产需求、安全库存的补货需求或MRP运算结果；采购入库直接影响库存数量和成本；采购付款则与财务的应付账款管理密切相关。系统通过模块间的数据集成，确保采购业务与整体业务流的一致性。

## 核心业务流程

### 1. 标准采购流程

```
采购需求 → 采购订单 → 供应商发货 → 采购入库 → 应付账款
```

**流程说明**：

1. 采购需求：系统根据销售订单、安全库存或MRP运算结果自动生成采购需求，也可以由用户手动创建
2. 采购订单：确认供应商、采购价格、采购数量和交货日期，生成正式的采购订单
3. 供应商发货：供应商根据订单安排发货，可能分批交货
4. 采购入库：收到货物后进行入库操作，检验合格后入库，更新库存
5. 应付账款：入库后生成应付账款，进行付款结算

### 2. 外协加工流程

```
加工需求 → 外协订单 → 发料 → 加工回货 → 成品入库 → 应付加工费
```

**特殊处理**：

- 需要进行发料操作，将原材料发给加工商（`UCMaterialRequisition`）
- 加工完成后需要收货入库（`UCFinishedGoodsInv`）
- 加工费用需要单独核算和结算

### 3. 采购退货流程

```
采购入库 → 质量问题 → 退货申请 → 退货出库 → 退款处理 → 账务调整
```

**特殊处理**：

- 退货需要填写退货原因
- 退货数量不能超过入库数量
- 退货可能涉及退款或冲减应付账款

### 4. 受托代销流程

```
委托代销 → 代销入库 → 代销结算 → 销售处理 → 账务处理
```

**特殊处理**：

- 代销商品不计入正式库存
- 销售后才结算货款
- 需要跟踪代销数量和结算情况

## 关键业务规则

### 供应商管理

1. **供应商准入**：新供应商需要经过资质审核才能正式合作
2. **供应商评级**：根据交货质量、价格竞争力、服务水平进行评级
3. **供应商价格**：可以为不同供应商设置不同产品的采购价格
4. **供应商账期**：与供应商约定的付款账期，影响付款计划

### 价格管理

1. **采购价格**：系统支持多个价格策略，包括固定价、协议价、市场价
2. **价格时效**：采购价格可以设置生效日期，过期后自动失效
3. **价格审批**：超过限价的采购价格需要审批
4. **历史价格**：系统记录采购历史价格，供决策参考

### 库存控制

1. **到货检验**：采购入库前需要进行质量检验，不合格品不能入库
2. **数量验收**：实际到货数量与订单数量可能存在差异，需要确认
3. **库存更新**：入库后自动更新库存数量和成本
4. **批次管理**：支持按批次跟踪入库商品

### 付款控制

1. **应付账期**：根据与供应商的约定，计算应付账款到期日
2. **付款申请**：大额付款需要提前申请审批
3. **付款核销**：付款时自动核销对应的应付账款
4. **账期预警**：临近付款期限时自动提醒

## 常见业务场景

### 场景1：常规采购

**适用情况**：根据生产需求向供应商采购原材料或成品

**操作步骤**：
1. 根据MRP或安全库存生成采购需求
2. 创建采购订单，选择供应商和产品
3. 确认价格和交货日期
4. 供应商发货后进行入库检验
5. 检验合格后办理入库
6. 根据账期进行付款结算

### 场景2：紧急采购

**适用情况**：生产急需的物料，需要快速采购

**操作步骤**：
1. 标记为紧急采购
2. 优先选择能快速供货的供应商
3. 简化审批流程
4. 加急入库检验
5. 快速办理入库

### 场景3：分批交货

**适用情况**：订单数量较大，供应商分多次交货

**操作步骤**：
1. 创建采购订单，录入总数量
2. 第一次到货：录入实际到货数量并入库
3. 系统自动更新订单的未交货数量
4. 第二次到货：重复上述操作
5. 订单全部到货后关闭订单

### 场景4：采购变更

**适用情况**：采购订单签订后需要修改

**操作步骤**：
1. 查询要变更的采购订单
2. 点击"变更"按钮
3. 修改采购数量、价格或交期
4. 记录变更原因
5. 提交审批（如果需要）

### 场景5：采购取消

**适用情况**：采购订单需要取消

**操作步骤**：
1. 查询要取消的订单
2. 点击"取消"按钮
3. 选择取消原因
4. 如果已入库，需要处理入库货物
5. 通知供应商和财务

## 数据统计与报表

### 关键指标

- **采购完成率**：已完成采购订单数/总订单数
- **准时交货率**：准时交货订单数/总订单数
- **平均采购价格**：总采购金额/采购数量
- **供应商准时率**：准时交货供应商数/总供应商数

### 常用报表

1. **采购分析报表**：按供应商、产品、时间段统计采购情况
2. **价格分析报表**：分析价格变动趋势和原因
3. **供应商绩效报表**：评估供应商交货质量和服务水平
4. **在途订单报表**：跟踪未完成采购订单的执行情况
5. **应付账款报表**：按供应商和账龄分析应付账款

## 常见问题

### Q1：采购订单可以修改吗？

A：已审核的采购订单不能直接修改，需要先取消审核。建议使用订单变更功能，这样可以保留变更历史。

### Q2：如何处理到货数量与订单数量不一致？

A：系统支持部分入库。如果实际到货数量少于订单数量，可以先入库实际数量，剩余数量等待后续到货或与供应商协商处理。

### Q3：如何设置供应商的价格？

A：在供应商档案中可以为不同产品设置不同的采购价格，也可以设置价格生效的有效期。系统会根据采购订单自动应用正确的价格。

### Q4：采购入库后发现质量问题如何处理？

A：需要先创建退货申请，然后进行退货出库操作。退货可以冲减应付账款或要求供应商退款。

### Q5：如何查询某个供应商的历史采购记录？

A：在采购订单查询界面，可以选择供应商进行筛选，显示该供应商的所有历史采购订单。
