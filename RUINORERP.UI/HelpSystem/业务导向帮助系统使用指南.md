# 业务导向帮助系统使用指南

## 帮助系统设计理念

本帮助系统以**业务场景**为核心，以**解决问题**为目标，帮助用户快速理解和完成业务操作。

### 核心特点

1. **业务流程导向**：按业务流程组织内容，而非按功能模块
2. **场景驱动**：针对具体业务场景提供详细操作指导
3. **实战导向**：强调"如何做"，而非"是什么"
4. **关联知识**：提供相关业务、相关字段、相关报表的链接

## 帮助内容结构

### 三级内容体系

```
模块级（Modules）
  ↓
窗体级（Forms）
  ↓
字段级（Fields）
```

#### 1. 模块级帮助

**作用**：介绍整个业务模块的全貌

**内容**：
- 模块概述
- 核心业务流程
- 关键业务规则
- 常见业务场景
- 数据统计与报表
- 系统集成说明

**适用场景**：
- 新员工培训
- 了解整体业务流程
- 跨模块协作

**示例**：
- 销售管理模块
- 库存管理模块
- 财务应收模块

#### 2. 窗体级帮助

**作用**：指导具体窗体的操作

**内容**：
- 窗体概述
- 窗体布局说明
- 基本操作流程
- 重要业务场景
- 关键字段说明（表格形式）
- 状态流转说明
- 常见问题
- 快捷键列表

**适用场景**：
- 首次使用某个窗体
- 忘记操作步骤
- 遇到业务问题

**示例**：
- 销售订单窗体
- 销售出库单窗体
- 客户档案窗体

#### 3. 字段级帮助

**作用**：深入解释关键字段的业务含义

**内容**：
- 字段说明
- 业务意义（重点！）
- 使用说明
- 重要提示
- 常见问题
- 相关字段
- 相关帮助

**适用场景**：
- 不理解字段的业务含义
- 不知道如何填写
- 填写后出现问题

**示例**：
- 客户字段
- 总成本字段
- 定制成本字段

## 帮助内容编写规范

### 1. 业务意义优先

**❌ 不好的写法**：
```
总成本：订单中所有产品的成本总和。
```

**✅ 好的写法**：
```
总成本：销售订单中所有产品的成本总和，是计算毛利的重要依据。

总成本由产品基础成本和定制成本组成：
- 产品基础成本：采用先进先出（FIFO）或加权平均方法计算的实际成本
- 定制成本：特殊材料、额外工时、设计费用等定制相关成本

业务意义：
- 定价参考：根据总成本确定合理的销售价格
- 利润分析：评估订单的盈利能力
- 成本控制：监控成本波动，及时发现问题
```

### 2. 场景驱动描述

**❌ 不好的写法**：
```
平台订单：勾选平台单选项，填写平台订单号。
```

**✅ 好的写法**：
```
平台订单：从电商平台（如淘宝、亚马逊、1688）接到的订单

业务场景：
- 在电商平台接收到客户订单
- 需要在系统中记录平台订单信息
- 出库后需要填写物流单号并同步到平台

操作步骤：
1. 勾选"平台单"复选框
2. 在"平台订单号"中填写平台订单号
3. 在"客户PO号"中填写客户采购订单号
4. 选择对应客户
5. 添加订单明细
6. 保存订单

系统处理：
- 平台订单不计入客户信用额度
- 出库后需要填写物流单号
- 需要等待平台确认后才确认收入
```

### 3. 关键业务规则

每个业务模块都要明确列出关键业务规则：

#### 示例：销售管理模块

```
关键业务规则

1. 库存管理
   - 订单创建后自动占用库存
   - 支持部分出库
   - 支持替代品出库
   - 订单取消自动释放库存

2. 价格管理
   - 自动应用客户价目表
   - 支持折扣规则
   - 记录历史价格
   - 特价需要审批

3. 信用控制
   - 订单金额不能超过客户信用额度
   - 逾期账款限制新增订单
   - 信用较差客户要求预付款

4. 交期管理
   - 根据预交期自动排产
   - 临近交期自动提醒
   - 延期交货需要填写原因
```

### 4. 常见问题

每个帮助文档都要包含常见问题（FAQ）：

#### 示例：销售订单

```
常见问题

Q: 订单审核后还能修改吗？
A: 已审核的订单不能直接修改，需要先取消审核。建议使用"订单变更"功能，这样可以保留变更历史记录。

Q: 订单可以部分出库吗？
A: 可以。在出库单中录入实际出库数量，系统会自动更新订单状态为"部分出库"，剩余数量可以后续出库。

Q: 平台订单与普通订单有什么区别？
A: 主要区别：
   - 平台订单需要填写平台订单号
   - 平台订单不计入客户信用额度
   - 需要填写物流单号
   - 需要等待平台确认
```

|-----|---------|------|

| 窗体帮助 | `{窗体类名}.md` | `UCSaleOrder.md` |

## 帮助键自动映射

系统会自动将文件路径映射为帮助键：

```
文件路径                           帮助键
-----------------------------------------------
Modules/SalesManagement.md     ->  Modules.SalesManagement
Forms/UCSaleOrder.md          ->  Forms.UCSaleOrder
Fields/tb_SaleOrder/TotalCost.md  ->  Fields.tb_SaleOrder.TotalCost
```

## 控件命名规范

系统使用标准命名规范，帮助系统自动匹配：

| 控件类型 | 前缀 | 示例 | 自动匹配 |
|---------|------|------|---------|
| 标签 | `lbl` | `lblCustomerVendor_ID` | → `CustomerVendor_ID` |
| 文本框 | `txt` | `txtOrderNo` | → `OrderNo` |
| 下拉框 | `cmb` | `cmbCurrency_ID` | → `Currency_ID` |
| 复选框 | `chk` | `chkIsCustomizedOrder` | → `IsCustomizedOrder` |
| 日期选择 | `dtp` | `dtpSaleDate` | → `SaleDate` |

**匹配规则**：去掉前缀后，与字段名或帮助键匹配

## 帮助系统使用方式

### 方式1: F1键帮助（推荐）

**操作步骤**：
1. 将光标放在任意控件上
2. 按 `F1` 键
3. 系统显示该控件的详细帮助

**匹配优先级**：
1. 字段级帮助（最详细）
2. 窗体级帮助
3. 模块级帮助

**示例**：
```
光标在: txtCustomerVendor_ID
按F1后: 显示 Fields/tb_SaleOrder/CustomerVendor_ID.md
```

### 方式2: 鼠标悬停

**操作步骤**：
1. 将鼠标悬停在控件上约500毫秒
2. 系统显示简短的智能提示

**适用场景**：
- 快速查看字段说明
- 记住常用字段的含义

### 方式3: 帮助按钮

**操作步骤**：
1. 点击输入框右侧的 `?` 图标
2. 系统显示详细帮助面板

**适用场景**：
- 需要查看完整帮助内容
- 按键不方便时

### 方式4: 帮助菜单

**操作步骤**：
1. 点击菜单栏"帮助"
2. 选择"帮助内容"
3. 在帮助目录中浏览

**适用场景**：
- 系统学习
- 查找相关帮助
- 离线查看

## 帮助内容维护

### 新增帮助内容

1. **创建帮助文件**：
   ```
   在 HelpContent/ 对应目录下创建 .md 文件
   ```

2. **编写帮助内容**：
   ```
   按照编写规范编写业务导向的帮助内容
   ```

3. **测试效果**：
   ```
   按 F1 键查看帮助是否正常显示
   ```

### 更新帮助内容

1. **编辑帮助文件**：
   ```
   直接修改 HelpContent/ 下的 .md 文件
   ```

2. **立即生效**：
   ```
   修改后立即生效，无需重新编译
   ```

3. **发布部署**：
   ```
   编译时自动嵌入到程序集，无需额外部署
   ```

### 删除帮助内容

1. **删除文件**：
   ```
   直接删除 HelpContent/ 下的 .md 文件
   ```

2. **检查引用**：
   ```
   确认没有其他帮助内容引用了该文件
   ```

## 最佳实践

### 1. 始终从业务角度出发

**❌ 错误**：
```
总成本字段：类型是decimal，精度是4位小数
```

**✅ 正确**：
```
总成本字段：订单中所有产品的成本总和，用于计算毛利
```

### 2. 提供具体操作步骤

**❌ 错误**：
```
创建销售订单：填写订单信息后保存
```

**✅ 正确**：
```
创建销售订单：
1. 点击"新建"按钮
2. 选择客户
3. 添加订单明细
4. 填写预交期
5. 点击"审核"按钮
```

### 3. 提供业务背景和原因

**❌ 错误**：
```
信用额度：客户的最大购买金额
```

**✅ 正确**：
```
信用额度：控制客户赊购金额的业务规则
- 目的：控制坏账风险
- 超限处理：系统会阻止保存或要求审批
- 查询：可以在客户档案中查看信用使用情况
```

### 4. 保持内容的时效性

- **定期更新**：系统功能变更时及时更新帮助
- **版本标记**：注明适用的系统版本
- **废弃说明**：已废弃的功能要特别说明
