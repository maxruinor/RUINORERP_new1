# 基本数据清理 - 使用说明

## 设计理念

**复用现有架构,避免重复造轮子**

本项目已经有完善的单据查询和删除架构(`BaseListGeneric<T>`, `BaseController<T>`),我们直接复用这些成熟的功能,而不是重新实现复杂的级联删除逻辑。

## 核心优势

### 1. 强类型支持
通过泛型 `<T>` 直接指定 `RUINORERP.Model` 中的实体类:
```csharp
// 清理客户数据
await engine.ExecuteCascadeDeleteAsync<tb_CRM_Customer>(customerIds);

// 清理产品数据  
await engine.ExecuteCascadeDeleteAsync<tb_Prod>(productIds);
```

### 2. ✅ 真正的自动级联删除
利用现有的 `BaseController.BaseDeleteByNavAsync()` 方法,该方法内部使用 SqlSugar 的 `DeleteNav()`:
```csharp
// BaseController 中的实现(以 tb_CRM_CustomerController 为例)
bool rs = await _db.DeleteNav<tb_CRM_Customer>(m => m.Customer_id == entity.Customer_id)
    .Include(m => m.tb_CRM_Contacts)        // ✅ 自动删除联系人
    .Include(m => m.tb_CRM_FollowUpPlanses) // ✅ 自动删除跟进计划
    .Include(m => m.tb_CRM_FollowUpRecordses) // ✅ 自动删除跟进记录
    .Include(m => m.tb_CRM_Collaborators)   // ✅ 自动删除协作人
    .ExecuteCommandAsync();
```

**实际效果示例**:
- 删除 1 个客户 → 同时删除该客户的 5 个联系人 + 3 条跟进记录 + 2 个协作人 = **共 11 条记录**
- 删除 1 个产品类目 → 同时删除该类目的所有产品 + 子类目
- 删除 1 个产品 → 同时删除产品详情 + 属性关系 + 包装信息

### 3. ✅ 深层级关联数据清理
通过 `DeleteDeepLevelRelationsByNavigationAsync` 自动扫描并删除多层级关联:
```csharp
// 删除产品分类时,会自动处理以下层级关系:
tb_ProdCategories (分类)
  ↓ OneToMany
tb_Prods (产品)
  ↓ OneToMany  
tb_ProdDetail (产品明细)
  ↓ OneToMany
tb_ManufacturingOrderDetail (生产订单明细)
```

**优势**:
- ✅ 自动识别所有导航属性,无需手动配置
- ✅ 递归处理多层级关系(默认5层)
- ✅ 先删子表再删主表,避免外键约束冲突
- ✅ 支持测试模式下的准确统计

### 4. ✅ 测试模式真正执行并回滚
测试模式下会**真正执行删除操作**,但通过事务回滚保证数据不被实际删除:
```csharp
// 测试模式:执行删除 → 统计影响范围 → 回滚事务
var result = await engine.ExecuteCascadeDeleteAsync<tb_Prod>(ids, isTestMode: true);
Console.WriteLine($"预计删除 {result.TotalDeletedCount} 条记录(含关联数据)");
// 数据未实际删除,可以安全预览

// 确认后正式执行
if (Confirm())
{
    await engine.ExecuteCascadeDeleteAsync<tb_Prod>(ids, isTestMode: false);
    // 数据被真正删除并提交
}
```

**测试模式改进** (v1.1):
- ✅ 模拟深层级删除,准确统计所有关联数据
- ✅ 递归检查最多2层深度的子表关系
- ✅ 提供详细的预计删除记录数

### 5. ✅ 事务保护与外键冲突处理
批量删除时使用 `IUnitOfWorkManage` 管理事务,确保原子性:
- 测试模式:执行后自动回滚
- 正式模式:全部成功后提交,失败则回滚

**外键冲突自动处理** (v1.1):
- ✅ 自动解析 SQL 外键约束错误信息
- ✅ 识别冲突的表和字段
- ✅ 针对性删除关联数据后重试
- ✅ 详细日志记录便于排查问题

## 使用方法

### UI 界面使用

1. **选择数据表**: 从下拉框选择要清理的数据表(如"产品基本信息表")
2. **刷新数据**: 点击"刷新"按钮加载数据
3. **选择记录**: 勾选要删除的记录(支持全选/取消/反选)
4. **删除选中**: 点击"删除选中"按钮执行级联删除
5. **确认操作**: 系统会显示确认对话框,确认后执行删除

### 代码调用

```csharp
// 1. 创建清理引擎
var engine = new DataCleanupEngine();

// 2. 准备要清理的ID列表
var customerIds = new List<long> { 1001, 1002, 1003 };

// 3. 执行级联删除
var result = await engine.ExecuteCascadeDeleteAsync<tb_CRM_Customer>(customerIds);

// 4. 查看结果
if (result.IsSuccess)
{
    Console.WriteLine($"成功删除 {result.TotalDeletedCount} 条主记录及其关联数据");
}
else
{
    Console.WriteLine($"删除失败: {result.ErrorMessage}");
}
```

### 监听进度和日志

```csharp
var engine = new DataCleanupEngine();

// 订阅进度事件
engine.OnProgressChanged += (sender, e) => 
{
    progressBar.Value = e.Percentage;
    lblStatus.Text = e.Message;
};

// 订阅日志事件
engine.OnLog += (sender, log) => 
{
    txtLog.AppendText(log + Environment.NewLine);
};

// 执行清理
await engine.ExecuteCascadeDeleteAsync<tb_Prod>(productIds);
```

## 工作原理

### 调用链

```
DataCleanupEngine.ExecuteCascadeDeleteAsync<T>()
    ↓
使用 TransactionScope 包装(测试模式会回滚)
    ↓
BaseController<T>.BaseDeleteByNavAsync(entity)  ← 复用现有方法
    ↓
SqlSugar DeleteNav().Include().ExecuteCommandAsync()  ← 真正级联删除
    ↓
测试模式: 事务回滚 | 正式模式: 事务提交
```

### 级联删除流程(以删除客户为例)

1. **用户选择**: 选择要删除的客户 ID: [1001]

2. **调用引擎**:
   ```csharp
   await engine.ExecuteCascadeDeleteAsync<tb_CRM_Customer>(new[] { 1001 }, isTestMode: false);
   ```

3. **内部处理**:
   ```csharp
   using (var scope = new TransactionScope(TransactionScopeAsyncFlowOption.Enabled))
   {
       var entity = new tb_CRM_Customer { Customer_id = 1001 };
       
       // 调用 Controller 的 BaseDeleteByNavAsync
       await controller.BaseDeleteByNavAsync(entity);
       // 内部执行:
       // DELETE FROM tb_CRM_Contacts WHERE Customer_id = 1001
       // DELETE FROM tb_CRM_FollowUpPlans WHERE Customer_id = 1001
       // DELETE FROM tb_CRM_FollowUpRecords WHERE Customer_id = 1001
       // DELETE FROM tb_CRM_Collaborator WHERE Customer_id = 1001
       // DELETE FROM tb_CRM_Customer WHERE Customer_id = 1001
       
       if (isTestMode)
       {
           // 测试模式:不调用 scope.Complete(),事务自动回滚
       }
       else
       {
           scope.Complete(); // 正式模式:提交事务
       }
   }
   ```

4. **结果**:
   - 测试模式:统计出将删除 11 条记录,但数据未实际删除 ✅
   - 正式模式:真正删除 11 条记录并提交 ✅

## 注意事项

### 1. 确保 Controller 存在
每个实体必须有对应的 Controller:
```
tb_CRM_Customer → tb_CRM_CustomerController
tb_Prod → tb_ProdController
```

### 2. 确保导航属性正确配置
在 Model 中正确配置导航属性:
```csharp
[SugarTable("tb_CRM_Customer")]
public class tb_CRM_Customer : BaseEntity
{
    [SugarColumn(IsPrimaryKey = true)]
    public long CustomerID { get; set; }
    
    // 导航属性 - 一对多
    [Navigate(NavigateType.OneToMany, nameof(tb_CRM_Contact.CustomerID))]
    public List<tb_CRM_Contact> tb_CRM_Contacts { get; set; }
}
```

### 3. 测试模式优先
建议先使用测试模式查看影响范围:
```csharp
// 先测试
var preview = await engine.ExecuteCascadeDeleteAsync<tb_Prod>(ids, isTestMode: true);
Console.WriteLine($"将删除 {preview.TotalDeletedCount} 条记录");

// 确认后正式执行
if (Confirm())
{
    await engine.ExecuteCascadeDeleteAsync<tb_Prod>(ids, isTestMode: false);
}
```

## 性能优化建议

### 批量处理
如果要删除大量数据,建议分批处理:

```csharp
var allIds = GetIdsToDelete(); // 假设有 10000 条
int batchSize = 100;

for (int i = 0; i < allIds.Count; i += batchSize)
{
    var batch = allIds.Skip(i).Take(batchSize).ToList();
    await engine.ExecuteCascadeDeleteAsync<tb_Prod>(batch);
    
    // 每批之间稍作停顿,避免数据库压力过大
    await Task.Delay(100);
}
```

### 事务控制
如果需要在单个事务中删除多个相关实体:

```csharp
using (var scope = new TransactionScope(TransactionScopeAsyncFlowOption.Enabled))
{
    await engine.ExecuteCascadeDeleteAsync<tb_SaleOrder>(orderIds);
    await engine.ExecuteCascadeDeleteAsync<tb_SaleOrderDetail>(detailIds);
    
    scope.Complete(); // 提交事务
}
```
