# 基本数据清理 - 使用说明

## 设计理念

**复用现有架构,避免重复造轮子**

本项目已经有完善的单据查询和删除架构(`BaseListGeneric<T>`, `BaseController<T>`),我们直接复用这些成熟的功能,而不是重新实现复杂的级联删除逻辑。

## 核心优势

### 1. 强类型支持
通过泛型 `<T>` 直接指定 `RUINORERP.Model` 中的实体类:
```csharp
// 清理客户数据
await engine.ExecuteCascadeDeleteAsync<tb_CRM_Customer>(customerIds);

// 清理产品数据  
await engine.ExecuteCascadeDeleteAsync<tb_Prod>(productIds);
```

### 2. 自动级联删除
利用现有的 `BaseController.BaseDeleteByNavAsync()` 方法,该方法内部使用 SqlSugar 的 `DeleteNav()`:
```csharp
// BaseController 中的实现
bool rs = await _db.DeleteNav<tb_CRM_Customer>(m => m.CustomerID == entity.CustomerID)
    .Include(m => m.tb_CRM_Contacts)        // 自动包含联系人
    .Include(m => m.tb_CRM_FollowUpRecords) // 自动包含跟进记录
    .ExecuteCommandAsync();
```

### 3. 无需手动配置关系
- ❌ 不需要手动定义外键关系
- ❌ 不需要编写复杂的表达式树
- ❌ 不需要维护级联删除计划

✅ 只需要在 Model 中使用导航属性,系统自动识别

## 使用方法

### UI 界面使用

1. **选择数据表**: 从下拉框选择要清理的数据表(如"产品基本信息表")
2. **刷新数据**: 点击"刷新"按钮加载数据
3. **选择记录**: 勾选要删除的记录(支持全选/取消/反选)
4. **删除选中**: 点击"删除选中"按钮执行级联删除
5. **确认操作**: 系统会显示确认对话框,确认后执行删除

### 代码调用

```csharp
// 1. 创建清理引擎
var engine = new DataCleanupEngine();

// 2. 准备要清理的ID列表
var customerIds = new List<long> { 1001, 1002, 1003 };

// 3. 执行级联删除
var result = await engine.ExecuteCascadeDeleteAsync<tb_CRM_Customer>(customerIds);

// 4. 查看结果
if (result.IsSuccess)
{
    Console.WriteLine($"成功删除 {result.TotalDeletedCount} 条主记录及其关联数据");
}
else
{
    Console.WriteLine($"删除失败: {result.ErrorMessage}");
}
```

### 监听进度和日志

```csharp
var engine = new DataCleanupEngine();

// 订阅进度事件
engine.OnProgressChanged += (sender, e) => 
{
    progressBar.Value = e.Percentage;
    lblStatus.Text = e.Message;
};

// 订阅日志事件
engine.OnLog += (sender, log) => 
{
    txtLog.AppendText(log + Environment.NewLine);
};

// 执行清理
await engine.ExecuteCascadeDeleteAsync<tb_Prod>(productIds);
```

## 工作原理

### 调用链

```
DataCleanupEngine.ExecuteCascadeDeleteAsync<T>()
    ↓
BaseController<T>.BaseDeleteByNavAsync(entity)  ← 复用现有方法
    ↓
SqlSugar DeleteNav().Include().ExecuteCommandAsync()  ← 自动级联
```

### 级联删除流程

以删除客户为例:

1. **用户选择**: 选择要删除的客户 ID: [1001, 1002]

2. **调用引擎**:
   ```csharp
   await engine.ExecuteCascadeDeleteAsync<tb_CRM_Customer>(new[] { 1001, 1002 });
   ```

3. **内部处理**:
   ```csharp
   foreach (var id in targetIds)
   {
       var entity = new tb_CRM_Customer { CustomerID = id };
       await controller.BaseDeleteByNavAsync(entity);  // 自动级联
   }
   ```

4. **SqlSugar 执行**:
   ```sql
   -- 自动删除关联数据
   DELETE FROM tb_CRM_Contacts WHERE CustomerID IN (1001, 1002)
   DELETE FROM tb_CRM_FollowUpRecords WHERE CustomerID IN (1001, 1002)
   DELETE FROM tb_CRM_Customer WHERE CustomerID IN (1001, 1002)
   ```

## 注意事项

### 1. 确保 Controller 存在
每个实体必须有对应的 Controller:
```
tb_CRM_Customer → tb_CRM_CustomerController
tb_Prod → tb_ProdController
```

### 2. 确保导航属性正确配置
在 Model 中正确配置导航属性:
```csharp
[SugarTable("tb_CRM_Customer")]
public class tb_CRM_Customer : BaseEntity
{
    [SugarColumn(IsPrimaryKey = true)]
    public long CustomerID { get; set; }
    
    // 导航属性 - 一对多
    [Navigate(NavigateType.OneToMany, nameof(tb_CRM_Contact.CustomerID))]
    public List<tb_CRM_Contact> tb_CRM_Contacts { get; set; }
}
```

### 3. 测试模式优先
建议先使用测试模式查看影响范围:
```csharp
// 先测试
var preview = await engine.ExecuteCascadeDeleteAsync<tb_Prod>(ids, isTestMode: true);
Console.WriteLine($"将删除 {preview.TotalDeletedCount} 条记录");

// 确认后正式执行
if (Confirm())
{
    await engine.ExecuteCascadeDeleteAsync<tb_Prod>(ids, isTestMode: false);
}
```

## 性能优化建议

### 批量处理
如果要删除大量数据,建议分批处理:

```csharp
var allIds = GetIdsToDelete(); // 假设有 10000 条
int batchSize = 100;

for (int i = 0; i < allIds.Count; i += batchSize)
{
    var batch = allIds.Skip(i).Take(batchSize).ToList();
    await engine.ExecuteCascadeDeleteAsync<tb_Prod>(batch);
    
    // 每批之间稍作停顿,避免数据库压力过大
    await Task.Delay(100);
}
```

### 事务控制
如果需要在单个事务中删除多个相关实体:

```csharp
using (var scope = new TransactionScope(TransactionScopeAsyncFlowOption.Enabled))
{
    await engine.ExecuteCascadeDeleteAsync<tb_SaleOrder>(orderIds);
    await engine.ExecuteCascadeDeleteAsync<tb_SaleOrderDetail>(detailIds);
    
    scope.Complete(); // 提交事务
}
```
