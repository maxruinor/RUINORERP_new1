# 数据清理工具 - 最终简化（只保留数据库元数据模式）

## 📅 简化日期
2026-04-24

## ✅ 简化内容

### 1. 移除实体导航模式

**原因**：
- ❌ 实体导航模式依赖 `Navigate` 属性，很多实体类未定义
- ❌ `ChildRelations` 为空，无法找到父子表关系
- ❌ 日志显示所有表的子表关联数都是 0
- ❌ 导致外键约束冲突，删除失败

**优势**：
- ✅ 数据库元数据模式直接从 SQL Server 系统表查询外键关系
- ✅ 不依赖实体类的 Navigate 属性定义
- ✅ 覆盖全面，自动识别所有外键依赖
- ✅ 已修复关键Bug（预查询主键ID）

## 📊 对比分析

### 两种模式的差异

| 特性 | 实体导航模式 | 数据库元数据模式 |
|------|------------|----------------|
| **依赖来源** | 实体类的 `[Navigate]` 属性 | SQL Server `sys.foreign_keys` 系统表 |
| **可靠性** | ❌ 低（依赖实体定义完整性） | ✅ 高（直接读取数据库） |
| **覆盖范围** | ❌ 部分（只有定义了Navigate的表） | ✅ 全部（所有外键关系） |
| **维护成本** | ❌ 高（需要同步更新实体类） | ✅ 低（自动识别） |
| **性能** | ⚠️ 中 | ✅ 好（预查询+批量删除） |
| **调试难度** | ❌ 难（ChildRelations为空） | ✅ 易（日志清晰） |
| **推荐度** | ❌ 不推荐 | ✅ 强烈推荐 |

## 🎯 核心优势

### 数据库元数据模式的优势

1. **自动识别外键关系**
   ```sql
   SELECT 
       fk.name AS ForeignKeyName,
       tp.name AS ReferencingTableName,     -- 子表
       cp.name AS ReferencingColumnName,    -- 外键列
       tr.name AS ReferencedTableName,      -- 父表
       cr.name AS ReferencedColumnName      -- 主键列
   FROM sys.foreign_keys AS fk
   INNER JOIN sys.foreign_key_columns AS fkc 
       ON fk.object_id = fkc.constraint_object_id
   INNER JOIN sys.tables AS tp 
       ON fkc.parent_object_id = tp.object_id
   INNER JOIN sys.columns AS cp 
       ON fkc.parent_object_id = cp.object_id AND fkc.parent_column_id = cp.column_id
   INNER JOIN sys.tables AS tr 
       ON fkc.referenced_object_id = tr.object_id
   INNER JOIN sys.columns AS cr 
       ON fkc.referenced_object_id = cr.object_id AND fkc.referenced_column_id = cr.column_id
   WHERE tr.name = @tableName
   ```

2. **预查询主键ID，避免外键冲突**
   ```csharp
   // 步骤1：递归查询所有层级的主键ID
   var idMap = new Dictionary<string, List<long>>();
   await QueryChildIds(rootTableName, targetIds, rootPkColumn);
   
   // 步骤2：按顺序删除（使用正确的主键ID）
   foreach (var tableInfo in deleteOrder)
   {
       var idsToDelete = idMap[tableInfo.TableName];
       await db.Deleteable<object>()
           .AS(tableInfo.TableName)
           .Where($"[{primaryKeyColumn}] IN (@ids)", new { ids = idsToDelete })
           .ExecuteCommandAsync();
   }
   ```

3. **拓扑排序确保正确的删除顺序**
   ```
   tb_SaleOut (根表)
     ├─ tb_SaleOutRe (阶段3)
     │   └─ tb_SaleOutReDetail (阶段1) ← 先删最深层
     └─ tb_SaleOutDetail (阶段2)
   ```

## 📝 使用流程

### 简化后的操作流程

```
1. 选择实体类型（如：tb_SaleOut）
   ↓
2. 刷新数据
   ↓
3. 勾选要删除的记录
   ↓
4. 选择执行模式：
   - 数据预览（测试模式，不删除）
   - 测试执行（事务回滚）
   - 正式执行（提交事务）
   ↓
5. 点击"删除选中"
   ↓
6. 查看日志确认结果
```

**注意**：不再需要选择"删除方式"，系统自动使用数据库元数据模式。
