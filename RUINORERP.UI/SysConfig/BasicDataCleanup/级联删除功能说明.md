# 级联删除功能说明

## 功能概述

实现了基于外键约束的级联数据删除功能，支持可视化的级联关系展示和阶梯式删除操作。

## 核心特性

### 1. 级联关系分析
- 自动分析实体间的外键依赖关系
- 生成正确的删除顺序（先删除子表，再删除主表）
- 支持多层级联关系追溯

### 2. 可视化日志显示
- 层级式显示删除流程
- 每层显示关键信息（主键值、名称字段）
- 实时状态反馈

### 3. 删除流程

#### 3.1 删除顺序
1. **第一层**：最底层的单据数据（如销售订单、采购订单等）
2. **第二层**：产品详情数据（tb_ProdDetail）
3. **第三层**：产品资料数据（tb_Prod）
4. **第四层**：供应商/客户数据（tb_CustomerVendor）

#### 3.2 删除逻辑
```
选择供应商记录
  ↓
查找所有引用该供应商的产品（通过 VendorID 等外键）
  ↓
查找所有产品详情（通过 ProdID 外键）
  ↓
查找所有相关单据（通过 ProdID、VendorID 等外键）
  ↓
按顺序逐级删除
```

## 使用方法

### 步骤 1：选择要删除的记录
1. 在数据清理界面选择实体类型（如"供应商表"）
2. 点击"数据预览"按钮加载数据
3. 勾选要删除的记录（支持全选、反选）

### 步骤 2：查看级联关系
1. 点击"查看级联关系"按钮
2. 系统自动分析并显示所有关联数据
3. 查看每一层级的数据详情

### 步骤 3：执行删除
1. 确认级联关系无误
2. 点击"开始删除"按钮
3. 系统按层级顺序依次删除
4. 实时查看删除进度和状态

## 界面说明

### 级联删除日志窗体 (FrmCascadeDeleteLog)

#### 表格列说明
| 列名 | 说明 |
|------|------|
| 层级 | 删除顺序层级（数字越小越先删除） |
| 实体 | 实体名称（带层级缩进） |

| 主键值 | 第一条记录的主键值 |
| 名称 | 第一条记录的名称字段值 |
| 外键字段 | 关联的外键字段名 |
| 记录数 | 待删除的记录数量 |
| 状态 | 当前状态（待删除/已删除/错误） |

#### 按钮说明
- **查看级联关系**：分析并显示关联数据
- **开始删除**：执行级联删除操作
- **导出日志**：将删除日志复制到剪贴板
- **关闭**：关闭窗体

## 技术实现

### 关键类

1. **EntityRelationshipAnalyzer**
   - 分析实体间的外键关系
   - 生成拓扑排序的删除顺序
   - 提供外键字段查询方法

2. **FrmCascadeDeleteLog**
   - 显示级联删除日志
   - 执行级联删除操作
   - 提供可视化反馈

3. **CascadeDeleteLogEntry**
   - 日志条目数据结构
   - 包含层级、表名、记录数等信息

### 核心算法

#### 1. 拓扑排序生成删除顺序
```csharp
public List<CleanupOrderNode> GenerateCleanupOrder(Type targetEntityType)
{
    // 找到所有依赖目标表的表
    FindDependentTables(targetEntityType, dependentTables, visited);
    
    // 拓扑排序，确保先删除依赖表
    Visit(dependentType);
    
    // 最后删除目标表
    Visit(targetEntityType);
}
```

#### 2. 外键关系分析
```csharp
public string GetForeignKeyField(Type relatedEntityType, Type mainEntityType)
{
    var info = GetEntityRelationship(relatedEntityType);
    var fkRelation = info.ForeignKeyRelations
        .FirstOrDefault(fk => fk.ReferencedTableName == mainTableName);
    return fkRelation?.ForeignKeyColumn;
}
```

#### 3. 级联删除执行
```csharp
private async Task DeleteSelectedRecordsAsync(List<long> selectedIds)
{
    var cleanupOrder = analyzer.GenerateCleanupOrder(_selectedEntityType);
    
    foreach (var node in cleanupOrder)
    {
        if (node.EntityType == _selectedEntityType)
        {
            // 目标表：根据 ID 删除
            await DeleteByIdsAsync(node.EntityType, selectedIds);
        }
        else
        {
            // 关联表：根据外键删除
            await DeleteRelatedAsync(node.EntityType, _selectedEntityType, selectedIds);
        }
    }
}
```

## 数据模型支持

### 外键特性
系统使用 `FKRelationAttribute` 标识外键关系：

```csharp
[FKRelation(FKTableName = "tb_CustomerVendor", FK_IDColName = "ID")]
public long VendorID { get; set; }
```

### 支持的实体
- tb_CustomerVendor（供应商/客户）
- tb_Prod（产品基本信息）
- tb_ProdDetail（产品详情）
- tb_ProdCategories（产品类目）
- tb_ProdProperty（产品属性）
- tb_ProdPropertyValue（产品属性值）
- tb_Location（库位）
- tb_StorageRack（货架）
- tb_Unit（单位）
- tb_ProductType（产品类型）
- tb_Department（部门）

## 安全保障

1. **二次确认**：删除前显示详细信息并要求确认
2. **事务支持**：可配置事务保证数据一致性
3. **状态反馈**：实时显示删除进度和状态
4. **错误处理**：异常时显示详细错误信息
5. **日志导出**：支持导出删除日志便于审计

## 注意事项

1. **数据备份**：删除前务必备份重要数据
2. **权限控制**：确保操作人员具有相应权限
3. **业务验证**：确认删除不会影响正在进行业务
4. **外键约束**：系统自动处理，但需确保数据模型正确配置

## 扩展建议

1. **批量操作优化**：对于大量数据可采用分批删除
2. **异步日志**：使用异步日志记录提高性能
3. **进度条显示**：添加进度条显示整体进度
4. **撤销功能**：实现回收站机制支持数据恢复
