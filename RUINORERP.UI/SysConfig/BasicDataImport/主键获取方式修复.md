# 主键获取方式修复

## 🔍 问题分析

### ❌ 错误的方式（修复前）

**位置**：`DynamicImporter.cs` 第995-1000行

```csharp
private string GetPrimaryKeyFieldName(Type entityType)
{
    // 从entityInfo获取主键字段
    var entityInfo = _entityInfoService.GetEntityInfoByTableName(entityType.Name);
    return entityInfo?.IdField ?? "ID";
}
```

**问题**：
1. **使用了错误的服务**：`_entityInfoService` 是单据编号和ID映射服务，不是实体元数据服务
2. **职责混淆**：该服务用于管理单据编号生成规则，不应该用于获取实体主键
3. **不可靠**：如果实体不在该服务的注册表中，会返回 `null`，导致使用硬编码的 `"ID"`
4. **违反规范**：违反了"动态获取实体主键规范"，应该使用项目提供的通用方法

---

## ✅ 正确的方式（修复后）

### 方案对比

| 优先级 | 方案 | 说明 | 可靠性 |
|--------|------|------|--------|
| 1️⃣ | **EntityRegistry** | 结构化实体元数据注册中心 | ⭐⭐⭐⭐⭐ |
| 2️⃣ | **BaseEntity.GetPrimaryKeyColName()** | 基类动态获取主键方法 | ⭐⭐⭐⭐ |
| 3️⃣ | **反射 SugarColumn 特性** | 直接从属性特性读取 | ⭐⭐⭐ |
| 4️⃣ | **默认值 "ID"** | 最终降级方案 | ⭐ |

### 修复后的代码

```csharp
/// <summary>
/// 获取实体类型的主键字段名
/// 【修复】使用 EntityRegistry 或 BaseEntity 动态获取主键，禁止硬编码
/// </summary>
/// <param name="entityType">实体类型</param>
/// <returns>主键字段名（数据库列名）</returns>
private string GetPrimaryKeyFieldName(Type entityType)
{
    if (entityType == null)
    {
        return "ID"; // 默认值
    }

    // 优先从 EntityRegistry 获取（结构化元数据）
    var metadata = RUINORERP.Model.EntityRegistry.Entities
        .FirstOrDefault(e => e.EntityType == entityType);
    
    if (metadata != null && !string.IsNullOrEmpty(metadata.PrimaryKeyName))
    {
        return metadata.PrimaryKeyName;
    }

    // 降级方案：使用 BaseEntity 的动态方法获取主键
    // 创建临时实例调用 GetPrimaryKeyColName()
    try
    {
        if (typeof(RUINORERP.Model.BaseEntity).IsAssignableFrom(entityType))
        {
            var instance = Activator.CreateInstance(entityType) as RUINORERP.Model.BaseEntity;
            if (instance != null)
            {
                string pkColName = instance.GetPrimaryKeyColName();
                if (!string.IsNullOrEmpty(pkColName))
                {
                    return pkColName;
                }
            }
        }
    }
    catch (Exception ex)
    {
        System.Diagnostics.Debug.WriteLine($"通过 BaseEntity 获取主键失败: {ex.Message}");
    }

    // 最后的降级方案：尝试从 SugarColumn 特性获取
    try
    {
        var pkProperty = entityType.GetProperties()
            .FirstOrDefault(p => p.GetCustomAttribute<SqlSugar.SugarColumn>()?.IsPrimaryKey == true);
        
        if (pkProperty != null)
        {
            var sugarColumn = pkProperty.GetCustomAttribute<SqlSugar.SugarColumn>();
            return sugarColumn?.ColumnName ?? pkProperty.Name;
        }
    }
    catch (Exception ex)
    {
        System.Diagnostics.Debug.WriteLine($"通过反射获取主键失败: {ex.Message}");
    }

    // 最终降级：返回默认值
    return "ID";
}
```

### 2. BaseEntity.GetPrimaryKeyColName()（备用）

**文件**：`RUINORERP.Model/Base/BaseEntity.cs`

**特点**：
- ✅ 所有继承自 `BaseEntity` 的实体都可用
- ✅ 动态反射，适应运行时变化
- ✅ 符合"动态获取实体主键规范"
- ⚠️ 需要创建实例，有轻微性能开销

**实现**（第795-818行）：
```csharp
public string GetPrimaryKeyColName()
{
    string PrimaryKeyColName = string.Empty;
    // 获取需要跟踪的属性列表（使用缓存）
    var properties = GetCachedProperties();
    // 记录所有需要跟踪的属性的当前值
    foreach (var property in properties)
    {
        var sugarColumnAttr = property.GetCustomAttribute<SugarColumn>(false);
        if (sugarColumnAttr != null)
        {
            if (sugarColumnAttr.IsPrimaryKey)
            {
                PrimaryKeyColName = sugarColumnAttr.ColumnName;
                return PrimaryKeyColName;
            }
        }
    }
    return PrimaryKeyColName;
}
```

**使用场景**：
- EntityRegistry 中未注册的实体
- 需要最准确的主键信息（实时反射）

### 3. 反射 SugarColumn 特性（兜底）

**特点**：
- ✅ 不依赖任何服务或基类
- ✅ 直接读取 SqlSugar 特性
- ⚠️ 性能较差（每次都要反射）
- ⚠️ 只能获取主键，无法获取其他元数据

**实现**：
```csharp
var pkProperty = entityType.GetProperties()
    .FirstOrDefault(p => p.GetCustomAttribute<SqlSugar.SugarColumn>()?.IsPrimaryKey == true);

if (pkProperty != null)
{
    var sugarColumn = pkProperty.GetCustomAttribute<SqlSugar.SugarColumn>();
    return sugarColumn?.ColumnName ?? pkProperty.Name;
}
```

**使用场景**：
- 非 `BaseEntity` 继承的实体
- 极端情况下的最后手段

## 🎯 为什么之前的方式是错误的？

### `_entityInfoService` 的真实用途

**服务名称**：`IEntityMappingService` / `ClientBizCodeService`

**实际功能**：
- 管理单据编号生成规则
- 管理实体ID与业务编号的映射关系
- 用于单据创建时的编号分配

**示例**：
```csharp
// 这不是用来获取主键的！
var entityInfo = _entityInfoService.GetEntityInfoByTableName("tb_SaleOrder");
// 返回的是单据编号配置，如：SO-20260426-0001
// 而不是主键字段名 "SaleOrderID"
```

**错误原因**：
1. **职责不符**：该服务关注的是"业务编号"，不是"数据库主键"
2. **覆盖不全**：只注册了需要编号生成的单据，不包含所有实体
3. **返回值错误**：`IdField` 可能为空或不准确

## 🎉 总结

这种设计确保了：
- ✅ **准确性**：优先使用权威的元数据源
- ✅ **兼容性**：多级降级，适应各种情况
- ✅ **规范性**：完全符合项目开发规范
