# 基础数据导入工具使用指南

## 📋 概述

基础数据导入工具支持两种导入模式：**单表导入**和**宽表导入**。本指南将帮助您理解两种模式的区别，并正确使用该工具。

---

## 🎯 两种导入模式对比

### 1️⃣ 单表导入（推荐新手）

**适用场景：**
- 一个Excel工作表只包含一个数据库表的数据
- 例如：供应商表、产品类目表、单位表等

**特点：**
- ✅ 简单易用，适合初学者
- ✅ 需要配置列映射关系（Excel列 → 数据库字段）
- ✅ 可以保存和复用映射配置
- ✅ 支持外键自动解析

**典型流程：**
```
选择文件 → 选择实体类型 → 配置列映射 → 解析数据 → 导入数据库
```

**示例：**
假设您有一个Excel文件 `供应商.xlsx`，包含以下列：
- A列：供应商编号
- B列：供应商名称
- C列：联系电话

您需要将这些数据导入到 `tb_CustomerVendor` 表中，就需要使用单表导入模式。

---

### 2️⃣ 宽表导入（高级功能）

**适用场景：**
- 一个Excel工作表包含多个关联表的数据
- 例如：产品信息 + 产品属性 + 产品图片等多张表的数据

**特点：**
- ⚡ 一次导入多张关联表
- ⚡ 使用预定义的Profile配置（无需每次配置）
- ⚡ 自动拆分数据到不同的表
- ⚠️ 需要提前创建Profile配置文件

**典型流程：**
```
选择文件 → 选择宽表Profile → 直接导入数据库
```

**示例：**
假设您有一个Excel文件 `产品宽表.xlsx`，包含以下列：
- A-D列：产品基本信息（导入 tb_Prod 表）
- E-H列：产品详情信息（导入 tb_ProdDetail 表）
- I-L列：产品属性信息（导入 tb_ProdProperty 表）

这种情况下，使用宽表导入模式可以一次性完成所有数据的导入。

---

## 🚀 快速开始

### 第一步：选择导入模式

打开工具后，在界面顶部会看到**导入模式选择器**：

```
导入模式: ○ 单表导入    ○ 宽表导入
         ↑              ↑
      (默认选中)     (高级功能)
```

**如何选择？**
- 🟢 **新手用户**：保持默认的"单表导入"即可
- 🔵 **有经验用户**：如果有宽表Profile配置，可以选择"宽表导入"

---

### 第二步：根据模式进行操作

#### 📌 单表导入模式操作流程

1. **选择Excel文件**
   - 点击"浏览"按钮选择Excel文件
   - 系统会自动加载文件中的工作表列表

2. **选择实体类型**
   - 从下拉框选择要导入的数据库表
   - 例如："供应商表"、"产品类目表"等

3. **配置列映射**（首次使用时）
   - 点击"解析配置"按钮
   - 在弹出的窗口中，将Excel列映射到数据库字段
   - 保存映射配置（可以命名以便下次使用）

4. **选择映射配置**
   - 从"解析配置"下拉框选择之前保存的配置
   - 如果是首次使用，需要先执行第3步

5. **解析数据**
   - 点击"解析"按钮
   - 系统会根据映射配置转换数据
   - 可以在"解析后数据"标签页查看结果

6. **生成结果预览**（可选）
   - 点击"生成结果预览"按钮
   - 系统会解析外键引用（如供应商ID、类目ID等）
   - 这一步是可选的，可以直接跳过进行导入

7. **导入数据库**
   - 点击"导入"按钮
   - 系统会将数据保存到数据库
   - 导入完成后会显示成功/失败统计

---

#### 📌 宽表导入模式操作流程

1. **选择Excel文件**
   - 点击"浏览"按钮选择Excel文件
   - 注意：文件格式必须符合Profile定义

2. **选择宽表Profile**
   - 从"解析配置"下拉框选择宽表Profile
   - Profile名称以"[宽表]"开头
   - 例如："[宽表] 04_产品宽表导入"

3. **直接导入**
   - 点击"导入"按钮
   - 系统会自动根据Profile配置拆分数据并导入多张表
   - 无需配置映射，无需解析数据

## ❓ 常见问题

### Q1: 我应该选择哪种模式？

**A:** 
- 如果您的Excel文件只包含一张表的数据 → 选择**单表导入**
- 如果您的Excel文件包含多张关联表的数据 → 选择**宽表导入**
- 不确定时，选择**单表导入**（更安全）

### Q2: 为什么我看不到"[宽表]"开头的配置？

**A:** 
- 宽表Profile需要预先创建
- 检查目录：`SysConfig/DataMigration/Profiles/`
- 如果没有Profile文件，请联系管理员创建

### Q3: 单表导入时，为什么要配置列映射？

**A:** 
- Excel的列名和数据库字段名通常不一致
- 例如：Excel列叫"供应商名称"，数据库字段叫"CustomerName"
- 列映射告诉系统如何对应这些字段
- 配置一次后可以保存，下次直接使用

### Q4: "生成结果预览"是必须的吗？

**A:** 
- **不是必须的**
- 它的作用是解析外键引用（如将"供应商名称"转换为"供应商ID"）
- 如果您的数据不包含外键，可以跳过这一步
- 如果包含外键，建议先预览确认解析正确

### Q5: 切换模式后，之前的配置会丢失吗？

**A:** 
- 不会丢失
- 单表模式的映射配置会保存在配置文件中
- 宽表模式的Profile配置也是独立的
- 切换模式只是改变当前使用的配置类型

## 💡 最佳实践

### 单表导入最佳实践

1. **首次使用前**
   - 准备一个示例Excel文件
   - 了解目标数据库表的结构
   - 规划好列映射关系

2. **配置映射时**
   - 为映射配置起一个有意义的名称
   - 例如："供应商导入_标准格式"
   - 方便后续查找和使用

3. **导入前检查**
   - 先解析数据，查看是否有错误
   - 对于包含外键的数据，先生成预览
   - 确认无误后再导入

4. **批量导入**
   - 如果有多个相同格式的Excel文件
   - 可以先配置好映射，然后依次导入
   - 无需重复配置

### 宽表导入最佳实践

1. **使用前准备**
   - 确保Profile配置已创建
   - 了解Profile的数据格式要求
   - 准备符合格式的Excel文件

2. **数据格式检查**
   - 严格按照Profile定义的列顺序
   - 不要随意增删列
   - 确保数据类型正确

3. **测试导入**
   - 先用小批量数据测试
   - 确认导入成功后再处理大批量数据

## 🔧 故障排除

### 问题1: 选择文件后，"解析"按钮仍然是灰色的

**可能原因：**
- 未选择实体类型
- 未配置或未选择映射配置

**解决方法：**
1. 确保已选择实体类型
2. 点击"解析配置"按钮配置映射
3. 或者从下拉框选择已保存的配置

### 问题2: 切换模式后，下拉框内容没有变化

**可能原因：**
- 界面刷新延迟

**解决方法：**
- 稍等片刻，系统会自动刷新
- 或者重新选择一次模式

### 问题3: 宽表模式下无法选择实体类型

**这是正常的！**
- 宽表模式不需要选择实体类型
- Profile配置中已经定义了所有相关的表
- 实体类型选择框会被禁用
