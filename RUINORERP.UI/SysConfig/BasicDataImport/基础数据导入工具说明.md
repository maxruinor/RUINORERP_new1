# 基础数据导入工具说明

## 📌 工具定位

**作用**: 项目初始化时，用于批量导入基础数据（客户、供应商、产品分类、商品等）  
**使用频率**: ⭐⭐ 低频（仅在项目初期或数据迁移时使用）  
**核心价值**: 提高数据录入效率，避免手动逐条输入
**设计原则**: **简单实用** - 专注于功能实现，避免过度设计和复杂特性

---

## 📋 支持的数据类型

| 数据类型 | 说明 | 特殊处理 |
|---------|------|----------|
| 客户/供应商 | tb_CustomerVendor | 自动生成编号、默认值设置 |
| 产品分类 | tb_ProdCategories | 层级关系、排序号自动递增 |
| 商品主档 | tb_Prod | 编号生成、单位转换 |
| 商品明细 | tb_ProdDetail | SKU生成、图片关联 |
| 业务员 | tb_Salesman | - |
| 仓库 | tb_Stock | - |

## 🚀 使用流程

### 步骤1: 选择Excel文件
- 点击"选择文件"按钮
- 系统自动预览原始数据（前100行）
- 显示行数统计和列信息

### 步骤2: 配置导入参数
- **字段映射**: 将Excel列映射到数据库字段
  - 支持文本、数字、日期、外键、图片等多种类型
  - 可保存/加载映射配置（XML格式）
- **图片设置**: 配置图片列的压缩和质量
- **验证规则**: 设置必填字段、唯一性约束等
- **存在性策略**: 选择重复数据处理方式（跳过/报错）

### 步骤3: 解析并预览
- 点击"生成结果预览"按钮
- 系统执行：
  1. ✅ Excel数据解析（按需读取列，优化性能）
  2. ✅ 外键值转换（缓存机制，加速查询）
  3. ✅ 数据预处理（统一入口，避免重复处理）
  4. ✅ 实体特殊字段处理（编号生成、默认值等）
  5. ✅ 数据验证（FluentValidation体系）
- 在表格中显示最终导入结果（含错误提示）

### 步骤4: 批量导入
- 点击"确认导入"按钮
- 系统执行：
  1. ✅ 唯一性检查（批量查询优化）
  2. ✅ 事务控制（保证数据一致性）
  3. ✅ 批量插入/更新（SqlSugar Storageable）
  4. ✅ 图片文件保存（压缩、调整大小）
  5. ✅ 进度实时反馈
- 显示导入结果统计（成功/失败数量）

## ✨ 核心特性

### 1. 智能外键处理
- 自动识别外键字段（如"客户名称"→"客户ID"）
- 缓存机制加速查询（ConcurrentDictionary）
- 支持多级外键（如分类→父分类）

### 2. 图片优化
- 自动压缩图片（最大1024x1024，质量85%）
- 按商品编码命名（避免冲突）
- 支持Excel中的嵌入图片和路径引用

### 3. 性能优化
- 按需读取Excel列（减少内存占用）
- 批量查询数据库（减少IO次数）
- 异步处理（UI不卡顿）

### 4. 数据验证
- 复用FluentValidation体系（与业务层一致）
- 实时错误提示（红色标记问题单元格）
- 唯一性检查（避免重复导入）

### 5. 配置持久化
- 映射配置保存到XML（可复用）
- 快速切换不同导入场景

## ⚠️ 注意事项

1. **Excel格式要求**
   - 第一行为列标题
   - 图片列需标注"[图片]"
   - 外键列需配置映射关系

2. **性能建议**
   - 单次导入建议不超过10,000行
   - 大数据量建议分批导入
   - 图片文件建议预先压缩

3. **数据备份**
   - 导入前请备份数据库
   - 建议在测试环境先验证
   - 使用事务保证数据一致性
