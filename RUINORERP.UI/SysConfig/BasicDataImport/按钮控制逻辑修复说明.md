# 基础数据导入按钮控制逻辑修复

## 问题描述

用户反馈"解析"和"导入"按钮不可用，界面显示混乱。

## 问题原因分析

### 1. 按钮控制逻辑混乱

之前的逻辑要求用户必须按照以下步骤操作：
1. 浏览选择文件
2. 点击"解析配置"配置映射
3. 点击"解析"按钮
4. **必须**点击"生成结果预览"按钮
5. 才能点击"导入"按钮

这导致：
- **导入按钮默认禁用**，只有点击"生成结果预览"后才启用
- 用户不理解为什么解析后不能直接导入
- 普通模式和宽表模式的按钮状态没有明确区分

### 2. 宽表模式处理不当

宽表模式（选择`[宽表] xxx`配置时）：
- "解析"按钮被禁用（正确）
- "导入"按钮也被禁用（错误）
- 实际上宽表模式有自己的导入流程，应该可以直接导入

## 修复方案

### 修复1：简化导入按钮启用逻辑

**修改前：**
```csharp
// 导入按钮：需要先生成结果预览
kbtnDynamicImport.Enabled = false;  // 默认禁用，只有生成预览后才启用
```

**修改后：**
```csharp
// 导入按钮：需要在解析后或生成预览后才能启用
// 注意：解析后就应该可以直接导入，不一定需要点"生成结果预览"
if (_parsedImportData != null && _parsedImportData.Rows.Count > 0)
{
    // 有解析数据就可以导入，不需要强制点"生成结果预览"
    kbtnDynamicImport.Enabled = true;
}
else
{
    kbtnDynamicImport.Enabled = false;
}
```

**效果：**
- 解析完成后，导入按钮自动启用
- "生成结果预览"变成可选步骤（用于查看外键解析后的效果）
- 用户体验更流畅

### 修复2：完善宽表模式按钮状态

**修改前：**
```csharp
// 宽表模式不需要选择实体类型和映射配置
kbtnDynamicParse.Enabled = false;  // 禁用解析按钮(宽表不需要预览解析)
kbtnDynamicMap.Enabled = false;    // 禁用映射配置按钮
// 缺少 kbtnGeneratePreview 和 kbtnDynamicImport 的状态设置
```

**修改后：**
```csharp
// 宽表模式不需要选择实体类型和映射配置
kbtnDynamicParse.Enabled = false;  // 禁用解析按钮(宽表不需要预览解析)
kbtnDynamicMap.Enabled = false;    // 禁用映射配置按钮
kbtnGeneratePreview.Enabled = false; // 禁用预览按钮
kbtnDynamicImport.Enabled = true;  // 宽表可以直接导入
```

**效果：**
- 宽表模式下，"导入"按钮可用
- 用户选择宽表配置后可以直接导入

### 修复3：重构 UpdateMappingControlStates 方法

将宽表模式和普通模式的按钮控制逻辑完全分离：

```csharp
private void UpdateMappingControlStates()
{
    bool hasEntityType = _selectedEntityType != null;
    bool hasRawData = _rawExcelData != null && _rawExcelData.Rows.Count > 0;
    bool hasMappingSelected = kcmbDynamicMappingName.SelectedIndex > 0;
    bool isWideTable = _isWideTableMode;

    // 宽表模式：禁用解析和映射配置按钮
    if (isWideTable)
    {
        kbtnDynamicParse.Enabled = false;
        kbtnDynamicMap.Enabled = false;
        kbtnGeneratePreview.Enabled = false;
        kbtnDynamicImport.Enabled = true;  // 宽表可以直接导入
        return;
    }

    // 普通模式：根据条件启用按钮
    // ...
}
```

### 修复4：优化用户提示信息

**解析完成后：**

```csharp
MainForm.Instance.ShowStatusText($"数据解析完成，共 {_parsedImportData.Rows.Count} 行数据，可以点击\"导入\"按钮直接导入");
```

## 新的按钮逻辑流程

### 普通导入模式

1. **浏览选择文件** → 显示原始数据
2. **选择实体类型** → "解析配置"按钮启用
3. **选择/配置映射** → "解析"按钮启用
4. **点击解析** → "导入"和"生成结果预览"按钮都启用
5. **可选：点击生成结果预览** → 查看外键解析后的效果
6. **点击导入** → 执行导入

### 宽表导入模式

1. **浏览选择文件** → 显示原始数据
2. **选择宽表配置**（如`[宽表] 04_产品宽表导入`）→ "导入"按钮直接启用
3. **点击导入** → 选择导入策略（一键导入/分步导入）→ 执行导入
