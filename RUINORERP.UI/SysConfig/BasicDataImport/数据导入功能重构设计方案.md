### 1.3 核心组件分析

#### 1.3.1 DynamicImporter (动态导入器)

**职责**: 根据映射配置将Excel数据导入到指定实体

**优点**:
- 支持列映射配置
- 外键自动解析
- 数据验证
- 批量插入

**缺点**:
- 单表导入，不支持一对多关系
- 无依赖解析队列
- 事务粒度较粗

#### 1.3.2 MultiAttributeProductImporter (多属性产品导入器)

**职责**: 处理产品-多属性SKU的复杂导入

**优点**:
- 支持一对多关系（产品-SKU）
- 属性提取规则
- 数据分组

**缺点**:
- 仅针对产品表，不够通用
- 与DynamicImporter代码重复
- 缺乏统一的事务管理

#### 1.3.3 ForeignKeyService (外键服务)

**职责**: 外键值获取和验证

**优点**:
- 预加载机制
- 缓存支持
- 模糊匹配

**缺点**:
- 无依赖顺序控制
- 循环依赖检测缺失

## 二、架构局限性

### 2.1 主要问题

#### 问题1: 功能分散，缺乏统一入口

```
当前状况:
- UCDataImportTool: 简单导入，无业务逻辑
- UCBasicDataImport: 完整功能，但仅支持基础数据
- MultiAttributeProductImporter: 专用导入器，无法复用

问题:
✗ 用户需要选择不同工具处理不同场景
✗ 代码重复严重
✗ 维护成本高
```

#### 问题2: 依赖关系处理不完善

```
当前状况:
- ForeignKeyService 仅处理单表外键
- 无导入队列概念
- 无法处理多级依赖（如：供应商→产品类目→产品）

问题场景:
Excel数据包含:
├── 供应商信息（新供应商）
├── 产品类目（新类目）
└── 产品信息（引用新供应商和类目）

当前处理:
✗ 导入产品时会因外键不存在而失败
✗ 需要分多次导入
✗ 用户体验差
```

#### 问题3: 一对多关系处理不通用

```
当前状况:
- MultiAttributeProductImporter 仅针对产品表
- 硬编码的产品结构逻辑
- 无法扩展到其他一对多场景（如：订单-明细）

需要的通用能力:
✓ 可配置的主从关系
✓ 灵活的数据分组策略
✓ 统一的事务管理
```

#### 问题4: 事务与错误处理不完善

```
当前状况:
- DynamicImporter 使用简单try-catch
- 失败记录仅记录到内存
- 无回滚机制
- 部分成功状态难以追踪

问题:
✗ 1000条记录导入，第500条失败，前499条已提交
✗ 无法知道哪些成功、哪些失败
✗ 无法重试失败的记录
```

#### 问题5: 配置化程度不足

```
当前状况:
- ColumnMapping 仅支持简单字段映射
- MultiAttributeImportConfig 专用配置
- 缺乏统一的导入任务配置

需要的配置能力:
✓ 导入任务定义（包含多个实体）
✓ 依赖关系配置
✓ 数据处理规则（转换、验证、过滤）
✓ 错误处理策略（跳过、回滚、记录）
```

### 3.2 核心设计原则

#### 原则1: 配置驱动

所有导入行为通过配置定义，无需修改代码即可支持新场景。

#### 原则2: 依赖自动解析

自动分析实体间的依赖关系，生成最优导入顺序。

#### 原则3: 统一事务管理

支持多级事务（任务级、实体级、记录级），确保数据一致性。

#### 原则4: 可扩展架构

通过插件机制支持自定义处理器、验证器、转换器。

#### 3.3.3 一对多数据处理

**MasterDetailImporter (主从导入器)**

```csharp
/// <summary>
/// 主从关系导入器
/// 处理一对多关系的数据导入
/// </summary>
public class MasterDetailImporter
{
    /// <summary>
    /// 导入主从数据
    /// </summary>
    public async Task<MasterDetailImportResult> ImportAsync(
        DataTable excelData,
        EntityImportConfig masterConfig,
        List<EntityImportConfig> detailConfigs,
        ImportContext context)
    {
        var result = new MasterDetailImportResult();
        
        // 1. 按分组键分组
        var groups = GroupData(excelData, masterConfig.MasterDetailConfig.GroupByField);
        
        foreach (var group in groups)
        {
            using (var transaction = context.BeginTransaction())
            {
                try
                {
                    // 2. 导入主表记录
                    var masterEntity = await ImportMasterAsync(
                        group.Value.First(), 
                        masterConfig, 
                        context);
                    
                    // 3. 导入子表记录
                    foreach (var detailConfig in detailConfigs)
                    {
                        await ImportDetailsAsync(
                            group.Value, 
                            masterEntity, 
                            detailConfig, 
                            context);
                    }
                    
                    transaction.Commit();
                    result.SuccessGroups++;
                }
                catch (Exception ex)
                {
                    transaction.Rollback();
                    result.FailedGroups.Add(new FailedGroup
                    {
                        GroupKey = group.Key,
                        ErrorMessage = ex.Message,
                        Rows = group.Value
                    });
                    
                    if (context.ErrorStrategy == ErrorHandlingStrategy.StopOnFirstError)
                        break;
                }
            }
        }
        
        return result;
    }
    
    /// <summary>
    /// 导入主表记录
    /// </summary>
    private async Task<BaseEntity> ImportMasterAsync(
        DataRow row, 
        EntityImportConfig config, 
        ImportContext context)
    {
        // 创建主表实体
        var entity = CreateEntity(row, config);
        
        // 检查是否存在
        var existing = await FindExistingAsync(entity, config);
        
        if (existing != null)
        {
            switch (config.UpdateStrategy)
            {
                case UpdateStrategy.Update:
                    entity.CopyTo(existing);
                    await context.Db.Updateable(existing).ExecuteCommandAsync();
                    return existing;
                    
                case UpdateStrategy.Skip:
                    return existing;
                    
                case UpdateStrategy.ThrowError:
                    throw new ImportException($"记录已存在: {config.UniqueFields}");
            }
        }
        
        // 插入新记录
        var id = await context.Db.Insertable(entity).ExecuteReturnIdentityAsync();
        entity.SetPrimaryKey(id);
        
        // 记录ID映射（用于子表外键）
        context.IdRemappingEngine.RegisterMapping(config.EntityType, GetUniqueKey(row, config), id);
        
        return entity;
    }
    
    /// <summary>
    /// 导入子表记录
    /// </summary>
    private async Task ImportDetailsAsync(
        List<DataRow> rows,
        BaseEntity masterEntity,
        EntityImportConfig detailConfig,
        ImportContext context)
    {
        foreach (var row in rows)
        {
            var detailEntity = CreateEntity(row, detailConfig);
            
            // 设置外键值
            var foreignKeyProperty = detailConfig.MasterDetailConfig.ForeignKeyField;
            var masterId = masterEntity.GetPrimaryKey();
            detailEntity.SetPropertyValue(foreignKeyProperty, masterId);
            
            // 插入子表记录
            await context.Db.Insertable(detailEntity).ExecuteCommandAsync();
        }
    }
}
```

/// <summary>
/// 事务管理器
/// 支持多级事务管理

public class TransactionManager
{
    private readonly ISqlSugarClient _db;
    private readonly TransactionConfig _config;
    
    /// <summary>
    /// 创建事务上下文
    /// </summary>
    public ImportTransaction CreateTransaction(TransactionLevel level)
    {
        return new ImportTransaction(_db, level);
    }
}

/// <summary>
/// 导入事务
/// </summary>
public class ImportTransaction : IDisposable
{
    private readonly ISqlSugarClient _db;
    private readonly TransactionLevel _level;
    private SqlSugarTransaction _sqlTransaction;
    
    public ImportTransaction(ISqlSugarClient db, TransactionLevel level)
    {
        _db = db;
        _level = level;
        _sqlTransaction = db.Ado.UseTran();
    }
    
    public void Commit()
    {
        _sqlTransaction.Commit();
    }
    
    public void Rollback()
    {
        _sqlTransaction.Rollback();
    }
    
    public void Dispose()
    {
        _sqlTransaction?.Dispose();
    }
}

/// <summary>
/// 事务级别
/// </summary>
public enum TransactionLevel
{
    /// <summary>
    /// 整个任务一个事务（要么全成功，要么全失败）
    /// </summary>
    Task,
    
    /// <summary>
    /// 每个实体一个事务
    /// </summary>
    Entity,
    
    /// <summary>
    /// 每组数据一个事务（主从关系）
    /// </summary>
    Group,
    
    /// <summary>
    /// 每条记录一个事务
    /// </summary>
    Record
}

**ErrorLogger (错误记录器)**

/// <summary>
/// 错误记录器
/// 详细记录导入过程中的错误信息

public class ErrorLogger
{
    private readonly List<ImportError> _errors = new List<ImportError>();
    
    /// <summary>
    /// 记录错误
    /// </summary>
    public void LogError(ImportError error)
    {
        _errors.Add(error);
    }
    
    /// <summary>
    /// 生成错误报告
    /// </summary>
    public ErrorReport GenerateReport()
    {
        return new ErrorReport
        {
            TotalErrors = _errors.Count,
            ErrorsByEntity = _errors.GroupBy(e => e.EntityName)
                .ToDictionary(g => g.Key, g => g.ToList()),
            ErrorsByType = _errors.GroupBy(e => e.ErrorType)
                .ToDictionary(g => g.Key, g => g.Count()),
            DetailedErrors = _errors
        };
    }
    
    /// <summary>
    /// 导出错误到Excel
    /// </summary>
    public void ExportToExcel(string filePath)
    {
        // 生成包含错误数据的Excel文件，方便用户修正后重新导入
    }
}

/// <summary>
/// 外键解析器（复用 ForeignKeyService）
/// </summary>
public class ForeignKeyResolver
{
    private readonly ForeignKeyService _foreignKeyService;
    
    public ForeignKeyResolver(ISqlSugarClient db)
    {
        _foreignKeyService = new ForeignKeyService(db);
    }
    
    /// <summary>
    /// 解析外键值（支持ID重映射）
    /// </summary>
    public object ResolveForeignKey(
        DataRow row, 
        ColumnMapping mapping, 
        IdRemappingEngine idRemapping,
        int rowNumber)
    {
        // 1. 先尝试从ID重映射引擎获取（处理新导入的依赖数据）
        var sourceValue = GetSourceValue(row, mapping);
        var remappedId = idRemapping.GetMappedId(
            mapping.ForeignConfig.ForeignKeyTable.Type, 
            sourceValue);
        
        if (remappedId != null)
            return remappedId;
        
        // 2. 从数据库查询（处理已存在的数据）
        return _foreignKeyService.GetForeignKeyValue(row, mapping, rowNumber, out _);
    }
}
