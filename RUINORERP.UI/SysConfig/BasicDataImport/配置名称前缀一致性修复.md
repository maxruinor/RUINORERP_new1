# 配置名称前缀一致性修复

## 🔍 问题描述

用户反馈配置文件保存和加载时，名称不一致：
- **保存的文件名**：`[配置] supper.xml`（带前缀）
- **期望的文件名**：`supper.xml`（不带前缀）

## 📊 问题分析

### 设计原则

在基础数据导入模块中，采用了**显示名称与存储名称分离**的设计：

| 场景 | 显示名称（UI） | 存储名称（文件系统） |
|------|--------------|-------------------|
| 下拉框显示 | `[配置] supper` | - |
| 文件保存 | - | `supper.xml` |
| 文件加载 | - | `supper.xml` |
| 配置管理 | `[配置] supper` | - |

**核心规则**：
- ✅ UI层使用 `[配置]` 前缀标识单表配置
- ✅ 文件系统使用原始名称（无前缀）
- ✅ 加载/保存时需要正确转换

### ❌ 发现的问题

#### 问题1：编辑配置时传递了带前缀的名称

**位置**：`UCBasicDataImport.cs` 第1407行

```csharp
// 错误代码
string mappingName = isEditMode ? kcmbDynamicMappingName.SelectedItem.ToString() : string.Empty;
frmMapping.OriginalMappingName = mappingName;
```

**问题**：
- `kcmbDynamicMappingName.SelectedItem` 返回 `"[配置] supper"`
- 直接赋值给 `OriginalMappingName`
- 导致删除旧文件时尝试删除 `[配置] supper.xml`（实际不存在）

#### 问题2：加载配置时未去除前缀

**位置**：`UCBasicDataImport.cs` 第1492-1493行

```csharp
// 错误代码
string mappingName = kcmbDynamicMappingName.SelectedItem.ToString();
_currentConfig = _columnMappingManager.LoadConfiguration(mappingName, _selectedEntityType);
```

**问题**：
- 传入 `LoadConfiguration` 的是 `"[配置] supper"`
- `ColumnMappingManager` 会尝试加载 `[配置] supper.xml`
- 但实际文件名是 `supper.xml`，导致加载失败

## ✅ 修复方案

### 修复1：KbtnDynamicMap_Click 方法

**文件**：`UCBasicDataImport.cs`

**修改前**：
```csharp
// 判断是编辑模式还是新增模式
bool isEditMode = kcmbDynamicMappingName.SelectedIndex > 0;
string mappingName = isEditMode ? kcmbDynamicMappingName.SelectedItem.ToString() : string.Empty;
```

**修改后**：
```csharp
// 判断是编辑模式还是新增模式
bool isEditMode = kcmbDynamicMappingName.SelectedIndex > 0;
string mappingName = string.Empty;

if (isEditMode)
{
    // 【修复】从下拉框获取配置名时需要去除"[配置] "前缀
    string selectedText = kcmbDynamicMappingName.SelectedItem.ToString();
    if (selectedText.StartsWith("[配置] "))
    {
        mappingName = selectedText.Substring(5); // 移除 "[配置] " 前缀
    }
    else
    {
        mappingName = selectedText;
    }
}
```

### 修复2：LoadSelectedMapping 方法

**文件**：`UCBasicDataImport.cs`

**修改前**：
```csharp
private void LoadSelectedMapping()
{
    try
    {
        if (kcmbDynamicMappingName.SelectedIndex <= 0)
        {
            return;
        }

        string mappingName = kcmbDynamicMappingName.SelectedItem.ToString();
        _currentConfig = _columnMappingManager.LoadConfiguration(mappingName, _selectedEntityType);
    }
    catch (Exception ex)
    {
        MessageBox.Show($"加载映射配置失败: {ex.Message}", "错误", MessageBoxButtons.OK, MessageBoxIcon.Error);
    }
}
```

**修改后**：
```csharp
private void LoadSelectedMapping()
{
    try
    {
        if (kcmbDynamicMappingName.SelectedIndex <= 0)
        {
            return;
        }

        // 【修复】从下拉框获取配置名时需要去除"[配置] "前缀
        string selectedText = kcmbDynamicMappingName.SelectedItem.ToString();
        string mappingName = selectedText;
        
        if (selectedText.StartsWith("[配置] "))
        {
            mappingName = selectedText.Substring(5); // 移除 "[配置] " 前缀
        }
        else if (selectedText.StartsWith("[Profile] "))
        {
            mappingName = selectedText.Substring(10); // 移除 "[Profile] " 前缀
        }
        
        _currentConfig = _columnMappingManager.LoadConfiguration(mappingName, _selectedEntityType);
    }
    catch (Exception ex)
    {
        MessageBox.Show($"加载映射配置失败: {ex.Message}", "错误", MessageBoxButtons.OK, MessageBoxIcon.Error);
    }
}
```

## ⚠️ 注意事项

### 1. 宽表Profile的前缀

宽表Profile使用 `[Profile]` 或 `[宽表]` 前缀，长度为10个字符：
```csharp
if (selectedText.StartsWith("[Profile] "))
{
    mappingName = selectedText.Substring(10);
}
```

### 2. 模板的前缀

模板使用 `[模板]` 前缀，长度为5个字符：
```csharp
if (selectedText.StartsWith("[模板] "))
{
    templateName = selectedText.Substring(5);
}
```

### 3. 兼容性处理

为了兼容旧格式配置（没有前缀的情况），保留了else分支：
```csharp
else
{
    mappingName = selectedText; // 直接使用，不做处理
}
```
