## ✅ 第二阶段:配置保存验证测试

### 测试1:外键配置完整性验证

**步骤**:
1. 打开基础数据导入界面

## 🔍 问题排查指南

### 问题1:编译错误

**可能原因**:
- ImportValidationAdapter.cs 未正确添加到项目
- 引用命名空间缺失

**解决方法**:
```csharp
// 确保文件顶部有以下引用
using RUINORERP.Business;
using RUINORERP.Model.Context;
using System;
using System.Collections.Generic;
using System.Linq;
```

---

### 问题2:验证器未找到

**可能原因**:
- 验证器命名不符合规范
- 验证器不在正确的命名空间下

**检查清单**:
- [ ] 验证器类名 = 实体名 + "Validator"
- [ ] 命名空间 = `RUINORERP.Business.Validator`
- [ ] 继承自 `BaseValidatorGeneric<T>`

---

### 问题3:验证未生效

**可能原因**:
- 实体没有对应的 Validator 类
- 验证器构造函数参数不正确

**调试方法**:
```csharp
// 在 ImportValidationAdapter.ValidateEntity 中添加日志
System.Diagnostics.Debug.WriteLine($"查找验证器: {validatorName}");
System.Diagnostics.Debug.WriteLine($"验证器类型: {validatorType?.FullName ?? "null"}");
```

---

### 问题4:性能问题

**症状**:
- 大数据量导入时速度明显变慢

**优化建议**:
1. 先预览验证,确认无误后再执行导入
2. 考虑批量验证机制(后续优化)
3. 检查验证器中是否有耗时操作(如数据库查询)
