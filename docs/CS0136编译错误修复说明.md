# CS0136编译错误修复说明

## ❌ 问题描述

在完善成本计算方法后,出现编译错误:

```
错误(活动) CS0136 无法在此范围中声明名为"totalQty"的局部变量或参数,
因为该名称在封闭局部范围中用于定义局部变量或参数
```

**错误位置**:
- `CostCalculations.cs` 第84行 (FIFO case)
- `CostCalculations.cs` 第113行 (月加权平均 case)
- `CostCalculations.cs` 第176行 (实际成本法 case)

---

## 🔍 根本原因

在C#的`switch`语句中,**所有case共享同一个作用域**。即使代码在不同的case分支中,也不能声明同名的局部变量。

### 错误示例

```csharp
switch (m)
{
    case 库存成本计算方式.先进先出法:
        int totalQty = inv.Quantity + currentQty;  // ❌ 第一次声明
        break;

    case 库存成本计算方式.月加权平均:
        int totalQty = inv.Quantity + currentQty;  // ❌ 重复声明 - 编译错误!
        break;
}
```

---

## ✅ 修复方案

为每个case添加**花括号`{}`**,创建独立的作用域块:

### 修复后的代码

```csharp
switch (m)
{
    case 库存成本计算方式.先进先出法:
    {  // ✅ 开启独立作用域
        int totalQty = inv.Quantity + currentQty;  // 只在当前case可见
        // ... 其他代码 ...
        break;
    }  // ✅ 关闭作用域

    case 库存成本计算方式.月加权平均:
    {  // ✅ 开启新的独立作用域
        int totalQty = inv.Quantity + currentQty;  // ✅ 可以重新声明
        // ... 其他代码 ...
        break;
    }  // ✅ 关闭作用域

    case 库存成本计算方式.移动加权平均法:
    {
        decimal totalQty = newQty;  // ✅ 可以使用相同变量名
        // ... 其他代码 ...
        break;
    }

    case 库存成本计算方式.实际成本法:
    {
        int totalQty = inv.Quantity + currentQty;  // ✅ 可以重新声明
        // ... 其他代码 ...
        break;
    }

    default:
        throw new NotImplementedException($"未实现的成本计算方式：{m}");
}
```

## 🎯 C# Switch作用域规则

### 规则1: Case共享作用域
```csharp
// ❌ 错误: 不同case中不能声明同名变量
switch (value)
{
    case 1:
        int x = 10;  // 第一次声明
        break;
    case 2:
        int x = 20;  // 编译错误: 重复声明!
        break;
}
```

### 规则2: 使用花括号创建独立作用域
```csharp
// ✅ 正确: 每个case有自己的作用域
switch (value)
{
    case 1:
    {
        int x = 10;  // 只在case 1中可见
        break;
    }
    case 2:
    {
        int x = 20;  // ✅ 可以重新声明
        break;
    }
}
```

### 规则3: Break必须在花括号内
```csharp
// ❌ 错误: break在花括号外
case 1:
{
    int x = 10;
}
break;  // 编译错误: case后面必须有语句

// ✅ 正确: break在花括号内
case 1:
{
    int x = 10;
    break;  // break在花括号内
}
```

## 💡 最佳实践

### 1. 始终为Case添加花括号

即使只有一个语句,也建议添加花括号:

```csharp
// 推荐
case 1:
{
    DoSomething();
    break;
}

// 不推荐(虽然语法正确,但不一致)
case 1:
    DoSomething();
    break;
```

### 2. 保持代码风格一致

如果部分case使用了花括号,所有case都应该使用:

```csharp
// ✅ 一致的风格
switch (value)
{
    case 1:
    {
        // 复杂逻辑
        break;
    }
    case 2:
    {
        // 简单逻辑也加花括号
        break;
    }
}
```

### 3. 避免在Switch外部声明变量

如果需要在多个case中使用同一个变量,在switch外部声明:

```csharp
int result;  // 在switch外部声明

switch (value)
{
    case 1:
        result = 10;
        break;
    case 2:
        result = 20;
        break;
}

Console.WriteLine(result);  // 可以在switch外部使用
```
