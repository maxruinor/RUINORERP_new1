#### ExecuteWithRetryAsync使用配置
```csharp
public async Task ExecuteWithRetryAsync(Func<Task> action, int? maxRetryCount = null, CancellationToken cancellationToken = default)
{
    int retryCount = 0;
    var maxRetries = maxRetryCount ?? _options.MaxRetryCount;  // ✅ 使用配置值
    
    while (true)
    {
        try
        {
            await action();
            return;
        }
        catch (Exception ex)
        {
            var sqlEx = GetInnermostSqlException(ex);
            bool isRetryable = sqlEx != null && IsRetryableSqlError(sqlEx.Number);
            
            if (!isRetryable || retryCount >= maxRetries)
            {
                throw;
            }
            
            retryCount++;
            var delayMs = (int)(100 * Math.Pow(2, retryCount));
            
            // ✅ 重试前重置事务状态
            ResetTransactionState();
            
            await Task.Delay(delayMs, cancellationToken);
        }
    }
}
```

## 🔍 关键技术决策

### 1. 为什么使用partial class而不是合并文件?

**优点**:
- ✅ 代码组织清晰,主文件(~1300行)和异步方法(~310行)分离
- ✅ 便于团队协作,减少合并冲突
- ✅ 符合单一职责原则,同步和异步逻辑分开

**缺点**:
- ⚠️ 需要确保两个文件都纳入构建(已通过.csproj自动处理)

### 2. 为什么SqlSugar的Ado方法不接受CancellationToken?

**原因**:
- SqlSugar底层使用ADO.NET,Azure SQL Driver旧版本不支持异步取消
- CommitTran/RollbackTran是快速操作,通常不需要取消

**解决方案**:
- 在更高层(BeginTranAsync)传递CancellationToken用于锁等待
- 数据库操作本身不使用CancellationToken,但整个方法支持取消

### 3. 为什么兼容C# 7.3而不是升级到8.0+?

**原因**:
- 项目目标框架是.NET Framework 4.8,默认C# 7.3
- 升级语言版本可能影响其他模块
- switch表达式不是必需功能,if-else同样清晰

## ⚠️ 注意事项

### 1. 不要混用同步和异步方法
```csharp
// ❌ 错误
await _unitOfWork.BeginTranAsync();
_unitOfWork.CommitTran();  // 混用可能导致死锁

// ✅ 正确
await _unitOfWork.BeginTranAsync();
await _unitOfWork.CommitTranAsync();
```

### 2. 重试必须包裹整个事务
```csharp
// ❌ 错误
await _unitOfWork.BeginTranAsync();
await _unitOfWork.ExecuteWithRetryAsync(async () => {
    await DoWork();  // 事务已开启,重试无意义
});

// ✅ 正确
await _unitOfWork.ExecuteWithRetryAsync(async () => {
    await _unitOfWork.BeginTranAsync();
    try {
        await DoWork();
        await _unitOfWork.CommitTranAsync();
    } catch {
        await _unitOfWork.RollbackTranAsync();
        throw;
    }
});
```

### 3. 及时释放资源
```csharp
// ✅ 推荐: using语句
using var unitOfWork = serviceProvider.GetRequiredService<IUnitOfWorkManage>();
await unitOfWork.BeginTranAsync();
// ... 业务逻辑
await unitOfWork.CommitTranAsync();
// 离开作用域时自动Dispose
```
