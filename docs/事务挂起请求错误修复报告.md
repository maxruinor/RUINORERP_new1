# 事务"挂起请求"错误修复报告

## 📋 问题概述

**错误信息**: 
```
System.Data.SqlClient.SqlException: 无法执行该事务操作，因为有挂起请求正在此事务上运行。
```

**发生场景**:
- 采购入库单审核/保存
- 生产需求分析提交
- 制令单审核/保存  
- 生产领料单审核/保存

**影响范围**: 所有使用 `UnitOfWorkManage` 进行异步事务操作的业务模块

---

## 🔍 根本原因分析

### 1. **ReaderWriterLockSlim 在异步环境中的问题**

```csharp
// ❌ 旧代码 - 存在异步上下文切换风险
private readonly ReaderWriterLockSlim _rwLock = new ReaderWriterLockSlim(LockRecursionPolicy.NoRecursion);

public void BeginTran(...)
{
    _rwLock.EnterWriteLock();  // 可能在await后丢失上下文
    try { ... }
    finally { _rwLock.ExitWriteLock(); }
}
```

**问题**:
- `ReaderWriterLockSlim` 不是为异步环境设计的
- 在 `async/await` 中,线程可能切换,导致锁状态混乱
- 多个异步操作可能竞争同一个锁对象

### 2. **DataReader未正确关闭**

当执行以下模式时:
```csharp
var data = await db.Queryable<T>().ToListAsync();  // 可能有未关闭的DataReader
_unitOfWork.BeginTran();  // ❌ 此时连接上还有挂起的读取操作
```

SQL Server会拒绝在同一连接上开启新事务,因为前一个命令还未完成。

### 3. **AsyncLocal连接复用问题**

虽然使用了 `AsyncLocal<ISqlSugarClient>`,但在某些边界情况下:
- 嵌套调用可能共享同一个连接实例
- 异常路径可能导致连接状态不一致

---

## ✅ 修复方案

### 修复1: 移除ReaderWriterLockSlim,改用轻量级Monitor锁

**文件**: `RUINORERP.Repository/UnitOfWorks/UnitOfWorkManage.cs`

```csharp
// ✅ 新代码 - 使用每个TransactionContext独立的锁对象
public class TransactionContext
{
    public object LockObject { get; set; } = new object();  // 每个事务独立的锁
}

public void BeginTran(IsolationLevel? isolationLevel = null)
{
    var context = GetOrCreateTransactionContext();
    var dbClient = GetDbClient();

    // ✅ 使用context自己的锁对象,避免跨上下文竞争
    lock (context.LockObject)
    {
        try
        {
            // ... 事务开启逻辑
        }
        catch (Exception ex)
        {
            // ... 错误处理
        }
    }  // 自动释放,无需finally
}
```

**优势**:
- `lock` 语句是CLR原生支持,对异步更友好
- 每个事务上下文有独立的锁,避免全局竞争
- 代码更简洁,减少死锁风险

### 修复2: 增强BeginTran防御性检查

```csharp
// 最外层事务：开启物理事务
if (context.Depth == 1)
{
    // ✅ 确保连接打开且无挂起的DataReader
    if (dbClient.Ado.Connection.State != System.Data.ConnectionState.Open)
    {
        dbClient.Ado.Connection.Open();
    }
    
    // ✅ 防御性检查: 清理可能存在的残留事务
    if (dbClient.Ado.Transaction != null)
    {
        _logger.LogWarning($"[Transaction-{context.TransactionId}] 检测到残留事务对象，先回滚");
        try { dbClient.Ado.RollbackTran(); } catch { }
    }

    // 设置命令超时时间
    dbClient.Ado.CommandTimeOut = DEFAULT_TRANSACTION_TIMEOUT;
    
    // 开启事务
    dbClient.Ado.BeginTran();
}
```

### 修复3: 优化GetDbClient注释和日志

```csharp
public ISqlSugarClient GetDbClient()
{
    // ✅ P0修复: 每个异步上下文使用独立的连接实例
    // 关键: AsyncLocal确保不同异步流有独立的连接,避免"挂起请求"冲突
    if (_asyncLocalClient.Value == null)
    {
        var originalConfig = (_sqlSugarClient as SqlSugarScope)?.CurrentConnectionConfig;
        
        if (originalConfig != null)
        {
            var newConfig = new ConnectionConfig
            {
                ConnectionString = originalConfig.ConnectionString,
                DbType = (SqlSugar.DbType)originalConfig.DbType,
                IsAutoCloseConnection = false, // ⚠️ 关键: 事务期间不自动关闭
                // ... 其他配置
            };
            
            _asyncLocalClient.Value = new SqlSugarClient(newConfig);
            _logger.LogDebug($"为异步上下文创建了新的数据库连接实例，线程 ID: {Thread.CurrentThread.ManagedThreadId}");
        }
    }

    return _asyncLocalClient.Value;
}
```

## ⚠️ 注意事项

### 1. 业务代码规范

**✅ 推荐做法**:
```csharp
public async Task<ReturnResults<T>> ApprovalAsync(T entity)
{
    // 1. 预处理验证(无事务)
    var existingData = await _db.Queryable<T>().Where(...).FirstAsync();
    
    // 2. 开启事务
    _unitOfWork.BeginTran();
    try
    {
        // 3. 核心业务逻辑
        await _db.Updateable(entity).ExecuteCommandAsync();
        
        // 4. 提交事务
        _unitOfWork.CommitTran();
        return ReturnResults.Success(entity);
    }
    catch (Exception ex)
    {
        // 5. 异常回滚
        _unitOfWork.RollbackTran();
        throw;
    }
}
```

**❌ 避免的做法**:
```csharp
// 不要在事务外执行长时间运行的查询
var largeData = await _db.Queryable<T>().ToListAsync();  // 可能返回大量数据
_unitOfWork.BeginTran();  // ❌ 此时连接可能还有挂起的DataReader
```

### 2. 数据库连接管理

确保 `IsAutoCloseConnection = false` 在事务期间:
```csharp
var config = new ConnectionConfig
{
    IsAutoCloseConnection = false,  // ⚠️ 事务期间必须为false
    // ...
};
```

### 3. 异常处理

所有事务操作必须有try-catch-finally保护:
```csharp
_unitOfWork.BeginTran();
try
{
    // 业务逻辑
    _unitOfWork.CommitTran();
}
catch
{
    _unitOfWork.RollbackTran();
    throw;
}
// 不需要finally,CommitTran/RollbackTran内部已处理状态重置
```
