## 🎯 修复模式总结

### 模式1: 需要用户确认的情况
**适用场景**: 超额、重复数据等需要用户确认后继续的操作

**实现方式**:
```csharp
// 业务层
return ValidationResult.NeedConfirm("标题", "详细消息", "确定按钮文本");

// Controller层
if (validationResult.NeedUserConfirm) {
    rmrs.ErrorMsg = $"NEED_CONFIRM:{title}:{message}";
    return rmrs;
}

// UI层
if (result.ErrorMsg.StartsWith("NEED_CONFIRM:")) {
    var parts = result.ErrorMsg.Split(new[] { ':' }, 3);
    if (MessageBox.Show(parts[2], parts[1], MessageBoxButtons.YesNo) == DialogResult.Yes) {
        // 用户确认，继续执行
    }
}
```

### 模式2: 直接返回错误
**适用场景**: 数据验证失败、业务规则违反等不需要确认的错误

**实现方式**:
```csharp
// 业务层
rs.ErrorMsg = errorMessage;
_logger.Debug(errorMessage);
return rs;

// UI层
if (!result.Succeeded) {
    MessageBox.Show(result.ErrorMsg, "错误", MessageBoxButtons.OK, MessageBoxIcon.Error);
}
```

### 模式3: 记录日志并存储到实体
**适用场景**: 提示信息、警告信息等非阻断性消息

**实现方式**:
```csharp
// 业务层
_logger.LogWarning($"提示信息: {msg}");
entity.Remark = $"【提示】{msg}";

// UI层
if (!string.IsNullOrEmpty(entity.Remark)) {
    // 可以选择性显示提示
    ShowToast(entity.Remark);
}
```

```csharp
private async void btnApproval_Click(object sender, EventArgs e)
{
    var result = await controller.ApprovalAsync(entity);
    
    if (!result.Succeeded)
    {
        // 检查是否需要用户确认
        if (result.ErrorMsg.StartsWith("NEED_CONFIRM:"))
        {
            var parts = result.ErrorMsg.Split(new[] { ':' }, 3);
            string title = parts[1];
            string message = parts[2];
            
            if (MessageBox.Show(message, title, 
                MessageBoxButtons.YesNo, 
                MessageBoxIcon.Question) == DialogResult.Yes)
            {
                // 用户确认，重新提交并跳过验证
                await ContinueWithConfirmation();
            }
            return;
        }
        
        // 普通错误，直接显示
        MessageBox.Show(result.ErrorMsg, "错误", 
            MessageBoxButtons.OK, MessageBoxIcon.Error);
    }
}
```
