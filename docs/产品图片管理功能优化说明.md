# 产品图片管理功能优化说明

## 优化目标
优化产品图片（包含SKU图片）的下载显示及上传功能，实现：
1. **上传前预览**：在编辑窗体中实时预览图片并显示状态标识
2. **查询时显示**：在DataGridView/TreeListView中显示图片缩略图
3. **状态区分**：编辑时显示状态标识（新增/修改/替换），查询时不显示

## 主要改动

### 1. ImageInfo类增强 (RUINORERP.Lib/BusinessImage/ImageInfo.cs)
- **新增Tag字段**：用于存储自定义状态信息（如：新增、修改、替换等）
- **用途**：支持在图片上显示不同的状态标识

```csharp
/// <summary>
/// 通用标记字段，用于存储自定义状态信息（如：新增、修改、替换等）
/// </summary>
public object Tag { get; set; }
```

### 2. SKU图片编辑器优化 (RUINORERP.UI/ProductEAV/frmSKUImageEdit.cs)

#### 2.1 预加载图片状态标记
在`LoadPreloadedImages()`方法中为图片添加状态标识：
- **FileId == 0**：标记为"新增"（绿色标识）
- **FileId > 0**：标记为"修改"（橙色标识）

```csharp
// 为预加载的图片添加状态标记
var info = item.Item2 ?? new ImageInfo();
if (info.FileId == 0)
{
    info.Tag = "新增"; // 使用Tag字段存储状态
}
else
{
    info.Tag = "修改";
}
```

### 3. 产品编辑窗体优化 (RUINORERP.UI/ProductEAV/frmProductEdit.cs)

#### 3.1 DataGridView单元格绘制优化
在`dataGridView1_CellPainting()`方法中增强状态显示：

**改进前**：
- 仅显示"*"号标记未保存更改

**改进后**：
- **新增图片**：显示绿色"新增"标签
- **替换图片**：显示橙红色"替换"标签（同时有删除和新增操作）
- **半透明背景**：提升视觉效果


## 技术要点

### 1. 状态判断逻辑
```csharp
// 新增：有缓存数据但无删除记录
bool isNew = imageDataList.Count > 0 && !skuImageDeletedCache.ContainsKey(detail);

// 替换：既有删除又有新增
bool isReplace = skuImageDeletedCache.ContainsKey(detail) && 
                  skuImageDeletedCache[detail].Count > 0 &&
                  imageDataList.Count > 0;
```

### 2. 内存管理
- 使用`using`语句确保Image对象正确释放
- 缩略图生成后立即释放原始流
- 窗体关闭时调用`magicPictureBox.ClearImage()`清理资源

### 3. 性能优化
- 查询时仅获取图片数量，不加载图片数据
- 预加载SKU图片到缓存，避免重复下载
- 异步加载图片，不阻塞UI线程

## 使用说明

### 编辑产品时
1. 打开产品编辑窗体
2. 点击SKU行的"图片"列，弹出图片编辑器
3. 拖拽或粘贴图片到编辑器
4. **状态标识自动显示**：
   - 绿色"新增"：新添加的图片
   - 橙红色"替换"：替换了原有图片
5. 点击"确定"保存更改（此时仅标记，未真正上传）
6. 保存产品时统一上传图片

### 查询产品时
1. 在产品查询界面查看列表
2. **SKU图片列显示**：
   - 简洁文本：如“3张”
   - 无状态标识，界面清爽
3. **无多余符号**：去除了勾选符号，更简洁

## 注意事项

1. **图片上传时机**：
   - 编辑时仅在内存中标记更改
   - 点击“确定”按钮时上传图片到服务器
   - 取消编辑会清除所有未保存的更改

2. **状态清理**：
   - 保存成功后自动清除状态标识
   - 取消编辑时调用`ResetImageChangeStatus()`重置状态

3. **兼容性**：
   - Tag字段为object类型，可存储任意类型数据
   - 向后兼容，不影响现有功能
