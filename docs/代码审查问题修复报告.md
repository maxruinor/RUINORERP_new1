private async void btnApproval_Click(object sender, EventArgs e)
{
    var result = await controller.ApprovalAsync(entity);
    
    if (!result.Succeeded)
    {

        if (result.ErrorMsg.StartsWith("NEED_CONFIRM:"))
        {
            var parts = result.ErrorMsg.Split(new[] { ':' }, 3);
            string title = parts[1];
            string message = parts[2];

    }

### Issue 3: 异常处理参数错误 ✅ 已修复

**问题描述**：
在 `UCSaleOrder.cs` L1385 调用 `TryShowUniqueConstraintError(null)` 时传递了 `null` 参数，虽然该方法支持可选参数，但语义上不够清晰。

**修复前**：
```csharp
// L1385 - 传递 null 参数
if (DatabaseExceptionHandler.TryShowUniqueConstraintError(null))
{
    MainForm.Instance.PrintInfoLog($"保存失败。", Color.Red);
    return false;
}
```

**修复后**：
```csharp
// L1385 - 使用默认参数（更清晰）
if (DatabaseExceptionHandler.TryShowUniqueConstraintError())
{
    MainForm.Instance.PrintInfoLog($"保存失败。", Color.Red);
    return false;
}
```

**说明**：
- `TryShowUniqueConstraintError` 方法签名为：`public static bool TryShowUniqueConstraintError(Exception ex = null)`
- 当参数为 `null` 或未传递时，方法会读取 `MainForm.LastUniqueConstraintError` 静态属性
- 在 catch 块中应该传递实际的异常对象：`TryShowUniqueConstraintError(ex)`
- 在非 catch 块中（如检查返回结果时），可以不传参数，使用方法签名中的默认值

**相关代码**：
```csharp
// catch 块中的正确用法（L1425）
catch (Exception ex)
{
    if (RUINORERP.UI.Common.DatabaseExceptionHandler.TryShowUniqueConstraintError(ex))
    {
        return false;
    }
    // ...
}
```
