### 优化1: 在欢迎流程中集成IP黑名单检查

**问题**: 
- 当前欢迎流程未检查IP黑名单
- 恶意IP可以建立TCP连接但不登录,浪费服务器资源
- 只能在登录阶段拦截,防护时机较晚

**解决方案**:

#### 修改文件: `SessionService.cs`

**位置**: `OnSessionConnectedAsync(IAppSession session)` 方法开头

**修改内容**:

```csharp
public async ValueTask OnSessionConnectedAsync(IAppSession session)
{
    try
    {
        // ✅ 新增：获取客户端IP并检查黑名单（在网络层早期拦截）
        string clientIp = GetClientIp(session);
        if (!string.IsNullOrEmpty(clientIp) && BlacklistManager.IsIpBanned(clientIp))
        {
            _logger.LogWarning($"[黑名单拦截] IP地址已被封禁，拒绝连接: {clientIp}");
            await session.CloseAsync(CloseReason.ServerShutdown);
            return;
        }

        // 检查是否已达到最大会话数
        if (ActiveSessionCount >= MaxSessionCount)
        {
            _logger.LogWarning($"达到最大会话数量限制: {MaxSessionCount}，拒绝新连接");
            await session.CloseAsync(CloseReason.ServerShutdown);
            return;
        }
        
        // ... 原有逻辑
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, $"处理会话连接事件时出错: SessionID={session?.SessionID}");
    }
}
```

**新增辅助方法**:

```csharp
/// <summary>
/// 获取客户端IP地址
/// </summary>
/// <param name="session">会话对象</param>
/// <returns>客户端IP地址</returns>
private string GetClientIp(IAppSession session)
{
    try
    {
        if (session?.RemoteEndPoint != null)
        {
            var ipEndpoint = session.RemoteEndPoint as System.Net.IPEndPoint;
            if (ipEndpoint != null)
            {
                return ipEndpoint.Address.ToString();
            }
        }
    }
    catch (Exception ex)
    {
        _logger.LogWarning(ex, "获取客户端IP地址失败");
    }
    
    return "0.0.0.0"; // 使用0.0.0.0表示未知IP
}
```

**优势**:
- ✅ 在网络层早期拦截恶意IP,减少无效会话创建
- ✅ 降低服务器资源消耗(CPU、内存、网络带宽)
- ✅ 提高系统安全性,防止IP级别的攻击
- ✅ 与现有BlacklistManager无缝集成

**日志示例**:

### 优化2: 添加自动封禁机制

**问题**:
- 当前黑名单需手动管理,无法应对暴力破解
- 登录失败多次后不会自动封禁IP
- 缺乏自动化安全防护机制

**解决方案**:

#### 修改文件: `LoginCommandHandler.cs`

**位置**: `ProcessLoginAsync()` 方法中的凭据验证失败分支

**修改内容**:

```csharp
// 验证用户凭据
var userInfo = await ValidateUserCredentialsAsync(loginRequest, cancellationToken);
if (userInfo == null)
{
    IncrementLoginAttempts(loginRequest.Username);
    
    // ✅ 新增：失败5次后自动封禁IP 1小时
    if (GetLoginAttempts(loginRequest.Username) >= MaxLoginAttempts)
    {
        BlacklistManager.BanIp(loginRequest.ClientIp, TimeSpan.FromHours(1));
        logger?.LogWarning($"[自动封禁] IP地址 {loginRequest.ClientIp} 因登录失败次数过多被封禁1小时，Username={loginRequest.Username}");
    }
    
    logger?.LogWarning($"[登录失败] 用户名或密码错误: Username={loginRequest.Username}, ClientIp={loginRequest.ClientIp}, Attempts={GetLoginAttempts(loginRequest.Username)}");
    return ResponseFactory.CreateSpecificErrorResponse(executionContext, "用户名或密码错误");
}
```

**工作机制**:

1. **失败计数**: 每次登录失败调用 `IncrementLoginAttempts()`
2. **阈值检查**: 当失败次数 >= 5次时触发自动封禁
3. **自动封禁**: 调用 `BlacklistManager.BanIp(ip, TimeSpan.FromHours(1))`
4. **封禁时长**: 1小时(可配置)
5. **自动解封**: 1小时后BlacklistManager自动清理过期封禁

**优势**:
- ✅ 自动化安全防护,无需人工干预
- ✅ 有效防止暴力破解攻击
- ✅ 可配置的封禁时长和失败阈值
- ✅ 与现有BlacklistManager无缝集成
- ✅ 支持手动解封(通过UI管理界面)

**日志示例**:
```

[Warning] [自动封禁] IP地址 192.168.1.100 因登录失败次数过多被封禁1小时，Username=admin
```

## ⚠️ 注意事项

### 1. 误封风险

**问题**: 合法用户可能因忘记密码而多次尝试,导致IP被封

**缓解措施**:
- ✅ 提供清晰的错误提示:"用户名或密码错误,剩余尝试次数:X"
- ✅ 支持管理员手动解封(通过BlacklistManagementControl)
- ✅ 封禁时长可配置,建议初始设置为30分钟而非1小时

**改进建议**:
```csharp
// 可选: 区分不同错误类型
if (failureReason == "PasswordExpired")
{
    // 密码过期不封禁IP
    return ResponseFactory.CreateSpecificErrorResponse(executionContext, "密码已过期,请联系管理员重置");
}
else if (failureReason == "AccountLocked")
{
    // 账号锁定不封禁IP
    return ResponseFactory.CreateSpecificErrorResponse(executionContext, "账号已被锁定");
}
else
{
    // 只有密码错误才计入失败次数
    IncrementLoginAttempts(loginRequest.Username);
}
```

- ✅ 提供手动解封功能
