**审查日期**: 2026-04-18

**问题级别**: 🔴 严重 - 已修复

## 📋 审查目标

确保各层依赖注入配置符合以下原则:
1. **职责分离**: 各层只注册自己的服务
2. **避免重复**: 同一服务不在多处注册
3. **生命周期一致**: 所有注册使用统一的生命周期策略
4. **架构清晰**: 上层引用下层的DI配置,不越层注册

## ❌ 发现的问题

### 问题1: IUnitOfWorkManage 被重复注册3次(最严重)

#### 问题描述

`IUnitOfWorkManage` 在以下位置被重复注册:

| 位置 | 文件 | 行号 | 生命周期 | 状态 |
|------|------|------|---------|------|
| Repository层 | `RepositoryDIConfig.cs` | 36-39 | InstancePerLifetimeScope | ✅ 保留 |
| UI层 (ConfigureContainer) | `UI/Startup.cs` | 561-566 | InstancePerLifetimeScope | ❌ 已删除 |
| UI层 (MainRegister) | `UI/Startup.cs` | 939-942 | InstancePerLifetimeScope | ❌ 已删除 |
| Server层 (Autofac) | `SmartReminderModule.cs` | 87-89 | InstancePerLifetimeScope | ❌ 已删除 |
| Server层 (MS.DI) | `Server/Startup.cs` | 604 | AddScoped | ❌ 已删除 |

#### 导致的后果

1. **生命周期不确定性**: Autofac以最后注册的为准,但注册顺序不明确
2. **连接泄漏风险**: 如果不同注册的生命周期不一致,可能导致连接未正确释放
3. **日志重复**: 每次解析可能创建新实例,导致"创建新连接"日志多次出现
4. **调试困难**: 无法确定实际使用的是哪个注册

#### 修复方案

**遵循架构原则**: 
- ✅ **Repository层注册** - UnitOfWorkManage是仓储层的核心服务,应由Repository层自己注册
- ❌ **UI层不注册** - UI层通过引用RepositoryDIConfig自动获得
- ❌ **Server层不注册** - Server层通过引用RepositoryDIConfig自动获得

### 问题2: HasAttachmentSyncService 生命周期不匹配

#### 问题描述

```csharp
// NetworkServicesDependencyInjection.cs
services.AddSingleton<HasAttachmentSyncService>();  // ❌ Singleton

// 但它依赖:
public HasAttachmentSyncService(IUnitOfWorkManage unitOfWorkManage, ...)
// IUnitOfWorkManage 是 Scoped (InstancePerLifetimeScope)
```

**Singleton 依赖 Scoped = 捕获依赖(Captive Dependency)**
- HasAttachmentSyncService 会持有第一个解析的 UnitOfWorkManage 实例
- 后续请求不会获得新的 UnitOfWorkManage,导致连接复用问题

#### 修复方案

```csharp
// ✅ 改为 Scoped,与 IUnitOfWorkManage 生命周期一致
services.AddScoped<HasAttachmentSyncService>();
builder.RegisterType<HasAttachmentSyncService>().AsSelf().InstancePerLifetimeScope();
```

### 问题3: GenericConfigService 生命周期冲突

#### 问题描述

```csharp
// BusinessDIConfig.cs (第206-210行)
builder.RegisterGeneric(typeof(GenericConfigService<>))
    .As(typeof(IGenericConfigService<>))
    .SingleInstance()              // ❌ 矛盾1: SingleInstance
    .InstancePerDependency()       // ❌ 矛盾2: InstancePerDependency
    .PropertiesAutowired();
```

**两个互斥的生命周期修饰符同时存在!**
- Autofac会以最后一个为准(`InstancePerDependency`)
- 但代码意图不明确,容易误导维护者

#### 修复方案

```csharp
// ✅ 明确指定为 SingleInstance (配置服务通常是单例)
builder.RegisterGeneric(typeof(GenericConfigService<>))
    .As(typeof(IGenericConfigService<>))
    .SingleInstance()
    .PropertiesAutowired();
```

```
┌─────────────────────────────────────────┐
│           UI Layer (客户端)              │

└──────────────┬──────────────────────────┘
               │ 依赖注入
┌──────────────▼──────────────────────────┐
│        Server Layer (服务器)             │
│  - 引用 RepositoryDIConfig.ConfigureContainer() │
│  - 不注册 UnitOfWorkManage               │
└──────────────┬──────────────────────────┘
               │ 依赖注入
┌──────────────▼──────────────────────────┐
│     Repository Layer (仓储层)            │
│  ✅ 注册 UnitOfWorkManage                │
│  ✅ 注册 BaseRepository<>                │
│  ✅ 生命周期: InstancePerLifetimeScope   │
└─────────────────────────────────────────┘
```

### 生命周期策略

| 服务类型 | 生命周期 | 说明 |
|---------|---------|------|
| UnitOfWorkManage | InstancePerLifetimeScope | 同一请求内共享,避免连接泄漏 |
| BaseRepository<> | InstancePerLifetimeScope | 与 UnitOfWorkManage 保持一致 |
| HasAttachmentSyncService | InstancePerLifetimeScope | 依赖 UnitOfWorkManage,必须一致 |
| EntityCacheManager | SingleInstance | 全局缓存,整个应用共享 |
| TableSchemaManager | SingleInstance | 表结构元数据,全局共享 |
| CommonController | SingleInstance | 通用控制器,无状态 |

## 💡 最佳实践建议

### 1. 各层DI配置原则

```csharp
// ✅ 正确: Repository层注册自己的服务
public static class RepositoryDIConfig
{
    public static void ConfigureContainer(ContainerBuilder builder)
    {
        builder.RegisterType<UnitOfWorkManage>()
            .As<IUnitOfWorkManage>()
            .InstancePerLifetimeScope();
    }
}

// ✅ 正确: UI层引用Repository层的配置
public static void ConfigureContainer(ContainerBuilder builder)
{
    RepositoryDIConfig.ConfigureContainer(builder);  // 引用下层
    // 只注册UI层特有的服务
}
```

### 2. 避免跨层注册

```csharp
// ❌ 错误: UI层注册Repository层的服务
builder.RegisterType<UnitOfWorkManage>()...  // UnitOfWorkManage属于Repository层

// ✅ 正确: 通过引用下层DI配置获得
RepositoryDIConfig.ConfigureContainer(builder);
```

### 3. 生命周期一致性

```csharp
// ❌ 错误: Singleton 依赖 Scoped
services.AddSingleton<ServiceA>();  // ServiceA 依赖 IUnitOfWorkManage (Scoped)

// ✅ 正确: 生命周期匹配
services.AddScoped<ServiceA>();  // 与依赖的服务生命周期一致
```

### 4. 避免矛盾的生命周期修饰符

```csharp
// ❌ 错误: 两个互斥的修饰符
builder.RegisterType<MyService>()
    .SingleInstance()
    .InstancePerDependency();  // 矛盾!

// ✅ 正确: 明确指定一个
builder.RegisterType<MyService>()
    .SingleInstance();
```
