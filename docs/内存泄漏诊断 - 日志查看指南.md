## 🔍 关键指标解读

### 1️⃣ 内存状态

#### 工作集内存（Working Set）
- **正常值**: <1000 MB
- **警告值**: 1000-2000 MB
- **危险值**: >2000 MB（您的系统：6144 MB 🔴）

#### 内存增长率
- **正常**: <1 MB/分钟
- **警告**: 1-5 MB/分钟
- **严重**: >5 MB/分钟（您的系统：15.5 MB/分钟 🔴）

### 2️⃣ Top 内存占用者

**重点关注标记 ⚠️【可能是泄漏源】的项目**

#### 会话管理（常见泄漏源）
- **正常**: 15 个用户 × 1 MB = 15 MB
- **异常**: 800 MB（您的系统）
- **原因**: 会话只增不减，未清理过期会话

#### 实体缓存（常见泄漏源）
- **正常**: <200 MB
- **异常**: 600 MB（您的系统）
- **原因**: 缓存无过期策略，无限增长

#### 线程栈（严重问题）
- **正常**: <100 个线程 × 2 MB = 200 MB
- **异常**: 250 个线程 × 2 MB = 500 MB（您的系统）
- **原因**: 定时器、后台任务未释放

#### GDI 对象（常见问题）
- **正常**: <5000 个对象 × 10 KB = 50 MB
- **异常**: 15000 个对象 × 10 KB = 150 MB（您的系统）
- **原因**: Bitmap/Graphics 未 Dispose

### 3️⃣ 泄漏检测

#### 是否检测到泄漏
- ✅ 否：内存增长正常
- 🔴 是（严重）：存在内存泄漏

#### 达到临界值时间
- **安全**: >360 分钟（6 小时）
- **警告**: 60-360 分钟
- **严重**: <60 分钟

## 🎯 根据日志定位问题

### 场景 1：会话管理泄漏（概率 80%）

#### 日志特征
```
#1 会话管理                   800 MB (13.0%)  ⚠️【可能是泄漏源】
```

#### 立即检查
1. 找到 `SessionService.cs`
2. 搜索是否有清理过期会话的代码
3. 检查是否有定时器定期清理

#### 修复代码
```csharp
// 在 SessionService.cs 中添加
private readonly Timer _cleanupTimer;

public SessionService()
{
    // 每 5 分钟清理一次过期会话
    _cleanupTimer = new Timer(CleanupExpiredSessions, null, 
        TimeSpan.FromMinutes(5), 
        TimeSpan.FromMinutes(5));
}

private void CleanupExpiredSessions()
{
    var expiredKeys = _sessions.Values
        .Where(s => s.LastActivity < DateTime.Now.AddMinutes(-30))
        .Select(s => s.SessionId)
        .ToList();
    
    foreach (var key in expiredKeys)
    {
        _sessions.TryRemove(key, out _);
    }
    
    Console.WriteLine($"✅ 清理了 {expiredKeys.Count} 个过期会话");
}
```

   ⚠️ 警告：线程数过多（250 个），可能存在线程泄漏！

#### 立即检查
1. 搜索所有 `new Timer(` 代码

#### 修复代码
```csharp
// 为使用定时器的类添加 IDisposable
public class MonitoringService : IDisposable
{
    private Timer _timer;
    private bool _disposed = false;
    
    public void StartMonitoring()
    {
        _timer = new Timer(CheckStatus, null, 0, 5000);
    }
    
    public void Dispose()
    {
        if (!_disposed)
        {
            _timer?.Dispose();
            _disposed = true;
        }
    }
}

// 为后台任务添加 CancellationToken
public class BackgroundWorker
{
    private CancellationTokenSource _cts;
    
    public void StartProcessing()
    {
        _cts = new CancellationTokenSource();
        
        Task.Run(async () =>
        {
            while (!_cts.Token.IsCancellationRequested)
            {
                await ProcessDataAsync(_cts.Token);
                await Task.Delay(1000, _cts.Token);
            }
        }, _cts.Token);
    }
    
    public void Stop()
    {
        _cts.Cancel();
        _cts.Dispose();
    }
}
```

public void DrawImage()
{
    var bitmap = new Bitmap("image.jpg");
    var graphics = Graphics.FromImage(bitmap);
    graphics.DrawString("Hello", font, brush, point);
    // bitmap 和 graphics 未释放
}

// ✅ 正确：使用 using 语句
public void DrawImage()
{
    using (var bitmap = new Bitmap("image.jpg"))
    using (var graphics = Graphics.FromImage(bitmap))
    {
        graphics.DrawString("Hello", font, brush, point);
    } // 自动释放
}

## 🔧 高级诊断工具

### dotnet-dump（最准确）

#### 1. 安装
```bash
dotnet tool install -g dotnet-dump
```

#### 2. 创建 Dump 文件
```bash
# 找到进程 ID
tasklist | findstr RUINORERP.Server
# 假设进程 ID 是 12345

# 创建 dump
dotnet-dump collect --process-id 12345 --output D:\Dumps\memory.dmp
```

#### 3. 分析 Dump
```bash
# 启动分析
dotnet-dump analyze D:\Dumps\memory.dmp

# 查看最大的对象
dumpheap -stat -min 10000000

# 查看 RUINORERP 命名空间的类型
dumpheap -type RUINORERP

# 查找 GC 根
gcroot <MethodTable>
```

## 📝 检查清单

### 会话管理
- [ ] 是否有 `Timer` 定期清理过期会话？
- [ ] 过期时间是否设置为 30 分钟？
- [ ] 用户退出时是否立即清理会话？
- [ ] 会话中是否存储了过大的对象？

### 缓存管理
- [ ] 缓存是否有 `AbsoluteExpiration`（绝对过期）？
- [ ] 缓存是否有 `SlidingExpiration`（滑动过期）？
- [ ] 缓存是否有 `SizeLimit`（数量限制）？
- [ ] 是否定期清理未使用的缓存？

### 定时器
- [ ] 所有 `Timer` 是否实现了 `IDisposable`？
- [ ] 是否在不再需要时调用 `Dispose()`？
- [ ] 是否避免重复创建定时器？

### 后台任务
- [ ] `Task.Run` 是否使用 `CancellationToken`？
- [ ] 后台线程是否有终止条件？
- [ ] 是否在关闭时取消所有任务？

### GDI 对象
- [ ] 所有 `Bitmap` 是否使用 `using` 语句？
- [ ] 所有 `Graphics` 是否使用 `using` 语句？
- [ ] 所有 `Font`、`Pen`、`Brush` 是否释放？

### 数据库连接
- [ ] 所有 `SqlSugarClient` 是否使用 `using` 语句？
- [ ] 连接字符串中的 `Max Pool Size` 是否合理（建议 50）？
- [ ] `DbContext` 是否及时释放？
