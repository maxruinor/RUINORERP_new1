## 🔍 常见内存泄漏原因和修复

### 1. 会话管理泄漏（最常见）

#### 症状
- 内存中累积大量 Session 对象
- 离线用户的会话未清理

#### 修复代码

找到 `SessionService.cs`，添加清理逻辑：

```csharp
// 添加定时器字段
private readonly Timer _cleanupTimer;

// 在构造函数中启动定时器
public SessionService()
{
    _cleanupTimer = new Timer(CleanupExpiredSessions, null, 
        TimeSpan.FromMinutes(5), 
        TimeSpan.FromMinutes(5));
}

// 添加清理方法
private void CleanupExpiredSessions()
{
    try
    {
        var expiredKeys = _sessions.Values
            .Where(s => s.LastActivity < DateTime.Now.AddMinutes(-30))
            .Select(s => s.SessionId)
            .ToList();
        
        foreach (var key in expiredKeys)
        {
            _sessions.TryRemove(key, out _);
        }
        
        Console.WriteLine($"清理了 {expiredKeys.Count} 个过期会话");
    }
    catch (Exception ex)
    {
        Console.WriteLine($"清理过期会话失败：{ex.Message}");
    }
}
```

### 2. 实体缓存泄漏

#### 症状
- 缓存只添加不清理
- 缓存没有过期策略

#### 修复代码

找到 `EntityCacheManager.cs`，修改缓存策略：

```csharp
// 修改缓存方法，添加过期时间
public void CacheEntity(string key, object entity)
{
    // 添加 30 分钟绝对过期时间
    var options = new MemoryCacheEntryOptions
    {
        AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(30),
        SlidingExpiration = TimeSpan.FromMinutes(5)
    };
    
    _memoryCache.Set(key, entity, options);
}
```

---

### 3. 事件订阅泄漏

#### 症状
- 对象订阅事件后未取消订阅
- 静态事件导致对象无法释放

#### 修复代码

为所有订阅事件的类实现 `IDisposable`：

```csharp
public class SomeService : IDisposable
{
    private readonly ISubscription _subscription;
    private bool _disposed = false;
    
    public SomeService(EventAggregator aggregator)
    {
        _subscription = aggregator.Subscribe<DataChangedEvent>(OnDataChanged);
    }
    
    public void Dispose()
    {
        if (!_disposed)
        {
            _subscription?.Dispose();
            _disposed = true;
        }
    }
}
```

---

### 4. 定时器泄漏

#### 症状
- 定时器创建后未释放
- 多个定时器重复创建

#### 修复代码

为所有使用定时器的类实现 `IDisposable`：

```csharp
public class MonitoringService : IDisposable
{
    private Timer _timer;
    private bool _disposed = false;
    
    public void StartMonitoring()
    {
        _timer = new Timer(CheckStatus, null, 0, 5000);
    }
    
    public void Dispose()
    {
        if (!_disposed)
        {
            _timer?.Dispose();
            _disposed = true;
        }
    }
}
```

---

### 5. 数据库连接泄漏

#### 症状
- 数据库连接未正确关闭
- 连接池耗尽

#### 修复代码

使用 `using` 语句：

```csharp
// ❌ 错误
public List<User> GetAllUsers()
{
    var db = new SqlSugarClient(config);
    return db.Queryable<User>().ToList();
    // db 未释放
}

// ✅ 正确
public List<User> GetAllUsers()
{
    using (var db = new SqlSugarClient(config))
    {
        return db.Queryable<User>().ToList();
    }
}
```

- **安全**: > 360 分钟
- **警告**: 60-360 分钟

### Top 内存占用者标记

## 📝 检查清单

### 会话管理
- [ ] 是否有会话过期清理逻辑？
- [ ] 过期时间设置是否合理（30 分钟）？
- [ ] 是否在用户退出时立即清理？
- [ ] 会话中是否存储了过大的对象？

### 缓存管理
- [ ] 缓存是否有绝对过期时间？
- [ ] 缓存是否有滑动过期时间？
- [ ] 缓存是否有数量限制？
- [ ] 是否定期清理未使用的缓存？

### 事件订阅
- [ ] 是否实现了 IDisposable？
- [ ] 是否在 Dispose 中取消订阅？
- [ ] 是否避免使用静态事件？

### 定时器
- [ ] 定时器是否实现了 IDisposable？
- [ ] 是否在不再需要时停止定时器？
- [ ] 是否避免重复创建定时器？

### 数据库连接
- [ ] 是否使用 using 语句？
- [ ] 连接字符串中的 Pool Size 是否合理？
- [ ] DbContext 是否及时释放？

### 后台任务
- [ ] Task.Run 是否使用 CancellationToken？
- [ ] 后台线程是否有终止条件？
- [ ] 是否避免创建过多线程？
