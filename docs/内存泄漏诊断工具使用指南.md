# 内存泄漏诊断工具使用指南

## 🚨 问题严重性

您的服务器内存占用达到 **6-7 GB**，这是非常严重的内存泄漏问题！

### 正常内存占用
- 空闲状态：200-400 MB
- 中等负载：400-800 MB
- 高负载：800-1500 MB

### 异常内存占用
- **6-7 GB = 严重内存泄漏** ⚠️
- 必须立即诊断和修复

---

## 📋 诊断步骤

### 第一步：启动诊断服务

#### 1. 确保服务已注册

在 `Startup.cs` 或依赖注入配置中添加：

```csharp
// 注册内存泄漏诊断服务
services.AddSingleton<MemoryLeakDiagnosticsService>();
```

#### 2. 启动诊断

诊断服务会自动启动，每 60 秒进行一次诊断。

## 🔍 常见内存泄漏原因

### 1. 会话管理泄漏 (SessionService)

#### 症状
- 会话对象只增不减
- 离线用户的会话未清理
- 会话中存储了大量数据

#### 检查代码
```csharp
// 检查 SessionService 是否有清理逻辑
public class SessionService
{
    private ConcurrentDictionary<string, SessionInfo> _sessions;
    
    // ❌ 错误：只添加不清理
    public void AddSession(string sessionId, SessionInfo session)
    {
        _sessions[sessionId] = session;
    }
    
    // ✅ 正确：定期清理过期会话
    public void CleanupExpiredSessions()
    {
        var expiredKeys = _sessions.Values
            .Where(s => s.LastActivity < DateTime.Now.AddMinutes(-30))
            .Select(s => s.SessionId)
            .ToList();
        
        foreach (var key in expiredKeys)
        {
            _sessions.TryRemove(key, out _);
        }
    }
}
```

#### 修复方案
```csharp
// 添加定时器清理过期会话
private readonly Timer _cleanupTimer;

public SessionService()
{
    _cleanupTimer = new Timer(CleanupExpiredSessions, null, 
        TimeSpan.FromMinutes(5), 
        TimeSpan.FromMinutes(5));
}
```

### 2. 实体缓存泄漏 (EntityCacheManager)

#### 症状
- 缓存只添加不清理
- 缓存没有过期策略
- 缓存键设计不当导致重复缓存

#### 检查代码
```csharp
// 检查 EntityCacheManager 的缓存策略
public class EntityCacheManager
{
    // ❌ 错误：无限缓存
    public void CacheEntity(string key, object entity)
    {
        _cache[key] = entity; // 永不过期
    }
    
    // ✅ 正确：添加过期时间
    public void CacheEntity(string key, object entity, TimeSpan expiration)
    {
        _cache.Set(key, entity, expiration);
    }
}
```

#### 修复方案
```csharp
// 使用 MemoryCache 的过期策略
var cacheEntryOptions = new MemoryCacheEntryOptions
{
    AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(30),
    SlidingExpiration = TimeSpan.FromMinutes(5),
    SizeLimit = 1000 // 限制缓存数量
};

_memoryCache.Set(key, entity, cacheEntryOptions);
```

### 3. 事件订阅泄漏

#### 症状
- 对象订阅事件后未取消订阅
- 事件处理器持有对象引用
- 静态事件导致对象无法释放

#### 检查代码
```csharp
// ❌ 错误：订阅事件但未取消
public class SomeService
{
    public SomeService(EventAggregator aggregator)
    {
        aggregator.Subscribe<DataChangedEvent>(OnDataChanged);
        // 没有取消订阅的逻辑
    }
    
    private void OnDataChanged(DataChangedEvent evt)
    {
        // 处理事件
    }
}

// ✅ 正确：实现 IDisposable 取消订阅
public class SomeService : IDisposable
{
    private readonly ISubscription _subscription;
    
    public SomeService(EventAggregator aggregator)
    {
        _subscription = aggregator.Subscribe<DataChangedEvent>(OnDataChanged);
    }
    
    public void Dispose()
    {
        _subscription?.Dispose(); // 取消订阅
    }
}
```

---

### 4. 定时器泄漏

#### 症状
- 定时器创建后未释放
- 定时器回调持有对象引用
- 多个定时器重复创建

#### 检查代码
```csharp
// ❌ 错误：定时器未释放
public class MonitoringService
{
    private Timer _timer;
    
    public void StartMonitoring()
    {
        _timer = new Timer(CheckStatus, null, 0, 5000);
        // 没有停止定时器的逻辑
    }
}

// ✅ 正确：实现 IDisposable 释放定时器
public class MonitoringService : IDisposable
{
    private Timer _timer;
    private bool _disposed = false;
    
    public void StartMonitoring()
    {
        _timer = new Timer(CheckStatus, null, 0, 5000);
    }
    
    public void Dispose()
    {
        if (!_disposed)
        {
            _timer?.Dispose();
            _disposed = true;
        }
    }
}
```

### 5. 数据库连接泄漏

#### 症状
- 数据库连接未正确关闭
- 连接池耗尽
- DbContext 未释放

#### 检查代码
```csharp
// ❌ 错误：SqlSugar 未释放
public class SomeRepository
{
    private SqlSugarClient _db;
    
    public List<User> GetAllUsers()
    {
        _db = new SqlSugarClient(config);
        return _db.Queryable<User>().ToList();
        // _db 未释放
    }
}

// ✅ 正确：使用 using 语句
public class SomeRepository
{
    public List<User> GetAllUsers()
    {
        using (var db = new SqlSugarClient(config))
        {
            return db.Queryable<User>().ToList();
        }
    }
}
```

### 6. 线程池泄漏

#### 症状
- Task.Run 创建大量任务
- 后台线程未正确终止
- CancellationToken 未使用

#### 检查代码
```csharp
// ❌ 错误：无限创建任务
public class BackgroundWorker
{
    public void StartProcessing()
    {
        while (true)
        {
            Task.Run(() => ProcessData()); // 无限创建任务
            Thread.Sleep(1000);
        }
    }
}

// ✅ 正确：使用 CancellationToken
public class BackgroundWorker
{
    private CancellationTokenSource _cts;
    
    public void StartProcessing()
    {
        _cts = new CancellationTokenSource();
        
        Task.Run(async () =>
        {
            while (!_cts.Token.IsCancellationRequested)
            {
                await ProcessDataAsync(_cts.Token);
                await Task.Delay(1000, _cts.Token);
            }
        }, _cts.Token);
    }
    
    public void Stop()
    {
        _cts.Cancel();
    }
}
```

## 📊 诊断报告解读

### 关键指标

#### 内存增长率
- **正常**: < 1 MB/分钟
- **警告**: 1-5 MB/分钟
- **严重**: > 5 MB/分钟

#### 达到临界值时间
- **安全**: > 360 分钟 (6 小时)
- **警告**: 60-360 分钟
- **严重**: < 60 分钟

#### Top 内存占用者
- 找出占用最大的前 5 个模块
- 标记为"⚠️ 可能是泄漏源"的需要重点检查

- [ ] 缓存是否有绝对过期时间？
- [ ] 缓存是否有滑动过期时间？
- [ ] 缓存是否有数量限制？
