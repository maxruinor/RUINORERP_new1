# 内存泄漏诊断工具 - 使用说明（优化版）

## ✅ 已优化功能

### 1. 诊断频率调整
- **原频率**: 每 60 秒输出一次日志
- **新频率**: 每 5 分钟输出一次日志（降低 83% 的日志量）

### 2. 添加开关控制

#### 启用/禁用诊断服务
- 默认：**启用**
- 禁用后：完全停止诊断，不输出任何日志
- 启用后：恢复 5 分钟诊断一次

#### 详细模式开关
- 默认：**关闭**（只在检测到泄漏时输出警告）
- 开启后：每次诊断都输出完整日志（类似之前的行为）
- 建议：平时关闭，需要详细分析时开启

### 3. UI 控制按钮

在工具栏添加了 **"内存诊断"** 按钮，点击可：
- ✅ 启用/禁用诊断服务
- ✅ 开启/关闭详细模式
- ✅ 立即运行诊断
- ✅ 查看当前状态

## 📋 使用步骤

### 第 1 步：注册服务（5 分钟）

打开 `Startup.cs` 第 405 行附近，添加：

```csharp
// 注册内存分布统计服务模块（使用 Module 方式统一管理）
builder.RegisterModule<MemoryDistributionModule>();

// 注册内存泄漏诊断服务模块（用于深度分析内存泄漏问题）
builder.RegisterModule<MemoryLeakDiagnosticsModule>(); // ← 添加这行
#endregion
```

### 第 2 步：重启服务器

保存并重启 RUINORERP.Server

### 第 3 步：使用 UI 控制

#### 方式 A：使用工具栏按钮（推荐）

1. 点击工具栏的 **"内存诊断"** 按钮
2. 显示下拉菜单：
   - ☑️ **启用诊断服务**（打勾表示已启用）
   - ☐ **详细模式**（打勾表示已开启）
   - **立即运行诊断**（手动触发诊断）
   - **当前状态：✅ 已启用**（只读）

#### 方式 B：使用代码控制

```csharp
// 禁用诊断服务
frmMainNew.Instance.IsMemoryLeakDiagnosticsEnabled = false;

// 启用诊断服务
frmMainNew.Instance.IsMemoryLeakDiagnosticsEnabled = true;

// 开启详细模式
frmMainNew.Instance.IsMemoryLeakDiagnosticsVerbose = true;

// 关闭详细模式
frmMainNew.Instance.IsMemoryLeakDiagnosticsVerbose = false;

// 立即运行诊断
var report = _memoryLeakDiagnosticsService.RunDiagnosticsNow();
```

## 🎯 推荐使用场景

### 日常运行（推荐配置）
- ✅ 诊断服务：**启用**
- ✅ 详细模式：**关闭**
- 📊 日志输出：只在检测到泄漏时显示警告
- 💡 优点：不影响正常日志，只在有问题时提醒

### 问题排查（临时配置）
- ✅ 诊断服务：**启用**
- ✅ 详细模式：**开启**
- 📊 日志输出：每 5 分钟完整报告
- 💡 优点：详细记录内存变化，便于分析问题
- ⚠️ 注意：问题解决后记得关闭详细模式

### 性能优先（极端情况）
- ✅ 诊断服务：**禁用**
- ✅ 详细模式：无所谓
- 📊 日志输出：无
- 💡 优点：完全不占用资源
- ⚠️ 缺点：失去内存泄漏监控能力

## 📊 日志级别说明

### 日志级别对应关系

| 级别 | 显示条件 | 说明 |
|------|---------|------|
| **Debug** | 正常情况，详细模式关闭 | 每 5 分钟一条简单日志 |
| **Information** | 详细模式开启 | 每 5 分钟完整报告 |
| **Warning** | 检测到内存泄漏 | 完整报告（无论详细模式） |
| **Error** | 诊断服务出错 | 错误详情 |

### 如何查看 Debug 日志

如果默认不显示 Debug 日志，需要调整日志级别：

1. 点击工具栏 **"调试模式"** 按钮
2. 选择 **"Debug"** 级别
3. 或修改配置文件 `appsettings.json`：
```json
"Logging": {
  "LogLevel": {
    "Default": "Debug"
  }
}
```

## 🔧 高级功能

### 立即运行诊断

当怀疑有内存泄漏时，可以手动触发诊断：

1. 点击 **"内存诊断"** 按钮
2. 选择 **"立即运行诊断"**
3. 查看日志输出：
```
[18:30:45 INF] 正在运行内存泄漏诊断...
[18:30:46 INF] 诊断完成 - 工作集：6144 MB, 增长率：15.50 MB/分钟
[18:30:46 WRN] ⚠️ 检测到内存泄漏！预计 258 分钟后达到临界值
```

### 查看诊断报告

完整诊断报告包含：

1. **内存状态** - 工作集、托管内存、私有内存等
2. **泄漏检测** - 是否泄漏、增长率、临界时间
3. **Top 占用者** - 前 10 个内存占用模块
4. **线程分析** - 线程数、线程池、后台线程
5. **建议措施** - 按优先级排序的修复建议
6. **快速诊断** - 针对最大占用者的具体检查项

## 📝 配置示例

### 场景 1：日常监控（推荐）

```csharp
// 在 frmMainNew 构造函数或初始化时设置
_memoryLeakDiagnosticsService.IsEnabled = true;   // 启用诊断
_memoryLeakDiagnosticsService.IsVerboseMode = false; // 关闭详细模式
```

**效果**：
- 每 5 分钟诊断一次
- 只在检测到泄漏时输出警告
- 不影响正常日志

### 场景 2：问题排查

```csharp
// 发现内存异常时
_memoryLeakDiagnosticsService.IsEnabled = true;   // 确保启用
_memoryLeakDiagnosticsService.IsVerboseMode = true;  // 开启详细模式
```

**效果**：
- 每 5 分钟输出完整报告
- 详细记录内存变化
- 便于分析泄漏原因

### 场景 3：性能关键

```csharp
// 高负载时临时禁用
_memoryLeakDiagnosticsService.IsEnabled = false;  // 禁用诊断
```

**效果**：
- 完全停止诊断
- 零日志输出
- 节省 CPU 资源

## 🚨 注意事项

1. **不要长期禁用诊断服务**
   - 诊断服务占用资源极少
   - 禁用后会失去内存泄漏监控能力
   - 建议在问题解决后立即启用

2. **详细模式用后及时关闭**
   - 详细模式会产生较多日志
   - 可能影响日志可读性
   - 建议仅在排查问题时使用

3. **发现泄漏立即处理**
   - 看到警告日志后立即分析
   - 使用"立即运行诊断"获取最新数据
   - 根据报告中的建议检查代码

## 📞 故障排除

### 问题 1：点击按钮无反应
**原因**：服务未正确注册  
**解决**：检查 Startup.cs 是否添加了模块注册

### 问题 2：菜单项显示不全
**原因**：菜单初始化失败  
**解决**：查看日志文件中的错误信息

### 问题 3：日志不输出
**原因**：日志级别设置过高  
**解决**：将日志级别调整为 Debug 或 Information

### 问题 4：诊断报告不准确
**原因**：运行时间太短，数据不足  
**解决**：等待至少 10 分钟，让诊断服务收集足够数据
