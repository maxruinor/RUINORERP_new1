# 列配置管理 - 快速参考指南

## 🚀 快速开始

### 1. 基本使用（3步完成）

```csharp
// 步骤1: 设置菜单ID
newSumDataGridView1.MenuId = currentMenuInfo.MenuID;

// 步骤2: 设置表格键名（通常用数据源类型名）
newSumDataGridView1.XmlFileName = "tb_Employee";

// 步骤3: 启用持久化（默认已是true，可省略）
newSumDataGridView1.NeedSaveColumnsXml = true;

// 完成！配置会自动保存和加载
```

## 💾 保存配置

### 自动保存（推荐）

```csharp
// 用户通过右键菜单"自定义显示列"修改配置后
// 系统会自动延时批量保存（5秒内或达到10个变更）
// 无需手动调用任何方法
```

### 手动立即保存（特殊场景）

```csharp
// 在关闭窗体前或关键操作前强制保存
await ColumnConfigManager.Instance.FlushChangesAsync();
```

## 📖 加载配置

### 自动加载

```csharp
// 数据绑定后会自动加载配置
newSumDataGridView1.DataSource = employeeList;
// 内部会自动调用 LoadColumnsListByCdcAsync()
```

### 手动加载

```csharp
// 重新加载最新配置
var config = await customizeGrid.LoadColumnsListByCdcAsync();
customizeGrid.SetColumns(config);
```

## 🔄 恢复默认配置

### 方式1: 通过UI

```
右键点击表格 → 自定义显示列 → 点击"恢复默认值"按钮
```

### 方式2: 通过代码


```csharp

    "tb_Employee", 
    menuId

{
    Console.WriteLine("数据库中有配置");

    Console.WriteLine("数据库中无配置，将使用默认配置");

```

```csharp

```

## ❌ 常见错误

### 错误1: 配置没有保存

**原因**: 未设置必要参数

```csharp
// ❌ 错误
newSumDataGridView1.NeedSaveColumnsXml = true;

// ✅ 正确
newSumDataGridView1.MenuId = menuInfo.MenuID;        // 必须
newSumDataGridView1.XmlFileName = "tb_Employee";     // 必须
newSumDataGridView1.NeedSaveColumnsXml = true;       // 可选（默认true）
```

### 错误2: 编译错误 - 找不到LoadColumnsListByCdc

**原因**: 方法已改为异步

```csharp
// ❌ 错误
var config = customizeGrid.LoadColumnsListByCdc();

// ✅ 正确
var config = await customizeGrid.LoadColumnsListByCdcAsync();
```

### 错误3: 配置不同步

**原因**: 两个入口使用了不同的存储

**解决**: 现在已统一为数据库存储，确保：
- 两个入口都设置了相同的 `MenuId` 和 `GridKeyName`
- 查看Debug日志确认使用的是数据库而非XML

## 🗄️ 数据库表结构

### tb_UIGridSetting

| 字段 | 类型 | 说明 |
|------|------|------|
| UIGID | bigint | 主键（雪花ID） |
| UIMenuPID | bigint | 菜单ID |
| GridKeyName | varchar(255) | 表格键名（如：tb_Employee） |
| ColsSetting | text | 列配置JSON |
| GridType | varchar(50) | 表格类型（NewSumDataGridView） |
| ColumnsMode | int | 列宽模式 |
| Created_at | datetime | 创建时间 |
| Modified_at | datetime | 修改时间 |

### 查询示例

```sql
-- 查看某个菜单的所有表格配置
SELECT * FROM tb_UIGridSetting 
WHERE UIMenuPID = 123456;

-- 查看某个表格的配置内容
SELECT GridKeyName, LEN(ColsSetting) as JsonLength
FROM tb_UIGridSetting 
WHERE GridKeyName = 'tb_Employee';
```

## 🔧 故障排查

### 问题: 重启后配置丢失

**检查清单**:
1. ✅ `NeedSaveColumnsXml` 是否为 `true`
2. ✅ `MenuId` 是否设置且大于0
3. ✅ `GridKeyName` 是否设置
4. ✅ 查看Debug日志是否有保存成功的记录
5. ✅ 查询数据库确认配置已保存

### 问题: 多个用户配置互相覆盖

**原因**: 使用了相同的 `MenuId`

**解决**: 每个用户的个性化配置应该使用不同的 `MenuId`（通过 `tb_UserPersonalized` 关联）

### 问题: 首次加载很慢

**原因**: 第一次需要从数据库查询

**解决**: 这是正常现象，后续加载会使用内存缓存，速度会非常快
