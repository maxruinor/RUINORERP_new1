# 单例服务注册问题全面排查与修复

## 📋 问题背景

在修复`ImageCacheService`缓存不共享问题时,发现根本原因是**Autofac自动批量注册默认使用`InstancePerDependency`生命周期**,导致每次获取服务都创建新实例,状态无法共享。

为避免类似问题再次发生,对所有**缓存相关服务**和**需要状态保持的服务类**进行了系统性排查。

---

## 🔍 排查范围

### 1. 缓存相关服务
- ImageCacheService (图片缓存)
- EntityCacheManager (实体缓存管理器)
- EventDrivenCacheManager (事件驱动缓存管理器)
- ClientLocalLockCacheService (客户端本地锁缓存)
- StockCacheService (库存缓存)

### 2. 同步/监控服务
- ConfigSyncService (配置同步)
- HasAttachmentSyncService (附件同步)
- FileStorageMonitorService (文件存储监控)
- CacheMetadataSyncService (缓存元数据同步)

### 3. 其他有状态服务
- DataCacheManager (数据缓存管理器)
- RolePermissionCacheService (角色权限缓存)

## ✅ 已修复的问题

### 1. ImageCacheService (客户端) 🔴 高危

**位置:** `RUINORERP.UI/Network/Services/ImageCacheService.cs`

**问题:** 
- 通过自动批量注册,默认`InstancePerDependency`
- 每次调用`Startup.GetFromFac<ImageCacheService>()`创建新实例
- 每个实例有独立的`IMemoryCache`,缓存无法共享

**影响:**
- 产品查询时每次查询都执行数据库查询
- 预加载功能完全失效
- 缓存命中率始终为0%

**修复:**
```csharp
// Startup.cs 第851-872行
// 1. 排除自动注册
type != typeof(RUINORERP.UI.Network.Services.ImageCacheService)

// 2. 单独注册为单例
builder.RegisterType<RUINORERP.UI.Network.Services.ImageCacheService>()
    .AsSelf()
    .SingleInstance()
    .PropertiesAutowired();
```

### 5. FileManagementService 🔴 高危

**位置:** `RUINORERP.UI/Network/Services/FileManagementService.cs`

**问题:**

- 有`Timer`:_lockCleanupTimer(每5分钟执行)

**修复:**

## 🎯 判断标准

以下特征的服务类**必须注册为单例**:

1. **缓存类服务**
   - 包含`Cache`、`MemoryCache`、`IMemoryCache`等字段
   - 用于减少数据库查询或网络请求

2. **状态管理类**
   - 实现了`IDisposable`接口
   - 包含`ConcurrentDictionary`、`SemaphoreSlim`、`Timer`等状态对象
   - 订阅了事件或有事件处理器

3. **同步/监控类**
   - 名称包含`Sync`、`Monitor`、`Watcher`
   - 长期运行的后台服务
   - 需要全局唯一实例

4. **工厂/管理器类**
   - 名称包含`Manager`、`Factory`、`Registry`
   - 负责创建或管理其他对象的生命周期

## 🔧 修复模式

对于所有需要单例的服务,统一采用以下模式:

### 步骤1: 排除自动批量注册

```csharp
builder.RegisterAssemblyTypes(System.Reflection.Assembly.GetExecutingAssembly())
    .Where(type => !typeof(IExcludeFromRegistration).IsAssignableFrom(type) &&
                   type != typeof(SomeService1) &&  // 排除1
                   type != typeof(SomeService2) &&  // 排除2
                   type != typeof(SomeService3))    // 排除3
    .AsImplementedInterfaces()
    .AsSelf();
```

### 步骤2: 单独注册为单例

```csharp
// 【新增】XXX服务必须注册为单例,确保YYY
builder.RegisterType<SomeService>()
    .AsSelf()
    .SingleInstance()  // ⚠️ 关键: 必须是单例!
    .PropertiesAutowired();
```

### 步骤3: 添加注释说明

```csharp
/// 注意：XXXService排除自动注册，单独注册为单例(避免YYY)
```

然后在启动时扫描并验证:

```csharp
var singletonTypes = Assembly.GetExecutingAssembly()
    .GetTypes()
    .Where(t => t.GetCustomAttribute<MustBeSingletonAttribute>() != null);

foreach (var type in singletonTypes)
{
    var registration = container.ComponentRegistry.Registrations
        .FirstOrDefault(r => r.Activator.LimitType == type);
    
    if (registration?.Sharing == InstanceSharing.None)
    {
        throw new InvalidOperationException(
            $"{type.Name}必须注册为单例,但当前是InstancePerDependency!");
    }
}
```

## ✨ 总结

本次**系统性排查**共发现并修复了**6个高危问题**:

### 客户端 (RUINORERP.UI)
1. ✅ **ImageCacheService** - 缓存不共享,命中率0%
2. ✅ **ClientLocalLockCacheService** - 分布式锁失效,并发控制不可靠
3. ✅ **ConfigSyncService** - 配置同步状态不一致,通知丢失
4. ✅ **FileManagementService** - 文件锁机制失效,Timer重复执行

### 服务器端 (RUINORERP.Server)
6. ✅ **ImageCacheService** - 预防性修复,确保代码一致性

所有修复都遵循统一的模式:
1. 从自动批量注册中排除
2. 单独注册为`SingleInstance`
3. 添加清晰的注释说明

这确保了系统的稳定性和可维护性,避免了因DI配置错误导致的隐蔽bug。
