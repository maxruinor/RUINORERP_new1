# 去重策略验证功能修复说明

## 修复日期
2026-04-26

## 问题概述
当用户启用去重功能(`EnableDeduplication = true`)但没有选择去重策略时,`DeduplicateStrategy` 枚举值为 `-1`(无效值),导致XML序列化失败,无法保存配置。

**错误信息**:
```
生成 XML 文档时出错。
实例验证错误:"-1"不是 RUINORERP.UI.SysConfig.BasicDataImport.DeduplicateStrategy 的有效值。
```

---

## 根本原因分析

### 1. ComboBox默认状态问题
- `kcmbDeduplicateStrategy` ComboBox在初始化时 `SelectedIndex = -1`
- 当用户勾选"去重复值"复选框但未选择策略时,直接使用 `SelectedIndex` 转换枚举会得到 `-1`
- XML序列化器无法将 `-1` 转换为有效的 `DeduplicateStrategy` 枚举值

### 2. 缺少输入验证
- 保存配置前没有验证去重功能的完整性
- 允许不完整配置进入序列化流程

### 3. 初始化不完善
- 窗体加载时未设置ComboBox默认值
- 新建映射时未重置去重设置

## DeduplicateStrategy 枚举定义

```csharp
public enum DeduplicateStrategy
{
    FirstOccurrence = 0,   // 保留第一条记录
    LastOccurrence = 1     // 保留最后一条记录
}
```

**有效值范围**: `0` 或 `1`  
**无效值**: `-1` (未选择)、`2+` (超出范围)

## 预防措施

### 1. 枚举类型安全
在使用枚举与UI控件交互时:
```csharp
// ❌ 危险做法 - 直接转换可能得到无效值
var strategy = (DeduplicateStrategy)comboBox.SelectedIndex;

// ✅ 安全做法 - 先验证再转换
if (comboBox.SelectedIndex >= 0)
{
    var strategy = (DeduplicateStrategy)comboBox.SelectedIndex;
}
else
{
    // 处理无效状态
}
```

### 2. 输入验证原则
- **前端验证**: 在UI层进行即时验证,提供友好提示
- **后端验证**: 在业务逻辑层再次验证,确保数据完整性
- **防御性编程**: 假设用户可能输入任何值,永远不要信任外部输入

### 3. 默认值策略
- 所有ComboBox在初始化时设置合理的默认值
- 避免使用 `-1` 作为有效业务状态
- 新建/重置时恢复到一致的初始状态

### 4. 单元测试建议

## 技术要点

### XML序列化与枚举
XmlSerializer在序列化枚举时会验证值的有效性:
- ✅ 有效枚举值 → 序列化为字符串或整数
- ❌ 无效枚举值 → 抛出 `InvalidOperationException`

### ComboBox SelectedIndex
- `-1`: 未选择任何项(初始状态或清除选择后)
- `0`: 第一个项
- `1`: 第二个项
- `N`: 第N+1个项

### 用户体验优化
- **即时反馈**: 在用户尝试保存时立即提示错误
- **明确指引**: 告诉用户具体需要做什么
- **自动聚焦**: 将焦点移到需要操作的控件,减少用户操作步骤
