## 模块概述

### 业务目标
实现**发票与收付款的强关联闭环管理**，确保：
- ✅ 收款时自动追踪待开票金额
- ✅ 付款时验证进项发票到位情况
- ✅ 支持一对多、多对多的灵活核销关系
- ✅ 实时计算未核销余额，防止超额核销

### 核心场景
1. **销项发票场景**：销售出库 → 生成应收 → 开具销项发票 → 客户付款 → 发票核销
2. **进项发票场景**：采购入库 → 生成应付 → 供应商开票 → 公司付款 → 发票核销
3. **预收预付场景**：预收款/预付款 → 后续开票 → 关联核销

### 表结构澄清

**重要说明**：项目中存在两个名称相似的表，需明确区分：

1. **tb_BillingInformation（开票资料表）** ✅ **保留使用**
   - 用途：存储往来单位的**开票基础信息档案**
   - 字段：Title（抬头）、TaxNumber（税号）、Address（地址）、BankAccount（银行账号）等
   - 关联：`CustomerVendor_ID` → 客户/供应商
   - 场景：给客户开发票时，选择该客户的开票资料
   
2. **tb_InvoiceInfo（重复表）** ❌ **已删除**
   - 原因：与 tb_BillingInformation 功能重叠
   - 处理：已创建迁移脚本 `SQLScripts/Drop_Duplicate_tb_InvoiceInfo.sql`
   - 建议：如需财务字段（信用天数、余额等），应添加到 `tb_CustomerVendor` 或动态计算

**正确引用关系**：
```
tb_CustomerVendor (往来单位)
    ↓ 1:N
tb_BillingInformation (开票资料档案) ← tb_FM_Invoice.BillingInfo_ID 引用
    
tb_FM_Invoice (发票单据)
    ↓ N:M (通过关联表)
tb_FM_PaymentRecord (收付款记录)
```

## 架构设计

### 数据流图
```
┌─────────────┐         ┌──────────────────┐         ┌──────────────┐
│  业务单据    │ ──────→ │  应收应付(tb_FM_  │ ──────→ │  收付款记录   │
│ (销售/采购)  │         │ ReceivablePayable)│         │(PaymentRecord)│
└─────────────┘         └────────┬─────────┘         └──────┬───────┘
                                 │                           │
                                 ↓                           ↓
                         ┌──────────────────┐         ┌──────────────┐
                         │   发票主表        │ ←────── │  关联关系表   │
                         │ (tb_FM_Invoice)  │         │(InvoicePaymen│
                         └──────────────────┘         │ tRelation)   │
                                                      └──────────────┘
```

### 关键设计原则
1. **中间关联表模式**：通过 `tb_FM_InvoicePaymentRelation` 解耦发票和收付款
2. **冗余字段优化查询**：在关联表中冗余 InvoiceNo、PaymentNo，避免频繁JOIN
3. **状态同步机制**：通过触发器或服务层同步更新主表的核销状态和金额
4. **部分核销支持**：允许一张发票分多次收付款，或一次收付款核销多张发票

## 业务流程

### 流程1：创建发票并关联应收
```
1. 用户创建销项发票（tb_FM_Invoice）
   - 设置 InvoiceType = 0（销项）
   - 填写 CustomerVendor_ID、TotalWithTax 等

2. 系统自动查找对应的应收应付记录（tb_FM_ReceivablePayable）
   - 通过 SourceBillId 匹配销售订单/出库单

3. 建立初始关联（可选）
   - 在 tb_FM_InvoicePaymentRelation 中创建记录
   - ARAPId 指向应收记录
   - VerificationStatus = 0（未核销）
```

### 流程2：收款并核销发票
```
1. 用户创建收款记录（tb_FM_PaymentRecord）
   - ReceivePaymentType = 1（收款）
   - 填写 TotalLocalAmount（实收金额）

2. 用户选择要核销的发票
   - 系统查询该客户的未完全核销发票列表
   - 显示 UnlinkedAmount > 0 的发票

3. 用户指定核销金额
   - 可以全额核销或部分核销
   - 系统验证：核销金额 ≤ 发票未关联金额

4. 系统创建关联记录
   - 插入 tb_FM_InvoicePaymentRelation
   - 更新发票的 TotalLinkedAmount、UnlinkedAmount
   - 更新发票的 VerificationStatus
   - 更新收付款的 TotalLinkedInvoiceAmount

5. 更新应收应付状态
   - 减少 tb_FM_ReceivablePayable.ForeignBalanceAmount
   - 如果余额为0，标记应收为"已结清"
```

### 流程3：查询发票核销状态
```sql
-- 查询某张发票的核销详情
SELECT 
    inv.InvoiceNo,
    inv.TotalWithTax AS 发票总额,
    inv.TotalLinkedAmount AS 已核销金额,
    inv.UnlinkedAmount AS 未核销金额,
    inv.VerificationStatus AS 核销状态,
    rel.PaymentNo AS 收付款单号,
    rel.LocalLinkedAmount AS 本次核销金额,
    rel.VerifiedDate AS 核销日期
FROM tb_FM_Invoice inv
LEFT JOIN tb_FM_InvoicePaymentRelation rel ON inv.InvoiceId = rel.InvoiceId
WHERE inv.InvoiceId = @InvoiceId
```

## 使用示例

### C# 代码示例

#### 示例1：创建发票-收付款关联
```csharp
public async Task CreateInvoicePaymentRelation(long invoiceId, long paymentId, decimal linkedAmount)
{
    using var db = GetDatabase();
    
    // 1. 获取发票信息
    var invoice = await db.Queryable<tb_FM_Invoice>()
        .Where(x => x.InvoiceId == invoiceId)
        .FirstAsync();
        
    if (invoice == null)
        throw new Exception("发票不存在");
        
    // 2. 验证可核销金额
    if (linkedAmount > invoice.UnlinkedAmount)
        throw new Exception($"核销金额超过未核销余额！可核销金额：{invoice.UnlinkedAmount}");
    
    // 3. 获取收付款信息
    var payment = await db.Queryable<tb_FM_PaymentRecord>()
        .Where(x => x.PaymentId == paymentId)
        .FirstAsync();
        
    // 4. 创建关联记录
    var relation = new tb_FM_InvoicePaymentRelation
    {
        InvoiceId = invoiceId,
        InvoiceNo = invoice.InvoiceNo,
        PaymentId = paymentId,
        PaymentNo = payment.PaymentNo,
        ARAPId = null, // 可从应收应付表获取
        ForeignLinkedAmount = linkedAmount / invoice.ExchangeRate,
        LocalLinkedAmount = linkedAmount,
        VerificationStatus = 0,
        VerifiedDate = DateTime.Now,
        Created_at = DateTime.Now,
        Created_by = CurrentUser.UserId
    };
    
    await db.Insertable(relation).ExecuteCommandAsync();
    
    // 5. 更新发票核销状态
    invoice.TotalLinkedAmount += linkedAmount;
    invoice.UnlinkedAmount -= linkedAmount;
    invoice.VerificationStatus = invoice.UnlinkedAmount <= 0 ? 2 : 1;
    invoice.ActualPaymentDate = DateTime.Now;
    
    await db.Updateable(invoice).ExecuteCommandAsync();
    
    // 6. 更新收付款关联金额
    payment.TotalLinkedInvoiceAmount += linkedAmount;
    payment.UnlinkedInvoiceAmount = payment.TotalLocalAmount - payment.TotalLinkedInvoiceAmount;
    
    await db.Updateable(payment).ExecuteCommandAsync();
}
```

#### 示例2：查询未核销发票列表
```csharp
public async Task<List<tb_FM_Invoice>> GetUnverifiedInvoices(long customerVendorId, int invoiceType)
{
    using var db = GetDatabase();
    
    return await db.Queryable<tb_FM_Invoice>()
        .Where(x => x.CustomerVendor_ID == customerVendorId)
        .Where(x => x.InvoiceType == invoiceType)
        .Where(x => x.UnlinkedAmount > 0)
        .Where(x => !x.isdeleted)
        .OrderByDescending(x => x.InvoiceDate)
        .ToListAsync();
}
```

#### 示例3：批量核销
```csharp
public async Task BatchVerifyInvoices(List<InvoiceVerificationItem> items)
{
    using var db = GetDatabase();
    using var tran = db.Ado.BeginTran();
    
    try
    {
        foreach (var item in items)
        {
            await CreateInvoicePaymentRelation(
                item.InvoiceId, 
                item.PaymentId, 
                item.LinkedAmount
            );
        }
        
        db.Ado.CommitTran();
    }
    catch (Exception ex)
    {
        db.Ado.RollbackTran();
        throw new Exception($"批量核销失败：{ex.Message}");
    }
}

public class InvoiceVerificationItem
{
    public long InvoiceId { get; set; }
    public long PaymentId { get; set; }
    public decimal LinkedAmount { get; set; }
}
```

## 注意事项

### ⚠️ 数据一致性
1. **事务保护**：所有涉及关联记录创建和状态更新的操作必须在事务中执行
2. **并发控制**：使用乐观锁或悲观锁防止并发核销导致超额
3. **回滚机制**：提供"取消核销"功能，支持误操作恢复

### ⚠️ 业务规则
1. **红字发票处理**：红字发票的核销金额为负数，需特殊处理
2. **多币种支持**：确保外币金额和本币金额的汇率一致性
3. **冲销记录**：收付款冲销时，需同步解除发票关联

### ⚠️ 权限控制
- 只有财务人员可以执行核销操作
- 核销记录不允许直接删除，只能通过"取消核销"功能
- 审计日志记录所有核销操作

## 总结

本模块通过引入**中间关联表**实现了发票与收付款的灵活关联，具有以下优势：

✅ **解耦设计**：发票和收付款独立维护，通过关联表建立关系

✅ **状态追踪**：实时计算核销状态和未核销余额  
✅ **可扩展性**：易于添加自动化规则、预警提醒等功能  
✅ **兼容性**：不影响现有业务流程，可逐步迁移
