# 图片缓存二进制数据修复方案

## 🔴 问题描述

### 现象
产品查询时,虽然`ImageCacheService`已注册为单例,但**每次查询都重新从服务器下载图片**,缓存命中率接近0%。

### 根本原因

**服务器端缓存的对象不包含二进制数据(`FileData`)**!

#### 问题分析流程

1. **客户端请求图片**: `FileBusinessService.DownloadImageAsync(fileId)`
2. **检查客户端缓存**: `ImageCacheService.GetImageInfo(fileId)` → 未命中
3. **HTTP请求到服务器**: `FileCommandHandler.HandleFileDownloadAsync()`
4. **服务器查询数据库**: `_fileStorageInfoController.QueryByNavAsync()` 
   - ❌ **问题**: `QueryByNavAsync`只查询元数据,**不加载`FileData`字段**
   - 原因: `FileData`在`tb_FS_FileStorageInfoFix.cs`中定义,没有SqlSugar映射特性
5. **服务器缓存对象**: `_imageCacheService.AddImageInfo(imageInfo)`
   - ❌ **问题**: 此时`imageInfo.FileData == null`
6. **服务器读取文件**: `File.ReadAllBytesAsync(filePath)` (第627行)
   - ✅ 加载二进制数据到`FileData`
7. **返回给客户端**: 包含完整`FileData`的响应
8. **客户端缓存**: `ImageCacheService.AddImageInfos(response.FileStorageInfos)`
   - ✅ 客户端缓存包含`FileData`

#### 关键矛盾

- **服务器端缓存**: 只有元数据,`FileData == null` ❌
- **客户端缓存**: 有完整数据,`FileData != null` ✅

**结果**: 
- 第一次查询: 客户端缓存未命中 → HTTP请求 → 服务器查询DB(无FileData) → 读取文件 → 返回 → 客户端缓存
- 第二次查询: 客户端缓存命中 ✅ → 直接返回(有FileData)
- **但如果客户端缓存失效**: 再次HTTP请求 → 服务器缓存命中(但FileData==null) → ❌ **需要重新读取文件!**

---

## ✅ 解决方案

### 核心思路

**在服务器端缓存之前,先加载`FileData`二进制数据**,确保缓存的对象是完整的。

### 修改位置

`e:\CodeRepository\SynologyDrive\RUINORERP\RUINORERP.Server\Network\CommandHandlers\FileCommandHandler.cs`

### 具体修复

#### 1. 修复`DownloadByFileIdAsync`方法

**修改前:**
```csharp
private async Task<tb_FS_FileStorageInfo> DownloadByFileIdAsync(long fileId, CancellationToken cancellationToken)
{
    // 先从缓存获取
    var cachedImageInfo = _imageCacheService.GetImageInfo(fileId);
    if (cachedImageInfo != null)
    {

```

**修改后:**
```csharp
private async Task<tb_FS_FileStorageInfo> DownloadByFileIdAsync(long fileId, CancellationToken cancellationToken)
{
    // ✅ 先从缓存获取(已包含FileData)
    var cachedImageInfo = _imageCacheService.GetImageInfo(fileId);
    if (cachedImageInfo != null && cachedImageInfo is tb_FS_FileStorageInfo cached && cached.FileData != null)
    {
        _logger?.LogDebug("✅ 缓存命中(含二进制数据): FileId={FileId}, Size={Size}KB", 
            fileId, cached.FileData.Length / 1024);
        return cached;
    }

    // 缓存未命中，从数据库查询元数据
    var fileInfos = await _fileStorageInfoController.QueryByNavAsync(...);
    if (fileInfos != null && fileInfos.Count > 0)
    {
        tb_FS_FileStorageInfo imageInfo = fileInfos[0];
        
        // ✅ 关键修复: 从文件系统加载二进制数据到FileData
        try
        {
            string fullRelativePath = Path.Combine(imageInfo.StoragePath, imageInfo.StorageFileName);
            string filePath = FileStorageHelper.ResolveToAbsolutePath(fullRelativePath);
            
            if (File.Exists(filePath))
            {
                imageInfo.FileData = await File.ReadAllBytesAsync(filePath, cancellationToken);
                _logger?.LogDebug("📦 从文件系统加载二进制数据: FileId={FileId}, Size={Size}KB", 
                    fileId, imageInfo.FileData.Length / 1024);
            }
            else
            {
                _logger?.LogWarning("⚠️ 物理文件不存在: {FilePath}", filePath);
            }
        }
        catch (Exception ex)
        {
            _logger?.LogError(ex, "❌ 加载文件二进制数据失败: FileId={FileId}", fileId);
        }
        
        // ✅ 添加到缓存(现在包含FileData)
        _imageCacheService.AddImageInfo(imageInfo);
        _logger?.LogDebug("💾 已将完整图片信息(含二进制)加入缓存: FileId={FileId}", fileId);
        return imageInfo;
    }
    return null;
}
```

#### 4. 优化`HandleFileDownloadAsync`避免重复IO

**修改前:**
```csharp
foreach (var fileInfo in fileList)
{
    // ❌ 无论FileData是否存在,都重新读取文件
    string filePath = FileStorageHelper.ResolveToAbsolutePath(...);
    fileInfo.FileData = await File.ReadAllBytesAsync(filePath, cancellationToken);  // 重复IO!
    successfulFiles.Add(fileInfo);
}
```

**修改后:**
```csharp
foreach (var fileInfo in fileList)
{
    try
    {
        // ✅ 优化: 如果FileData已存在(从缓存或DownloadByFileIdAsync加载),跳过重复IO
        if (fileInfo.FileData != null && fileInfo.FileData.Length > 0)
        {
            _logger?.LogDebug("✅ FileData已存在,跳过重复读取: FileId={FileId}, Size={Size}KB", 
                fileInfo.FileId, fileInfo.FileData.Length / 1024);
            successfulFiles.Add(fileInfo);
            continue;
        }
        
        // FileData为空,需要从文件系统读取
        string filePath = FileStorageHelper.ResolveToAbsolutePath(...);
        if (!File.Exists(filePath))
        {
            // 错误处理...
            continue;
        }

        fileInfo.FileData = await File.ReadAllBytesAsync(filePath, cancellationToken);
        successfulFiles.Add(fileInfo);
    }
    catch (Exception ex)
    {
        // 错误处理...
    }
}
```

## 🎯 关键改进点

1. **服务器端缓存完整性**: 缓存的对象包含完整的`FileData`二进制数据
2. **避免重复IO**: 如果`FileData`已存在,跳过文件读取操作
3. **三层缓存协同**:
   - 客户端内存缓存(IMemoryCache) - 最快
   - 服务器端内存缓存(IMemoryCache) - 次快
   - 文件系统 - 最慢(仅在缓存未命中时访问)

## ⚠️ 注意事项

1. **内存占用**: 缓存二进制数据会增加内存使用,需监控内存压力
2. **缓存过期策略**: 建议设置合理的滑动过期时间(如4小时)和绝对过期时间(如8小时)
3. **大文件处理**: 对于超大图片(>10MB),考虑是否缓存或采用其他策略
4. **缓存清理**: 文件更新或删除时,需要同步清理缓存
