## 3. 核心状态流转详解

### 3.1 正常流程

```
1. 草稿 → 提交 → 待审核 → 审核通过 → 已审核 → 完结
```

### 3.2 撤销流程（关键场景）

**业务场景：** 客户之前无法付款做了坏账处理，后来客户又愿意支付了。

```
1. 已审核（坏账已核销）
   ↓
2. 执行"撤销坏账"操作
   ↓
3. 系统处理：
   - 恢复应收款的余额（加上之前核销的金额）
   - 更新ARAPStatus为"待支付"（原状态）
   - 生成红字核销记录（IsReversedWriteOff=true）
   - 坏账申请单状态变为"已撤销"
   ↓
4. 财务人员继续进行正常的收款流程
   ↓
5. 客户付款
   ↓
6. 正常核销应收款
```

### 3.3 反审核流程

**业务场景：** 审核后发现有问题，需要打回重新修改。

```
1. 已审核
   ↓
2. 执行"反审核"操作
   ↓
3. 系统处理：
   - 坏账申请单状态变为"待审核"
   - 不恢复应收款余额（还在坏账状态）
   ↓
4. 修改后重新提交审核
```

### 4.2 初始化坏账状态转换规则

在 `GlobalStateRulesManager.cs` 的初始化方法中添加：

```csharp
/// <summary>
/// 初始化坏账申请单状态转换规则
/// </summary>
private void InitializeBadDebtStatusTransitionRules()
{
    if (_statusTransitionRules.ContainsKey(BadDebtStatusType))
        return;

    _statusTransitionRules[BadDebtStatusType] = new Dictionary<object, List<object>>
    {
        // 草稿 -> 新建、作废
        { BadDebtStatus.草稿, new List<object>
            {
                BadDebtStatus.待审核,
                BadDebtStatus.已作废
            }
        },

        // 待审核 -> 已审核、草稿(驳回)、作废
        { BadDebtStatus.待审核, new List<object>
            {
                BadDebtStatus.已审核,
                BadDebtStatus.草稿,  // 驳回
                BadDebtStatus.已作废
            }
        },

        // 已审核 -> 已撤销、作废
        // 关键：支持撤销操作
        { BadDebtStatus.已审核, new List<object>
            {
                BadDebtStatus.已撤销,
                BadDebtStatus.已作废
            }
        },

        // 已撤销 -> 终态（可手动完结或继续收款）
        { BadDebtStatus.已撤销, new List<object>() },

        // 已完结 -> 终态，不可转换
        { BadDebtStatus.已完结, new List<object>() },

        // 已作废 -> 终态，不可转换
        { BadDebtStatus.已作废, new List<object>() }
    };
}
```

## 5. ARAPStatus 状态调整

### 5.1 现有ARAPStatus分析

```csharp
public enum ARAPStatus
{
    草稿 = 1,
    待审核 = 2,
    待支付 = 3,
    部分支付 = 4,
    全部支付 = 5,
    坏账 = 6,
    已冲销 = 7
}
```

### 5.2 ARAPStatus流转调整

**问题：** 现有设计中 `坏账` 是终态，无法转换回其他状态。

**解决方案：** 调整ARAPStatus的流转规则，支持从"坏账"状态转换回"待支付"或"部分支付"状态。

```csharp
/// <summary>
/// 初始化应收应付状态转换规则（调整后）
/// </summary>
private void InitializeARAPStatusTransitionRules_Adjusted()
{
    _statusTransitionRules[typeof(ARAPStatus)] = new Dictionary<object, List<object>>
    {
        // 草稿 -> 待审核
        { ARAPStatus.草稿, new List<object>
            {
                ARAPStatus.待审核,
                ARAPStatus.已作废
            }
        },

        // 待审核 -> 待支付
        { ARAPStatus.待审核, new List<object>
            {
                ARAPStatus.待支付,
                ARAPStatus.已作废
            }
        },

        // 待支付 -> 部分支付、全部支付、坏账
        { ARAPStatus.待支付, new List<object>
            {
                ARAPStatus.部分支付,
                ARAPStatus.全部支付,
                ARAPStatus.坏账,
                ARAPStatus.已冲销
            }
        },

        // 部分支付 -> 全部支付、坏账、已冲销
        { ARAPStatus.部分支付, new List<object>
            {
                ARAPStatus.全部支付,
                ARAPStatus.坏账,
                ARAPStatus.已冲销
            }
        },

        // 全部支付 -> 已冲销
        { ARAPStatus.全部支付, new List<object>
            {
                ARAPStatus.已冲销
            }
        },

        // 坏账 -> 待支付（撤销坏账时恢复）
        // 关键调整：坏账不再是终态
        { ARAPStatus.坏账, new List<object>
            {
                ARAPStatus.待支付,       // 撤销坏账时
                ARAPStatus.部分支付     // 部分撤销时
            }
        },

        // 已冲销 -> 终态
        { ARAPStatus.已冲销, new List<object>() }
    };
}
```

## 6. 状态转换业务逻辑

### 6.1 审核通过（坏账核销）

**触发条件：** 待审核 → 已审核

**业务逻辑：**
1. 校验坏账金额 ≤ 应收/应付单未核销金额
2. 更新应收/应付单：
   - `LocalBalanceAmount = LocalBalanceAmount - BadDebtAmount`
   - `LocalPaidAmount = LocalPaidAmount + BadDebtAmount`
   - `ARAPStatus = 坏账`
3. 创建核销记录（tb_FM_BadDebtWriteOff）
4. 更新坏账申请单：
   - `BadDebtStatus = 已审核`
   - `IsWriteOff = true`
   - `WriteOffAt = 当前时间`

### 6.2 撤销坏账（关键流程）

**触发条件：** 已审核 → 已撤销

**业务逻辑：**
1. 校验是否可以撤销：
   - 坏账申请单状态必须是"已审核"
   - 关联的应收/应付单状态必须是"坏账"
2. 更新应收/应付单：
   - `LocalBalanceAmount = LocalBalanceAmount + BadDebtAmount`（恢复余额）
   - `LocalPaidAmount = LocalPaidAmount - BadDebtAmount`（减少已核销）
   - `ARAPStatus = 待支付`（恢复原状态）
3. 创建红字核销记录（tb_FM_BadDebtWriteOff）：
   - `WriteOffLocalAmount = -BadDebtAmount`（负数表示撤销）
   - `IsReversedWriteOff = true`
   - `SettlementType = 坏账撤销核销(11)`
4. 更新坏账申请单：
   - `BadDebtStatus = 已撤销`
   - `IsReversed = true`
   - `ReversedAt = 当前时间`
   - `ReversedBy = 当前用户`
   - `ReversedReason = 撤销原因`
5. 记录操作日志

### 6.3 反审核

**触发条件：** 已审核 → 待审核

**业务逻辑：**
1. 校验是否可以反审核：
   - 坏账申请单状态必须是"已审核"
   - 且未被撤销（`IsReversed = false`）
2. 更新坏账申请单：
   - `BadDebtStatus = 待审核`
3. 不恢复应收/应付单余额（还在坏账状态）

### 6.4 完结

**触发条件：** 已审核/已撤销 → 已完结

**业务逻辑：**
1. 更新坏账申请单：
   - `BadDebtStatus = 已完结`
   - `ProcessedAt = 当前时间`

## 8. 权限控制

### 8.1 按钮级别权限

| 按钮 | 权限角色 | 说明 |
|-----|---------|------|
| 提交 | 财务人员 | 将草稿提交审核 |
| 审核 | 财务主管 | 审核坏账申请 |
| 反审核 | 财务主管 | 将审核单打回 |
| 撤销坏账 | 财务主管 | 撤销坏账恢复余额 |
| 完结 | 财务经理 | 完结坏账单据 |
| 作废 | 财务经理 | 作废坏账单据 |

### 8.2 数据级别权限

| 字段 | 权限控制 | 说明 |
|-----|---------|------|
| 坏账金额 | 仅创建人可修改 | 审核后不可改 |
| 坏账原因 | 仅创建人可修改 | 审核后不可改 |
| 撤销原因 | 撤销人填写 | 必填 |
