##### 问题1: 内存泄漏风险 ⚠️ → ✅ 已修复
**问题描述**: 
- CellFormatting事件中频繁创建Bitmap对象
- 虽然有缓存机制,但旧图片释放不及时
- 大数据量预览时可能导致内存持续增长

**修复方案**:
- 添加`_currentDisplayImages`字典追踪当前显示的图片
- 实现`RegisterDisplayImage`方法,在替换图片时及时释放旧图片
- 增强`ClearImageCache`方法,同时清理缓存和显示追踪

**修复文件**: [UCBasicDataImport.cs](file://e:\CodeRepository\SynologyDrive\RUINORERP\RUINORERP.UI\SysConfig\BasicDataImport\UCBasicDataImport.cs#L48-L140)

**代码变更**:
```csharp
// 新增字段
private readonly Dictionary<string, Image> _currentDisplayImages = new Dictionary<string, Image>();
private readonly object _displayImagesLock = new object();

// 新增方法
private void RegisterDisplayImage(Image image, string key)
{
    lock (_displayImagesLock)
    {
        if (_currentDisplayImages.TryGetValue(key, out var oldImage))
        {
            if (!_imageCache.ContainsKey(key))
            {
                oldImage?.Dispose(); // 及时释放旧图片
            }
        }
        _currentDisplayImages[key] = image;
    }
}
```

```csharp
// 加强的验证规则
private bool IsValidSqlIdentifier(string identifier)
{
    if (identifier.Length < 1 || identifier.Length > 128)
        return false;
    
    if (!Regex.IsMatch(identifier, @"^[a-zA-Z_][a-zA-Z0-9_]*$"))
        return false;
        
    if (identifier.Contains("__")) // 禁止连续下划线
        return false;
        
    return true;
}

// 新增白名单验证
private bool IsKnownEntityField(Type entityType, string fieldName)
{
    var property = entityType.GetProperty(fieldName, 
        BindingFlags.Public | BindingFlags.Instance);
    return property != null;
}
```

##### 问题3: 唯一性检查性能低下 ⚠️ → ✅ 已修复
**问题描述**:
- 每个唯一性字段都执行一次数据库查询
- 如果有3个唯一性字段,就会查询3次数据库
- 大数据量时性能瓶颈明显

**修复方案**:
- 实现`BatchQueryExistingUniqueValues`方法,一次性查询所有字段
- 使用UNION ALL合并多个字段的查询
- 参数化查询防止SQL注入
- 失败时自动降级到逐个查询(容错机制)

**性能提升**: 
- 原来: N个字段 = N次数据库查询
- 现在: N个字段 = 1次数据库查询
- 预计性能提升: **60-80%** (取决于字段数量)

**修复文件**: [UCBasicDataImport.cs](file://e:\CodeRepository\SynologyDrive\RUINORERP\RUINORERP.UI\SysConfig\BasicDataImport\UCBasicDataImport.cs#L1571-L1755)
