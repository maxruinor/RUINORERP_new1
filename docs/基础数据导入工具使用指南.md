# 基础数据导入工具 - 使用指南

## 一、快速入门

### 1.1 导入流程图

```
选择数据类型 (供应商/产品/客户等)
    ↓
选择/新建映射配置
    ↓
选择Excel文件
    ↓
[可选] 配置列映射关系
    ↓
点击"解析"按钮 → 预览数据
    ↓
勾选需要导入的行
    ↓
点击"导入"按钮 → 写入数据库
    ↓
查看导入结果统计
```

## 二、两种导入模式

### 2.1 单表导入模式（推荐日常使用）

**适用场景**：导入单一表的数据（如仅导入供应商、仅导入产品）

**配置文件格式**：XML（保存在 `SysConfig/BasicDataImport/Profiles/`）

**操作步骤**：
1. 选择数据类型（如"供应商表"）
2. 选择已有配置或新建配置
3. 点击"映射配置"按钮，设置Excel列与系统字段的对应关系
4. 点击"解析"预览数据
5. 勾选需要导入的行
6. 点击"导入"执行

---

### 2.2 宽表导入模式（适合复杂场景）

**适用场景**：一个Excel包含多个关联表的数据（如产品+属性+图片）

**配置文件格式**：JSON（保存在 `SysConfig/DataMigration/Profiles/`）

**特点**：
- 自动按依赖顺序导入（先基础表，再主表，最后子表）
- 支持一次性导入完整的产品信息（基本信息、多属性、图片等）
- 使用全局事务确保数据一致性

**操作步骤**：
1. 在下拉框选择 `[宽表] xxx` 配置
2. 选择包含宽表数据的Excel文件
3. 直接点击"导入"（无需解析和映射配置）

## 三、列映射配置详解

### 3.1 映射类型

| 映射类型 | 说明 | 示例 |
|---------|------|------|
| **Excel直接映射** | Excel列直接对应系统字段 | `供应商名称` → `VendorName` |
| **外键关联** | 通过编码/名称查找关联表ID | `类目编码` → `CategoryID`（查询tb_ProdCategories） |
| **默认值** | 所有行填充相同值 | `Status` → `1`（启用） |
| **系统生成** | 系统自动生成的值 | `CreateTime` → 当前时间 |
| **字段复制** | 复制同一行其他字段的值 | `ShortName` ← `VendorName` |
| **列拼接** | 多个Excel列组合成一个字段 | `规格` ← `颜色` + `-` + `尺寸` |

### 3.2 外键关联配置示例

**场景**：导入产品时，需要通过"类目编码"查找类目ID

**配置步骤**：
1. 选择映射类型：`外键关联`
2. 配置关联表：`tb_ProdCategories`
3. 配置关联字段：`CategoryID`（目标字段，存储主键）
4. 配置来源列：`CategoryCode`（Excel中的编码列）
5. 配置查询字段：`CategoryCode`（在关联表中查询的字段）

**运行时逻辑**：
```
Excel读取 "CATEGORY001" 
  → 查询 SELECT ID FROM tb_ProdCategories WHERE CategoryCode = 'CATEGORY001'
  → 获取到 ID = 123456789
  → 写入产品的 CategoryID 字段
```

## 四、数据去重功能

### 4.1 去重策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **跳过（Skip）** | 数据库中已存在的记录不导入 | 增量导入，避免重复 |
| **更新（Update）** | 更新已存在的记录（暂未实现） | 数据同步 |
| **报错（Error）** | 发现重复则中止导入 | 严格的数据一致性要求 |

### 4.2 配置去重字段

**方式一**：在映射配置中勾选"业务键"字段
- 打开映射配置界面
- 勾选作为唯一标识的字段（如"供应商编号"、"产品编码"）
- 设置存在性策略（跳过/更新/报错）

**方式二**：在导入配置中设置去重字段列表
```xml
<EnableDeduplication>true</EnableDeduplication>
<DeduplicateFields>
  <Field>VendorCode</Field>
  <Field>VendorName</Field>
</DeduplicateFields>
```

## 五、图片导入功能

### 5.1 支持的图片存储方式

| 存储方式 | 说明 | 适用场景 |
|---------|------|---------|
| **文件路径** | 保存图片到磁盘，数据库存路径 | 推荐，节省数据库空间 |
| **Base64** | 图片转Base64字符串存入数据库 | 小型图片，便于迁移 |
| **二进制** | 图片二进制数据存入数据库 | 不推荐，影响性能 |

### 5.2 图片命名规则

| 规则 | 说明 | 示例文件名 |
|------|------|-----------|
| **列值命名** | 使用指定列的值命名 | `供应商A.jpg` |
| **GUID** | 使用全局唯一标识符 | `a1b2c3d4e5f6.jpg` |
| **时间戳** | 使用时间+行号 | `20260427103045_1.jpg` |
| **自动递增** | 使用行号 | `Image_1.jpg` |

### 5.3 配置图片字段

1. 在映射配置中添加图片字段
2. 选择数据源类型：`Excel图片`
3. 配置图片提取规则：
   - 命名规则（如"按产品编号命名"）
   - 存储类型（文件路径/Base64/二进制）
   - 输出目录（默认 `ImportImages/实体名/`）

## 六、常见问题

### Q1: 导入时提示"外键值未找到"
**原因**：Excel中的外键值（如供应商名称、类目编码）在关联表中不存在

**解决方法**：
1. 检查Excel中的编码/名称是否正确
2. 先导入基础数据（如先导入供应商，再导入产品）
3. 在映射配置中检查外键关联配置是否正确

---

### Q2: 导入后数据重复
**原因**：未配置去重策略或业务键

**解决方法**：
1. 在映射配置中勾选"业务键"字段（如供应商编号、产品编码）
2. 设置存在性策略为"跳过"
3. 或在导入配置中启用去重功能

---

### Q3: 宽表导入失败
**原因**：依赖的基础数据不存在

**解决方法**：
1. 检查宽表配置中的 `DependencyTables` 是否正确
2. 确保Excel中包含所有必需的数据列
3. 查看错误日志，确认是哪个表导入失败

---

### Q4: 图片未导入
**原因**：图片配置错误或Excel中无图片

**解决方法**：
1. 检查映射配置中是否正确配置了图片字段
2. 确认Excel中确实包含图片（非链接）
3. 检查图片输出目录是否有写入权限

## 七、性能优化建议

### 7.1 大批量导入

| 优化项 | 建议 |
|-------|------|
| **分批导入** | 每批1000-5000行，避免内存溢出 |
| **关闭去重** | 如果确定数据无重复，可关闭去重提升速度 |
| **预加载外键** | 导入前预加载外键缓存，减少数据库查询 |
| **使用宽表模式** | 复杂场景使用宽表模式，减少多次导入的开销 |

### 7.2 外键缓存

系统自动缓存外键数据，避免重复查询：
```
首次查询：SELECT ID, CategoryCode FROM tb_ProdCategories
后续查询：直接从内存缓存查找（毫秒级）
```

## 八、配置文件说明

### 8.1 单表配置（XML）

**位置**：`SysConfig/BasicDataImport/Profiles/`

**示例**：
```xml
<ImportConfiguration>
  <MappingName>供应商导入配置</MappingName>
  <EntityType>tb_CustomerVendor</EntityType>
  <EnableDeduplication>true</EnableDeduplication>
  <DeduplicateFields>
    <Field>VendorCode</Field>
  </DeduplicateFields>
  <ColumnMappings>
    <ColumnMapping>
      <ExcelColumn>供应商名称</ExcelColumn>
      <SystemField>
        <Key>VendorName</Key>
        <Value>供应商名称</Value>
      </SystemField>
      <DataSourceType>Excel</DataSourceType>
    </ColumnMapping>
  </ColumnMappings>
</ImportConfiguration>
```

### 8.2 宽表配置（JSON）

**位置**：`SysConfig/DataMigration/Profiles/`

**示例**：
```json
{
  "ProfileName": "产品宽表导入",
  "MasterTable": {
    "TargetTable": "tb_Prod",
    "SheetIndex": 0,
    "BusinessKeys": ["ProdCode"],
    "ColumnMappings": [...]
  },
  "DependencyTables": [
    {
      "TargetTable": "tb_ProdCategories",
      "BusinessKeys": ["CategoryCode"]
    }
  ],
  "ChildTables": [
    {
      "TargetTable": "tb_ProdDetail",
      "ForeignKey": "ProdID"
    }
  ]
}
```

## 九、技术支持

### 9.1 日志查看

导入过程中的详细日志输出到：
- **Debug输出**：Visual Studio 输出窗口
- **状态栏**：主界面底部状态栏显示关键信息

### 9.2 错误排查

1. **检查配置文件格式**：XML/JSON 语法是否正确
2. **检查数据库连接**：是否能正常访问数据库
3. **检查Excel格式**：列名是否与映射配置一致
4. **查看详细异常**：异常信息通常包含具体错误原因
