## 问题概述
BasicDataImport文件夹下的多个窗体无法在VS2022中打开标准设计器,主要原因是Krypton控件命名空间不一致和事件注册缺失。

**修复**: 统一改为新版命名空间 `Krypton.Toolkit`  
**影响文件**: 
- `FrmAttributeRulesConfig.Designer.cs` (第31-40行, 第176-185行)

```csharp
this.kryptonPanel1 = new ComponentFactory.Krypton.Toolkit.KryptonPanel();
```

## 根本原因分析

### 原因1: Krypton版本迁移遗留问题
项目从旧版 `ComponentFactory.Krypton.Toolkit` 迁移到新版 `Krypton.Toolkit`,但部分Designer文件未同步更新。

**检测规则**:
- 主文件(.cs)使用 `using Krypton.Toolkit;`
- Designer文件必须使用相同的命名空间
- 两者不一致会导致设计器加载失败

### 原因2: 手动添加事件未同步到Designer
当开发者在主文件中手动添加事件处理方法(如Load事件),但未通过Visual Studio设计器界面添加时,Designer文件不会自动生成事件注册代码。

**最佳实践**:
- 所有事件处理应通过设计器界面添加
- 或手动在Designer文件的InitializeComponent方法中添加事件注册

## 预防措施

### 1. 代码审查清单
今后修改WinForms窗体时,需检查:
- [ ] Designer文件中的控件命名空间与主文件一致
- [ ] 所有事件处理方法都在Designer中注册
- [ ] Dispose方法正确释放资源
- [ ] partial class声明完整

### 2. 自动化检测脚本
建议在CI/CD流程中添加检测:
```bash
# 检测Designer文件中是否存在ComponentFactory引用
grep -r "ComponentFactory.Krypton" *.Designer.cs
```

### 3. 开发规范
- **强制**: 所有Krypton控件统一使用 `Krypton.Toolkit` 命名空间
- **推荐**: 事件处理优先通过设计器界面添加
- **禁止**: 直接在主文件中手动添加事件而不更新Designer

## 技术参考

### Krypton Toolkit版本对照
| 版本 | 命名空间 | NuGet包 |
|------|---------|---------|
| 旧版 (< 5.x) | ComponentFactory.Krypton.Toolkit | ComponentFactory.Krypton.Toolkit |
| 新版 (>= 60.x) | Krypton.Toolkit | Krypton.Toolkit.Suite.Extended |

### WinForms设计器工作原理
1. Designer文件包含 `InitializeComponent()` 方法
2. 该方法在设计时由Visual Studio自动生成和维护
3. 手动修改Designer文件可能导致设计器无法正常加载
4. 推荐使用"重构"功能而非直接编辑Designer代码
