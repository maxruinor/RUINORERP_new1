## 功能概述

### 什么是宽表导入?

宽表导入允许您从**单个Excel文件**中导入**多个关联的数据库表**,系统会自动处理:
- ✅ 数据拆分(将宽表拆分为多个目标表)
- ✅ 外键映射(名称自动转ID)
- ✅ 依赖顺序(先插基础表,再插主表,最后插子表)
- ✅ 自动创建(数据库中不存在的基础数据自动创建)

### 核心优势

| 传统方式 | 宽表导入 |
|---------|---------|
| 需要分别准备多个Excel文件 | 只需一个Excel文件 |
| 手动查询外键ID | 自动名称转ID |
| 需要按顺序导入多个表 | 一键自动处理 |
| 容易出错,效率低 | 自动化,效率高 |

## 快速开始

### 步骤1: 准备Excel文件

创建一个包含所有数据的Excel文件,例如 `product_import.xlsx`:

| 产品编码 | 产品名称 | 供应商名称 | 供应商编码 | 类目名称 | SKU编码 | 规格 |
|---------|---------|-----------|-----------|---------|--------|------|
| P001 | 手机 | 华为 | HW001 | 电子产品 | SKU001 | 黑色/128G |
| P001 | 手机 | 华为 | HW001 | 电子产品 | SKU002 | 白色/256G |
| P002 | 笔记本 | 联想 | LX001 | 电子产品 | SKU003 | i7/16G |

### 步骤2: 打开基础数据导入界面

在系统中找到: **系统设置 → 系统工具 → 基础数据导入**

### 步骤3: 创建或选择Profile配置

#### 方式A: 使用现有Profile (推荐新手)

1. 点击"映射配置"下拉框
2. 选择以 `[宽表]` 开头的配置项
   - 例如: `[宽表] 04_产品宽表导入`

#### 方式B: 创建新的Profile (高级用户)

1. 点击 **"管理宽表Profile"** 按钮
2. 在弹出的编辑器中配置:
   - **基本信息**: Profile名称、描述
   - **主表配置**: 目标表、业务键、列映射
   - **依赖表配置**: 添加供应商表、类目表等
   - **子表配置**: 添加SKU明细表等
3. 点击"保存"按钮

### 步骤4: 选择Excel文件

点击"浏览"按钮,选择准备好的Excel文件

### 步骤5: 执行导入

点击"导入"按钮,系统会弹出对话框让您选择导入策略:

```
┌─────────────────────────────────────┐
│  宽表导入策略选择                    │
├─────────────────────────────────────┤
│  【是】一键导入                      │
│       自动处理所有表(推荐新手)        │
│                                     │
│  【否】分步导入                      │
│       先导入基础表,再导入主表         │
│       (推荐高级用户)                  │
│                                     │
│  【取消】不执行导入                   │
└─────────────────────────────────────┘
```

## Profile配置编辑器使用

### 打开编辑器

点击UI上的 **"管理宽表Profile"** 按钮

### 界面说明

编辑器包含4个标签页:

#### 1. 基本信息

- **Profile名称**: 唯一标识,用于在下拉框中显示
- **描述**: 可选,说明此Profile的用途

#### 2. 主表配置

- **目标表**: 选择要导入的主表(如 `tb_Prod`)
- **业务键**: 用于判断记录是否已存在的字段,多个用逗号分隔(如 `ProdCode`)
- **列映射表格**:
  - Excel列名: Excel中的表头
  - 数据库字段: 对应的数据库列名
  - 数据类型: String/Long/DateTime等
  - 转换规则: 外键引用格式为 `REF:表名:字段名`
  - 外键配置: 显示是否已配置外键

**操作按钮**:
- **添加列**: 打开列映射配置对话框
- **编辑列**: 编辑选中的列映射
- **删除列**: 删除选中的列映射

#### 3. 依赖表配置

依赖表是需要先于主表导入的基础数据表(如供应商表、类目表)

**操作按钮**:
- **添加依赖表**: 选择表并配置业务键

#### 4. 子表配置

子表是一对多关系中的"多"方(如产品明细表)

**配置项**:
- 表名: 子表名称
- 父表: 关联的父表名称
- 关联字段: 子表中引用父表ID的字段
- 业务键: 子表的业务唯一键

**操作按钮**:
- **添加子表**: 配置子表信息

### 保存Profile

1. 填写完所有配置后,点击 **"保存"** 按钮
2. 系统会将配置保存为JSON文件
3. 文件位置: `SysConfig/DataMigration/Profiles/{ProfileName}.json`

## 外键关联配置

### 什么是外键关联?

外键关联是指Excel中填写的是**业务名称**(如"华为"),但数据库中存储的是**物理ID**(如123)。

系统需要自动完成: **名称 → ID** 的转换。

### 配置外键关联

在列映射配置对话框中:

1. 勾选 **"外键关联"** 选项
2. 点击 **"配置外键"** 按钮
3. 在外键配置对话框中:
   - **关联表**: 选择外键引用的表(如 `tb_CustomerVendor`)
   - **主键字段**: 选择该表的主键(通常是 `ID`)
   - **源字段**: 选择用于匹配的字段(如 `VendorName`)
   - **□ 不存在时自动创建**: 勾选后,如果数据库中不存在该名称,会自动创建新记录

4. 点击"确定"保存

### 示例

假设产品表(tb_Prod)中有供应商ID字段(SupplierId):

```
Excel列: 供应商名称
数据库字段: SupplierId
转换规则: REF:tb_CustomerVendor:VendorName
外键配置:
  - 关联表: tb_CustomerVendor
  - 主键字段: ID
  - 源字段: VendorName
  - 自动创建: ✓
```

导入时的处理流程:
1. 读取Excel中的"供应商名称" = "华为"
2. 查询 `tb_CustomerVendor` 表中 `VendorName='华为'` 的记录
3. 如果找到,获取其ID(如123)
4. 如果没找到且勾选了"自动创建",则插入新记录并返回新ID
5. 将ID(123)写入 `tb_Prod.SupplierId`

## 导入策略选择

### 策略一: 一键导入 (推荐新手)

**特点**:
- ✅ 操作简单,一步到位
- ✅ 系统自动处理所有表
- ✅ 适合数据量不大(<5000行)

**操作流程**:
1. 选择Profile
2. 选择Excel文件
3. 点击"导入"
4. 在对话框中选择"是"(一键导入)
5. 等待完成

**适用场景**:
- 测试环境
- 数据量较小
- 希望快速完成导入

### 策略二: 分步导入 (推荐高级用户)

**特点**:
- ✅ 灵活可控,每步可验证
- ✅ 适合大数据量(>10000行)
- ✅ 便于排查问题

**操作流程**:

**步骤1: 导入基础表**
1. 点击"导入"
2. 在对话框中选择"否"(分步导入)
3. 在步骤选择中选择"是"(步骤1: 导入基础表)
4. 等待完成,查看结果

**步骤2: 验证基础数据**
```sql
-- 可以在数据库中查询确认
SELECT * FROM tb_CustomerVendor WHERE VendorName = '华为'
SELECT * FROM tb_ProdCategories WHERE CategoryName = '电子产品'
```

**步骤3: 导入主表**
1. 再次点击"导入"
2. 选择"否"(分步导入)
3. 在步骤选择中选择"否"(步骤2: 导入主表)
4. 等待完成

**步骤4: 导入子表** (如需要)
- 当前版本暂不支持单独导入子表
- 建议使用一键导入模式

**适用场景**:
- 生产环境
- 数据量大
- 需要逐步验证
- 调试和排查问题

## 常见问题

### Q1: 如何选择使用哪种导入策略?

**A**: 
- **数据量 < 5000行**: 使用一键导入
- **数据量 > 10000行**: 使用分步导入
- **首次使用**: 先用小数据量测试一键导入,熟悉后再用分步导入

### Q2: 导入失败怎么办?

**A**:
1. 查看错误提示信息
2. 检查Excel文件格式是否正确
3. 检查Profile配置是否正确
4. 使用分步导入,逐步排查是哪一步出错
5. 查看系统日志获取详细错误信息

### Q3: 如何修改已有的Profile配置?

**A**:
1. 点击"管理宽表Profile"按钮
2. 目前需要手动编辑JSON文件
3. 文件位置: `SysConfig/DataMigration/Profiles/{ProfileName}.json`
4. 修改后重新加载即可

> **注意**: 图形化编辑已有Profile的功能正在开发中

### Q4: 外键关联配置后,导入时提示"外键值不存在"?

**A**:
1. 检查是否勾选了"不存在时自动创建"
2. 如果没有勾选,需要先导入基础数据
3. 或者使用分步导入,先导入依赖表

### Q5: 可以同时导入多个宽表吗?

**A**:
- 可以,但建议逐个导入
- 每个宽表导入完成后,再导入下一个
- 避免并发导入导致的数据冲突

### Q6: Profile配置文件可以共享吗?

**A**:
- 可以,JSON文件可以直接复制到其他环境
- 确保目标环境的数据库表结构一致
- 确保表名和字段名相同

## 最佳实践

### 1. 先测试后生产

- 在测试环境验证Profile配置
- 使用小数据量测试导入流程
- 确认无误后再在生产环境使用

### 2. 备份重要数据

- 导入前备份相关表
- 特别是基础数据表(供应商、类目等)
- 以便出错时可以恢复

### 3. 合理使用业务键

- 业务键应该是唯一的业务标识
- 例如: 产品编码、供应商名称
- 避免使用可能重复的字段

### 4. 监控导入结果

- 查看导入报告中的成功/失败记录数
- 检查数据库中的数据是否正确
- 特别关注外键关联是否正确

### 5. 定期优化Profile

- 根据实际使用情况调整配置
- 删除不再使用的Profile
- 添加常用的新Profile
