## 🏗️ 架构设计亮点

### 1. 统一架构 - 宽表是单表的超集

```csharp
// 宽表配置
{
  "MasterTable": {...},
  "DependencyTables": [...],  // 可以有0个或多个
  "ChildTables": [...]        // 可以有0个或多个
}

// 单表配置 (宽表的特例)
{
  "MasterTable": {...},
  "DependencyTables": [],     // 空数组
  "ChildTables": []           // 空数组
}
```

**优势**:
- ✅ 一套代码支持两种场景
- ✅ 向后兼容原有单表导入
- ✅ 平滑升级路径

### 2. 混合来源解析 - 智能外键映射

```
Excel填写"华为" → 系统处理流程:
1. 查缓存 → 命中则返回ID
2. 查数据库 → 找到则缓存并返回ID
3. 自动创建 → 插入新记录并返回新ID
```

**优势**:
- ✅ 减少数据库查询次数
- ✅ 支持自动创建缺失的基础数据
- ✅ 提高导入效率

### 3. 依赖驱动执行 - 保证数据完整性

```
执行顺序:
1. DependencyTables (字典表) → 先建立基础数据
2. MasterTable (主表) → 引用字典表ID
3. ChildTables (子表) → 引用主表ID
```

**优势**:
- ✅ 保证外键引用完整性
- ✅ 避免插入顺序错误
- ✅ 支持复杂的多表关联

### 4. 灵活的导入策略

```
用户可选择:
- 一键导入: 自动化处理,适合新手
- 分步导入: 逐步验证,适合高级用户
```

**优势**:
- ✅ 满足不同用户需求
- ✅ 便于问题排查
- ✅ 提升用户体验

## 🎯 核心功能特性

### 功能1: 单文件多表导入

**传统方式**:
```
需要3个Excel文件:
- suppliers.xlsx (供应商)
- categories.xlsx (类目)
- products.xlsx (产品+SKU)

手动按顺序导入3次
```

**宽表导入**:
```
只需1个Excel文件:
- product_wide.xlsx (包含所有数据)

一键自动处理,系统自动拆分
```

### 功能2: 智能外键映射

**传统方式**:
```
Excel中需要填写供应商ID: 123
用户需要先查询供应商表获取ID
容易出错,效率低
```

**宽表导入**:
```
Excel中直接填写供应商名称: "华为"
系统自动查询/创建,返回ID: 123
准确高效,用户体验好
```

### 功能3: 自动创建基础数据

**场景**: Excel中有新供应商"小米",数据库中不存在

**处理方式**:
```
如果勾选了"自动创建":
1. 自动在tb_CustomerVendor中插入"小米"
2. 获取新生成的ID
3. 在产品表中引用该ID

如果不勾选:
抛出异常,提示用户先导入基础数据
```

### 功能4: Profile可视化配置

**之前**: 手动编辑JSON文件,容易出错

**现在**: 
- ✅ 图形化界面配置
- ✅ 下拉框选择表和字段
- ✅ 实时验证配置
- ✅ 一键保存为JSON

## 📊 性能与优化

### 预期性能指标

| 数据量 | 预计耗时 | 说明 |
|--------|---------|------|
| 100行 | ~1-2秒 | 快速导入 |
| 1000行 | ~5-10秒 | 可接受 |
| 5000行 | ~20-30秒 | 建议使用分步导入 |
| >10000行 | ~1分钟+ | 必须使用分步导入 |

### 已实施的优化

1. **批量预加载**: 一次性查询所有依赖表到内存
2. **缓存机制**: 避免重复查询相同的外键值
3. **事务控制**: 每个表独立事务,失败时回滚

### 后续优化方向

1. **Bulk Insert**: 使用SqlSugar的批量插入API
2. **并行处理**: 多个依赖表并行导入
3. **异步流式处理**: 超大文件分块处理

## ⚠️ 注意事项

### 1. 数据库表结构要求

- ✅ 表必须存在
- ✅ 字段名必须匹配
- ✅ 主键必须是自增ID或Guid
- ❌ 不支持复合主键

### 2. Excel文件格式要求

- ✅ 第一行必须是表头
- ✅ 表头名称需与Profile配置一致
- ✅ 支持.xlsx和.xls格式
- ❌ 不支持加密的Excel文件

### 3. 外键关联限制

- ✅ 支持一对一关系
- ✅ 支持一对多关系
- ❌ 暂不支持多对多关系
- ❌ 暂不支持循环引用

### 4. 并发导入

- ⚠️ 不建议同时导入多个宽表
- ⚠️ 可能导致数据冲突
- ✅ 建议逐个导入,等待完成

## 🐛 已知问题与限制

### 当前版本的限制

1. **子表单独导入**: 暂不支持单独导入子表
   - ** workaround**: 使用一键导入模式

2. **Profile编辑**: 只能创建新Profile,不能编辑已有Profile
   - ** workaround**: 手动编辑JSON文件

3. **AI智能映射**: 已预留接口,但未集成
   - **计划**: 下一版本集成IIntelligentMappingService

4. **进度显示**: 缺少实时进度条
   - **计划**: 下一版本添加进度反馈

### 待优化的功能

1. **大数据量处理**: >50000行时性能下降
2. **错误恢复**: 部分失败后无法续传
3. **导入历史**: 没有导入记录查询
4. **模板下载**: 没有标准模板下载功能
