# 库存快照功能 - 快速参考卡

## 🎯 一分钟上手

### 打开管理界面
```
服务器管理端 → 左侧导航 → "库存快照管理"
```

### 三个标签页
1. **配置管理** - 查看和修改执行时间
2. **手动触发** - 立即执行测试
3. **帮助说明** - 查看详细文档

---

## ⚙️ 常用操作

### 修改执行时间（推荐夜间）

**快捷方式**：
- 点击"凌晨1点" / "凌晨2点" / "凌晨3点" / "凌晨4点"

**自定义**：
- 点击"自定义时间"按钮
- 输入格式：HH:mm:ss（如 02:30:00）

### 启用/禁用任务
- 点击"启用/禁用"按钮
- 或直接编辑 `ScheduledTasks.json`

### 立即测试
1. 切换到"手动触发"标签
2. 点击"立即执行库存快照"
3. 查看日志确认结果

---

## 🔧 配置文件位置

```
RUINORERP.Server\bin\Debug\net8.0\ScheduledTasks.json
```

首次运行后自动生成，修改后需重启服务。

## 📊 默认配置

| 项目 | 值 | 说明 |
|------|-----|------|
| 执行时间 | 01:00:00 | 凌晨1点 |
| 调试模式 | true | 每5分钟执行（已优化为true用于测试） |
| 保留月数 | 12个月 | 自动清理过期数据 |
| 任务状态 | 启用 | 默认开启 |

## 🐛 常见问题

### Q1: 工作流不执行？
- 检查任务是否启用
- 确认服务已重启
- 查看日志是否有错误

### Q2: 如何验证执行成功？
- 查看日志输出
- 运行SQL测试脚本
- 检查数据库快照表

### Q3: 执行时间不准确？
- 查看"下次执行时间"显示
- 确认服务器系统时间正确
- 重启服务重新调度

## 📝 SQL快速查询

```sql
-- 查看最新快照
SELECT TOP 10 * FROM tb_InventorySnapshot 
ORDER BY SnapshotTime DESC;

-- 统计今天快照数量
SELECT COUNT(*) FROM tb_InventorySnapshot 
WHERE CAST(SnapshotTime AS DATE) = CAST(GETDATE() AS DATE);

-- 查看快照表大小
EXEC sp_spaceused 'tb_InventorySnapshot';
```

完整测试脚本：`SQLScripts/库存快照测试.sql`
