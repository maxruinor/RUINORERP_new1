# 性能监控使用指南

## 1. TransactionScope 与 SqlSugar 事务对比

### 1.1 .NET Framework 4.8 + TransactionScope

**官方文档说明**:
- ✅ .NET Framework 4.5.1+ 已完全支持 `TransactionScopeAsyncFlowOption.Enabled`
- ⚠️ **SqlSugar 官方不推荐混用** TransactionScope 和 SqlSugar 事务

```csharp
// ❌ 不推荐：双重事务管理
using (var scope = new TransactionScope(TransactionScopeAsyncFlowOption.Enabled))
{
    var db = new SqlSugarClient(config);
    db.Ado.BeginTran();  // SqlSugar 事务
    // ... 操作
    db.Ado.CommitTran();
    scope.Complete();    // TransactionScope 事务
}
// 问题：可能导致事务嵌套过深或冲突
```

### 1.2 SqlSugar 推荐方案

```csharp
// ✅ 方案 1: UseTran 封装（推荐）
var result = db.Ado.UseTran(() =>
{
    db.Ado.ExecuteCommand("DELETE FROM Student");
    db.Insertable(entity).ExecuteCommand();
});
if (!result.IsSuccess)
{
    // 处理失败
}

// ✅ 方案 2: 手动事务控制
try
{
    db.Ado.BeginTran();
    
    db.Ado.ExecuteCommand("UPDATE TableA SET ...");
    db.Insertable(entity).ExecuteCommand();
    
    db.Ado.CommitTran();
}
catch (Exception ex)
{
    db.Ado.RollbackTran();
    throw;
}

// ✅ 方案 3: 异步事务
var asyncResult = await db.Ado.UseTranAsync(async () =>
{
    await db.Ado.ExecuteCommandAsync("DELETE FROM Student");
    await db.Insertable(entity).ExecuteCommandAsync();
});
```

## 2. _appContext.Db vs _unitOfWorkManage.GetDbClient()

### 2.1 核心区别

| 对比项 | `_appContext.Db` | `_unitOfWorkManage.GetDbClient()` |
|--------|-----------------|----------------------------------|
| **类型** | `SqlSugarScope` (全局单例) | `ISqlSugarClient` (每上下文独立) |
| **来源** | 依赖注入全局实例 | AsyncLocal 独立实例 |
| **线程安全** | ✅ 支持 | ✅ 支持 |
| **事务支持** | ⚠️ 需手动 BeginTran | ✅ 自动关联当前事务 |
| **连接管理** | 自动关闭 | UnitOfWork 管理 |
| **使用场景** | 无状态查询 | 事务性操作 |

### 2.2 正确使用示例

```csharp
public class tb_SaleOutController<T>
{
    private readonly IUnitOfWorkManage _unitOfWorkManage;
    private readonly ApplicationContext _appContext;
    
    // ✅ 正确：事务性操作使用 _unitOfWorkManage
    public async Task<int> UpdateStatusAsync(int id, int status)
    {
        try
        {
            _unitOfWorkManage.BeginTran();
            
            // 使用 _unitOfWorkManage.GetDbClient() 保证在事务中
            await _unitOfWorkManage.GetDbClient()
                .Updateable<tb_SaleOut>()
                .SetColumns(e => new { e.Status })
                .Where(e => e.Id == id)
                .ExecuteCommandAsync();
            
            _unitOfWorkManage.CommitTran();
            return 1;
        }
        catch
        {
            _unitOfWorkManage.RollbackTran();
            throw;
        }
    }
    
    // ✅ 正确：无状态查询可以使用 _appContext.Db
    public async Task<tb_SaleOut> GetByIdAsync(int id)
    {
        // 只读查询，不需要事务
        return await _appContext.Db
            .Queryable<tb_SaleOut>()
            .FirstAsync(e => e.Id == id);
    }
}
```

### 2.3 禁止混用

```csharp
// ❌ 错误：混用导致事务失效
public async Task WrongUsageAsync()
{
    _unitOfWorkManage.BeginTran();
    
    // 正确：在事务中
    await _unitOfWorkManage.GetDbClient()
        .Updateable<tb_SaleOut>(entity1)
        .ExecuteCommandAsync();
    
    // ❌ 错误：_appContext.Db 不在事务中！
    await _appContext.Db
        .Updateable<tb_SaleOut>(entity2)
        .ExecuteCommandAsync();
    
    _unitOfWorkManage.CommitTran();
    // 结果：entity1 的更新会提交，entity2 的更新不会回滚
}
```

## 4. 性能优化建议

### 4.1 识别慢事务

```csharp
var stats = TransactionMetrics.GetStatistics();

// 找出 P95 > 1 秒的事务
var slowTransactions = stats.TopLongTransactions
    .Where(t => t.DurationSeconds > 1.0)
    .ToList();

foreach (var tx in slowTransactions)
{
    Console.WriteLine($"慢事务：{tx.Caller}, 耗时：{tx.DurationSeconds:F2}秒");
}
```

### 4.2 识别热点表

```csharp
var hotTables = stats.HotTables
    .Where(t => t.TransactionCount > 1000)
    .OrderByDescending(t => t.AverageDuration)
    .ToList();

foreach (var table in hotTables)
{
    Console.WriteLine($"热点表：{table.TableName}, " +
                     $"次数：{table.TransactionCount}, " +
                     $"平均：{table.AverageDuration:F3}秒");
}
```

### 4.3 分析死锁模式

```csharp
var deadlocks = stats.RecentDeadlocks
    .GroupBy(d => d.TableName)
    .Select(g => new 
    {
        TableName = g.Key,
        Count = g.Count(),
        AvgDuration = g.Average(d => d.Duration.TotalMilliseconds)
    })
    .OrderByDescending(x => x.Count)
    .ToList();

foreach (var dl in deadlocks)
{
    Console.WriteLine($"表：{dl.TableName}, " +
                     $"死锁次数：{dl.Count}, " +
                     $"平均耗时：{dl.AvgDuration:F0}ms");
}
```

## 5. 最佳实践

### 5.1 事务使用规范

```csharp
// ✅ 短事务原则
public async Task<int> QuickUpdateAsync(int id, int value)
{
    try
    {
        _unitOfWorkManage.BeginTran();
        
        var affected = await _unitOfWorkManage.GetDbClient()
            .Updateable<Entity>()
            .SetColumns(e => new { e.Value })
            .Where(e => e.Id == id)
            .ExecuteCommandAsync();
        
        _unitOfWorkManage.CommitTran();
        return affected;
    }
    catch
    {
        _unitOfWorkManage.RollbackTran();
        throw;
    }
}

// ❌ 避免长事务
public async Task LongTransactionAsync()
{
    _unitOfWorkManage.BeginTran();
    
    // ❌ 包含 HTTP 请求
    await httpClient.GetAsync("...");
    
    // ❌ 包含文件 IO
    await File.ReadAllTextAsync("...");
    
    // ❌ 包含复杂计算
    for (int i = 0; i < 10000; i++) { ... }
    
    _unitOfWorkManage.CommitTran();
}
```

### 5.2 内层异常传播

```csharp
// ✅ 内层失败立即抛出
public async Task<Result> ParentMethodAsync()
{
    try
    {
        _unitOfWorkManage.BeginTran();
        
        var childResult = await ChildMethodAsync();
        if (!childResult.Succeeded)
        {
            throw new BusinessException($"内层方法失败：{childResult.ErrorMsg}");
        }
        
        _unitOfWorkManage.CommitTran();
        return Result.Success();
    }
    catch
    {
        _unitOfWorkManage.RollbackTran();
        throw;
    }
}
```

### 5.3 死锁重试

```csharp
// ✅ 使用内置重试机制
public async Task ProcessWithRetryAsync()
{
    await _unitOfWorkManage.ExecuteWithRetryAsync(async () =>
    {
        _unitOfWorkManage.BeginTran();
        try
        {
            // 业务逻辑
            await _unitOfWorkManage.GetDbClient()
                .Insertable(entity)
                .ExecuteCommandAsync();
            
            _unitOfWorkManage.CommitTran();
        }
        catch
        {
            _unitOfWorkManage.RollbackTran();
            throw;
        }
    }, maxRetryCount: 3);
}
```

// 在 UnitOfWorkManage 中已配置
if (duration > 10)
{

## 7. 故障排查流程

### 7.1 性能下降排查

1. **查看统计报告**
   ```csharp
   var report = TransactionMetrics.ExportReport();
   Console.WriteLine(report);
   ```

2. **识别 Top 长事务**
   ```csharp
   var stats = TransactionMetrics.GetStatistics();
   var topSlow = stats.TopLongTransactions.Take(10);
   ```

3. **定位代码位置**
   ```csharp
   foreach (var tx in topSlow)
   {
       Console.WriteLine($"{tx.Caller} - {tx.DurationSeconds:F2}秒");
   }
   ```

4. **优化代码**
   - 拆分长事务
   - 添加索引
   - 减少锁竞争

### 7.2 死锁频繁排查

1. **查看死锁统计**
   ```csharp
   var deadlocks = stats.RecentDeadlocks
       .GroupBy(d => d.TableName)
       .OrderByDescending(g => g.Count())
       .ToList();
   ```

2. **分析死锁 SQL**
   ```csharp
   foreach (var dl in deadlocks)
   {
       Console.WriteLine($"表：{dl.Key}, 次数：{dl.Count()}");
       foreach (var item in dl)
       {
           Console.WriteLine($"  SQL: {item.Sql}");
       }
   }
   ```

3. **优化策略**
   - 调整事务顺序
   - 使用乐观锁
   - 添加必要的索引

## 8. 总结

### 8.1 关键要点

1. **不要混用** `_appContext.Db` 和 `_unitOfWorkManage.GetDbClient()`
2. **事务内操作**必须使用 `_unitOfWorkManage.GetDbClient()`
3. **无状态查询**可以使用 `_appContext.Db`
4. **禁用 TransactionScope** + SqlSugar 组合
5. **启用性能监控**定期查看统计报告
