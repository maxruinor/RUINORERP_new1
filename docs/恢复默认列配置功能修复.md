# 恢复默认列配置功能修复

## ❌ 问题描述

在 `frmColumnsSets.cs` 的"恢复默认列配置"功能中存在两个严重问题：

1. **恢复默认后没有保存配置到数据库**
2. **恢复默认后没有应用到 NewSumDataGridView**

用户点击"恢复默认值"按钮后，虽然UI上显示了默认配置，但：
- 配置没有保存到 `tb_UIGridSetting` 表
- 下次打开窗口时仍然显示旧的配置
- DataGridView 的列没有立即更新

---

## ✅ 修复方案

### 修复后的完整代码

```csharp
/// <summary>
/// 恢复默认列配置
/// </summary>
private async void btnRestoreDefaultConfig_Click(object sender, EventArgs e)
{
    try
    {
        // 1. 检查是否有初始化的列配置
        if (InitColumnDisplays == null || InitColumnDisplays.Count == 0)
        {
            if (InitializeDefaultColumn != null && InitColumnDisplays.Count == 0)
            {
                InitializeDefaultColumn(InitColumnDisplays);
            }
        }

        // 2. 清空当前ListView
        listView1.Items.Clear();

        // 3. 重新加载ListView内容 - 使用 InitColumnDisplays 作为默认配置
        foreach (ColDisplayController keyValue in InitColumnDisplays)
        {
            if (!keyValue.Disable)
            {
                ListViewItem lvi = new ListViewItem();
                lvi.Checked = keyValue.Visible;
                lvi.Name = keyValue.ColName;
                lvi.Tag = keyValue;
                lvi.Text = keyValue.ColDisplayText;
                lvi.ImageKey = keyValue.ColDisplayIndex.ToString();
                listView1.Items.Add(lvi);
            }
        }

        // ✅ 修复1: 同步更新 ColumnDisplays 集合
        ColumnDisplays.Clear();
        ColumnDisplays.AddRange(InitColumnDisplays);

        // ✅ 修复2: 如果有 DataGridViewSetTarget，立即应用配置到表格
        if (DataGridViewSetTarget != null)
        {
            // 更新 DataGridView 的 ColumnDisplays
            DataGridViewSetTarget.ColumnDisplays.Clear();
            DataGridViewSetTarget.ColumnDisplays.AddRange(InitColumnDisplays);
            
            // 重新绑定列样式
            DataGridViewSetTarget.BindColumnStyle();
            
            // 保存默认配置到数据库
            SaveDefaultConfigToDatabase();
        }
        else
        {
            // 即使没有 DataGridViewSetTarget，也尝试保存配置
            SaveDefaultConfigToDatabase();
        }
    }
    catch (Exception ex)
    {
        MessageBox.Show($"恢复默认列配置失败：{ex.Message}", "错误", 
            MessageBoxButtons.OK, MessageBoxIcon.Error);
    }
}

/// <summary>
/// 保存默认配置到数据库
/// </summary>
private async void SaveDefaultConfigToDatabase()
{
    try
    {
        // 尝试从 DataGridViewSetTarget 获取 MenuId 和 GridKeyName
        long? menuId = null;
        string gridKeyName = null;

        if (DataGridViewSetTarget != null)
        {
            menuId = DataGridViewSetTarget.MenuId;
            gridKeyName = DataGridViewSetTarget.XmlFileName; // XmlFileName 属性会自动设置 GridKeyName
        }

        // 验证必要参数
        if (!menuId.HasValue || menuId.Value == 0 || string.IsNullOrEmpty(gridKeyName))
        {
            System.Diagnostics.Debug.WriteLine("[恢复默认] 警告: 缺少 MenuId 或 GridKeyName，无法保存到数据库");
            return;
        }

        // 使用 ColumnConfigManager 保存到数据库
        await RUINORERP.UI.Common.ColumnConfigManager.Instance.ResetToDefaultAsync(
            gridKeyName,
            menuId.Value,
            InitColumnDisplays
        );

        System.Diagnostics.Debug.WriteLine($"[恢复默认] 成功保存默认配置到数据库: {gridKeyName}_{menuId}");
    }
    catch (Exception ex)
    {
        System.Diagnostics.Debug.WriteLine($"[恢复默认] 保存失败: {ex.Message}");
    }
}
```

## 🎯 关键技术点

### 1. 异步方法的使用

```csharp
private async void btnRestoreDefaultConfig_Click(object sender, EventArgs e)
```

**为什么用 async void？**
- 这是事件处理方法，签名固定为 `void EventHandler(object, EventArgs)`
- 需要调用异步的 `ResetToDefaultAsync()` 方法
- 事件处理是唯一可以使用 `async void` 的场景

### 2. ColumnConfigManager.ResetToDefaultAsync

```csharp
await ColumnConfigManager.Instance.ResetToDefaultAsync(
    gridKeyName,
    menuId.Value,
    InitColumnDisplays
);
```

**这个方法做了什么？**
1. 更新内存缓存 `_configCache[cacheKey] = defaultConfig`
2. 立即保存到数据库（不延时）
3. 清除待提交队列中的旧配置

### 3. 参数获取

```csharp
menuId = DataGridViewSetTarget.MenuId;
gridKeyName = DataGridViewSetTarget.XmlFileName;
```

**为什么这样获取？**
- `MenuId`：NewSumDataGridView 的属性，用于关联菜单
- `XmlFileName`：设置时会自动同步到 `GridKeyName`
- 这两个参数是保存到数据库的必要条件

## 📝 注意事项

### 1. 必须设置 MenuId 和 GridKeyName

```csharp
// 在窗体初始化时设置
newSumDataGridView1.MenuId = currentMenuInfo.MenuID;
newSumDataGridView1.XmlFileName = "tb_SaleOrder";
```

如果未设置，会输出警告日志：
```
[恢复默认] 警告: 缺少 MenuId 或 GridKeyName，无法保存到数据库
```

### 2. InitColumnDisplays 必须正确初始化

`InitColumnDisplays` 应该在窗体初始化时设置，包含默认的列配置。通常由以下代码设置：

```csharp
// 在 CustomizeGrid.SetColumns() 中
set.InitColumnDisplays = InitColumnDisplays;
```

### 3. 异步操作不会阻塞UI

由于使用了 `async void`，保存操作在后台执行，不会阻塞UI线程。用户可以立即看到UI变化，而数据库保存异步进行。
