## 快速开始

### 场景 1: 使用统计服务层 (推荐用于简单统计)

```csharp
// 1. 创建服务实例
var db = MainForm.Instance.AppContext.Db;
var cacheService = new MemoryCacheService();
var statsService = new AggregationService(db, cacheService);

// 2. 构建查询参数
var query = new StatisticsQuery
{
    EntityType = "tb_SaleOrder",
    TimeField = "Created_at",
    StartTime = DateTime.Now.AddMonths(-1),
    EndTime = DateTime.Now,
    Dimensions = new List<string> { "Employee_ID" },
    Metrics = new List<string> { "TotalAmount", "OrderID" }
};

// 3. 执行查询
var chartData = await statsService.ExecuteQueryAsync(query);

// 4. 渲染图表
DisplayChart(chartData);
```

### 场景 2: 使用数据适配器 (推荐用于复杂业务逻辑)

```csharp
// 1. 创建适配器
var db = MainForm.Instance.AppContext.Db;
var adapter = new OrderAnalysisAdapter(db);

// 2. 构建数据请求
var request = new DataRequest
{
    TimeField = "Created_at",
    StartTime = DateTime.Now.AddMonths(-1),
    EndTime = DateTime.Now,
    ChartType = ChartType.Column,
    Dimensions = new List<DimensionConfig>
    {
        new DimensionConfig("Created_at", "订单日期", DimensionType.DateTime)
    },
    Metrics = new List<MetricConfig>
    {
        new MetricConfig("OrderID", "订单数", MetricType.Count, MetricUnit.笔),
        new MetricConfig("TotalAmount", "金额", MetricType.Sum, MetricUnit.元)
    }
};

// 3. 获取数据
var chartData = await adapter.GetDataAsync(request);

// 4. 使用 ChartBuilderFactory 渲染
var builder = ChartBuilderFactory.CreateBuilder(request, adapter);
var chartControl = await builder.BuildChart(request);
```

## 核心组件说明

### 1. IStatisticsService (统计服务接口)

**位置**: `RUINORERP.Business\Services\Statistics\IStatisticsService.cs`

**用途**: 提供通用的统计查询能力，支持动态维度 + 指标组合。

**适用场景**:
- 简单的 SUM/COUNT/AVG聚合
- 需要缓存加速的查询
- 跨多个实体的通用统计

**示例**:
```csharp
public class SalesStatsService : IStatisticsService
{
    private readonly AggregationService _innerService;
    
    public async Task<ChartData> ExecuteQueryAsync(StatisticsQuery query)
    {
        // 可以添加自定义逻辑
        if (query.EntityType == "SaleOrder")
        {
            // 特殊处理销售订单
        }
        
        return await _innerService.ExecuteQueryAsync(query);
    }
}
```

### 2. BaseChartAdapter (数据适配器基类)

**位置**: `RUINORERP.UI\ChartFramework\DataProviders\Adapters\BaseChartAdapter.cs`

**用途**: 为特定业务场景提供数据查询和转换逻辑。

**与 IStatisticsService 的区别**:
- Adapter 更贴近业务，可编写复杂 SQL 关联
- Adapter 支持自定义数据转换规则
- Service 更通用，适合标准化统计

### 3. QueryPanelGenerator (查询面板生成器)

**位置**: `RUINORERP.UI\ChartFramework\QueryPanel\QueryPanelGenerator.cs`

**用途**: 根据参数对象自动生成查询界面。

**使用示例**:
```csharp
public class SalesQueryParams
{
    [QueryParameter(DisplayName = "业务员", Order = 1)]
    public long? Employee_ID { get; set; }
    
    [QueryParameter(DisplayName = "开始日期", Order = 2)]
    public DateTime StartDate { get; set; }
    
    [QueryParameter(DisplayName = "结束日期", Order = 3)]
    public DateTime EndDate { get; set; }
}

// 自动生成查询面板
var params = new SalesQueryParams();
var panel = QueryPanelGenerator.Generate(params);
```

## 最佳实践

### 1. 选择合适的架构层级

| 需求场景 | 推荐方案 | 理由 |
|---------|---------|------|
| 简单聚合统计 | IStatisticsService | 通用、自带缓存 |
| 多表关联分析 | BaseChartAdapter | 灵活编写 SQL |
| 超大数据集 | ProgressiveLoader | 分层加载 |
| 需要钻取 | DrillDownService | 内置交互 |

### 2. 性能优化建议

- **缓存策略**: 热点数据设置 30 分钟 TTL
- **分页限制**: 单次查询最多返回 1000 条记录
- **索引优化**: 确保时间字段和分组字段有索引
- **异步查询**: 始终使用 `async/await`避免 UI 阻塞

### 3. 错误处理

```csharp
try
{
    var chartData = await service.ExecuteQueryAsync(query);
}
catch (ArgumentException ex)
{
    // 参数验证失败
    MessageBox.Show($"查询参数错误：{ex.Message}");
}
catch (SqlSugarException ex)
{
    // 数据库错误
    MessageBox.Show($"数据库查询失败：{ex.Message}");
}
catch (Exception ex)
{
    // 其他错误
    MessageBox.Show($"未知错误：{ex.Message}");
}
```

## 常见问题

**Q: 什么时候使用 StatisticsService vs Adapter?**  
A: 如果只需要简单的 GROUP BY + 聚合，用 Service；如果需要 JOIN 多个表或复杂计算，用 Adapter。

**Q: 如何支持实时刷新？**  
A: 可以使用 SignalR 推送数据变更，然后调用 `chartControl.UpdateData(newData)`。

**Q: 大数据量导致内存溢出怎么办？**  
A: 使用 `ProgressiveLoader` 进行分批加载，并启用虚拟滚动。
