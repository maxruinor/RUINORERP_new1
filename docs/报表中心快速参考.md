## 常见错误及解决方案

### 错误 1: "必须指定实体类型"

**原因**: `StatisticsQuery.EntityType`为空

**解决**:
```csharp
var query = new StatisticsQuery
{
    EntityType = "tb_SaleOrder" // ✅ 必须设置
};
```

### 错误 2: "必须至少选择一个聚合指标"

**原因**: `Metrics` 集合为空

**解决**:
```csharp
query.Metrics = new[] { "TotalAmount" }; // ✅ 至少一个
```

### 错误 3: SQL 注入风险

**错误代码**:
```csharp
// ❌ 危险!
query.Where($"{fieldName} = '{value}'");
```

**正确做法**:
```csharp
// ✅ 使用参数化查询
query.Where($"{fieldName} = @{fieldName}", 
    new Dictionary<string, object> { [fieldName] = value });
```

### 错误 4: 内存泄漏

**原因**: 事件订阅未释放

**解决**:
```csharp
// 使用弱引用管理事件
private readonly WeakReference _eventRef;

public void Subscribe()
{
    if (_eventRef.Target is EventHandler handler)
    {
        handler -= OnEvent;
    }
}
```

## 最佳实践速查

### ✅ DO (推荐)

1. **继承 BaseChartAdapter** 而非直接实现 IDataProvider
2. **使用缓存** 加速重复查询
3. **异步编程** 避免 UI 阻塞
4. **参数验证** 在 Service 层进行
5. **异常处理** 包装为友好提示

### ❌ DON'T (避免)

1. ~~在 UI 层直接写 SQL~~
2. ~~硬编码表名和字段名~~
3. ~~一次性加载全部数据~~
4. ~~忽略缓存 TTL 设置~~
5. ~~捕获 Exception 却不记录日志~~
