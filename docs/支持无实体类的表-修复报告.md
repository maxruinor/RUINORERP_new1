# 🔧 支持无实体类的表 - 修复报告

## 📌 问题描述

**错误信息**：
```
删除失败：加载实体 tb_BOMChangeLog 失败: 无法找到实体类型: tb_BOMChangeLog
```

**根本原因**：
- 数据库中有 `tb_BOMChangeLog` 表
- 但 **Model项目中没有对应的实体类**
- 原代码强制要求所有表都必须有实体类

## 🔍 问题分析

### 之前的实现

```csharp
// ❌ 强制要求实体类存在
public static EntityMetadata GetMetadata(string entityName)
{
    var assembly = Assembly.Load("RUINORERP.Model");
    var entityType = assembly.GetTypes().FirstOrDefault(t => 
        t.Name == entityName && typeof(BaseEntity).IsAssignableFrom(t));

    if (entityType == null)
        throw new Exception($"无法找到实体类型: {entityName}");  // ← 直接抛异常！
    
    return GetMetadata(entityType);
}
```

### 调用位置

```csharp
// 位置1：预查询子表主键ID时
var childMetadata = ModelMetadataHelper.GetMetadata(childTable);  // ❌ 需要实体

// 位置2：删除阶段获取主键列名时
var tableMetadata = ModelMetadataHelper.GetMetadata(tableInfo.TableName);  // ❌ 需要实体
```

## ✅ 解决方案

### 新实现：两级查询策略

```csharp
/// <summary>
/// 根据表名获取元数据（优先从实体，其次从数据库查询）
/// </summary>
public static async Task<EntityMetadata> GetMetadataAsync(ISqlSugarClient db, string tableName)
{
    // 1️⃣ 先尝试从实体程序集加载
    var cacheKey = $"RUINORERP.Model.{tableName}";
    if (_cache.TryGetValue(cacheKey, out var cached)) return cached;

    try
    {
        var assembly = Assembly.Load("RUINORERP.Model");
        var entityType = assembly.GetTypes().FirstOrDefault(t => 
            t.Name == tableName && typeof(BaseEntity).IsAssignableFrom(t));

        if (entityType != null)
        {
            return GetMetadata(entityType);  // 有实体类，使用实体元数据
        }
    }
    catch
    {
        // 实体不存在，忽略，继续下一步
    }

    // 2️⃣ 实体不存在，从数据库查询表结构
    return await GetMetadataFromDatabaseAsync(db, tableName);
}

/// <summary>
/// 从数据库查询表的元数据（主键列名）
/// </summary>
private static async Task<EntityMetadata> GetMetadataFromDatabaseAsync(ISqlSugarClient db, string tableName)
{
    var cacheKey = $"DB.{tableName}";
    if (_cache.TryGetValue(cacheKey, out var cached)) return cached;

    try
    {
        // 查询主键列名
        var sql = @"
            SELECT c.COLUMN_NAME
            FROM INFORMATION_SCHEMA.TABLE_CONSTRAINTS tc
            INNER JOIN INFORMATION_SCHEMA.KEY_COLUMN_USAGE c 
                ON tc.CONSTRAINT_NAME = c.CONSTRAINT_NAME
            WHERE tc.TABLE_NAME = @tableName 
              AND tc.CONSTRAINT_TYPE = 'PRIMARY KEY'";

        var primaryKeyColumn = await db.Ado.GetStringAsync(sql, new { tableName = tableName });

        var metadata = new EntityMetadata
        {
            TableName = tableName,
            PrimaryKeyName = primaryKeyColumn ?? "PrimaryKeyID"
        };

        _cache[cacheKey] = metadata;
        return metadata;
    }
    catch
    {
        // 查询失败，默认使用 PrimaryKeyID
        var metadata = new EntityMetadata
        {
            TableName = tableName,
            PrimaryKeyName = "PrimaryKeyID"
        };

        _cache[cacheKey] = metadata;
        return metadata;
    }
}
```

## 🎯 核心改进

### 1. 不再强制依赖实体类

| 情况 | 之前的行为 | 现在的行为 |
|-----|---------|---------|
| 有实体类 | ✅ 从实体读取元数据 | ✅ 从实体读取元数据 |
| 无实体类 | ❌ 抛异常，删除失败 | ✅ 从数据库查询主键 |

### 2. 支持所有数据库表

```
数据库表                  实体类          清理工具支持
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
tb_BOM                    ✅ 有           ✅ 支持
tb_BOMMaterial            ✅ 有           ✅ 支持
tb_BOMChangeLog           ❌ 无           ✅ 支持（新）
tb_UnknownTable           ❌ 无           ✅ 支持（新）
```

### 3. 智能降级策略

```
GetMetadataAsync(db, tableName)
    ↓
[1] 尝试从 Model 程序集加载实体
    ├─ 成功 → 使用实体元数据（完整信息）
    └─ 失败 → 继续下一步
        ↓
[2] 从数据库查询表结构
    ├─ 成功 → 使用数据库元数据（主键列名）
    └─ 失败 → 使用默认值 PrimaryKeyID
```

## 📌 注意事项

### 1. 默认主键名

如果数据库查询失败或表没有主键约束，会使用默认值 `PrimaryKeyID`。

**建议**：确保所有表都遵循主键命名规范（`PrimaryKeyID`）。

### 2. 性能考虑

- 首次查询会从数据库读取表结构
- 后续查询使用缓存，性能无影响
- 缓存Key：`DB.{tableName}`

### 3. 外键关系

外键关系始终从数据库查询，不受实体类影响。
