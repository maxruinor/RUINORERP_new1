# 数据导入功能优化说明

## 📋 优化概述

本次优化实现了完整的数据导入流程：**Excel预览 → 配置映射 → 生成结果预览（显示外键真实值）→ 用户确认 → 保存到数据库**

## ✨ 核心改进

### 1. 新增"生成结果预览"按钮

**位置**: 在"解析"按钮旁边

**功能**:
- 预加载所有外键数据到缓存
- 解析外键字段为真实的ID值
- 显示关联信息（如：`供应商A → ID:123`）
- 标记未找到的外键（如：`未找到: 供应商B`）

**使用流程**:
```
1. 选择Excel文件和工作表
2. 选择目标实体类型
3. 配置列映射（包括外键关联）
4. 点击"解析"按钮 → 仅做数据转换，外键显示占位符
5. 点击"生成结果预览"按钮 → 查询外键并显示真实值
6. 检查预览数据无误后，点击"导入"按钮保存到数据库
```

### 2. 外键数据显示增强

**优化前**:
```
[通过关联外键:供应商名称:XXX->找tb_CustomerVendor.ID]
```

**优化后**:
```
系统字段列: 123 (真实ID)
关联信息列: 供应商A → ID:123
错误列(如有): 未找到: 供应商B
```

### 3. 两阶段导入流程

#### 阶段一：数据解析与预览
- **解析按钮**: 将Excel数据转换为系统字段格式，外键显示占位符
- **生成结果预览按钮**: 查询外键并显示真实值，供用户检查

#### 阶段二：确认与保存
- **导入按钮**: 用户确认数据无误后，执行数据库保存操作
- 导入前会弹出确认对话框，提示用户检查预览数据

### 4. 分步导入支持（宽表模式）

对于包含多个关联表的Excel（如同时包含供应商和产品信息）：

**策略一：一键导入**
- 自动按依赖顺序处理所有表
- 先导入基础表（供应商/类目），再导入主表（产品）

**策略二：分步导入**
- **步骤1**: 仅导入基础表（供应商/类目）
- **步骤2**: 仅导入主表（产品），自动引用步骤1中保存的ID

## 🔧 技术实现

### 核心方法

#### 1. `kbtnGeneratePreview_Click` - 生成结果预览
```csharp
private async void kbtnGeneratePreview_Click(object sender, EventArgs e)
{
    // 1. 预加载所有外键数据到缓存
    PreloadForeignKeyData();
    
    // 2. 创建带外键解析的数据副本
    DataTable previewData = _parsedImportData.Copy();
    
    // 3. 解析外键字段为真实值
    ResolveForeignKeysInPreview(previewData);
    
    // 4. 显示预览并启用导入按钮
    dgvParsedImportData.DataSource = previewData;
    kbtnDynamicImport.Enabled = true;
}
```

#### 2. `ResolveForeignKeysInPreview` - 解析外键为真实值
```csharp
private void ResolveForeignKeysInPreview(DataTable previewData)
{
    // 遍历所有外键映射
    foreach (var mapping in foreignKeyMappings)
    {
        // 从缓存中获取外键ID
        object foreignKeyId = _foreignKeyService.GetForeignKeyId(
            targetTable,
            targetField,
            sourceValue.ToString(),
            out errorMessage
        );
        
        if (foreignKeyId != null)
        {
            // 更新系统字段的值为真实的ID
            row[systemFieldName] = foreignKeyId;
            
            // 添加关联信息显示列
            row[$"{systemFieldName}_关联信息"] = $"{sourceValue} → ID:{foreignKeyId}";
        }
        else
        {
            // 标记错误
            row[$"{systemFieldName}_错误"] = $"未找到: {errorMessage}";
        }
    }
}
```

#### 3. `kbtnDynamicImport_Click` - 导入前确认
```csharp
private async void kbtnDynamicImport_Click(object sender, EventArgs e)

```

### 外键缓存机制

使用 `ForeignKeyCacheService` 实现高效的外键查询：

1. **预加载**: 在生成结果预览时，批量查询所有关联表数据
2. **缓存**: 将查询结果存储在内存中（ConcurrentDictionary）
3. **复用**: 导入时直接从缓存获取，避免重复查询数据库

```csharp
// 缓存结构
ConcurrentDictionary<string, ConcurrentDictionary<string, object>> _foreignKeyCache;
// Key: "表名_字段名" (如: "tb_CustomerVendor_VendorName")
// Value: 字段值到ID的映射 (如: {"供应商A": 123, "供应商B": 456})
```

## 📊 使用示例

### 场景：导入产品信息（含供应商外键）

#### Excel数据结构
| 产品编码 | 产品名称 | 供应商名称 | 价格 |
|---------|---------|-----------|------|
| P001    | 产品A   | 供应商甲   | 100  |
| P002    | 产品B   | 供应商乙   | 200  |

#### 操作步骤

1. **选择文件**: 浏览并选择Excel文件
2. **选择工作表**: 选择包含数据的Sheet
3. **选择实体类型**: 选择"产品基本信息表"
4. **配置列映射**:
   - 产品编码 → ProductCode (唯一值)
   - 产品名称 → ProductName
   - 供应商名称 → VendorID (外键关联: tb_CustomerVendor.VendorName)
   - 价格 → Price

5. **点击"解析"**: 
   - 数据转换完成，供应商ID列显示: `[通过关联外键:供应商名称:供应商甲->找tb_CustomerVendor.VendorName]`

6. **点击"生成结果预览"**:
   - 预加载供应商数据到缓存
   - 解析外键，供应商ID列显示真实值: `123`
   - 关联信息列显示: `供应商甲 → ID:123`

7. **检查预览数据**:
   - 确认外键ID正确
   - 检查是否有"错误"列标记未找到的供应商

8. **点击"导入"**:
   - 弹出确认对话框
   - 确认后执行数据库保存
   - 显示导入结果统计

## 🎯 优势总结

1. **用户体验提升**: 明确的三步骤流程（解析 → 预览 → 导入）
2. **数据准确性**: 导入前可看到真实的外键ID，避免错误
3. **错误定位**: 未找到的外键会被明确标记，方便修正
4. **性能优化**: 外键缓存机制，避免重复查询
5. **灵活性**: 支持单表和多表导入，支持分步操作

## ⚠️ 注意事项

1. **必须先点击"生成结果预览"**: 导入按钮会在解析后被禁用，只有生成预览后才启用
2. **外键数据必须存在**: 如果外键值在数据库中不存在，会在预览中标记为错误
3. **缓存时效性**: 外键缓存在每次生成预览时刷新，确保数据最新
4. **大数据量处理**: 对于大量数据，预加载可能需要一些时间，请耐心等待
