# 数据导入功能使用指南

## 📖 快速开始

### 基本流程

```
1️⃣ 选择Excel文件
   ↓
2️⃣ 选择工作表和数据类型
   ↓
3️⃣ 配置列映射（AI智能或手动）
   ↓
4️⃣ 点击"解析"按钮
   ↓
5️⃣ 点击"生成结果预览"按钮 ⭐ 新增
   ↓
6️⃣ 检查预览数据（外键显示真实ID）
   ↓
7️⃣ 点击"导入"按钮保存到数据库
```

---

## 🎯 详细操作步骤

### 步骤1：选择Excel文件

1. 点击 **"浏览"** 按钮
2. 选择要导入的Excel文件（支持 .xls 和 .xlsx）
3. 系统自动加载工作表列表

### 步骤2：选择工作表和数据类型

1. **选择工作表**: 从下拉框中选择包含数据的Sheet
2. **选择数据类型**: 选择要导入的目标表
   - 供应商表
   - 客户表
   - 产品类目表
   - 产品基本信息表
   - 等等...

### 步骤3：配置列映射

#### 方式一：AI智能映射（推荐）

1. 点击 **"AI 辅助映射"** 按钮
2. 系统自动分析Excel列名和数据库字段
3. 查看并调整AI建议的映射关系

#### 方式二：智能匹配

1. 点击 **"自动匹配"** 按钮
2. 系统根据列名相似度自动匹配
3. 显示匹配置信度（高/中/低）

#### 方式三：手动配置

1. 在左侧选择 **Excel列**
2. 在右侧选择 **系统字段**
3. 点击 **"添加映射"** 按钮
4. 配置字段属性：
   - ✅ 是否为唯一值
   - 🔗 外键关联配置
   - 📷 是否为图片列
   - 💡 默认值设置

### 步骤4：解析数据

点击 **"解析"** 按钮，系统将：
- ✅ 读取Excel全部数据
- ✅ 根据映射配置转换数据
- ✅ 执行去重处理（如果启用）
- ✅ 显示转换后的数据预览

> ⚠️ **注意**: 此时外键字段显示的是占位符文本，如：
> `[通过关联外键:供应商名称:XXX->找tb_CustomerVendor.ID]`

### 步骤5：生成结果预览 ⭐ 核心功能

点击 **"生成结果预览"** 按钮，系统将：
1. 🔍 **预加载外键数据**: 批量查询所有关联表
2. 🔄 **解析外键为真实值**: 将占位符替换为实际ID
3. 📊 **显示关联信息**: 添加"关联信息"列，显示映射关系
4. ❌ **标记错误**: 未找到的外键会标记为错误

**预览效果示例**:

| 产品编码 | 产品名称 | VendorID | VendorID_关联信息 | Price |
|---------|---------|----------|------------------|-------|
| P001    | 产品A   | 123      | 供应商甲 → ID:123 | 100   |
| P002    | 产品B   | 456      | 供应商乙 → ID:456 | 200   |
| P003    | 产品C   |          |                  | 150   |
|         |         |          | VendorID_错误: 未找到: 供应商丙 |       |

### 步骤6：检查预览数据

仔细检查以下内容：
- ✅ 外键ID是否正确
- ✅ 是否有"错误"列标记未找到的外键
- ✅ 数据格式是否符合预期
- ✅ 必填字段是否都有值

### 步骤7：导入到数据库

1. 点击 **"导入"** 按钮
2. 系统弹出确认对话框
3. 确认后开始导入
4. 等待导入完成
5. 查看导入结果统计

## 🔗 外键关联配置详解

### 什么是外键关联？

外键关联是指当前表的某个字段引用了另一个表的主键。例如：
- 产品表的 `VendorID` 字段引用供应商表的 `ID`
- 通过供应商名称查找对应的供应商ID

### 如何配置外键关联？

1. 在列映射配置中，选择系统字段（如 `VendorID`）
2. 打开属性配置对话框
3. 选择数据来源类型为 **"外键关联"**
4. 配置外键信息：
   - **关联表**: tb_CustomerVendor（供应商表）
   - **关联字段**: VendorName（供应商名称）
   - **来源列**: Excel中的"供应商名称"列

### 外键匹配规则

系统会按以下顺序查找外键：

1. **精确匹配**: `WHERE VendorName = '供应商甲'`
2. **模糊匹配**: `WHERE VendorName LIKE '%供应商甲%'`（仅针对名称字段）
3. **缓存查询**: 从预加载的缓存中快速获取

### 外键未找到怎么办？

如果在预览中看到"错误"列标记：

1. **检查Excel数据**: 确认供应商名称拼写正确
2. **检查数据库**: 确认该供应商已存在于数据库中
3. **先导入基础数据**: 
   - 如果供应商不存在，先导入供应商表
   - 再导入产品表，此时就能找到对应ID

## 📦 宽表导入（多表同时导入）

### 什么是宽表导入？

宽表导入是指一个Excel文件中包含多个关联表的数据。例如：
- 同时包含供应商信息和产品信息
- 需要先导入供应商，再导入产品

### 如何使用宽表导入？

1. **选择宽表Profile**: 在下拉框中选择以 `[宽表]` 开头的配置
2. **选择导入策略**:
   
   **策略一：一键导入**（推荐新手）
   - 自动按依赖顺序处理所有表
   - 先导入基础表，再导入主表
   
   **策略二：分步导入**（推荐高级用户）
   - 步骤1: 仅导入基础表（供应商/类目）
   - 步骤2: 仅导入主表（产品）

### 分步导入示例

**场景**: Excel包含供应商和产品信息

**步骤1: 导入供应商**
1. 选择宽表Profile
2. 选择"分步导入" → "步骤1: 仅导入基础表"
3. 系统提取供应商数据并导入
4. 完成后提示可以执行步骤2

**步骤2: 导入产品**
1. 再次点击"导入"按钮
2. 选择"分步导入" → "步骤2: 仅导入主表"
3. 系统自动引用步骤1中保存的供应商ID
4. 导入产品数据

## 💡 最佳实践

### 1. 首次导入建议

- ✅ 先用小批量数据测试（10-20条）
- ✅ 仔细检查预览数据
- ✅ 确认无误后再导入全量数据

### 2. 外键数据处理

- ✅ 先导入被引用的基础表（供应商、类目等）
- ✅ 再导入引用这些基础数据的主表（产品）
- ✅ 使用"生成结果预览"功能验证外键是否正确

### 3. 数据去重

- ✅ 在列映射配置中启用"去除重复数据"
- ✅ 配置去重字段（如产品编码、供应商编号）
- ✅ 选择去重策略（保留第一条/最后一条）

### 4. 性能优化

- ✅ 大数据量导入时，关闭其他应用程序
- ✅ 避免同时导入多个大文件
- ✅ 使用SSD硬盘提升读写速度

### 5. 错误处理

- ✅ 导入失败时查看详细错误信息
- ✅ 修正Excel数据后重新导入
- ✅ 对于部分失败的记录，可以导出错误清单单独处理

## ❓ 常见问题

### Q1: 为什么"导入"按钮是灰色的？

**A**: 必须先点击"生成结果预览"按钮，系统才会启用导入按钮。这是为了确保您在导入前已经检查过数据。

### Q2: 外键ID显示为空怎么办？

**A**: 
1. 检查Excel中的外键值（如供应商名称）是否正确
2. 确认该值在数据库中是否存在
3. 如果不存在，需要先导入基础数据

### Q3: 如何知道哪些外键未找到？

**A**: 在"生成结果预览"后，查看带有"_错误"后缀的列，里面会显示未找到的外键信息。

### Q4: 可以撤销导入操作吗？

**A**: 目前不支持自动撤销。建议在导入前备份数据库，或使用事务机制。

### Q5: 导入速度慢怎么办？

**A**: 
1. 检查网络连接（如果是远程数据库）
2. 减少一次性导入的数据量
3. 确保数据库索引正常
4. 关闭不必要的应用程序

### Q6: AI映射不准确怎么办？

**A**: 
1. 使用"自动匹配"功能作为备选
2. 手动调整不准确的映射
3. 保存为模板，下次直接使用

## 🎓 进阶技巧

### 1. 创建导入模板

1. 配置好列映射
2. 点击"保存映射配置"
3. 输入模板名称
4. 下次导入时直接选择该模板

### 2. 批量导入多个文件

1. 准备好所有Excel文件
2. 使用相同的映射配置
3. 依次导入每个文件
4. 或者编写脚本自动化处理

### 3. 自定义转换规则

在列映射配置中，可以设置：
- 默认值
- 枚举值转换
- 日期格式转换
- 字符串拼接

### 4. 监控导入进度

- 查看状态栏显示的当前操作
- 导入完成后查看详细统计
- 记录导入日志用于审计
