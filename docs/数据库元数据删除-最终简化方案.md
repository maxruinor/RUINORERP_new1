# 数据库元数据删除 - 最终简化方案

## 🎯 核心问题

之前的实现失败原因是：**删除父表时，使用的是父表的外键ID，但子表需要的是父表的主键ID**。

### 错误示例

```
要删除: tb_SaleOut (销售出库单) WHERE SaleOut_MainID IN (ID1, ID2)
  ↓ 
删除子表: tb_SaleOutRe (销售出库返回单) WHERE SaleOut_MainID IN (ID1, ID2)  ✅ 正确
  ↓
删除孙表: tb_SaleOutReDetail (明细) WHERE SaleOutRe_ID IN (ID1, ID2)  ❌ 错误！
                                            ↑ 这里应该是 SaleOutRe 的主键ID，不是 SaleOut_MainID
```

---

## ✅ 正确的解决方案

### 核心思路：每层都查询主键ID

```
第1步：删除最深层子表
  - 用根表ID查询第1层子表的主键ID
  - 用第1层主键ID查询第2层子表的主键ID
  - 用第2层主键ID删除第3层子表（最深层）

第2步：逐层向上删除
  - 用第2层主键ID删除第2层子表
  - 用第1层主键ID删除第1层子表
  - 用根表ID删除根表
```

### 具体实现

#### 步骤 1：收集所有依赖关系（已完成✅）

使用您提供的 SQL #3：
```sql
-- 谁引用了我？
SELECT parent.name AS ReferencingTableName, fc.name AS ReferencingColumnName
FROM sys.foreign_keys fk
INNER JOIN sys.foreign_key_columns fkc ON fk.object_id = fkc.constraint_object_id
INNER JOIN sys.tables parent ON fkc.parent_object_id = parent.object_id
INNER JOIN sys.columns fc ON fkc.parent_object_id = fc.object_id AND fkc.parent_column_id = fc.column_id
WHERE fk.referenced_object_id = OBJECT_ID('tb_SaleOut')
```

得到依赖关系：
```
tb_SaleOut
  ├─ tb_SaleOutRe (外键: SaleOut_MainID)
  │   └─ tb_SaleOutReDetail (外键: SaleOutRe_ID)
  └─ tb_SaleOutDetail (外键: SaleOut_MainID)
```

#### 步骤 2：生成删除顺序（已完成✅）

拓扑排序结果（自底向上）：
```
阶段 1: tb_SaleOutReDetail.SaleOutRe_ID
阶段 2: tb_SaleOutDetail.SaleOut_MainID
阶段 3: tb_SaleOutRe.SaleOut_MainID
阶段 4: tb_SaleOut (根表)
```

#### 步骤 3：执行删除（⚠️ 需要修改）

**关键修复**：在删除每一层之前，先查询出该层的主键ID！

```csharp
// 初始ID：根表的ID
var rootIds = new List<long> { ID1, ID2 };

// 阶段 1：删除 tb_SaleOutReDetail
// ⚠️ 需要先查询 tb_SaleOutRe 的主键ID
var saleOutReIds = await db.Ado.SqlQueryAsync<long>(
    "SELECT SaleOutRe_ID FROM tb_SaleOutRe WHERE SaleOut_MainID IN (@ids)",
    new { ids = rootIds }
);
// 然后删除明细
await db.Deleteable<object>()
    .AS("tb_SaleOutReDetail")
    .Where("[SaleOutRe_ID] IN (@ids)", new { ids = saleOutReIds })
    .ExecuteCommandAsync();

// 阶段 2：删除 tb_SaleOutDetail
// ✅ 直接用根表ID
await db.Deleteable<object>()
    .AS("tb_SaleOutDetail")
    .Where("[SaleOut_MainID] IN (@ids)", new { ids = rootIds })
    .ExecuteCommandAsync();

// 阶段 3：删除 tb_SaleOutRe
// ✅ 直接用根表ID
await db.Deleteable<object>()
    .AS("tb_SaleOutRe")
    .Where("[SaleOut_MainID] IN (@ids)", new { ids = rootIds })
    .ExecuteCommandAsync();

// 阶段 4：删除根表 tb_SaleOut
await db.Deleteable<object>()
    .AS("tb_SaleOut")
    .Where("[SaleOut_MainID] IN (@ids)", new { ids = rootIds })
    .ExecuteCommandAsync();
```

## 🔧 代码修改方案

### 方案 A：简单直接（推荐）

修改 `ExecuteCascadeDeleteByDbAsync` 方法，在删除每个表之前，检查是否需要查询主键ID：

```csharp
for (int stage = 0; stage < deleteOrder.Count; stage++)
{
    var tableInfo = deleteOrder[stage];
    
    // 根表跳过
    if (tableInfo.TableName == rootTableName)
        continue;
    
    // ⚠️ 关键：判断当前表的外键列是否等于currentLevelIds对应的列
    // 如果不等，说明需要先查询主键ID
    
    List<long> idsToDelete;
    
    if (tableInfo.ForeignKeyColumn == currentFkColumn)
    {
        // 外键列匹配，直接使用当前ID
        idsToDelete = currentLevelIds;
    }
    else
    {
        // 外键列不匹配，需要查询主键ID
        Log($"[阶段 {stage + 1}] 外键列不匹配，需要查询主键ID...");
        Log($"[阶段 {stage + 1}] 当前外键列: {currentFkColumn}");
        Log($"[阶段 {stage + 1}] 目标外键列: {tableInfo.ForeignKeyColumn}");
        
        // 查询父表的主键ID
        var parentMetadata = ModelMetadataHelper.GetMetadata(tableInfo.TableName);
        string parentPkColumn = parentMetadata?.PrimaryKeyName ?? "PrimaryKeyID";
        
        Log($"[阶段 {stage + 1}] 父表主键列: {parentPkColumn}");
        Log($"[阶段 {stage + 1}] SQL: SELECT [{parentPkColumn}] FROM [{tableInfo.TableName}] WHERE [{currentFkColumn}] IN (...)");
        
        var pkIds = await db.Ado.SqlQueryAsync<long>(
            $"SELECT [{parentPkColumn}] FROM [{tableInfo.TableName}] WHERE [{currentFkColumn}] IN (@ids)",
            new { ids = currentLevelIds }
        );
        
        idsToDelete = pkIds.ToList();
        Log($"[阶段 {stage + 1}] 查询结果: {idsToDelete.Count} 条记录");
    }
    
    // 执行删除
    var deleted = await db.Deleteable<object>()
        .AS(tableInfo.TableName)
        .Where($"[{tableInfo.ForeignKeyColumn}] IN (@ids)", new { ids = idsToDelete })
        .ExecuteCommandAsync();
    
    Log($"[阶段 {stage + 1}] 删除 {deleted} 条记录");
    
    // 更新currentLevelIds和currentFkColumn
    currentLevelIds = idsToDelete;
    currentFkColumn = tableInfo.ForeignKeyColumn;
}
```

### 方案 B：预查询所有主键ID（更清晰）

在执行删除前，先递归查询所有层级的主键ID：

```csharp
// 构建 ID 映射表：表名 -> 主键ID列表
var idMap = new Dictionary<string, List<long>>();
idMap[rootTableName] = targetIds;

// 递归查询所有子表的主键ID
async Task QueryChildIds(string parentTable, List<long> parentIds, string parentFkColumn)
{
    if (!allDependencies.ContainsKey(parentTable))
        return;
    
    foreach (var fk in allDependencies[parentTable])
    {
        string childTable = fk.ReferencingTableName;
        string childFkColumn = fk.ReferencingColumnName;
        
        // 获取子表的主键列
        var childMetadata = ModelMetadataHelper.GetMetadata(childTable);
        string childPkColumn = childMetadata?.PrimaryKeyName ?? "PrimaryKeyID";
        
        // 查询子表的主键ID
        var childIds = await db.Ado.SqlQueryAsync<long>(
            $"SELECT [{childPkColumn}] FROM [{childTable}] WHERE [{childFkColumn}] IN (@ids)",
            new { ids = parentIds }
        );
        
        idMap[childTable] = childIds.ToList();
        
        // 递归查询下一层
        await QueryChildIds(childTable, childIds.ToList(), childFkColumn);
    }
}

// 开始查询
await QueryChildIds(rootTableName, targetIds, rootPkColumn);

// 按删除顺序执行删除
foreach (var tableInfo in deleteOrder)
{
    if (tableInfo.TableName == rootTableName)
        continue;
    
    var idsToDelete = idMap.ContainsKey(tableInfo.TableName) 
        ? idMap[tableInfo.TableName] 
        : new List<long>();
    
    if (idsToDelete.Count > 0)
    {
        var deleted = await db.Deleteable<object>()
            .AS(tableInfo.TableName)
            .Where($"[{tableInfo.ForeignKeyColumn}] IN (@ids)", new { ids = idsToDelete })
            .ExecuteCommandAsync();
        
        Log($"[阶段] 删除 {tableInfo.TableName} {deleted} 条记录");
    }
}
```
