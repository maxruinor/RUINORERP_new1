{

    var hasForeignKey = await CheckHasForeignKeys(tableName);
    
    // 2. 智能选择 TRUNCATE 或 DELETE
    if (hasForeignKey)

}

## 🎯 核心功能

### 功能1：按选中行删除（级联删除）
**使用场景**：删除具体的几条记录及其关联数据

**操作流程**：
1. 选择实体类型（如 tb_SaleOrder）
2. 勾选要删除的行
3. 选择"按选中行删除（级联）"
4. 点击"删除选中"
5. 系统自动级联删除所有关联数据

**技术实现**：
- 使用数据库元数据模式
- 预查询所有层级的主键ID
- 拓扑排序确定删除顺序
- 自底向上批量删除

---

### 功能2：整表清空（TRUNCATE/DELETE）
**使用场景**：快速清空整个表的所有数据

**操作流程**：
1. 选择实体类型（如 tb_BuyingRequisition）
2. 选择"整表清空（TRUNCATE）"
3. 点击"删除选中"
4. 确认清空操作
5. 系统自动选择TRUNCATE或DELETE

**技术实现**：
```csharp
// 检查外键约束
SELECT COUNT(*) FROM sys.foreign_keys 
WHERE referenced_object_id = OBJECT_ID(@tableName)
   OR parent_object_id = OBJECT_ID(@tableName)

// 无外键：TRUNCATE TABLE（快300倍，重置自增ID）
TRUNCATE TABLE [tb_YourTable]

// 有外键：DELETE FROM（安全，保留自增ID）
DELETE FROM [tb_YourTable]
```

## ⚠️ 注意事项

### 1. TRUNCATE 的限制
- ❌ 不能用于有外键约束的表
- ❌ 不会触发DELETE触发器
- ❌ 不能带WHERE条件
- ✅ 速度极快，重置自增ID

### 2. DELETE 的特点
- ✅ 可以处理有外键的表
- ✅ 会触发DELETE触发器
- ✅ 可以带WHERE条件
- ⚠️ 速度较慢，保留自增ID

### 3. 智能切换逻辑
```csharp
if (表有外键约束)
    使用 DELETE FROM（更安全）
else
    使用 TRUNCATE TABLE（更快速）
```
