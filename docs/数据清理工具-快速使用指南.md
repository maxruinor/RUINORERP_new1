# 数据清理工具 - 快速使用指南

## 🎯 工具简介

数据清理工具是一个强大的级联删除工具，支持两种删除模式：
- **实体导航模式**：基于 Model 层的 Navigate 属性（推荐）
- **数据库元数据模式**：直接读取数据库外键关系

## 🚀 快速开始

### 步骤 1：打开工具

在系统菜单中找到：**系统设置 → 系统工具 → 基础数据清理**

### 步骤 2：选择要清理的数据表

1. 在"实体类型"下拉框中选择要清理的表
2. 可以使用"快速搜索"框快速定位

![选择实体](images/select-entity.png)

### 步骤 3：刷新数据

点击 **"刷新"** 按钮，加载该表的所有记录

### 步骤 4：选择要删除的记录

使用表格左侧的 CheckBox 勾选要删除的记录：

- **全选**：勾选所有记录
- **取消选择**：取消所有勾选
- **反选**：反转当前选择状态

### 步骤 5：预览删除影响

点击 **"数据预览"** 按钮，查看预计删除的记录数量

> 💡 此操作不会删除任何数据，仅显示影响范围

### 步骤 6：选择删除模式

在右上角选择删除方式：

| 模式 | 适用场景 | 推荐度 |
|------|---------|--------|
| 实体导航模式 | 标准业务实体 | ⭐⭐⭐⭐⭐ |
| 数据库元数据模式 | 复杂表结构、缺少Navigate定义 | ⭐⭐⭐ |

### 步骤 7：测试执行

点击 **"测试执行"** 按钮，验证删除逻辑

> ⚠️ 此模式会执行删除但自动回滚，不会真正删除数据

### 步骤 8：正式删除

确认无误后，点击 **"删除选中"** 按钮

系统会弹出二次确认对话框，确认后开始正式删除。

## 📋 功能详解

### 1. 实体选择

#### 分类浏览
实体按业务模块分组显示：
```
--- 基础资料 ---
  客户信息
  供应商信息
  商品信息
  
--- 销售管理 ---
  销售订单
  销售出库单
```

#### 快速搜索
在搜索框输入关键词，自动过滤匹配的实体：
- 支持拼音搜索
- 支持中文名称搜索
- 支持表名搜索
- 单个匹配结果自动选中

### 2. 数据预览

表格显示以下信息：
- ✅ CheckBox 列：用于选择记录
- ✅ 主键字段：记录的唯一标识
- ✅ 常用字段：最多显示20个字段
- ✅ 字段标题：优先使用 AdvQueryAttribute 的 ColDesc

### 3. 删除模式对比

#### 实体导航模式（推荐）

**工作原理**：
```
Model 实体类
  ↓
Navigate 属性定义
  ↓
自动识别依赖关系
  ↓
拓扑排序确定删除顺序
```

**优点**：
- ✅ 符合业务逻辑设计
- ✅ 准确可靠
- ✅ 避免误删

**缺点**：
- ❌ 依赖实体定义的完整性

**适用场景**：
- 标准业务实体
- 有完整 Navigate 属性定义
- 需要精确控制删除范围

#### 数据库元数据模式

**工作原理**：
```
SQL Server 系统表
  ↓
sys.foreign_keys 查询
  ↓
获取所有外键关系
  ↓
拓扑排序确定删除顺序
```

**优点**：
- ✅ 覆盖所有外键关系
- ✅ 不依赖实体定义
- ✅ 适用于复杂场景

**缺点**：
- ❌ 可能删除意外关联的数据
- ❌ 需要仔细验证

**适用场景**：
- 复杂表结构
- 缺少 Navigate 定义
- 需要强制清理

### 4. 执行模式

#### 数据预览
- **作用**：查看预计删除数量
- **安全性**：✅ 完全安全，不执行任何删除
- **用途**：评估影响范围

#### 测试执行
- **作用**：验证删除逻辑
- **安全性**：✅ 事务自动回滚
- **用途**：测试删除流程

#### 正式执行
- **作用**：真正删除数据
- **安全性**：⚠️ 数据永久删除，不可恢复
- **用途**：生产环境使用

## ⚠️ 安全注意事项

### 必须遵守的规则

1. **备份先行**
   ```
   重要！在执行正式删除前，务必备份数据
   ```

2. **测试验证**
   ```
   始终先使用"测试执行"验证删除逻辑
   ```

3. **权限控制**
   ```
   确保只有授权人员可以访问此工具
   ```

4. **谨慎操作**
   ```
   仔细阅读确认对话框，确认无误后再执行
   ```

### 风险提示

⚠️ **高风险操作**
- 级联删除会影响多个关联表
- 删除后数据无法恢复
- 可能影响正在使用的业务数据

💡 **建议**
- 在非高峰时段执行
- 小批量测试后再大批量执行
- 保留操作日志备查

## 🔍 常见问题

### Q1: 为什么有些记录删除失败？

**A**: 可能的原因：
1. 数据正在被其他单据引用
2. 外键约束冲突
3. 数据库权限不足

**解决方法**：
- 查看日志中的详细错误信息
- 检查是否有未完成的业务流程
- 联系管理员处理

### Q2: 如何选择删除模式？

**A**: 推荐策略：
1. 优先使用**实体导航模式**
2. 如果删除不完整，尝试**数据库元数据模式**
3. 比较两种模式的结果，选择更合适的

### Q3: 删除速度慢怎么办？

**A**: 优化建议：
1. 分批删除（每次100-500条）
2. 在非高峰时段执行
3. 检查数据库性能
4. 关闭不必要的应用程序

### Q4: 如何查看删除日志？

**A**: 
1. 切换到"运行日志"标签页
2. 查看所有操作的详细记录
3. 可以右键清除日志

### Q5: 能否撤销删除操作？

**A**: 
- ❌ 正式删除后无法撤销
- ✅ 测试执行会自动回滚
- 💡 建议从备份恢复数据
