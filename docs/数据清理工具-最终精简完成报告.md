## 🎯 当前实现方案

### 单一体系：纯数据库元数据模式

**核心流程**：
```
1. 从 SQL Server 系统表查询外键关系
   ↓
2. 递归收集所有依赖表
   ↓
3. 生成删除顺序（自底向上）
   ↓
4. 预查询所有层级的主键ID
   ↓
5. 按顺序批量删除
   ↓
6. 最后删除根表
```

**关键SQL**：
```sql
-- 查询外键关系
SELECT 
    fk.name AS ForeignKeyName,
    tp.name AS ReferencingTableName,     -- 子表
    cp.name AS ReferencingColumnName,    -- 外键列
    tr.name AS ReferencedTableName,      -- 父表
    cr.name AS ReferencedColumnName      -- 主键列
FROM sys.foreign_keys AS fk
INNER JOIN sys.foreign_key_columns AS fkc ...
WHERE tr.name = @tableName
```

**优势**：
- ✅ 覆盖全面（不依赖实体类定义）
- ✅ 可靠性高（直接读取数据库）
- ✅ 维护简单（无需同步更新实体）
- ✅ 已验证成功（用户测试通过）

## 🔧 两种删除模式

### 模式1：按选中行删除（级联）
**使用场景**：删除具体的几条记录及其关联数据

**操作流程**：
1. 选择实体类型（如 tb_SaleOrder）
2. 勾选要删除的行
3. 选择"按选中行删除（级联）"
4. 点击"删除选中"
5. 系统自动级联删除所有关联数据

**技术实现**：
- 使用数据库元数据模式
- 预查询所有层级的主键ID
- 按拓扑顺序批量删除

---

### 模式2：整表清空（TRUNCATE/DELETE）
**使用场景**：快速清空整个表的所有数据

**操作流程**：
1. 选择实体类型（如 tb_BuyingRequisition）
2. 选择"整表清空（TRUNCATE）"
3. 点击"删除选中"
4. 确认操作
5. 系统自动选择TRUNCATE或DELETE

**智能切换逻辑**：
```csharp
if (表有外键约束)
    使用 DELETE FROM（更安全）
else
    使用 TRUNCATE TABLE（更快速）
```
