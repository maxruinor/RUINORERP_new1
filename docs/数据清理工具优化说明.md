## 使用建议

### 推荐工作流程

1. **准备阶段**
   ```
   选择实体类型 → 刷新数据 → 勾选要删除的记录
   ```

2. **预览阶段**
   ```
   点击"数据预览" → 查看预计删除数量 → 确认影响范围
   ```

3. **测试阶段**
   ```
   选择删除模式 → 点击"测试执行" → 验证删除逻辑
   ```

4. **执行阶段**
   ```
   确认无误 → 点击"删除选中" → 等待完成
   ```

### 删除模式选择建议

| 模式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **实体导航模式** | 标准业务实体，有完整的 Navigate 属性定义 | 准确、可靠、符合业务逻辑 | 依赖实体定义的完整性 |
| **数据库元数据模式** | 复杂表结构、缺少 Navigate 定义、需要强制清理 | 直接读取数据库外键、覆盖全面 | 可能删除意外关联的数据 |

**推荐**: 优先使用**实体导航模式**，仅在特殊情况下使用数据库元数据模式。

## 注意事项

⚠️ **重要警告**

1. **数据备份**: 执行正式删除前务必备份重要数据
2. **测试先行**: 始终先使用"测试执行"验证删除逻辑
3. **权限控制**: 确保只有授权人员可以执行删除操作
4. **事务安全**: 删除失败会自动回滚，不会留下部分删除的数据
5. **性能考虑**: 大批量删除建议在非高峰时段执行

## 技术细节

### 数据库元数据查询 SQL

```sql
-- 查询引用指定表的所有外键
SELECT 
    fk.name AS ForeignKeyName,
    tp.name AS ReferencingTableName,     -- 引用方表名（子表）
    cp.name AS ReferencingColumnName,    -- 引用方外键列
    tr.name AS ReferencedTableName,      -- 被引用方表名（父表）
    cr.name AS ReferencedColumnName      -- 被引用方主键列
FROM sys.foreign_keys AS fk
INNER JOIN sys.foreign_key_columns AS fkc 
    ON fk.object_id = fkc.constraint_object_id
INNER JOIN sys.tables AS tp 
    ON fkc.parent_object_id = tp.object_id
INNER JOIN sys.columns AS cp 
    ON fkc.parent_object_id = cp.object_id AND fkc.parent_column_id = cp.column_id
INNER JOIN sys.tables AS tr 
    ON fkc.referenced_object_id = tr.object_id
INNER JOIN sys.columns AS cr 
    ON fkc.referenced_object_id = cr.object_id AND fkc.referenced_column_id = cr.column_id
WHERE tr.name = @tableName
```

### 拓扑排序算法

1. 构建依赖图（父表 → 子表）
2. 计算每个节点的出度（子表数量）
3. 找出度为 0 的节点（叶子节点/最底层子表）
4. 删除这些节点并更新父节点的出度
5. 重复步骤 3-4 直到所有节点处理完毕
6. 得到自底向上的删除顺序
