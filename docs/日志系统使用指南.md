# 日志系统使用指南

## 1. 日志服务概述

系统提供两种日志服务：
- **AuditLogService**: 审计日志服务（通用审计）
- **FMAuditLogService**: 财务审计日志服务（财务专用）

## 2. 核心特性

### 2.1 性能优化
- ✅ 异步非阻塞写入（使用 `Task.Run` + fire-and-forget）
- ✅ 队列限流机制（防止内存泄漏）
- ✅ 批量写入（默认每50条或1秒刷新）
- ✅ 超时保护（30秒超时）
- ✅ 指数退避重试（最多3次）
- ✅ Log4Net 高频日志缓冲（Debug/Trace）

### 2.2 可靠性
- ✅ 丢失日志告警
- ✅ 队列满自动丢弃（防止系统崩溃）
- ✅ 应用关闭自动刷新
- ✅ 性能监控指标

## 3. 日志服务使用方法

### 3.1 在控制器中使用（推荐）

```csharp
// 在控制器中注入服务
public class tb_ProductController
{
    private readonly IAuditLogService _auditLogService;
    private readonly ILogger<tb_ProductController> _logger;

    public tb_ProductController(
        IAuditLogService auditLogService,
        ILogger<tb_ProductController> logger)
    {
        _auditLogService = auditLogService;
        _logger = logger;
    }

    // 记录实体操作
    public async Task<ReturnResults> SaveProduct(tb_Product product)
    {
        try
        {
            // 业务逻辑...
            await _repository.AddAsync(product);

            // ✅ 记录审计日志（异步，不阻塞业务）
            await _auditLogService.LogAsync("创建产品", product, "用户手动创建");

            return ReturnResults.Success();
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "保存产品失败");
            return ReturnResults.Error(ex.Message);
        }
    }
}
```

### 3.2 在业务服务中使用

```csharp
public class ProductService
{
    private readonly IAuditLogService _auditLogService;
    private readonly ILogger<ProductService> _logger;

    public ProductService(
        IAuditLogService auditLogService,

    {
        _auditLogService = auditLogService;
        _logger = logger;

    public async Task DeleteProductAsync(long productId)
    {
```

```csharp
}
```

## 6. Microsoft.Extensions.Logging 使用（常规日志）

### 6.1 控制器中使用

```csharp
public class MyController
{
    private readonly ILogger<MyController> _logger;

    public MyController(ILogger<MyController> logger)
    {
        _logger = logger;
    }

    public void ProcessData()
    {
        // ✅ Information 级别（重要业务日志）
        _logger.LogInformation("开始处理数据");

        try
        {
            // 业务逻辑...
            _logger.LogInformation("数据处理完成");
        }
        catch (Exception ex)
        {
            // ✅ Error 级别（错误日志）
            _logger.LogError(ex, "数据处理失败");
        }
    }
}
```

### 6.2 不同日志级别

```csharp

// ✅ Information - 重要信息（直接写入）

// ✅ Warning - 警告信息（直接写入）
```

## 8. 配置说明

日志服务配置在 `Startup.cs` 中：

```csharp
services.Configure<AuditLogOptions>(options =>
{
    options.BatchSize = 50;           // 批量写入大小
    options.FlushInterval = 1000;     // 刷新间隔1秒
    options.EnableAudit = true;       // 启用审计
    options.MaxQueueSize = 10000;     // 队列最大容量
    options.FlushTimeout = 30000;     // 刷新超时30秒
    options.MaxRetryCount = 3;        // 最大重试次数
    options.EnableMetrics = true;     // 启用性能监控
});
```

## 9. 最佳实践

### ✅ 推荐做法

1. **使用异步方法**：优先使用 `await LogAsync()`
2. **合理选择日志级别**：
   - Trace/Debug: 调试信息（会被缓冲）
   - Information: 重要业务操作
   - Warning: 潜在问题
   - Error: 错误信息
   - Critical: 系统严重错误
3. **使用参数化日志**：`_logger.LogInformation("用户 {Id} 登录", userId)`
4. **依赖注入接口**：注入 `IAuditLogService` 而非具体类
5. **不要在日志中记录敏感信息**：密码、身份证号等

### ❌ 避免做法

1. ❌ 同步调用 `LogAsync().Wait()`（除非必要）
2. ❌ 在循环中记录大量日志（考虑使用参数化批量记录）
3. ❌ 在热路径中使用 Trace 级别（影响性能）
4. ❌ 记录过于详细的调试信息到生产环境
5. ❌ 在日志中嵌入大量数据（影响性能）

## 10. 故障排查

### 10.1 日志丢失

如果发现日志丢失，检查：
1. 队列是否已满（查看 `GetMetrics()` 返回的 `QueueSize`）
2. 是否达到 `MaxQueueSize` 限制
3. 是否触发 `OnLogDropped` 回调

### 10.2 性能问题

如果日志影响性能：
1. 降低日志级别（设置 `CurrentLogLevel = LogLevel.Warning`）
2. 增加 `BatchSize` 和 `FlushInterval`
3. 检查是否有死循环记录日志
4. 查看 `GetPerformanceReport()` 分析性能指标

### 10.3 数据库写入失败

如果日志写入数据库失败：
1. 检查数据库连接
2. 检查 `FlushTimeout` 是否过短
3. 查看 `_logger` 输出的错误信息
4. 增加 `MaxRetryCount` 配置

## 11. 迁移指南

### 11.1 从旧代码迁移

**旧代码：**
```csharp
FMAuditLogHelper helper = new FMAuditLogHelper();
helper.CreateAuditLog("操作", entity);
```

**新代码：**
```csharp
// 通过依赖注入获取
private readonly IAuditLogService _auditLogService;

// 使用异步方法
await _auditLogService.LogAsync("操作", entity);
```

### 11.2 逐步迁移策略

1. **第一阶段**：保持旧代码不变，新代码使用新方式
2. **第二阶段**：逐步替换旧代码为依赖注入方式
3. **第三阶段**：移除 `FMAuditLogHelper`（如果不再需要）
