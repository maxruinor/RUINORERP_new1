- **位置**: SessionInfo.cs, HeartbeatCommandHandler.cs, NetPacketPipelineFilter.cs

#### 📄 报告2观点
- **缺失**: ❌ 完全未提及此问题

#### 💻 代码验证结果

**验证点1: SessionInfo.cs**

**验证点2: HeartbeatCommandHandler.cs**

**问题分析**:

**结论**:
- ✅ 报告1准确识别问题
- ✅ 这是真实存在的性能问题
- ⚠️ 100MB估计偏高，实际约10-30MB

- ✅ P0优先级：改用依赖注入获取ILogger

```csharp
// 改造后（通过构造函数注入）

{

    public SessionInfo(ILogger<SessionInfo> logger)
    {
        _logger = logger;
    }
}
```

```csharp
{
    private const int MaxQueueSize = 100;      // 报告1建议
    private const int MaxPropertySize = 1024 * 1024; // 报告2建议（1MB）
    
    public void AddSendData(EncryptedData gde)
    {
        // 限制队列大小
        while (DataQueue.Count >= MaxQueueSize)
        {
            DataQueue.TryDequeue(out _);
        }
        DataQueue.Enqueue(gde.Head);
    }
    
    public bool SetSessionProperty(string key, object value)
    {
        long size = EstimateObjectSize(value);
        if (size > MaxPropertySize)
        {
            _logger?.LogWarning($"会话属性过大: {size/1024}KB, 拒绝存储");
            return false;
        }
        Properties[key] = value;
        return true;
    }
}
```

### 七、登录尝试记录清理策略保守

#### 📄 报告1观点
- **问题**: 清理策略过于保守，可能保留过多历史数据
- **代码**: LoginCommandHandler.cs#L1115

#### 📄 报告2观点
- **缺失**: ❌ 未提及此问题

#### 💻 代码验证结果

**验证点1: 清理逻辑**
```csharp
// LoginCommandHandler.cs 第1115行 ✅ 确认
if (_loginAttempts.TryGetValue(username, out var attempts) && 
    attempts < ConservativeCleanupThreshold)
{
    keysToRemove.Add(username);
}
```

**问题分析**:
- 只有失败次数 < 阈值才清理
- 高失败次数的记录会永久保留
- 可能导致内存泄漏

**结论**:
- ✅ 报告1准确识别问题
- ⚠️ 这是安全与性能的权衡

**建议采纳**:
- ✅ P2优先级：改进清理策略
- 建议方案：
```csharp
// 添加最大保留时间（如24小时）
private static readonly TimeSpan MaxRetentionDuration = TimeSpan.FromHours(24);

foreach (var kvp in _loginAttemptTimes)
{
    var username = kvp.Key;
    var firstAttemptTime = kvp.Value;
    
    // 超过24小时的记录强制清理，无论失败次数
    if (now - firstAttemptTime > MaxRetentionDuration)
    {
        keysToRemove.Add(username);
    }
    // 或者保留原有的保守策略
    else if (now - firstAttemptTime > LoginAttemptExpiryDuration && 
             attempts < ConservativeCleanupThreshold)
    {
        keysToRemove.Add(username);
    }
}
```
