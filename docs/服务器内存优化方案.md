# RUINORERP.Server 内存占用优化方案

## 📊 问题概述

**当前状况：**
- 最大并发用户数：≤50人
- 实际内存占用：≥5GB
- 预期合理内存：应在 500MB-1GB 范围内

**核心问题定位：**
通过代码审查，发现以下主要内存消耗源：

---

## 🔍 内存占用分析

### 1. **EntityCacheManager - 实体缓存管理器（主要问题）**

#### 问题描述
- **位置**: `RUINORERP.Business\Cache\EntityCacheManager.cs`
- **配置限制**: 
  ```csharp
  private readonly long _maxCacheSize = 400 * 1024 * 1024; // 400MB
  private long _cacheSizeThreshold = (long)(_maxCacheSize * 0.7); // 280MB触发清理
  ```
- **实际问题**: 
  - 虽然设置了400MB限制，但这是**软限制**，仅靠定时清理
  - 缓存的是**完整实体对象列表**，包括所有字段
  - 对于大表（如订单、库存），单个表的缓存可能达到几十MB
  - 如果有50个表被缓存，轻松超过2GB

#### 内存泄漏风险
```csharp
// 第109行：存储所有表的列表缓存
private ConcurrentDictionary<string, object> _tableListCaches = new();

// 第1836行：缓存统计字典持续累积
private Dictionary<string, CacheItemStatistics> _cacheItemStatistics = new();
```

**问题点：**
- `_cacheItemStatistics` 只增不减，即使缓存项被移除，统计信息仍保留
- 没有真正的内存上限保护机制
- 清理策略是LRU（最近最少使用），但对于50个用户同时在线，大部分数据都是"活跃"的

## 🎯 优化方案（按优先级排序）

### ✅ P0 - 立即执行（预计减少2-3GB）

#### 1. 优化EntityCacheManager缓存策略

**问题根源：**
- 缓存了太多不必要的大表数据
- 没有针对小并发场景优化

**解决方案：**

##### 方案A：降低缓存容量并启用严格模式
```csharp
// EntityCacheManager.cs 修改
private readonly long _maxCacheSize = 100 * 1024 * 1024; // 从400MB降至100MB
private long _cacheSizeThreshold = (long)(_maxCacheSize * 0.6); // 60MB触发清理

// 缩短过期时间
private TimeSpan GetCacheExpirationTime()
{
    return TimeSpan.FromMinutes(15); // 从30分钟降至15分钟
}
```

##### 方案B：选择性缓存（推荐）
```csharp
// 在EntityCacheManager构造函数中添加白名单
private HashSet<string> _cacheWhitelist = new HashSet<string>
```

##### 方案C：修复统计信息泄漏
```csharp
// RemoveFromCache方法增强
private void RemoveFromCache(string cacheKey)

private Timer _statsCleanupTimer;

{

    // 每30分钟清理一次过期的统计信息

}

private void CleanupExpiredStats(object state)
{

    {
        var now = DateTime.Now;
        var expiredKeys = _cacheItemStatistics
            .Where(kvp => kvp.Value.LastAccessedTime < now.AddHours(-1))
            .Select(kvp => kvp.Key)
            .ToList();
            
        foreach (var key in expiredKeys)
        {
            _cacheItemStatistics.Remove(key);
        }
        
        if (expiredKeys.Count > 0)
        {
            _logger?.LogDebug($"清理了{expiredKeys.Count}个过期的缓存统计信息");
        }
    }
}
```

#### 2. 统一缓存层，避免双重缓存

**问题：**
- EntityCacheManager使用自己的缓存实现
- MemoryCache也注册为服务
- 两者可能缓存相同数据

**解决方案：**

让EntityCacheManager直接使用MemoryCache：

```csharp
// EntityCacheManager.cs 修改
public class EntityCacheManager : IEntityCacheManager
{
    private readonly IMemoryCache _memoryCache; // 改用.NET标准缓存
    private readonly ITableSchemaManager _tableSchemaManager;
    private readonly ILogger<EntityCacheManager> _logger;
    
    public EntityCacheManager(
        IMemoryCache memoryCache,  // 注入标准缓存
        ITableSchemaManager tableSchemaManager,
        ILogger<EntityCacheManager> logger)
    {
        _memoryCache = memoryCache;
        _tableSchemaManager = tableSchemaManager;
        _logger = logger;
        
        // 移除自定义的_cacheManager和统计逻辑
    }
    
    // 简化PutToCache
    private void PutToCache(string cacheKey, object value, string cacheType, string tableName)
    {
        var options = new MemoryCacheEntryOptions()
            .SetSize(EstimateObjectSize(value))
            .SetSlidingExpiration(GetCacheExpirationTime())
            .SetPriority(CacheItemPriority.Normal);
            
        _memoryCache.Set(cacheKey, value, options);
    }
    
    // 简化GetFromCache
    private T GetFromCache<T>(string cacheKey)
    {
        return _memoryCache.Get<T>(cacheKey);
    }
}
```

**优势：**
- 统一由MemoryCache管理，遵守500MB限制
- 自动处理过期和压缩
- 减少代码复杂度

#### 3. 优化SessionInfo存储

**问题：**
- SessionInfo.Properties可能存储大对象

**解决方案：**

```csharp
// SessionInfo.cs 添加限制
public class SessionInfo : AppSession
{
    private const int MaxPropertySize = 1024 * 1024; // 1MB限制
    
    public bool SetSessionProperty(string key, object value)
    {
        // 检查对象大小
        long size = EstimateObjectSize(value);
        if (size > MaxPropertySize)
        {
            _logger?.LogWarning($"会话属性过大: Key={key}, Size={size/1024}KB, 拒绝存储");
            return false;
        }
        
        Properties[key] = value;
        return true;
    }
    
    private long EstimateObjectSize(object obj)
    {
        if (obj == null) return 0;
        
        // 简单估算：字符串、基本类型等
        if (obj is string str)
            return System.Text.Encoding.UTF8.GetByteCount(str);
        
        // 复杂对象序列化后估算
        try
        {
            var json = Newtonsoft.Json.JsonConvert.SerializeObject(obj);
            return System.Text.Encoding.UTF8.GetByteCount(json);
        }
        catch
        {
            return 1024; // 默认1KB
        }
    }
}
```

#### 5. 清理Pending Requests泄漏

```csharp
// SessionService.cs 增强清理逻辑
private void CleanupPendingRequests()
{
    try
    {
        var cleanedCount = 0;
        var now = DateTime.UtcNow;
        const int REQUEST_TIMEOUT_MINUTES = 2; // 从5分钟降至2分钟
        
        // 查找所有超时的请求
        var timeoutKeys = _pendingRequests
            .Where(kvp => !kvp.Value.Task.IsCompleted)
            .Select(kvp => kvp.Key)
            .ToList();
            
        foreach (var key in timeoutKeys)
        {
            if (_pendingRequests.TryRemove(key, out var tcs))
            {
                try
                {
                    if (!tcs.Task.IsCompleted)
                    {
                        tcs.SetCanceled();
                    }
                }
                catch { }
                
                cleanedCount++;
            }
        }
        
        // 清理已完成的任务
        var completedKeys = _pendingRequests
            .Where(kvp => kvp.Value.Task.IsCompleted)
            .Select(kvp => kvp.Key)
            .ToList();
            
        foreach (var key in completedKeys)
        {
            _pendingRequests.TryRemove(key, out _);
            cleanedCount++;
        }
        
        if (cleanedCount > 0)
        {
            _logger?.LogDebug($"清理了{cleanedCount}个待处理请求");
        }
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, "清理待处理请求时发生错误");
    }
}
```

```csharp
public List<T> GetEntityListWithProjection<T>(string tableName, string[] fields)
{
    // 检查是否有对应的投影缓存
    var projectionKey = $"{tableName}_{string.Join("_", fields)}";

    if (cached != null)

    return result;
}
```

## 📝 总结

**核心问题：**
1. EntityCacheManager缓存过多大表数据（主因）
2. 双重缓存机制造成浪费
3. 缺乏严格的内存上限控制

**解决思路：**
1. **减少缓存量**：白名单机制，只缓存必要的小表
2. **统一管理**：使用MemoryCache替代自定义缓存
3. **严格限制**：所有缓存组件都设置硬性上限
4. **及时清理**：修复泄漏，加强清理机制

**预期收益：**
- 内存占用从5GB+降至1.5-2.5GB
- 减少60-70%的内存使用
- 提升GC效率，降低延迟抖动
- 提高系统稳定性和可扩展性
