# 服务器内存占用优化分析报告

**生成日期**: 2026-04-20

---

## 一、问题概述

服务器内存占用达到 **5GB以上**，而最大在线用户数不超过50人。需分析内存占用原因并提供优化建议。

---

## 二、主要内存占用问题分析

### 1. MemoryCache 未设置 Size（严重 ⚠️）

**位置**: Startup.cs#L619-627

虽然配置了 `SizeLimit = 500MB`，但大多数缓存操作没有使用 `.SetSize()`：

- SmartReminderMonitor.cs:62 - quickTestKey 缓存
- SmartReminderMonitor.cs:311 - policies 缓存
- StockCacheService.cs:162, 446 - stock 缓存

这会导致缓存大小无法正确计算，无法触发自动清理。

---

### 2. SessionInfo 中的 DataQueue 潜在问题

**位置**: Network/Models/SessionInfo.cs#L96-97

```csharp
public ConcurrentQueue<byte[]> DataQueue = new ConcurrentQueue<byte[]>();
```

每个Session都维护一个 `DataQueue`，如果客户端接收缓慢，队列会堆积大量数据。

---

### 3. 静态 LoggerFactory 重复创建（严重 ⚠️）

**问题文件**:
- SessionInfo.cs:29
- HeartbeatCommandHandler.cs:33  
- NetPacketPipelineFilter.cs:33

```csharp
private static readonly ILogger _logger = LoggerFactory.Create(builder => builder.AddConsole()).CreateLogger("SessionInfo");
```

每个类都创建独立的日志器工厂，造成不必要的内存开销。

---

### 4. ImageCache 过期时间过长

**位置**: Network/Services/ImageCacheService.cs

```csharp
// 8小时绝对过期 + 4小时滑动过期
: base(memoryCache, slidingExpirationMinutes: 240, absoluteExpirationHours: 8)
```

---

### 5. frmMainNew 中的全局数据结构无限制

**位置**: frmMainNew.cs#L109-132

```csharp
public ConcurrentDictionary<long, BaseConfig> UpdateConfigDataList;
public ConcurrentDictionary<long, ReminderData> ReminderBizDataList;
public ConcurrentDictionary<string, string> workflowlist;
public Queue<ReminderData> MessageList;
```

这些集合没有设置最大容量限制，可能无限增长。

---

### 6. 登录尝试记录清理策略保守

**位置**: Network/CommandHandlers/LoginCommandHandler.cs#L1100-1126

```csharp
// 保守策略：失败次数<阈值才清理
if (_loginAttempts.TryGetValue(username, out var attempts) && attempts < ConservativeCleanupThreshold)
```

---

### 7. 大量的 Singleton 服务

项目中有大量单例服务（20+），包括多个Timer，每个Timer都是独立的内存占用：

- SessionService: 2个Timer
- SmartReminderMonitor: 1个Timer  
- SmartReminderService: 1个Timer
- ServerLockManager: 2个Timer
- PerformanceDataStorageService: 1个Timer
- TopServerClientService: 3个Timer
- ...

---

## 三、优化建议

### 优化1：修复 MemoryCache Size 设置（高优先级）

为所有缓存操作添加 SetSize：

```csharp
var options = new MemoryCacheEntryOptions()
    .SetAbsoluteExpiration(TimeSpan.FromMinutes(5))
    .SetSize(1);  // 确保添加此行
_cache.Set(key, value, options);
```

### 优化2：限制 DataQueue 大小

```csharp
public class SessionInfo : AppSession
{
    private const int MaxQueueSize = 100;
    
    public void AddSendData(EncryptedData gde)
    {
        while (DataQueue.Count > MaxQueueSize)
        {
            DataQueue.TryDequeue(out _);
        }
    }
}
```

### 优化3：移除静态 LoggerFactory

改用依赖注入方式获取ILogger。

### 优化4：调整 ImageCache 过期时间

建议改为更短的时间（如1小时绝对过期，30分钟滑动过期）。

### 优化5：为全局集合设置容量限制

### 优化6：改进登录记录清理策略

---

## 四、建议实施优先级

| 优先级 | 优化项 | 预期效果 |
|--------|--------|----------|
| 🔴 P0 | 修复 MemoryCache Size 设置 | 防止缓存无限增长 |
| 🔴 P0 | 移除静态 LoggerFactory | 减少~100MB内存 |
| 🟠 P1 | 限制 DataQueue 大小 | 防止数据堆积 |
| 🟠 P1 | 调整 ImageCache 过期时间 | 加快缓存周转 |
| 🟡 P2 | 为全局集合设置容量限制 | 防止数据无限增长 |
| 🟡 P2 | 改进登录记录清理策略 | 防止历史数据堆积 |

## 五、现有监控能力

项目已有以下监控能力：

1. **MemoryMonitoringService** - 基础内存监控
   - 每30秒检查内存使用

   - 警告/临界阈值通知

2. **ServerMonitorControl** - 服务器监控UI
   - 显示工作集内存
   - 显示托管内存
   - 性能监控Tab页

3. **PerformanceDataStorageService** - 客户端性能数据存储
   - 存储客户端上报的性能数据
   - 支持数据清理
