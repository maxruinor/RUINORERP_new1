## 🔍 根本原因分析

### 1. 工作流设计缺陷 🔴 **严重**

#### 问题代码 (已修复前)
```csharp
// ❌ 原始代码 - 存在严重内存泄漏
public class DailyTaskWorkflow : IWorkflow
{
    static DateTime executionTime = DateTime.Today.AddHours(10).AddMinutes(50);
    TimeSpan NextTime = executionTime - DateTime.Now;
    
    public void Build(IWorkflowBuilder<object> builder)
    {
        builder
            .Recur(data => NextTime, data => data.ToString() != "")  // ❌ 条件永远为 true
            .Do(recur => recur.StartWith<DailyTaskStep>())
            .OnError(WorkflowErrorHandling.Retry, TimeSpan.FromMinutes(20));
    }
}
```

#### 问题分析
1. **静态字段累积**: `executionTime` 是静态字段，每次工作流重新注册时不会重置
2. **无限循环条件**: `data => data.ToString() != ""` 永远返回 true，导致 Recur 无限执行
3. **工作流实例泄漏**: 每次 Recur 都会创建新的工作流实例，但旧实例没有被释放
4. **错误处理不当**: `WorkflowErrorHandling.Retry` 会在失败时不断重试，累积更多实例

### 2. 后台任务未正确释放 ⚠️

从日志"判断条件是否停止=False"出现多次，说明:
- 后台任务在累积
- 任务完成后没有正确清理资源
- 可能存在定时器、事件订阅未注销

### 3. 缓存管理策略不当 ⚠️

```csharp
// Fireasy.Common.Caching.MemoryCacheManager
public int Capacity { get; set; } = 1000;  // ❌ 容量限制可能不够
```

问题:
- 容量限制为 1000，但如果缓存对象很大，会占用大量内存
- 过期缓存清理可能不够及时
- 没有内存压力下的主动清理机制

### 4. 缺少内存监控和 GC 策略 ⚠️

系统在内存达到 1559MB 时才开始警告，此时已经太晚:
- 缺少早期预警机制
- 没有主动触发 GC 的策略
- 没有内存阈值保护

## ✅ 修复方案

### 修复 1: 工作流重构 (已完成) ✅

**文件**: `RUINORERP.Server\Workflow\WFScheduled\DailyTaskWorkflow.cs`

#### 核心改进
1. **移除静态字段**: 使用工作流数据类传递状态
2. **正确的终止条件**: 使用 `ShouldContinue` 标志控制循环
3. **状态跟踪**: 记录执行次数和下次执行时间
4. **异常处理优化**: 使用 `WorkflowErrorHandling.Continue` 避免无限重试

```csharp
public class DailyTaskData
{
    public DateTime NextExecutionTime { get; set; }
    public int ExecutionCount { get; set; }
    public bool ShouldContinue { get; set; } = true;
}

public void Build(IWorkflowBuilder<DailyTaskData> builder)
{
    builder
        .StartWith(context =>
        {
            // 初始化执行时间 (每天上午 10:50)
            var executionTime = DateTime.Today.AddHours(10).AddMinutes(50);
            if (executionTime < DateTime.Now)
            {
                executionTime = executionTime.AddDays(1);
            }
            context.Workflow.Data.NextExecutionTime = executionTime;
            context.Workflow.Data.ExecutionCount = 0;
            return ExecutionResult.Next();
        })
        .Recur(data => (data.NextExecutionTime - DateTime.Now).TotalMilliseconds > 0 
                      ? (data.NextExecutionTime - DateTime.Now) 
                      : TimeSpan.FromMinutes(1), 
               data => data.ShouldContinue)  // ✅ 正确的终止条件
        .Do(recur => recur
            .StartWith<DailyTaskStep>()
            .Then(context =>
            {
                // 更新下次执行时间
                var executionTime = DateTime.Today.AddHours(10).AddMinutes(50);
                if (executionTime < DateTime.Now)
                {
                    executionTime = executionTime.AddDays(1);
                }
                context.Workflow.Data.NextExecutionTime = executionTime;
                context.Workflow.Data.ExecutionCount++;
                return ExecutionResult.Next();
            })
            .OnError(WorkflowErrorHandling.Continue))  // ✅ 避免无限重试
        .Then(context =>
        {
            frmMainNew.Instance.PrintInfoLog($"每日任务工作流结束，总执行次数：{context.Workflow.Data.ExecutionCount}");
            return ExecutionResult.Next();
        });
}
```

### 修复 4: 添加内存监控和主动 GC 📋

**建议创建**: `RUINORERP.Server\Monitoring\MemoryMonitorService.cs`

```csharp
/// <summary>
/// 内存监控服务
/// 监控系统内存使用，主动触发垃圾回收
/// </summary>
public class MemoryMonitorService : IDisposable
{
    private readonly ILogger<MemoryMonitorService> _logger;
    private readonly Timer _memoryCheckTimer;
    private readonly long _warningThresholdMB;
    private readonly long _criticalThresholdMB;

    public MemoryMonitorService(
        ILogger<MemoryMonitorService> logger,
        long warningThresholdMB = 1024,      // 1GB 警告
        long criticalThresholdMB = 2048)     // 2GB 严重
    {
        _logger = logger;
        _warningThresholdMB = warningThresholdMB;
        _criticalThresholdMB = criticalThresholdMB;

        // 每 30 秒检查一次内存
        _memoryCheckTimer = new Timer(CheckMemoryUsage, null, 
            TimeSpan.FromSeconds(30), TimeSpan.FromSeconds(30));
    }

    private void CheckMemoryUsage(object state)
    {
        try
        {
            var process = Process.GetCurrentProcess();
            var memoryMB = process.WorkingSet64 / (1024 * 1024);

            if (memoryMB >= _criticalThresholdMB)
            {
                _logger.LogError($"内存使用严重：当前使用 {memoryMB} MB，正在执行强制垃圾回收");
                ForceGarbageCollection();
            }
            else if (memoryMB >= _warningThresholdMB)
            {
                _logger.LogWarning($"内存使用警告：当前使用 {memoryMB} MB");
            }
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "检查内存使用失败");
        }
    }

    private void ForceGarbageCollection()
    {
        Task.Run(() =>
        {
            // 分代垃圾回收，从最高代开始
            GC.Collect(2, GCCollectionMode.Forced, blocking: true);
            GC.WaitForPendingFinalizers();
            GC.Collect(2, GCCollectionMode.Forced, blocking: true);
            
            _logger.LogInformation("强制垃圾回收完成");
        });
    }

    public void Dispose()
    {
        _memoryCheckTimer?.Dispose();
    }
}
```
