### M4. 异步方法中的同步阻塞 - 可能导致死锁

**位置**: `SessionService.cs` 第1662-1679行（Dispose方法）

```csharp
// ❌ 问题代码
Parallel.ForEach(sessionIds, new ParallelOptions { MaxDegreeOfParallelism = Environment.ProcessorCount }, sessionId =>
{
    if (_sessions.TryGetValue(sessionId, out var session))
    {
        try
        {
            var closeTask = session.CloseAsync(CloseReason.ServerShutdown);
            if (closeTask.AsTask().Wait(100))  // 同步等待异步操作
            {
                Interlocked.Increment(ref closedSessions);
            }
        }
        catch (Exception ex)
        {
            _logger.LogWarning(ex, $"关闭会话时出错: {sessionId}");
        }
    }
});
```

**影响**:
- `Task.Wait()`可能导致线程池饥饿
- 在ASP.NET等SynchronizationContext环境下可能死锁
- 超时时间100ms可能太短，导致大量会话未正常关闭

**修复建议**:
```csharp
public async Task DisposeAsync()
{
    if (_disposed) return;
    
    _disposed = true;
    _cleanupTimer?.Dispose();
    
    var sessionIds = _sessions.Keys.ToList();
    int totalSessions = sessionIds.Count;
    int closedSessions = 0;
    
    // 并行异步关闭
    var tasks = sessionIds.Select(async sessionId =>
    {
        if (_sessions.TryGetValue(sessionId, out var session))
        {
            try
            {
                await session.CloseAsync(CloseReason.ServerShutdown).ConfigureAwait(false);
                Interlocked.Increment(ref closedSessions);
            }
            catch (Exception ex)
            {
                _logger.LogWarning(ex, $"关闭会话时出错: {sessionId}");
            }
        }
    });
    
    await Task.WhenAll(tasks).ConfigureAwait(false);
    
    _sessions.Clear();
    
    _logger.LogInformation($"统一会话管理器资源已释放，共处理{totalSessions}个会话，成功关闭{closedSessions}个");
}
```

**优先级**: 🟡 P2 - 计划修复

### M5. 心跳间隔动态计算逻辑过于复杂

**位置**: `HeartbeatCommandHandler.cs` 第219-256行

```csharp
// ⚠️ 逻辑复杂，难以理解和维护
private int CalculateNextHeartbeatInterval(SessionInfo sessionInfo)
{
    const int defaultIntervalMs = 15000;
    const int minIntervalMs = 10000;
    const int maxIntervalMs = 60000;

    if (sessionInfo == null)
    {
        return defaultIntervalMs;
    }

    var timeSinceLastActivity = DateTime.Now - sessionInfo.LastActivityTime;
    var intervalMs = defaultIntervalMs;

    if (timeSinceLastActivity.TotalMinutes < 10)
    {
        intervalMs = minIntervalMs;  // 活跃用户用短间隔
    }
    else if (timeSinceLastActivity.TotalMinutes < 60)
    {
        intervalMs = defaultIntervalMs;
    }
    else
    {
        intervalMs = maxIntervalMs;  // 不活跃用户用长间隔
    }

    return intervalMs;
}
```

**问题**:
- 对于活跃用户反而使用更短的心跳间隔（10秒），增加了服务器负载
- 通常应该是不活跃用户才需要更频繁的心跳来检测断线
- 逻辑与注释相反

**修复建议**:
```csharp
private int CalculateNextHeartbeatInterval(SessionInfo sessionInfo)
{
    const int activeIntervalMs = 30000;     // 活跃用户30秒
    const int idleIntervalMs = 15000;       // 空闲用户15秒
    const int longIdleIntervalMs = 60000;   // 长时间空闲60秒

    if (sessionInfo == null)
    {
        return activeIntervalMs;
    }

    var timeSinceLastActivity = DateTime.Now - sessionInfo.LastActivityTime;

    if (timeSinceLastActivity.TotalMinutes < 5)
    {
        return activeIntervalMs;      // 5分钟内活跃，降低心跳频率
    }
    else if (timeSinceLastActivity.TotalMinutes < 30)
    {
        return idleIntervalMs;        // 30分钟内空闲，适中频率
    }
    else
    {
        return longIdleIntervalMs;    // 长时间空闲，进一步降低频率
    }
}
```
