## 🚀 三步走优化法

### 第1步：识别高风险代码

**检查清单**:
- [ ] 方法内有 `BeginTran()` / `CommitTran()` / `RollbackTran()`
- [ ] 有批量更新操作（循环中的 Update/Insert/Delete）
- [ ] 涉及库存、订单、财务核销等核心业务
- [ ] 事务内有外部 API 调用或复杂计算

### 第2步：应用优化模式

#### 模式A：资源访问排序（必做）
```csharp
// ❌ 错误：无序更新
foreach (var item in items)
{
    await UpdateRecord(item);
}

// ✅ 正确：按 ID 排序
var sortedItems = items.OrderBy(i => i.Id).ToList();
foreach (var item in sortedItems)
{
    await UpdateRecord(item);
}
```

**关键点**:
- ✅ 必须在事务内排序
- ✅ 排序字段必须是唯一且稳定的（如主键 ID）
- ✅ 适用于所有批量更新场景

#### 模式B：缩小事务范围
```csharp
// ❌ 错误：长事务
_unitOfWorkManage.BeginTran();
var externalData = await CallExternalApi(); // 耗时
await UpdateDatabase(data);
_unitOfWorkManage.CommitTran();

// ✅ 正确：短事务
var externalData = await CallExternalApi(); // 事务外
_unitOfWorkManage.BeginTran();
await UpdateDatabase(data); // 仅数据库操作
_unitOfWorkManage.CommitTran();
```

#### 模式C：批量预加载
```csharp
// ✅ 事务外一次性加载
var requiredIds = items.Select(i => i.Id).Distinct().ToList();
var dataList = await _db.Queryable<Entity>()
    .Where(e => requiredIds.Contains(e.Id))
    .ToListAsync();
var dataDict = dataList.ToDictionary(e => e.Id);

// ✅ 事务内直接使用字典
_unitOfWorkManage.BeginTran();
foreach (var item in items)
{
    var entity = dataDict[item.Id];
    // 更新操作...
}
_unitOfWorkManage.CommitTran();
```

### 第3步：添加死锁检测

```csharp
catch (Exception ex)
{
    bool isDeadlock = IsDeadlockException(ex);
    
    if (isDeadlock)
    {
        _logger.LogWarning($"检测到死锁 - 单号: {entity?.No}, 异常: {ex.Message}");
        
        TransactionMetrics.RecordDeadlock(
            "表名", 
            "操作名", 
            TimeSpan.FromSeconds(0), 
            ex.Message,
            entity?.No);
    }
    
    _unitOfWorkManage.RollbackTran();
    throw;
}

// 辅助方法（添加到类末尾）
private bool IsDeadlockException(Exception ex)
{
    if (ex == null) return false;
    
    string message = ex.Message.ToLower();
    return message.Contains("deadlock") || 
           message.Contains("1205") || 
           message.Contains("lock") ||
           message.Contains("timeout");
}
```

## 🔧 重试机制使用

### 方式1：DeadlockRetryHelper
```csharp
public async Task<ReturnResults<T>> ApprovalAsync(T entity)
{
    return await DeadlockRetryHelper.ExecuteWithDeadlockRetry(
        async () => await ApprovalAsyncInternal(entity),
        _logger,
        $"审核操作 - {entity.Id}"
    );
}
```

### 方式2：UnitOfWorkManage 内置
```csharp
// 同步
_unitOfWorkManage.ExecuteWithRetry(() => {
    // 业务逻辑...
}, maxRetryCount: 3);

// 异步
await _unitOfWorkManage.ExecuteWithRetryAsync(async () => {
    // 异步业务逻辑...
}, maxRetryCount: 3);
```

## ⚠️ 常见陷阱

### ❌ 陷阱1：忘记排序
```csharp
// 错误：直接遍历 Dictionary
foreach (var group in inventoryGroups)
{
    // 更新库存...
}

// 正确：先转换为有序列表
var sortedGroups = inventoryGroups
    .OrderBy(g => g.Key.ProdDetailID)
    .ThenBy(g => g.Key.LocationID)
    .ToList();

foreach (var group in sortedGroups)
{
    // 更新库存...
}
```

### ❌ 陷阱2：事务范围过大
```csharp
// 错误：包含非数据库操作
_unitOfWorkManage.BeginTran();
var result = await CallExternalApi(); // 外部调用
var calculated = PerformComplexCalculation(result); // 复杂计算
await UpdateDatabase(calculated);
_unitOfWorkManage.CommitTran();

// 正确：仅包裹数据库操作
var result = await CallExternalApi(); // 事务外
var calculated = PerformComplexCalculation(result); // 事务外
_unitOfWorkManage.BeginTran();
await UpdateDatabase(calculated); // 仅数据库操作
_unitOfWorkManage.CommitTran();
```

### ❌ 陷阱3：N+1 查询问题
```csharp
// 错误：循环中查询数据库
foreach (var item in items)
{
    var entity = await _db.Queryable<Entity>()
        .Where(e => e.Id == item.Id)
        .FirstAsync(); // N 次查询
    // 更新操作...
}

// 正确：批量预加载
var ids = items.Select(i => i.Id).Distinct().ToList();
var entities = await _db.Queryable<Entity>()
    .Where(e => ids.Contains(e.Id))
    .ToListAsync(); // 1 次查询
var entityDict = entities.ToDictionary(e => e.Id);

foreach (var item in items)
{
    var entity = entityDict[item.Id];
    // 更新操作...
}
```

## 📊 监控与告警

### 查看死锁统计
```csharp
// 获取全局统计
var stats = TransactionMetrics.GetStatistics();
Console.WriteLine($"总死锁次数: {stats.DeadlockCount}");

// 获取按表统计
var tableStats = TransactionMetrics.GetTableStatistics();
foreach (var stat in tableStats)
{
    Console.WriteLine($"表 {stat.TableName}: 死锁 {stat.DeadlockCount} 次");
}

// 获取最近死锁
var recentDeadlocks = TransactionMetrics.GetRecentDeadlocks(10);
foreach (var deadlock in recentDeadlocks)
{
    Console.WriteLine($"[{deadlock.Timestamp}] {deadlock.TableName}.{deadlock.Operation}");
}
```

### 设置告警阈值
```csharp
// 在定时任务中检查
var stats = TransactionMetrics.GetStatistics();
if (stats.DeadlockCount > 10) // 1小时内超过10次
{
    SendAlert($"死锁告警：1小时内发生 {stats.DeadlockCount} 次死锁");
}
```

## 💡 记忆口诀

> **一排二小三预加，四检五记六重试**

1. **一排**: 批量更新必排序
2. **二小**: 事务范围要最小化
3. **三预加**: 批量预加载减少查询
4. **四检**: 检测死锁异常
5. **五记**: 记录死锁日志
6. **六重试**: 使用自动重试机制
