# 状态保护机制推广示例 - 审核方法改造

## 概述

本文档展示如何将 EntityStateProtector 应用到审核（Approval）、提交（Submit）等业务场景中。

## 示例1：销售订单审核方法改造

### 改造前（假设的现有实现）

```csharp
public override async Task<ReturnResults<tb_SaleOrder>> ApprovalAsync(tb_SaleOrder entity)
{
    var result = new ReturnResults<tb_SaleOrder>();
    
    _unitOfWorkManage.BeginTran();  // ❌ 没有状态保护
    
    try
    {
        // 业务校验
        if (entity.DataStatus != (int)DataStatus.确认)
        {
            result.ErrorMsg = "只有确认状态的订单才能审核";
            return result;
        }
        
        // 修改状态
        entity.ApprovalStatus = (int)ApprovalStatus.审核通过;
        entity.ApprovalTime = DateTime.Now;
        entity.ApprovalUser = CurrentUser.UserId;
        
        // 复杂业务逻辑...
        await ProcessPrePayment(entity);
        await UpdateInventory(entity);
        
        _unitOfWorkManage.CommitTran();
        
        result.Succeeded = true;
        result.ReturnObject = entity;
    }
    catch (Exception ex)
    {
        _unitOfWorkManage.RollbackTran();  // ❌ 数据库回滚了，但内存实体状态未恢复
        result.ErrorMsg = $"审核失败: {ex.Message}";
    }
    
    return result;
}
```

**问题**：
- 如果 `ProcessPrePayment` 或 `UpdateInventory` 抛出异常
- 数据库事务回滚，数据恢复到审核前
- 但内存中的 `entity.ApprovalStatus` 仍然是"审核通过"
- UI 显示的状态与数据库不一致

### 改造后（使用完整保护）

```csharp
public override async Task<ReturnResults<tb_SaleOrder>> ApprovalAsync(tb_SaleOrder entity)
{
    var result = new ReturnResults<tb_SaleOrder>();
    
    // ✅ 第1步：开启事务并保护实体状态
    BeginTranWithProtection(entity as BaseEntity);
    
    try
    {
        // 业务校验
        if (entity.DataStatus != (int)DataStatus.确认)
        {
            result.ErrorMsg = "只有确认状态的订单才能审核";
            return result;
        }
        
        // 修改状态
        entity.ApprovalStatus = (int)ApprovalStatus.审核通过;
        entity.ApprovalTime = DateTime.Now;
        entity.ApprovalUser = CurrentUser.UserId;
        
        // 复杂业务逻辑...
        await ProcessPrePayment(entity);
        await UpdateInventory(entity);
        
        // ✅ 第2步：提交事务并清理快照
        CommitTranWithCleanup();
        
        result.Succeeded = true;
        result.ReturnObject = entity;
    }
    catch (Exception ex)
    {
        // ✅ 第3步：回滚事务并恢复实体状态
        RollbackTranWithRestore();
        result.ErrorMsg = $"审核失败: {ex.Message}";
    }
    
    return result;
}
```

**改进**：
- ✅ 事务回滚时，`entity.ApprovalStatus` 自动恢复到"待审核"
- ✅ 内存状态与数据库保持一致
- ✅ 只需改动3行代码

### 改造后（使用选择性保护 - 性能优化版）

如果 `tb_SaleOrder` 是一个大型实体（包含大量导航属性和集合），可以使用选择性保护：

```csharp
public override async Task<ReturnResults<tb_SaleOrder>> ApprovalAsync(tb_SaleOrder entity)
{
    var result = new ReturnResults<tb_SaleOrder>();
    
    // ✅ 只保护会被修改的状态字段，性能更优
    BeginTranWithSelectiveProtection(entity,
        e => e.ApprovalStatus,      // 审核状态
        e => e.ApprovalTime,        // 审核时间
        e => e.ApprovalUser,        // 审核人
        e => e.ApprovalComments     // 审核意见
    );
    
    try
    {
        // 业务校验
        if (entity.DataStatus != (int)DataStatus.确认)
        {
            result.ErrorMsg = "只有确认状态的订单才能审核";
            return result;
        }
        
        // 修改状态
        entity.ApprovalStatus = (int)ApprovalStatus.审核通过;
        entity.ApprovalTime = DateTime.Now;
        entity.ApprovalUser = CurrentUser.UserId;
        
        // 复杂业务逻辑...
        await ProcessPrePayment(entity);
        await UpdateInventory(entity);
        
        CommitTranWithCleanup();
        
        result.Succeeded = true;
        result.ReturnObject = entity;
    }
    catch (Exception ex)
    {
        RollbackTranWithRestore();
        result.ErrorMsg = $"审核失败: {ex.Message}";
    }
    
    return result;
}
```

**性能优势**：
- 完整克隆：~10ms（100个属性）
- 选择性克隆（4个字段）：~0.5ms
- **性能提升 95%**

```csharp
    _unitOfWorkManage.BeginTran();
    
    try
    {
        foreach (var entity in entities)

            // 业务逻辑...

        result.Succeeded = true;
    }
    catch (Exception ex)
    {
        _unitOfWorkManage.RollbackTran();  // ❌ 所有实体的状态都未恢复

    }
    
    return result;
}
```

### 改造后

```csharp
public async Task<ReturnResults<bool>> BatchApprovalAsync(List<tb_SaleOrder> entities)
{
    var result = new ReturnResults<bool>();
    
    // ✅ 批量保护所有实体
    BeginTranWithProtection(entities.Cast<BaseEntity>().ToArray());
    
    try
    {
        foreach (var entity in entities)
        {
            entity.ApprovalStatus = (int)ApprovalStatus.审核通过;
            entity.ApprovalTime = DateTime.Now;
            
            // 业务逻辑...
        }
        
        CommitTranWithCleanup();
        result.Succeeded = true;
    }
    catch (Exception ex)
    {
        RollbackTranWithRestore();  // ✅ 所有实体状态都恢复
        result.ErrorMsg = ex.Message;
    }
    
    return result;
}
```

## 示例3：提交方法改造

### 典型场景

提交操作通常会修改 `DataStatus` 从"草稿"到"确认"：

```csharp
public override async Task<ReturnResults<tb_PurOrder>> SubmitAsync(tb_PurOrder entity)
{
    var result = new ReturnResults<tb_PurOrder>();
    
    // ✅ 选择性保护：只保护状态字段
    BeginTranWithSelectiveProtection(entity,
        e => e.DataStatus,
        e => e.SubmitTime,
        e => e.SubmitterId
    );
    
    try
    {
        // 数据完整性校验
        if (!ValidateRequiredFields(entity))
        {
            result.ErrorMsg = "请填写必填字段";
            return result;
        }
        
        // 修改状态
        entity.DataStatus = (int)DataStatus.确认;
        entity.SubmitTime = DateTime.Now;
        entity.SubmitterId = CurrentUser.UserId;
        
        // 业务逻辑...
        await ValidateBudget(entity);
        
        CommitTranWithCleanup();
        result.Succeeded = true;
    }
    catch (Exception ex)
    {
        RollbackTranWithRestore();
        result.ErrorMsg = $"提交失败: {ex.Message}";
    }
    
    return result;
}
```

## 示例4：执行/反执行方法改造

### 执行方法

```csharp
public override async Task<ReturnResults<tb_StockOut>> ConfirmExecutionAsync(tb_StockOut entity)
{
    var result = new ReturnResults<tb_StockOut>();
    
    // ✅ 保护执行相关字段
    BeginTranWithSelectiveProtection(entity,
        e => e.ExecutionStatus,
        e => e.ExecutionTime,
        e => e.ExecutorId
    );
    
    try
    {
        // 库存扣减
        await DeductInventory(entity);
        
        // 修改状态
        entity.ExecutionStatus = (int)ExecutionStatus.已执行;
        entity.ExecutionTime = DateTime.Now;
        entity.ExecutorId = CurrentUser.UserId;
        
        CommitTranWithCleanup();
        result.Succeeded = true;
    }
    catch (Exception ex)
    {
        RollbackTranWithRestore();  // ✅ 库存和状态都恢复
        result.ErrorMsg = $"执行失败: {ex.Message}";
    }
    
    return result;
}
```

### 反执行方法

```csharp
public override async Task<ReturnResults<tb_StockOut>> AntiConfirmExecutionAsync(tb_StockOut entity)
{
    var result = new ReturnResults<tb_StockOut>();
    
    BeginTranWithSelectiveProtection(entity,
        e => e.ExecutionStatus,
        e => e.ExecutionTime,
        e => e.ExecutorId
    );
    
    try
    {
        // 库存回滚
        await RollbackInventory(entity);
        
        // 回退状态
        entity.ExecutionStatus = (int)ExecutionStatus.未执行;
        entity.ExecutionTime = null;
        entity.ExecutorId = null;
        
        CommitTranWithCleanup();
        result.Succeeded = true;
    }
    catch (Exception ex)
    {
        RollbackTranWithRestore();
        result.ErrorMsg = $"反执行失败: {ex.Message}";
    }
    
    return result;
}
```

## UI层适配

无论使用哪种保护方式，UI 层的错误处理都是一样的：

```csharp
protected override async Task<bool> ApprovalAsync()
{
    if (EditEntity == null)
        return false;
    
    // 调用 Controller
    var result = await Controller.ApprovalAsync(EditEntity);
    
    if (result.Succeeded)
    {
        EditEntity.AcceptChanges();
        Refreshs();  // 重新加载数据
        return true;
    }
    else
    {
        // ✅ 显示错误
        KryptonMessageBox.Show(
            $"审核失败!\n\n{result.ErrorMsg}",
            "错误",
            MessageBoxButtons.OK,
            MessageBoxIcon.Error
        );
        
        // ✅ 关键：刷新UI状态，显示恢复后的实体状态
        UpdateAllUIStates(EditEntity);
        
        return false;  // 让基类恢复按钮状态
    }
}
```

## 推广检查清单

在将状态保护机制应用到新的业务方法时，请检查：

### Controller 层
- [ ] 使用 `BeginTranWithProtection()` 或 `BeginTranWithSelectiveProtection()` 开启事务
- [ ] 使用 `CommitTranWithCleanup()` 提交事务
- [ ] 在 catch 块中使用 `RollbackTranWithRestore()` 回滚
- [ ] 不要在同一个方法中多次调用保护方法

### UI 层
- [ ] 失败分支调用 `UpdateAllUIStates(EditEntity)`
- [ ] 返回 `false` 让基类恢复按钮状态
- [ ] 显示明确的错误消息

### 性能考虑
- [ ] 小型实体（<50属性）：使用完整保护
- [ ] 大型实体（>100属性）：使用选择性保护
- [ ] 批量操作：监控内存占用
