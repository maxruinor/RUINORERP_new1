# 用户静止时间计算与上报逻辑修复说明

## 问题背景

在 UserManagementControl 中显示的"静止时间"旨在反映用户未操作鼠标或键盘的时长。该数据由客户端计算并通过心跳包发送至服务器，服务器端处理后更新会话状态。

## 发现的问题

### 1. 客户端问题（已修复）

**位置**: `RUINORERP.UI/MainForm.cs`

**问题描述**:
- 在 `timer1_Tick` 定时器中持续更新 `AppContext.CurUserInfo.静止时间`
- 这个定时更新的值可能不是最新的，因为心跳包发送时会重新获取

**修复方案**:
```csharp
// 注释掉定时器中的更新，避免使用过时的缓存值
// MainForm.Instance.AppContext.CurUserInfo.静止时间 = GetLastInputTime();
```

**理由**: 
- 静止时间应该在发送心跳包时实时获取（已在 ClientCommunicationService 中实现）
- 避免多处更新导致的数据不一致

### 2. 服务器端问题（已修复）

**位置**: `RUINORERP.Server/Controls/UserManagementControl.cs`

**问题描述**:
- 在 `UpdateVisibleSessionsHeartbeatAndIdleTime()` 方法中，错误地使用 `DateTime.Now - sessionInfo.LastHeartbeat` 计算空闲时间
- 这完全忽略了客户端上报的真实静止时间
- 导致显示的是"距离上次心跳的时间"而不是"用户静止时间"

**关键区别**:
- **用户静止时间**: 用户最后一次鼠标/键盘活动至今的时间（客户端通过 Windows API 获取）
- **心跳间隔**: 距离上次收到心跳包的时间（服务器端计算）
- 两者完全不同！用户可能在活跃操作，但心跳包可能因网络延迟而晚到

**修复方案**:
```csharp
// 优先使用客户端上报的静止时间
TimeSpan idleTimeSpan;
if (userInfo != null && userInfo.静止时间 > 0)
{
    // 使用客户端上报的静止时间（秒）
    idleTimeSpan = TimeSpan.FromSeconds(userInfo.静止时间);
}
else
{
    // 回退方案：使用服务器端最后心跳时间计算
    idleTimeSpan = DateTime.Now - sessionInfo.LastHeartbeat;
}
```

## 完整数据流

### 1. 客户端数据采集

**文件**: `RUINORERP.UI/MainForm.cs`

```csharp
/// <summary>
/// 取得最后一次输入时间（秒）
/// 使用Windows API GetLastInputInfo获取用户最后一次鼠标或键盘活动至今的时间
/// </summary>
public static long GetLastInputTime()
{
    try
    {
        LASTINPUTINFO vLastInputInfo = new LASTINPUTINFO();
        vLastInputInfo.cbSize = Marshal.SizeOf(vLastInputInfo);

        if (!GetLastInputInfo(ref vLastInputInfo))
        {
            return 0; // API调用失败
        }

        // 计算从最后一次输入到现在的时间差（毫秒）
        long timeInMilliseconds = Environment.TickCount - (long)vLastInputInfo.dwTime;
        
        // 转换为秒
        long timeInSeconds = timeInMilliseconds / 1000;

        // 处理可能的溢出或负数情况
        if (timeInSeconds < 0)
        {
            timeInSeconds = 0;
        }

        FreeTimeExecuteCnce = timeInSeconds > 0;
        return timeInSeconds;
    }
    catch (Exception ex)
    {
        System.Diagnostics.Debug.WriteLine($"GetLastInputTime异常: {ex.Message}");
        return 0;
    }
}
```

**技术要点**:
- 使用 Windows API `GetLastInputInfo` 获取系统级别的最后输入时间
- 返回值为秒，便于传输和显示
- 包含异常处理和溢出保护
- 线程安全（静态方法，无状态）

## 技术细节

### Windows API GetLastInputInfo

```c
typedef struct tagLASTINPUTINFO {
    UINT cbSize;      // 结构体大小
    DWORD dwTime;     // 最后一次输入的时间戳（毫秒）
} LASTINPUTINFO;
```

**工作原理**:
- 系统维护一个全局的最后输入时间戳
- 任何鼠标移动、点击、键盘按键都会更新这个时间戳
- `GetLastInputInfo` 返回这个时间戳
- 通过 `Environment.TickCount - dwTime` 计算时间差

**优势**:
- 系统级别监控，无需安装钩子
- 性能开销极小
- 覆盖所有输入设备（鼠标、键盘、触摸屏等）
- 线程安全

**注意事项**:
- `Environment.TickCount` 约49.7天会溢出（已处理）
- 需要管理员权限才能准确获取（通常不需要）
- 只在当前会话中有效
