### 2.2 任务2：配置同步 ⚠️ 需要优化

#### 代码位置
`FrmLogin.cs` 第1373-1394行

#### 当前实现
```csharp
_ = Task.Run(async () =>
{
    try
    {
        if (_configSyncService != null)
        {
            MainForm.Instance.PrintInfoLog("正在请求最新配置文件...");
            bool configSyncSuccess = await _configSyncService.RequestCommonConfigsAsync();
            if (configSyncSuccess)
            {
                MainForm.Instance.PrintInfoLog("配置文件请求发送成功，等待服务器响应");
            }

            // 初始化性能监控模块
            InitializePerformanceMonitoring();
        }
    }
    catch (Exception ex)
    {
        MainForm.Instance.logger?.LogError(ex, "请求最新配置文件时发生异常");
    }
});
```

#### 优点 ✅
1. ✅ 有空值检查 (`_configSyncService != null`)
2. ✅ 有异常处理
3. ✅ 包含性能监控初始化

#### 问题 ⚠️

**P1 - 缺少超时控制（严重）**
```csharp
// ❌ 问题：RequestCommonConfigsAsync内部有30秒超时
// 但如果网络卡住，仍可能阻塞很久
bool configSyncSuccess = await _configSyncService.RequestCommonConfigsAsync();
```

**分析底层实现**：
```csharp
// ConfigSyncService.RequestLatestConfigAsync 第91-95行
var response = await _communicationService.SendCommandWithResponseAsync<GeneralResponse>(
    GeneralCommands.ConfigSync,
    request,
    ct,  // ← 这里接收CancellationToken
    30000); // ← 30秒超时
```

**问题**：
- 虽然有30秒超时，但在Task.Run中没有传递CancellationToken
- 如果连续多次登录，可能累积多个未完成的请求

**建议修复**：
```csharp
_ = Task.Run(async () =>
{
    using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(30));
    try
    {
        if (_configSyncService != null)
        {
            _logger?.LogInformation("正在请求最新配置文件...");
            bool configSyncSuccess = await _configSyncService.RequestCommonConfigsAsync(
                forceRefresh: false, 
                ct: cts.Token);  // ✅ 传递CancellationToken
            
            if (configSyncSuccess)
            {
                _logger?.LogInformation("配置文件同步成功");
            }
            else
            {
                _logger?.LogWarning("配置文件同步失败，将使用本地缓存");
            }

            // 初始化性能监控模块
            InitializePerformanceMonitoring();
        }
    }
    catch (OperationCanceledException)
    {
        _logger?.LogWarning("配置文件同步超时或已取消");
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, "请求最新配置文件时发生异常");
    }
});
```

**P2 - 性能监控初始化位置不当**
```csharp
// ❌ 问题：性能监控初始化放在配置同步的try块内
// 如果配置同步失败，性能监控也不会初始化
InitializePerformanceMonitoring();
```

**建议**：
```csharp
// ✅ 性能监控应该独立初始化，不依赖配置同步
_ = Task.Run(async () =>
{
    try
    {
        // 配置同步
        if (_configSyncService != null)
        {
            // ...
        }
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, "配置同步失败");
    }
    finally
    {
        // ✅ 无论配置同步是否成功，都初始化性能监控
        InitializePerformanceMonitoring();
    }
});
```

**P3 - 日志信息不准确**
```csharp
// ❌ "配置文件请求发送成功，等待服务器响应"
// 实际上 SendCommandWithResponseAsync 是同步等待响应的
// 到这里已经收到响应了
MainForm.Instance.PrintInfoLog("配置文件请求发送成功，等待服务器响应");
```

**建议改为**：
```csharp
_logger?.LogInformation("配置文件同步成功");
```

## 三、共性问题总结

### 3.1 缺少统一的超时控制 ❌

**问题**：三个后台任务都没有明确的超时控制

**影响**：
- 如果服务器无响应，任务可能无限等待
- 多次登录后可能累积大量未完成的任务
- 占用线程池资源

**解决方案**：为每个任务添加CancellationToken和超时时间

```csharp
// 统一模式
_ = Task.Run(async () =>
{
    using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(超时时间));
    try
    {
        // 执行操作，传递cts.Token
    }
    catch (OperationCanceledException)
    {
        _logger?.LogWarning("操作超时或已取消");
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, "操作失败");
    }
});
```

### 3.2 日志级别不一致 ⚠️

**问题**：
- 有时使用 `MainForm.Instance.PrintInfoLog()`
- 有时使用 `MainForm.Instance.logger?.LogInformation()`
- 有时使用 `MainForm.Instance.logger?.LogDebug()`

**建议统一**：
```csharp
// 推荐使用 logger，更灵活
private static readonly ILogger _logger = LogManager.GetLogger(typeof(FrmLogin));

// 或者注入ILogger
public FrmLogin(ILogger<FrmLogin> logger)
{
    _logger = logger;
}
```

### 3.3 缺少任务跟踪机制 ⚠️

**问题**：启动后台任务后，无法跟踪任务状态

**建议**：添加简单的任务跟踪
```csharp
// 可选：记录任务ID便于调试
var taskId = Guid.NewGuid();
_logger?.LogDebug("启动后台任务: {TaskId}, 类型: CacheSync", taskId);

_ = Task.Run(async () =>
{
    try
    {
        // ...
        _logger?.LogDebug("后台任务完成: {TaskId}", taskId);
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, "后台任务失败: {TaskId}", taskId);
    }
});
```

_ = Task.Run(async () =>

    using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(30));

    {

            if (configSyncSuccess)

                _logger?.LogInformation("配置文件同步成功");

            else
            {

            }
        }
    }
    catch (OperationCanceledException)
    {
        _logger?.LogWarning("配置同步超时或已取消");
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, "配置同步失败");
    }
    finally
    {
        // ✅ 无论配置同步是否成功，都初始化性能监控
        InitializePerformanceMonitoring();
    }

// 3. 锁状态获取（后台执行）
_ = Task.Run(async () =>
{
    using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(15));
    try
    {
        var lockManagementService = Startup.GetFromFac<ClientLockManagementService>();
        if (lockManagementService != null)
        {
            _logger?.LogDebug("开始获取锁状态列表...");
            
            // 注意：需要修改 GetLockStatusListAsync 方法签名，添加CancellationToken参数

            if (lockResponse != null && lockResponse.IsSuccess)

            }
            else
            {
                _logger?.LogWarning("获取锁状态列表失败: {ErrorMessage}",
                    lockResponse?.Message ?? "未知错误");
            }
        }
    }
    catch (OperationCanceledException)
    {
        _logger?.LogWarning("获取锁状态列表超时");
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, "获取锁状态列表时发生异常");
