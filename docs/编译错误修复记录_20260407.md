### 错误1: CS4033 - await在非异步方法中使用

**错误信息**: 
```
"await" 运算符只能用于异步方法中。请考虑用 "async" 修饰符标记此方法，并将其返回类型更改为 "Task"。
```

**位置**: 第592行

**原因**: `StopHeartbeat()`是同步方法（void返回），但内部使用了`await`关键字

**修复方案**: 
将异步等待逻辑移到锁外，并使用`.Result`进行同步等待

```csharp
// 修复前
private void StopHeartbeat()
{
    lock (_heartbeatLock)
    {
        _isHeartbeatRunning = false;
        _heartbeatCts?.Cancel();
        await SafeWaitForHeartbeatTaskCompletionAsync(); // ❌ 不能在同步方法中使用await
    }
}

// 修复后
private void StopHeartbeat()
{
    Task heartbeatTaskToWait = null;
    
    lock (_heartbeatLock)
    {
        if (!_isHeartbeatRunning) return;
        
        _isHeartbeatRunning = false;
        _heartbeatCts?.Cancel();
        
        // 获取任务引用并清空
        heartbeatTaskToWait = _heartbeatTask;
        _heartbeatTask = null;
    }

    // 在锁外同步等待任务完成
    if (heartbeatTaskToWait != null && !heartbeatTaskToWait.IsCompleted)
    {
        try
        {
            var timeoutTask = Task.Delay(2000);
            var completedTask = Task.WhenAny(heartbeatTaskToWait, timeoutTask).Result; // ✅ 使用.Result同步等待
            
            if (completedTask == timeoutTask)
            {
                _logger?.LogWarning("心跳任务停止超时，强制终止");
            }
        }
        catch (AggregateException ae)
        {
            foreach (var ex in ae.InnerExceptions)
            {
                if (!(ex is OperationCanceledException))
                {
                    _logger?.LogDebug(ex, "心跳任务异常结束");
                }
            }
        }
        catch (Exception ex)
        {
            _logger?.LogDebug(ex, "等待心跳任务完成时发生异常");
        }
    }
}
```

**技术要点**:
1. **锁内只做最少操作**：只取消任务和获取引用，不执行耗时操作
2. **锁外等待**：避免在lock语句中等待，防止死锁
3. **同步等待**：使用`.Result`在同步方法中等待Task完成
4. **超时保护**：设置2秒超时，避免无限期等待

### 错误2: CS1996 - lock语句中不能使用await

**错误信息**: 
```
无法在 lock 语句体中等待
```

**位置**: 第592行（与错误1相同）

**原因**: C#语言规范禁止在`lock`语句块中使用`await`，因为：
- `await`会释放当前线程，可能导致死锁
- `lock`需要保证同一线程持有锁，但`await`后会恢复在不同线程上

**修复**: 已在错误1中一并修复

**核心原则**:
```
❌ 错误做法
lock (obj)
{
    await SomeAsyncMethod(); // 编译错误！
}

✅ 正确做法
Task taskToWait;
lock (obj)
{
    taskToWait = GetTask(); // 只获取任务引用
}
await taskToWait; // 在锁外等待
```

### 错误4: CS1503 - SocketErrorCode类型不匹配

**错误信息**: 
```
参数 1: 无法从"System.Net.Sockets.SocketError"转换为"System.Net.Sockets.SocketErrorCode"
```

**位置**: 第1193行

**原因**: `SocketException.ErrorCode`返回的是`int`类型（SocketError枚举），而不是`SocketErrorCode`枚举

**修复**:
```csharp
// 修复前
ex is System.Net.Sockets.SocketException socketEx && IsNetworkRelatedSocketError(socketEx.SocketErrorCode)

// 修复后
ex is System.Net.Sockets.SocketException socketEx && IsNetworkRelatedSocketError((System.Net.Sockets.SocketError)socketEx.ErrorCode)
```

**说明**: 
- .NET Framework中`SocketException`没有`SocketErrorCode`属性
- 应该使用`ErrorCode`属性并强制转换为`SocketError`枚举

### 错误5 & 6: CS1501 - Contains方法签名错误

**错误信息**: 
```
"Contains"方法没有采用 2 个参数的重载
```

**位置**: 第1201-1202行

**原因**: `string.Contains(string, StringComparison)`是.NET Core/.NET 5+的方法，.NET Framework不支持

**修复**:
```csharp
// 修复前
ex.Message.Contains("connection", StringComparison.OrdinalIgnoreCase)
ex.Message.Contains("timeout", StringComparison.OrdinalIgnoreCase)

// 修复后
ex.Message.IndexOf("connection", StringComparison.OrdinalIgnoreCase) >= 0
ex.Message.IndexOf("timeout", StringComparison.OrdinalIgnoreCase) >= 0
```

**说明**: 
- `IndexOf`方法在所有.NET版本中都支持`StringComparison`参数
- 通过检查返回值是否>=0来实现相同的功能

## 技术要点总结

### 1. .NET Framework vs .NET Core/5+ API差异

| API | .NET Framework | .NET Core/5+ |
|-----|---------------|--------------|
| 字符串包含检查 | `IndexOf(str, comparison) >= 0` | `Contains(str, comparison)` |
| Socket异常错误码 | `SocketException.ErrorCode` (int) | `SocketException.SocketErrorCode` (枚举) |
| Socket错误枚举 | `SocketError` | `SocketError` / `SocketErrorCode` |

### 2. 异步方法命名规范

- 异步方法应以`Async`后缀结尾
- 调用异步方法必须使用`await`关键字
- 避免在同步方法中调用异步方法（会导致死锁）

### 3. 类型转换最佳实践

- 使用显式类型转换时，确保源类型和目标类型兼容
- 对于枚举转换，使用强制转换`(EnumType)value`
- 优先使用框架提供的API，避免手动转换
