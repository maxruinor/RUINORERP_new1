# 自动超时功能 - 快速参考卡

## 🚀 快速开始

### 1. 基本用法(推荐)

```csharp
// ✅ 最佳实践: using + 超时
using var unitOfWork = serviceProvider.GetRequiredService<IUnitOfWorkManage>();

await unitOfWork.BeginTranAsync(timeoutSeconds: 60);
try
{
    await DoWork();
    await unitOfWork.CommitTranAsync();
}
catch
{
    await unitOfWork.RollbackTranAsync();
    throw;
}
```

### 2. 自定义超时

```csharp
// 短事务: 30秒
await _unitOfWork.BeginTranAsync(timeoutSeconds: 30);

// 长事务: 5分钟
await _unitOfWork.BeginTranAsync(timeoutSeconds: 300);

// 使用默认值(60秒)
await _unitOfWork.BeginTranAsync();
```

## ⚙️ 配置

### appsettings.json

```json
{
  "UnitOfWork": {
    "EnableAutoTransactionTimeout": true,      // 启用超时
    "ForceRollbackOnTimeout": true,             // 超时后强制回滚
    "DefaultTransactionTimeoutSeconds": 60,     // 默认60秒
    "LongTransactionWarningSeconds": 60,        // 60秒警告
    "CriticalTransactionWarningSeconds": 300    // 300秒严重警告
  }
}
```

## ⚠️ 注意事项

### ❌ 不要这样做

```csharp
// 错误: 依赖超时作为正常流程
await _unitOfWork.BeginTranAsync(timeoutSeconds: 5);
await LongRunningWork();  // 期望6秒后自动回滚
```

### ✅ 应该这样做

```csharp
// 正确: 主动管理事务
await _unitOfWork.BeginTranAsync(timeoutSeconds: 60);
try
{
    await LongRunningWork();
    await _unitOfWork.CommitTranAsync();  // 明确提交
}
catch
{
    await _unitOfWork.RollbackTranAsync();  // 明确回滚
    throw;
}
```

## 🔍 故障排查

### 问题1: 看到"事务超时"日志

**原因**: 事务执行超过配置的超时时间

**解决**:
1. 检查业务逻辑是否耗时过长
2. 优化SQL查询
3. 如确实需要更长时间,增加 timeoutSeconds

### 问题2: 看到"Dispose时发现长事务"日志

**原因**: 忘记调用 Commit/Rollback

**解决**:
1. 检查代码是否有 try-catch-finally
2. 确保所有路径都调用 Commit 或 Rollback
3. 使用 using 语句自动管理

### 问题3: 看到"长事务警告"日志

**原因**: 事务执行时间较长但未超时

**解决**:
1. 这是警告,不是错误
2. 如果业务确实需要这么长时间,可以忽略
3. 建议优化以提升性能
