## 📝 快速配置指南

### 场景1: 销售人员只看成品

#### 1️⃣ 创建权限策略
```sql
INSERT INTO tb_RowAuthPolicy 
(PolicyName, TargetTable, TargetEntity, IsJoinRequired, 
 JoinTable, JoinOnClause, FilterClause, DefaultRuleEnum)
VALUES 
('产品-仅成品', 'tb_ProductInfo', 'ProductInfo', 1,
 'tb_ProductType', 
 'tb_ProductInfo.Type_ID = tb_ProductType.Type_ID',
 'tb_ProductType.ForSale = 1',
 8);
```

#### 2️⃣ 分配给角色
```sql
INSERT INTO tb_P4RowAuthPolicyByRole (PolicyId, RoleID, MenuID)
SELECT p.PolicyId, r.RoleID, m.MenuID
FROM tb_RowAuthPolicy p, tb_RoleInfo r, tb_MenuInfo m
WHERE p.PolicyName = '产品-仅成品'
  AND r.RoleName = '销售人员'
  AND m.MenuName LIKE '%产品%';
```

#### 3️⃣ 生成的SQL
```sql
SELECT * FROM tb_ProductInfo p
WHERE EXISTS (
    SELECT 1 FROM tb_ProductType pt 
    WHERE pt.Type_ID = p.Type_ID 
    AND pt.ForSale = 1
)
```

### 场景2: 项目组成员只看本项目数据

#### 1️⃣ 创建权限策略
```sql
INSERT INTO tb_RowAuthPolicy 
(PolicyName, TargetTable, TargetEntity, IsJoinRequired, 
 JoinTable, JoinOnClause, IsParameterized, ParameterizedFilterClause, DefaultRuleEnum)
VALUES 
('销售订单-仅本项目组', 'tb_SalesOrder', 'SalesOrder', 1,
 'tb_ProjectGroupEmployees', 
 'tb_SalesOrder.ProjectGroup_ID = tb_ProjectGroupEmployees.ProjectGroup_ID',
 1, 'tb_ProjectGroupEmployees.Employee_ID = {EmployeeId} AND Assigned = 1',
 11);
```

#### 2️⃣ 分配给角色
```sql
INSERT INTO tb_P4RowAuthPolicyByRole (PolicyId, RoleID, MenuID)
SELECT p.PolicyId, r.RoleID, m.MenuID
FROM tb_RowAuthPolicy p, tb_RoleInfo r, tb_MenuInfo m
WHERE p.PolicyName = '销售订单-仅本项目组'
  AND r.RoleName = '项目组成员'
  AND m.MenuName LIKE '%销售订单%';
```

#### 3️⃣ 生成的SQL
```sql
SELECT * FROM tb_SalesOrder so
WHERE EXISTS (
    SELECT 1 FROM tb_ProjectGroupEmployees pge 
    WHERE pge.ProjectGroup_ID = so.ProjectGroup_ID 
    AND pge.Employee_ID = 1001  -- 当前用户
    AND pge.Assigned = 1
)
```

## 💡 常见问题速查

### Q1: 为什么权限规则不生效?
**检查**:
1. 用户是否有分配角色?
2. 角色是否配置了权限规则?
3. 数据库中`IsEnabled=1`?
4. 缓存是否已清除?
5. 查看日志中的FilterClause输出

### Q2: 查询性能下降怎么办?
**优化**:
1. 检查是否缺少索引 → 执行索引脚本
2. 调整缓存过期时间 → 延长到1小时
3. 简化复杂规则 → 避免过多JOIN
4. 启用性能监控 → 定位瓶颈

### Q3: 如何为特定用户设置例外?
**方案**:
1. 启用用户级权限(`tb_P4RowAuthPolicyByUser`)
2. 为该用户配置特殊规则
3. 用户级权限优先级高于角色级

### Q4: 项目组变更后权限未更新?
**解决**:
1. 手动清除缓存: `_cacheManager.Remove(key)`
2. 或等待缓存自动过期(30分钟)
3. 或在项目组管理代码中添加缓存清除逻辑
