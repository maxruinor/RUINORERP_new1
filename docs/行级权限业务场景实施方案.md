## 一、业务场景概述

### 场景1: 产品可见性控制

**需求**: 销售人员仅能查看"成品",自动过滤掉"半成品"、"包装材料"等其他类型产品。

**应用点**:
- 产品查询界面
- 销售订单选择产品
- 报价单、出库单等产品选择场景

**技术要点**:
- 产品类型存储在`tb_ProductType`表
- 通过`ForSale`字段或`TypeName`判断是否为成品
- 需要JOIN产品类型表并添加过滤条件

---

### 场景2: 项目组维度数据隔离

**需求**: 在销售订单等业务中,增加"项目组"维度的权限控制,用户只能查看所属项目组的数据。

**应用点**:
- 销售订单查询
- 生产计划管理
- 项目任务分配

**技术要点**:
- 员工与项目组是多对多关系(`tb_ProjectGroupEmployees`)
- 需要获取用户的所有项目组ID列表
- 生成`ProjectGroup_ID IN (1,2,3)`或EXISTS子查询

## 三、场景1:产品可见性控制

### 3.1 方案设计

#### 推荐方案: 基于ForSale字段的EXISTS子查询

**过滤条件**:
```sql
EXISTS (SELECT 1 FROM tb_ProductType pt 
        WHERE pt.Type_ID = ProductInfo.Type_ID 
        AND pt.ForSale = 1)
```

**优点**:
- 性能优(可利用索引)
- 语义清晰(ForSale=true表示可销售=成品)
- 配置简单

## 四、场景2:项目组维度数据隔离

### 4.1 方案设计

#### 核心思路

**权限规则**: 用户只能查看其所属项目组相关的数据

**过滤条件**(推荐EXISTS方式):
```sql
EXISTS (
    SELECT 1 FROM tb_ProjectGroupEmployees pge 
    WHERE pge.ProjectGroup_ID = SalesOrder.ProjectGroup_ID 
    AND pge.Employee_ID = {EmployeeId}
    AND pge.Assigned = 1
)
```

#### 步骤2: 在RowAuthService中填充项目组信息

**文件**: `RUINORERP.Business\RowLevelAuthService\RowAuthService.cs`

修改`BuildRowAuthContext`方法:

```csharp
private RowAuthContext BuildRowAuthContext(long menuId, string entityName)
{
    var context = new RowAuthContext
    {
        UserId = GetCurrentUserId(),
        RoleId = _appContext.CurrentRole?.RoleID ?? 0,
        RoleIds = _appContext.Roles.Select(r => r.RoleID).ToList(),
        EmployeeId = _appContext.CurUserInfo?.UserInfo?.Employee_ID,
        DepartmentId = null,
        MenuId = menuId,
        EntityName = entityName
    };
    
    // ✅ 新增: 获取用户所属的项目组ID列表
    if (context.EmployeeId.HasValue && context.EmployeeId.Value > 0)
    {
        context.ProjectGroupIds = _db.Queryable<tb_ProjectGroupEmployees>()
            .Where(pge => pge.Employee_ID == context.EmployeeId.Value 
                       && pge.Assigned)  // 只查询已分配的
            .Select(pge => pge.ProjectGroup_ID)
            .ToList();
        
        _logger.LogDebug("用户 {EmployeeId} 所属项目组: {ProjectGroupIds}",
            context.EmployeeId, string.Join(",", context.ProjectGroupIds));
    }
    
    return context;
}
```
