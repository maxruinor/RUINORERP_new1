# 单据转换标识符(ConversionIdentifier)修复总结

## 问题描述

在单据转换体系中,`ConversionIdentifier` 用于区分同一源/目标类型下的不同转换逻辑。例如:
- `PreReceivedPaymentToPaymentRecordConverter`: Normal (预收款转收款)
- `PreReceivedPaymentToRefundConverter`: Refund (预收款退还余款)
- `SaleOrderCancelConverter`: Cancel (销售订单取消作废)

但在实际运行中,存在以下问题:

### 1. 基类设计缺陷

**位置**: `DocumentConverterBase.cs` 第230-238行

**原代码**:
```csharp
public virtual string ConversionIdentifier => GetConversionIdentifier();

protected virtual string GetConversionIdentifier()
{
    return string.Empty;  // ❌ 返回空字符串
}
```

**问题**:
- `GetConversionIdentifier()` 返回 `string.Empty` 而非 `null`
- 导致工厂层在生成Key时可能产生混淆
- 子类重写属性时,基类的 `GetConversionIdentifier()` 方法不会被调用,造成设计混乱

### 2. UI层获取标识符失败

**位置**: `BaseBillEditGeneric.cs` 第6711行

**原代码**:
```csharp
var converter = getConverterMethod.Invoke(converterFactory, new object[] { null });
if (converter != null)
{
    var identifier = identifierProperty.GetValue(converter) as string;
    return identifier;
}
```

**问题**:
- 当同一源/目标类型有多个转换器时(如Normal和Refund),`GetConverter(null)` 只返回第一个匹配的
- 如果第一个不是目标转换器类型,会导致获取到错误的标识符或null
- 没有遍历缓存找到正确的转换器实例

## 修复方案

### 修复1: 简化基类设计

**文件**: `DocumentConverterBase.cs`

**修改后**:
```csharp
/// <summary>
/// 获取转换唯一标识符
/// 默认为 null，子类可以根据业务逻辑重写（如 "Normal", "Refund", "Offset"）
/// 
/// 重要说明:
/// - 返回 null 表示不使用标识符区分,工厂层会使用 "Source:Target" 作为Key
/// - 返回非空字符串时,工厂层会使用 "Source:Target:Identifier" 作为Key
/// - 子类应直接重写此属性,而不是重写 GetConversionIdentifier() 方法
/// </summary>
public virtual string ConversionIdentifier => null;
```

**改进点**:
- ✅ 删除了多余的 `GetConversionIdentifier()` 方法
- ✅ 默认返回 `null` 而非 `string.Empty`,语义更清晰
- ✅ 添加了详细的注释说明使用规则

### 修复2: 增强UI层标识符获取逻辑

**文件**: `BaseBillEditGeneric.cs` - `GetConversionIdentifierFromConverterType` 方法

**修改后**:
```csharp
// 【关键修复】：先尝试不带标识符获取第一个匹配的转换器
var converter = getConverterMethod.Invoke(converterFactory, new object[] { null });
if (converter != null && converter.GetType() == converterType)
{
    // 如果获取到的转换器类型匹配，直接读取其标识符
    var identifier = identifierProperty.GetValue(converter) as string;
    return identifier;
}

// 如果第一个不匹配，需要遍历工厂缓存找到正确的转换器
var cacheField = converterFactory.GetType().GetField("_convertersCache", 
    System.Reflection.BindingFlags.NonPublic | System.Reflection.BindingFlags.Instance);
if (cacheField != null)
{
    var cache = cacheField.GetValue(converterFactory) as ConcurrentDictionary<string, object>;
    if (cache != null)
    {
        var baseKey = $"{sourceType.FullName}:{targetType.FullName}";
        foreach (var kvp in cache)
        {
            // 查找 Key 以 baseKey 开头且转换器类型匹配的项
            if (kvp.Key.StartsWith(baseKey + ":") && kvp.Value.GetType() == converterType)
            {
                var identifier = identifierProperty.GetValue(kvp.Value) as string;
                return identifier;
            }
        }
    }
}
```

**改进点**:
- ✅ 首先检查获取到的转换器类型是否匹配
- ✅ 如果不匹配,遍历工厂缓存找到正确的转换器实例
- ✅ 通过Key格式(`baseKey:`)精确定位带标识符的转换器
- ✅ 添加详细的调试日志

## 验证流程

### 1. 转换器注册阶段

**位置**: `DocumentConverterFactory.AutoDiscoverAndRegister()`

**流程**:
```
1. 扫描所有程序集,找到实现 IDocumentConverter<,> 的类型
2. 创建转换器实例(通过DI容器)
3. 调用 GetConverterKey(converter) 生成Key
   - 如果 ConversionIdentifier 为 null/empty: Key = "Source:Target"
   - 如果 ConversionIdentifier 有值: Key = "Source:Target:Identifier"
4. 将转换器存入缓存: _convertersCache[Key] = converter
```

**示例**:
```
PreReceivedPaymentToPaymentRecordConverter:
  Key = "RUINORERP.Model.tb_FM_PreReceivedPayment:RUINORERP.Model.tb_FM_PaymentRecord:Normal"

PreReceivedPaymentToRefundConverter:
  Key = "RUINORERP.Model.tb_FM_PreReceivedPayment:RUINORERP.Model.tb_FM_PaymentRecord:Refund"
```

### 2. 获取可用转换选项阶段

**位置**: `ActionManager.GetAvailableActionsAsync()`

**流程**:
```
1. 调用 _converterFactory.GetAvailableConversions<TSource>(source)
2. 工厂遍历缓存,找到所有 SourceType 匹配的转换器
3. 为每个转换器创建 ConversionOption,包含:
   - DisplayName (动态生成或子类重写)
   - ConversionIdentifier (从 IConverterMeta 获取)
   - ConversionType (DocumentGeneration 或 ActionOperation)
4. 返回给UI层显示为菜单项
```

### 3. 用户点击菜单执行转换阶段

**位置**: `BaseBillEditGeneric.PerformDocumentConversionAsync()`

**流程**:
```
1. 从菜单项的 ActionOption 中获取 ConverterType
2. 调用 GetConversionIdentifierFromConverterType(converterType) 获取标识符
   - 【修复后】正确从缓存中找到对应转换器并读取其 ConversionIdentifier
3. 创建 ActionOptions,设置 ConversionIdentifier
4. 调用 ActionManager.ExecuteActionAsync(source, options)
5. ActionManager 使用 options.ConversionIdentifier 获取正确的转换器
6. 执行转换器的 ValidateConversionAsync() 进行验证
7. 执行转换器的 ConvertAsync() 或 ExecuteActionOperationAsync()
```

### 4. 验证和执行阶段

**位置**: `ActionManager.ExecuteActionAsync()`

**流程**:
```
1. 验证: _converterFactory.ValidateConversionAsync<TSource, TTarget>(source, options.ConversionIdentifier)
   - 工厂根据 identifier 精确查找转换器
   - 调用 converter.ValidateConversionAsync(source)
   
2. 获取转换器: _converterFactory.GetConverter<TSource, TTarget>(converterIdentifier)
   - 工厂根据 identifier 精确查找转换器
   
3. 执行转换: converter.ConvertAsync(source)
   - 对于 DocumentGeneration: 生成新单据
   - 对于 ActionOperation: 执行业务操作
```

## 关键设计原则

### 1. 标识符的使用规则

- **不需要区分的场景**: `ConversionIdentifier = null` (默认)
  - 工厂Key: `"Source:Target"`
  - 示例: `ManufacturingOrderToFinishedGoodsInvConverter`

- **需要区分的场景**: `ConversionIdentifier = "xxx"` (子类重写)
  - 工厂Key: `"Source:Target:xxx"`
  - 示例: 
    - `PreReceivedPaymentToPaymentRecordConverter` → "Normal"
    - `PreReceivedPaymentToRefundConverter` → "Refund"
    - `SaleOrderCancelConverter` → "Cancel"

### 2. 子类实现规范

```csharp
public class MyConverter : DocumentConverterBase<TSource, TTarget>
{
    // ✅ 正确: 直接重写属性
    public override string ConversionIdentifier => "MyAction";
    
    // ❌ 错误: 不要重写已删除的方法
    // protected override string GetConversionIdentifier() { ... }
}
```

### 3. 工厂层Key生成逻辑

```csharp
private string GetConverterKey(object converter)
{
    var baseKey = $"{sourceType.FullName}:{targetType.FullName}";
    
    // 只有当 ConversionIdentifier 不为 null/empty 时才添加到Key
    if (converter is IConverterMeta meta && !string.IsNullOrEmpty(meta.ConversionIdentifier))
    {
        return $"{baseKey}:{meta.ConversionIdentifier}";
    }
    
    return baseKey;
}
```
