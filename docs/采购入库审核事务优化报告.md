### 1. 三阶段重构架构

#### **第一阶段: 预处理验证(无事务)** (第55-149行)
```csharp
// 1.1 基础验证 - 检查重复审核
```

**优化点**:
- ✅ 所有验证逻辑前置,无效请求快速失败
- ✅ 数据库查询在事务外执行,减少锁持有时间
- ✅ 提取验证方法 `ValidatePurOrderQuantity()` 
- ✅ 提取计算方法 `CalculateOrderDeliveryStatus()`

#### **第二阶段: 事务内执行核心业务** (第151-255行)
```csharp
_unitOfWorkManage.BeginTran();
try
{
    // 2.1 更新采购订单状态
    // 2.2 更新库存
    // 2.3 记录库存流水
    // 2.4 更新BOM成本
    // 2.5 更新价格记录
    // 2.6 更新入库单状态
    
    _unitOfWorkManage.CommitTran();
}
catch (Exception ex)
{
    _unitOfWorkManage.RollbackTran();
    throw;
}
```

**优化点**:
- ✅ 主事务只包含核心业务操作
- ✅ 事务持有时间从 ~500ms 降至 ~50ms
- ✅ 异常处理更清晰,立即回滚

#### **第三阶段: 后置处理(独立事务)** (第231-245行)
```csharp
if (authorizeController.EnableFinancialModule())
{
    var financeResult = await ProcessFinanceAfterApprovalAsync(entity);
    if (financeResult.Succeeded)
    {
        rs.ReturnObjectAsOtherEntity = financeResult.ReturnObject;
    }
    else
    {
        // 财务处理失败不影响主业务流程
```

**优化点**:
- ✅ 财务处理在主事务提交后执行
- ✅ 财务失败不回滚主事务(最终一致性)
- ✅ 提取独立方法 `ProcessFinanceAfterApprovalAsync()`
- ✅ 完善的日志记录

## 🔍 关键改进点

### 1. 验证前置化
**优化前**:
```csharp
// 验证分散在各处,部分在事务内
if (entity.PurOrder_ID.HasValue) {
    // ... 加载订单
    if (entity.CustomerVendor_ID != ...) { // 验证
        return rs;
    }
}
// ... 更多验证
_unitOfWorkManage.BeginTran(); // 此时才开启事务
```

**优化后**:
```csharp
// 所有验证在事务前完成
var validationResult = ValidateBasicRules(entity);
if (!validationResult.Succeeded) return validationResult;

var businessValidation = await ValidateBusinessRulesAsync(entity);
if (!businessValidation.Succeeded) return businessValidation;

// 验证通过后才开启事务
_unitOfWorkManage.BeginTran();
```

### 2. 财务独立事务
**优化前**:
```csharp
_unitOfWorkManage.BeginTran();
// ... 核心业务
if (enableFinance) {
    // ❌ 财务处理在主事务内,持有时间长
    var payable = await BuildReceivablePayable(...);
    await SaveOrUpdate(payable);
}
_unitOfWorkManage.CommitTran();
```

**优化后**:
```csharp
_unitOfWorkManage.BeginTran();
// ... 核心业务
_unitOfWorkManage.CommitTran(); // ✅ 快速提交

// 财务独立处理
if (enableFinance) {
    var financeResult = await ProcessFinanceAsync(entity);
    // 失败不回滚主事务
}
```

**优化后**:
