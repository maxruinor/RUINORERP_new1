## 💡 经验教训

### 成功经验

1. **渐进式迁移策略正确** ✅
   - 分阶段验证，避免大规模破坏
   - 优先保证核心层编译通过
   - 逐步推进，降低风险

2. **Model 层设计合理** ✅
   - 成功移除 SkiaSharp、OfficeOpenXml 等 UI 依赖
   - 保持 POCO 纯粹性
   - 添加必要的扩展方法

3. **C# 版本兼容性重视** ✅
   - 及时修复 target-typed new 问题
   - 避免使用 C# 8 不支持的特性
   - 使用传统语法确保兼容性

4. **批量替换工具** ✅
   - PowerShell 正则替换提高效率
   - 注意控制台缓冲区问题
   - 建议写入脚本文件执行

### 踩过的坑

1. **中文标点符号** ❌
   - MetricUnit 枚举中的中文逗号
   - **教训**: 始终使用英文标点

2. **C# 语言版本特性** ❌
   - primary constructor 在 C# 8 不可用
   - **教训**: 检查项目的 LangVersion 设置

3. **DateTime 扩展方法缺失** ❌
   - StartOfWeek 等方法不是.NET Framework 内置的
   - **教训**: 提前检查依赖项

4. **PowerShell 控制台缓冲区** ❌
   - 超长命令导致 SetCursorPosition 错误
   - **教训**: 复杂操作应写入.ps1 文件
