# 报表图表架构设计指南 v1.0

## 🎯 架构目标

**核心原则**: 简单、统一、可扩展、易复制

### 设计思想

1. **统一的基类** - 所有 Adapter 继承同一个基类 `SqlDataProviderBase`
2. **标准的模板** - 固定的代码结构和实现模式
3. **清晰的层次** - 基础设施 → 业务适配 → 呈现
4. **易于复制** - 新业务只需模仿现有示例 (CustomerAnalysisAdapter)

---

## 📋 标准实现模板

### Template A: 简单统计 (单表)

**适用场景**: 单表统计、简单分组

**文件路径**: `Adapters/{Business}/{Name}AnalysisAdapter.cs`

```csharp
using RUINORERP.Model;
using RUINORERP.Model.ChartFramework.Models;
using RUINORERP.UI.ChartFramework.Adapters;
using SqlSugar;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Threading.Tasks;

namespace RUINORERP.UI.ChartFramework.DataProviders.Adapters.CRM
{
    /// <summary>
    /// {业务名称} 数据适配器
    /// </summary>
    public class {Name}AnalysisAdapter : SqlDataProviderBase
    {
        public {Name}AnalysisAdapter() { }
        
        public {Name}AnalysisAdapter(ISqlSugarClient db) : base(db) { }

        /// <summary>
        /// 主表名
        /// </summary>
        protected override string PrimaryTableName => "{table_name}";

        /// <summary>
        /// 可用的维度配置
        /// </summary>
        public override IEnumerable<DimensionConfig> GetDimensions() => new[]
        {
            new DimensionConfig("{field1}", "{中文名称 1}", DimensionType.String),
            new DimensionConfig("{field2}", "{中文名称 2}", DimensionType.DateTime)
        };

        /// <summary>
        /// 可用的指标配置
        /// </summary>
        public override IEnumerable<MetricConfig> GetMetrics() => new[]
        {
            new MetricConfig("Count", "数量", MetricType.Count, MetricUnit.Count),
            new MetricConfig("Amount", "金额", MetricType.Sum, MetricUnit.Amount)
        };

        /// <summary>
        /// 获取数据 (标准实现)
        /// </summary>
        public async override Task<ChartData> GetDataAsync(DataRequest request)
        {
            // Step 1: 构建查询
            var query = _db.Queryable<{EntityClass}>()
                .WhereIF(request.Employee_ID.HasValue, x => x.Employee_ID == request.Employee_ID.Value)
                .WhereIF(request.StartTime.HasValue, x => x.Created_at >= request.StartTime)
                .WhereIF(request.EndTime.HasValue, x => x.Created_at <= request.EndTime);

            // Step 2: 构建分组字段
            var groupFields = new List<string>();
            groupFields.Add(SqlTimeGroupBuilder.GetGroupByTimeField(request.TimeField, request.RangeType));
            
            if (request.Dimensions.Any())
            {
                groupFields.AddRange(request.Dimensions.Select(d => d.FieldName));
            }

            var groupByModels = groupFields.Select(f => new GroupByModel { FieldName = f }).ToList();

            // Step 3: 执行查询
            var result = await query
                .GroupBy(groupByModels)
                .Select(g => (dynamic)new
                {
                    TimeGroup = SqlFunc.DateValue(g.Created_at.Value, DateType.Month),
                    Category = g.{category_field},
                    Count = SqlFunc.AggregateCount(1),
                    Amount = SqlFunc.AggregateSum(g.amount_field ?? 0)
                })
                .ToListAsync();

            // Step 4: 转换为图表数据
            return TransformToChartData(result, request);
        }

        /// <summary>
        /// 转换为图表数据
        /// </summary>
        protected override ChartData TransformToChartData(List<dynamic> rawData, DataRequest request)
        {
            var chartData = new ChartData
            {
                Title = "{图表标题}",
                ChartType = ChartType.Column,
                ValueType = ValueType.Absolute
            };

            // 处理 X 轴标签
            chartData.CategoryLabels = rawData
                .Select(x => FormatLabel(x.TimeGroup, request))
                .Distinct()
                .OrderBy(x => x)
                .ToArray();

            // 处理系列
            if (request.Dimensions.Any(d => d.Type == DimensionType.String))
            {
                var seriesGroups = rawData.GroupBy(x => x.Category);
                foreach (var group in seriesGroups)
                {
                    var series = new DataSeries
                    {
                        Name = GetCategoryName(group.Key),
                        Values = new List<double>()
                    };

                    foreach (var label in chartData.CategoryLabels)
                    {
                        var value = group.FirstOrDefault(x => FormatLabel(x.TimeGroup, request) == label)?.Count ?? 0;
                        series.Values.Add((double)value);
                    }

                    chartData.Series.Add(series);
                }
            }
            else
            {
                var series = new DataSeries
                {
                    Name = "数量",
                    Values = rawData.Select(x => (double)x.Count).ToList()
                };
                chartData.Series.Add(series);
            }

            return chartData;
        }

        #region Helper Methods

        private string FormatLabel(object value, DataRequest request)
        {
            if (value is DateTime dt)
            {
                return request.RangeType switch
                {
                    TimeRangeType.Daily => dt.ToString("yyyy-MM-dd"),
                    TimeRangeType.Monthly => dt.ToString("yyyy-MM"),
                    TimeRangeType.Quarterly => $"Q{(dt.Month - 1) / 3 + 1}-{dt.Year}",
                    TimeRangeType.Yearly => dt.ToString("yyyy"),
                    _ => dt.ToString("yyyy-MM-dd")
                };
            }
            return value?.ToString() ?? "未知";
        }

        private string GetCategoryName(object key)
        {
            return key?.ToString() ?? "未知";
        }

        #endregion
    }
}
```

## 🔧 如何使用

### Step 1: 选择适配器

根据业务需求选择合适的 Adapter:

```csharp
// 客户分析
var adapter = new CustomerAnalysisAdapter();

// 订单分析
var adapter = new OrderAnalysisAdapter();

// 跟进计划
var adapter = new FollowUpPlansAdapter();
```

### Step 2: 配置请求参数

```csharp
var request = new DataRequest
{
    Title = "客户增长分析",
    ChartType = ChartType.Column,
    TimeField = "Created_at",
    RangeType = TimeRangeType.Monthly,
    StartTime = DateTime.Now.AddMonths(-6),
    EndTime = DateTime.Now,
    Dimensions = new List<DimensionConfig>
    {
        new DimensionConfig("Employee_ID", "业务员", DimensionType.String)
    },
    Metrics = new List<MetricConfig>
    {
        new MetricConfig("Count", "客户数量", MetricType.Count, MetricUnit.Count)
    }
};
```

### Step 3: 获取数据

```csharp
var chartData = await adapter.GetDataAsync(request);
```

### Step 4: 渲染图表

```csharp
var chartBuilder = ChartBuilderFactory.Create(chartData.ChartType);
var chartControl = chartBuilder.Build(chartData);
panel.Controls.Add(chartControl);
```

## 💡 最佳实践

### 1. 命名规范

✅ **推荐**:
- `CustomerAnalysisAdapter` - 综合分析
- `CustomerTrendAdapter` - 趋势分析
- `CustomerStatsAdapter` - 统计分析

❌ **避免**:
- `CustomerDataAdapter` - 太泛
- `CustomerChartAdapter` - 混淆

### 2. 维度设计

✅ **好的维度**:
- 业务员、区域、产品类别 (业务相关)
- 时间 (日期、月份、季度)

❌ **不好的维度**:
- ID 字段 (用户看不懂)
- 太多维度 (>5 个)

### 3. 指标设计

✅ **好的指标**:
- Count (数量)
- Sum (汇总)
- Average (平均)

❌ **不好的指标**:
- 复杂的计算公式
- 需要多次查询的指标

### 4. 代码组织

✅ **推荐结构**:
```csharp
public class XxxAdapter : SqlDataProviderBase
{
    // 1. 构造函数
    public XxxAdapter() { }
    public XxxAdapter(ISqlSugarClient db) : base(db) { }
    
    // 2. 主表名
    protected override string PrimaryTableName => "...";
    
    // 3. 维度配置
    public override IEnumerable<DimensionConfig> GetDimensions() => ...;
    
    // 4. 指标配置
    public override IEnumerable<MetricConfig> GetMetrics() => ...;
    
    // 5. 数据获取
    public async override Task<ChartData> GetDataAsync(DataRequest request) { ... }
    
    // 6. 数据转换
    protected override ChartData TransformToChartData(...) { ... }
    
    // 7. 辅助方法
    #region Helper Methods
    private string FormatLabel(...) { ... }
    #endregion
}
```
