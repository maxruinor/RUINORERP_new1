# 中文注释乱码问题修复说明

## 问题描述

在第一次批量更新 ControllerPartial.cs 文件的异步事务方法时，由于使用了错误的文件编码方式，导致所有中文注释变成了乱码。

### 错误示例
```csharp
// 原始内容（正常）
// 生成：CodeBuilder (http://www.fireasy.cn/codebuilder)
// 项目：信息系统

// 错误处理后（乱码）
// 鐢熸垚锛欳odeBuilder (http://www.fireasy.cn/codebuilder)
// 椤圭洰锛氫俊鎭郴缁?
```

## 根本原因

PowerShell 的 `Set-Content -Encoding UTF8` 命令默认使用 **UTF8 without BOM**（无字节顺序标记），而 Visual Studio 和许多编辑器期望的是 **UTF8 with BOM**。当文件没有 BOM 时，某些编辑器会错误地以其他编码（如 GBK）解读 UTF8 字节序列，导致中文显示为乱码。

## 解决方案

### ✅ 正确做法：使用 UTF8 with BOM

```powershell
# 创建 UTF8 with BOM 编码器
$utf8WithBom = New-Object System.Text.UTF8Encoding $true

# 读取文件（使用 UTF8）
$content = [System.IO.File]::ReadAllText($file.FullName, [System.Text.Encoding]::UTF8)

# 修改内容...

# 写入文件（使用 UTF8 with BOM）
[System.IO.File]::WriteAllText($file.FullName, $content, $utf8WithBom)
```

### ❌ 错误做法：使用 Set-Content

```powershell
# 这会导致中文乱码！
Set-Content -Path $file.FullName -Value $content -Encoding UTF8 -NoNewline
```

## 技术细节

### UTF8 with BOM vs UTF8 without BOM

| 特性 | UTF8 with BOM | UTF8 without BOM |
|------|---------------|------------------|
| 字节顺序标记 | 有 (EF BB BF) | 无 |
| Windows 兼容性 | ✅ 好 | ⚠️ 可能有问题 |
| Visual Studio | ✅ 完全支持 | ⚠️ 可能误判编码 |
| Linux/macOS | ✅ 支持 | ✅ 标准做法 |
| PowerShell Set-Content | ❌ 不支持 | ✅ 默认行为 |

### 为什么会出现乱码？

1. **原始文件**：UTF8 with BOM 编码，包含中文注释
2. **错误处理**：使用 `Set-Content -Encoding UTF8` 写入（无 BOM）
3. **编辑器读取**：Visual Studio 检测到无 BOM，尝试用系统默认编码（GBK）解读
4. **结果**：UTF8 字节被当作 GBK 解读，显示为乱码

例如：
- 中文字符 "生成" 的 UTF8 编码：`E7 94 9F E6 88 90`
- 被当作 GBK 解读后：`鐢熸垚`

## 修复过程

### 第一步：恢复原始文件
```bash
git checkout -- RUINORERP.Business/*ControllerPartial.cs
```

### 第二步：使用正确的编码重新处理
```powershell
$utf8WithBom = New-Object System.Text.UTF8Encoding $true
$files = Get-ChildItem -Filter "*ControllerPartial.cs"

foreach ($file in $files) {
    $content = [System.IO.File]::ReadAllText($file.FullName, [System.Text.Encoding]::UTF8)
    $modified = $false
    
    # 替换事务方法并添加 await
    if ($content -match '_unitOfWorkManage\.BeginTran\(\)') {
        $content = $content -replace '_unitOfWorkManage\.BeginTran\(\)', 'await _unitOfWorkManage.BeginTranAsync()'
        $modified = $true
    }
    
    if ($content -match '_unitOfWorkManage\.CommitTran\(\)') {
        $content = $content -replace '_unitOfWorkManage\.CommitTran\(\)', 'await _unitOfWorkManage.CommitTranAsync()'
        $modified = $true
    }
    
    if ($content -match '_unitOfWorkManage\.RollbackTran\(\)') {
        $content = $content -replace '_unitOfWorkManage\.RollbackTran\(\)', 'await _unitOfWorkManage.RollbackTranAsync()'
        $modified = $true
    }
    
    # 使用 UTF8 with BOM 写入
    if ($modified) {
        [System.IO.File]::WriteAllText($file.FullName, $content, $utf8WithBom)
    }
}
```

## 最佳实践建议

### 1. PowerShell 文件操作编码规范

```powershell
# ✅ 推荐：使用 .NET File API + UTF8 with BOM
$utf8WithBom = New-Object System.Text.UTF8Encoding $true
$content = [System.IO.File]::ReadAllText($path, [System.Text.Encoding]::UTF8)
# ... 修改内容 ...
[System.IO.File]::WriteAllText($path, $content, $utf8WithBom)

# ❌ 避免：使用 Set-Content（无 BOM）
Set-Content -Path $path -Value $content -Encoding UTF8
```

### 2. Git 配置建议

在 `.gitattributes` 中添加：
```
*.cs text eol=crlf encoding=utf-8-bom
```

### 3. Visual Studio 设置

确保 Visual Studio 使用正确的编码：
- 工具 → 选项 → 文本编辑器 → 文件编码
- 选择 "Unicode (UTF-8 with signature) - Codepage 65001"
