# 事务管理优化 - 快速参考卡

## 🎯 核心改进一览

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **并发检测** | 防止事务内并行操作导致线程安全问题 | ⚠️ 高 |
| **死锁重试** | 自动重试死锁事务（最多 3 次，指数退避） | ⚠️ 高 |
| **连接释放** | IDisposable 确保连接及时关闭 | ⚠️ 中 |
| **隔离级别** | 支持自定义事务隔离级别 | ⚠️ 中 |
| **事务超时** | 默认 60 秒超时限制 | ⚠️ 中 |
| **日志增强** | 记录死锁参数和详细上下文 | ✅ 低 |

## 📝 代码变更速查

### 1. BeginTran 支持隔离级别

```csharp
// 旧代码
unitOfWork.BeginTran();

// 新代码 - 使用指定隔离级别
unitOfWork.BeginTran(IsolationLevel.ReadCommitted);

// 新代码 - 使用快照隔离（需数据库支持）
unitOfWork.BeginTran(IsolationLevel.Snapshot);
```

### 2. 并发检测 - 会自动拦截

```csharp
unitOfWork.BeginTran();

// ❌ 以下代码现在会抛出异常
await Task.WhenAll(
    db.InsertAsync(obj1),  // 并发操作 1
    db.InsertAsync(obj2)   // 并发操作 2
);

// ✅ 正确做法 - 顺序执行
await db.InsertAsync(obj1);
await db.InsertAsync(obj2);

unitOfWork.CommitTran();
```

### 3. 死锁自动重试 - 透明生效

```csharp
// CommitTran 内部已集成重试机制，无需修改代码
unitOfWork.BeginTran();
// ... 业务操作
unitOfWork.CommitTran(); // 遇到死锁会自动重试（最多 3 次）
```

### 4. 手动控制重试

```csharp
// 同步版本
unitOfWork.ExecuteWithRetry(() => {
    unitOfWork.BeginTran();
    // ... 业务
    unitOfWork.CommitTran();
}, maxRetryCount: 5);

// 异步版本
await unitOfWork.ExecuteWithRetryAsync(async () => {
    unitOfWork.BeginTran();
    await SomeOperation();
    unitOfWork.CommitTran();
});
```

### 5. 使用 using 确保释放

```csharp
// 推荐方式 1：using 语句
using (var unitOfWork = scope.ServiceProvider.GetService<UnitOfWorkManage>())
{
    unitOfWork.BeginTran();
    // ... 业务
    unitOfWork.CommitTran();
} // 自动调用 Dispose，确保连接释放

// 推荐方式 2：DI 容器托管（注册为 Scoped）
public class MyService
{
    private readonly UnitOfWorkManage _unitOfWork;
    
    public MyService(UnitOfWorkManage unitOfWork)
    {
        _unitOfWork = unitOfWork; // 容器会在适当时机调用 Dispose
    }
}
```

## ⚠️ 常见错误示例

### 错误 1：事务内并行操作

```csharp
unitOfWork.BeginTran();

// ❌ 错误：会抛出 InvalidOperationException
var tasks = new[] {
    Task.Run(() => db.Insert(obj1)),
    Task.Run(() => db.Insert(obj2))
};
await Task.WhenAll(tasks);

unitOfWork.CommitTran();
```

**修复方案：**
```csharp
// 方案 A：顺序执行
unitOfWork.BeginTran();
await db.Insert(obj1);
await db.Insert(obj2);
unitOfWork.CommitTran();

// 方案 B：每个任务独立事务
var tasks = items.Select(async item => {
    using (var scope = serviceScopeFactory.CreateScope())
    {
        var uow = scope.ServiceProvider.GetService<UnitOfWorkManage>();
        uow.BeginTran();
        await ProcessItem(item);
        uow.CommitTran();
    }
});
await Task.WhenAll(tasks);
```

### 错误 2：忘记释放连接

```csharp
// ❌ 错误：未使用 using，可能导致连接泄漏
var unitOfWork = container.Resolve<UnitOfWorkManage>();
unitOfWork.BeginTran();
// ... 如果这里抛异常，Dispose 不会被调用
unitOfWork.CommitTran();

// ✅ 正确：using 确保 Dispose 被调用
using (var unitOfWork = container.Resolve<UnitOfWorkManage>())
{
    unitOfWork.BeginTran();
    // ...
    unitOfWork.CommitTran();
}
```

### 错误 3：滥用重试机制

```csharp
// ❌ 错误：对所有操作都重试，浪费资源
unitOfWork.ExecuteWithRetry(() => {
    // 包含业务验证逻辑
    if (!IsValid(data))
        throw new ValidationException(); // 这种异常不应重试
});

// ✅ 正确：只对可能死锁的操作重试
try
{
    unitOfWork.BeginTran();
    // ... 业务操作
    unitOfWork.CommitTran();
}
catch (SqlException ex) when (ex.Number == 1205)
{
    // 专门处理死锁
    logger.LogWarning("发生死锁，考虑调整业务逻辑或索引");
    throw;
}
```

## 🔧 配置建议

### 应用配置（appsettings.json）

```json
{
  "Transaction": {
    "DefaultTimeoutSeconds": 60,
    "MaxRetryCount": 3,
    "EnableCallerMethod": true
  }
}
```

### 数据库配置（SQL Server）

```sql
-- 启用读已提交快照隔离（强烈推荐）
ALTER DATABASE YourDatabase SET READ_COMMITTED_SNAPSHOT ON;

-- 设置锁超时时间（可选）
SET LOCK_TIMEOUT 30000; -- 30 秒

-- 查看当前死锁统计
SELECT * FROM sys.dm_tran_locks 
WHERE resource_type = 'DATABASE';
```

## 📊 监控指标

### 关键指标

1. **死锁重试率** = 死锁重试次数 / 总事务数
   - 正常：< 1%
   - 警告：1-5%
   - 危险：> 5%（需要优化）

2. **并发违规次数** = 检测到的违规操作数
   - 目标：0（所有违规都应修复）

3. **事务超时率** = 超时时务数 / 总事务数
   - 正常：< 2%
   - 警告：> 5%

4. **平均事务耗时**
   - 健康：< 1 秒
   - 关注：1-10 秒
   - 优化：> 10 秒

### 日志查询示例

```kusto
// 查询死锁事件
```

## 🎓 最佳实践总结

### ✅ DO（推荐）

1. 在事务内顺序执行数据库操作
2. 使用 using 确保 UnitOfWork 释放
3. 为高频并发场景启用快照隔离
4. 记录和分析所有死锁事件
5. 保持事务短小精悍（< 1 秒）
6. 在 Repository 层统一处理事务

### ❌ DON'T（避免）

1. 不要在事务中使用 `Task.WhenAll`、`Parallel.ForEach`
2. 不要捕获死锁异常后静默失败
3. 不要在长事务中执行外部 API 调用
4. 不要忽略并发检测警告
5. 不要在不需要的地方使用高隔离级别
6. 不要忘记回滚未完成的事务
