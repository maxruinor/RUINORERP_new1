# 反审核与反执行拆分 - 快速参考

## 核心概念

| 方法 | 职责 | 状态变化 | 库存操作 | 调用时机 |
|------|------|---------|---------|---------|
| **AntiApprovalAsync** | 反审核（状态回退） | 确认→新建 | ❌ 无 | 未执行的单据 |
| **AntiConfirmExecutionAsync** | 反执行（库存回滚） | 完结→确认 | ✅ 反向操作 | 已执行的单据 |

## 业务流程

### 场景 A：未执行的单据
```
确认状态 
  │
  ├─ 调用 AntiApprovalAsync
  │
  ▼
新建状态 ✓
```

### 场景 B：已执行的单据
```
完结状态
  │
  ├─ 调用 AntiConfirmExecutionAsync
  │  └─ 库存回滚 + 状态回退
  ▼
确认状态
  │
  ├─ 调用 AntiApprovalAsync
  │  └─ 状态回退
  ▼
新建状态 ✓
```

## 关键代码片段

### 反审核（AntiApprovalAsync）
```csharp
// 1. 检查状态
if (entity.DataStatus != (int)DataStatus.确认)
    return Error("只有已审核的单据才能反审核");

// 2. 检查是否已执行
if (entity.DataStatus == (int)DataStatus.完结)
    return Error("此单据已执行，请先反执行再反审核");

// 3. 更新状态（仅状态字段）
entity.DataStatus = (int)DataStatus.新建;
entity.ApprovalResults = false;
entity.ApprovalStatus = (int)ApprovalStatus.未审核;
```

### 反执行（AntiConfirmExecutionAsync）
```csharp
// 1. 检查状态
if (entity.DataStatus != (int)DataStatus.完结)
    return Error("只有已执行的单据才能反执行");

// 2. 权限检查
if (!HasPermission("ReverseExecute"))
    return Error("没有反执行权限");

// 3. 预加载库存（死锁优化）
var invDict = PreloadInventory(details);

// 4. 反向操作库存
foreach (var detail in details)
{
    inv.Quantity += detail.Qty;  // 增加库存
    CreateReverseTransaction(detail, inv);
}

// 5. 回退状态
entity.DataStatus = (int)DataStatus.确认;
```

## 常见错误处理

### 错误 1：完结状态直接反审核
```
❌ 错误：此单据已执行，无法直接反审核！
✅ 正确：先反执行，再反审核
```

### 错误 2：确认状态反执行
```
❌ 错误：只有已执行的单据才能反执行！
✅ 正确：确认状态应该用反审核
```

### 错误 3：无权限反执行
```
❌ 错误：您没有反执行的权限！
✅ 正确：联系管理员授权
```

## 事务控制

```csharp
using (var scope = new TransactionScope(TransactionScopeAsyncFlowOption.Enabled))
{
    try
    {
        // 业务操作
        await OperationAsync(entity);
        
        // 状态变更
        await UpdateStatusAsync(entity);
        
        // 提交事务
        scope.Complete();
    }
    catch (Exception ex)
    {
        // 自动回滚
        throw;
    }
}
```

## 测试用例

### Test 1: 未执行单据的反审核
```csharp
[Test]
public async Task AntiApproval_WithoutExecution_ShouldSucceed()
{
    var entity = new tb_ProdBorrowing { DataStatus = (int)DataStatus.确认 };
    
    var result = await controller.AntiApprovalAsync(entity);
    
    Assert.IsTrue(result.Succeeded);
    Assert.AreEqual((int)DataStatus.新建, entity.DataStatus);
}
```

### Test 2: 已执行单据的反审核失败
```csharp

public async Task AntiApproval_WithExecution_ShouldFail()

    var entity = new tb_ProdBorrowing { DataStatus = (int)DataStatus.完结 };
    
    var result = await controller.AntiApprovalAsync(entity);
    
    Assert.IsFalse(result.Succeeded);
    Assert.AreEqual("此单据已执行...", result.ErrorMsg);
}
```

### Test 3: 反执行成功
```csharp
[Test]
public async Task AntiConfirmExecution_ShouldSucceed()
{
    var entity = new tb_ProdBorrowing { DataStatus = (int)DataStatus.完结 };
    MockInventoryService.Setup(x => x.Reverse(It.IsAny<long>()));
    
    var result = await controller.AntiConfirmExecutionAsync(entity);
    
    Assert.IsTrue(result.Succeeded);
    Assert.AreEqual((int)DataStatus.确认，entity.DataStatus);
}
```

## 性能优化要点

1. ✅ **死锁预防**：事务外批量预加载库存
2. ✅ **批量操作**：减少数据库往返次数
3. ✅ **事务控制**：确保原子性和一致性
4. ✅ **异常处理**：失败时自动回滚
