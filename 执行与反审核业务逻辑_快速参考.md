# 执行与反审核业务逻辑 - 快速参考

## 核心变更

### 1. 操作流程拆分

**原流程：**
```
审核 = 状态变化 + 库存操作
反审核 = 状态回退 + 库存冲减
```

**新流程：**
```
审核 = 仅状态变化（新建→确认）
执行 = 实际库存操作（确认→完结）
反执行 = 库存回滚操作（完结→确认）
反审核 = 仅状态回退（确认→新建）
```

### 2. 反审核处理策略

| 场景 | 状态 | 库存 | 反审核处理 |
|------|------|------|-----------|
| 未执行 | 确认 | 未变动 | ✅ 直接反审核到新建 |
| 已执行 | 完结 | 已变动 | ❌ 不允许直接反审核<br>需先【反执行】→确认<br>再【反审核】→新建 |

### 3. 状态流转图

```
                    ┌──────────────────────────────────┐
                    │                                  │
                    ▼                                  │
草稿 → 新建 → 确认 → 完结                            │
         ▲    │     │    │                            │
         │    │反审  │反执│反结                        │
         │    │     │行  │案                          │
         │    │     ▼    ▼                            │
         │    └─── 确认 ◄────────────────────────────┘
         │          │
         │          │反审
         │          │
         └──────────┘
```

## 代码实现模板

### 反审核按钮

```csharp
private async void toolStripBtnReverseReview_Click(object sender, EventArgs e)
{
    var entity = GetCurrentEntity();
    var currentStatus = entity.GetCurrentStatus();
    
    // 检查是否已执行（完结状态）
    if (currentStatus == DataStatus.完结)
    {
        MessageBox.Show(
            "⚠️ 此单据已执行，无法直接反审核！\n\n" +
            "请先执行【反执行】操作回滚库存，然后再反审核。");
        return;
    }
    
    // 只有确认状态可以反审核
    if (currentStatus != DataStatus.确认)
    {
        MessageBox.Show("只有已审核的单据才能反审核！");
        return;
    }
    
    // 二次确认
    if (MessageBox.Show(
        "确定要反审核此单据吗？",
        MessageBoxButtons.OKCancel,
        MessageBoxIcon.Question) != DialogResult.OK)
        return;
    
    try
    {
        // 只改变状态，不涉及库存
        await stateManager.SetStatusByActionAsync(entity, MenuItemEnums.反审);
        await SaveEntityAsync(entity);
        MessageBox.Show("反审核成功！");
        RefreshUI();
    }
    catch (Exception ex)
    {
        MessageBox.Show($"反审核失败：{ex.Message}");
    }
}
```

### 反执行按钮（新增）

```csharp
private async void toolStripBtnReverseExecute_Click(object sender, EventArgs e)
{
    var entity = GetCurrentEntity();
    var currentStatus = entity.GetCurrentStatus();
    
    // 只有完结状态可以反执行
    if (currentStatus != DataStatus.完结)
    {
        MessageBox.Show("只有已执行的单据才能反执行！");
        return;
    }
    
    // 权限检查
    if (!HasPermission("ReverseExecute"))
    {
        MessageBox.Show("您没有反执行的权限！");
        return;
    }
    
    // 警告确认
    if (MessageBox.Show(
        "⚠️ 警告：反执行将回滚所有库存操作！\n\n确定要继续吗？",
        MessageBoxButtons.OKCancel,
        MessageBoxIcon.Warning) != DialogResult.OK)
        return;
    
    try
    {
        using (var scope = new TransactionScope(TransactionScopeAsyncFlowOption.Enabled))
        {
            // 1. 先执行库存回滚
            await ReverseInventoryOperationAsync(entity);
            
            // 2. 回退状态到确认
            await stateManager.SetBusinessStatusAsync(entity, DataStatus.确认);
            
            // 3. 保存
            await SaveEntityAsync(entity);
            
            // 4. 提交事务
            scope.Complete();
            
            MessageBox.Show("反执行成功！\n单据状态已回退到【已审核】。");
            RefreshUI();
        }
    }
    catch (Exception ex)
    {
        MessageBox.Show($"反执行失败：{ex.Message}\n单据状态仍为【完结】。");
        _logger.Error(ex, $"反执行单据 {entity.PrimaryKeyID} 失败");
    }
}
```

### 执行按钮

```csharp
private async void toolStripBtnExecute_Click(object sender, EventArgs e)
{
    var entity = GetCurrentEntity();
    var currentStatus = entity.GetCurrentStatus();
    
    if (currentStatus != DataStatus.确认)
    {
        MessageBox.Show("只有已审核的单据才能执行！");
        return;
    }
    
    try
    {
        using (var scope = new TransactionScope(TransactionScopeAsyncFlowOption.Enabled))
        {
            // 1. 先执行业务操作（如修改库存）
            await ExecuteInventoryOperationAsync(entity);
            
            // 2. 更新状态为完结
            await stateManager.SetStatusByActionAsync(entity, MenuItemEnums.执行);
            
            // 3. 保存
            await SaveEntityAsync(entity);
            
            // 4. 提交事务
            scope.Complete();
            
            MessageBox.Show("执行成功，单据已完结！");
            RefreshUI();
        }
    }
    catch (Exception ex)
    {
        MessageBox.Show($"执行失败：{ex.Message}\n单据状态仍为【已审核】。");
        _logger.Error(ex, $"执行单据 {entity.PrimaryKeyID} 失败");
    }
}
```

## 关键要点

### 1. 职责分离

- **审核/反审核**：只管理状态，不碰库存
- **执行/反执行**：管理库存 + 状态

### 2. 事务控制

```csharp
using (var scope = new TransactionScope(TransactionScopeAsyncFlowOption.Enabled))
{
    // 业务操作
    await InventoryOperationAsync(entity);
    
    // 状态变更
    await stateManager.SetStatusByActionAsync(entity, action);
    
    // 保存
    await SaveEntityAsync(entity);
    
    scope.Complete();
}
```

### 3. 异常处理

- 业务操作失败 → 抛出异常 → 状态不变
- 状态变更失败 → 抛出异常 → 业务回滚

### 4. 权限控制

```csharp
// 反执行需要特殊权限
```

## 总结

### 反审核处理流程

```
确认状态（未执行）
  │
  ├─ 反审核 → 新建 ✓
  │
  ▼
执行 → 完结
  │
  ├─ 反执行 → 确认
  │           │
  │           └─ 反审核 → 新建 ✓
  │
  └─ 反结案 → 确认
              │
              └─ 反审核 → 新建 ✓
```

这样的设计确保了：
1. ✅ 状态与业务清晰分离
2. ✅ 反审核逻辑简单明了
3. ✅ 数据一致性有保障
4. ✅ 审计追踪完整
