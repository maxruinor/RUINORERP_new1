# ERP系统代码审查报告

## 审查原则

1. **事务边界原则**：事务内不应该有任何用户交互
2. **快速失败原则**：数据库操作应该立即返回结果，不应该等待用户输入
3. **职责分离原则**：数据访问层不应该直接处理UI展示

---

## 一、事务边界原则问题

### 问题说明

在数据库事务执行过程中，如果弹出 `MessageBox` 等用户交互组件，会导致以下严重问题：

1. **线程阻塞**：弹出消息框会阻塞当前线程，等待用户操作
2. **事务无法回滚**：由于线程被阻塞，事务无法正常回滚
3. **数据库锁表**：未回滚的事务会持有数据库锁，导致其他操作被阻塞
4. **用户体验极差**：用户不点击确定就无法继续操作，整个系统陷入假死状态

### 已修复问题

#### 1. MainForm.cs - SQL唯一键约束异常处理

**文件位置**：`RUINORERP.UI/MainForm.cs` (L1685-L1793)

**问题描述**：
- 在 SqlSugar AOP 错误回调 `SqlsugarSetup_RemindEvent` 中弹出 `MessageBox.Show`
- 当发生唯一键约束冲突时，消息框弹出导致事务无法回滚
- 用户如果不点击确定，数据库表会被一直锁定

**修复方案**：
- 移除 AOP 回调中的所有 `MessageBox.Show` 调用
- 让异常正确传播到业务层，由业务层处理事务回滚
- 添加静态属性 `LastUniqueConstraintError` 存储错误信息供业务层使用
- 始终返回 `false` 让 SqlSugar 不吞掉异常

**修复后代码示例**：
```csharp
private bool SqlsugarSetup_RemindEvent(SqlSugarException ex)
{
    // 处理唯一键约束异常，将错误信息存储到静态属性
    HandleUniqueConstraintException(ex);

    // 关键：始终返回 false，让异常继续传播
    // 这样事务才能正确回滚，不会锁表
    return false;
}
```

## 四、修复建议

### 建议方案

#### 1. 使用 Result/ReturnResults 模式

将验证结果封装到返回值中，由 UI 层决定如何展示：

```csharp
// 业务层
public class ValidationResult {
    public bool NeedUserConfirm { get; set; }
    public string ConfirmMessage { get; set; }
    public bool IsValid { get; set; }
    public string ErrorMessage { get; set; }
}

public async Task<ValidationResult> ValidateAsync(T entity) {
    if (超额) {
        return new ValidationResult {
            NeedUserConfirm = true,
            ConfirmMessage = "金额超额，确认继续吗？"
        };
    }
    return new ValidationResult { IsValid = true };
}

// UI层
var result = await controller.ValidateAsync(entity);
if (result.NeedUserConfirm) {
    if (MessageBox.Show(result.ConfirmMessage, "确认") == DialogResult.Yes) {
        // 继续执行
    }
}
```

#### 2. 使用事件/回调机制

在业务层抛出验证事件，由 UI 层订阅并处理：

```csharp
// 业务层
public event Func<ValidationContext, Task<DialogResult>> OnNeedUserConfirm;

public async Task ProcessAsync() {
    if (需要确认) {
        var confirmResult = await OnNeedUserConfirm?.Invoke(context);
        if (confirmResult != DialogResult.Yes) return;
    }
}

// UI层
controller.OnNeedUserConfirm += async (context) => {
    return MessageBox.Show(context.Message, "确认", MessageBoxButtons.YesNo);
};
```

#### 3. 使用异常携带验证信息

```csharp
// 业务层
if (超额) {
    throw new BusinessValidationException("超额验证", "金额超过订单总额，需要确认") {
        NeedUserConfirm = true,
        ConfirmMessage = "超额确认"
    };
}

// UI层
try {
    await controller.ProcessAsync();
}
catch (BusinessValidationException ex) when (ex.NeedUserConfirm) {
    if (MessageBox.Show(ex.ConfirmMessage) == DialogResult.Yes) {
        // 重试
    }
}
```
