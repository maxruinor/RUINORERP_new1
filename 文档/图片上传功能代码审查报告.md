### 问题1：大文件未压缩直接上传，浪费带宽

#### 问题位置
- **文件**: `BaseBillEditGeneric.cs`
- **行号**: 1572-1573
- **代码**:
```csharp
// 检查文件大小限制
if (imageData.Length > 10 * 1024 * 1024) // 10MB限制
```

#### 问题描述
1. 图片在上传前未进行任何压缩处理
2. 10MB限制仅作为检查，超过限制则拒绝上传，未进行压缩尝试
3. 大图片直接上传浪费网络带宽，用户体验差

#### 修复方案
在 `BaseBillEditGeneric.cs` 的 `UploadUpdatedImagesAsync` 方法中添加图片压缩逻辑：

```csharp
/// <summary>
/// 压缩图片（如果超过指定尺寸）
/// </summary>
/// <param name="imageData">原始图片数据</param>
/// <param name="maxWidth">最大宽度，默认1920</param>
/// <param name="maxHeight">最大高度，默认1080</param>
/// <param name="quality">JPEG质量，默认85</param>
/// <returns>压缩后的图片数据</returns>
private byte[] CompressImageIfNeeded(byte[] imageData, int maxWidth = 1920, int maxHeight = 1080, int quality = 85)
{
    if (imageData == null || imageData.Length == 0)
        return imageData;

    try
    {
        using (var ms = new MemoryStream(imageData))
        using (var originalImage = Image.FromStream(ms, false, false))
        {
            // 检查是否需要压缩
            if (originalImage.Width <= maxWidth && originalImage.Height <= maxHeight)
                return imageData;

            // 计算缩放比例
            var ratio = Math.Min((float)maxWidth / originalImage.Width,
                                (float)maxHeight / originalImage.Height);
            var newWidth = (int)(originalImage.Width * ratio);
            var newHeight = (int)(originalImage.Height * ratio);

            using (var resized = new Bitmap(newWidth, newHeight))
            using (var graphics = Graphics.FromImage(resized))
            {
                graphics.InterpolationMode = System.Drawing.Drawing2D.InterpolationMode.HighQualityBicubic;
                graphics.SmoothingMode = System.Drawing.Drawing2D.SmoothingMode.HighQuality;
                graphics.PixelOffsetMode = System.Drawing.Drawing2D.PixelOffsetMode.HighQuality;
                graphics.DrawImage(originalImage, 0, 0, newWidth, newHeight);

                using (var outputMs = new MemoryStream())
                {
                    var jpegEncoder = GetEncoder(ImageFormat.Jpeg);
                    if (jpegEncoder != null)
                    {
                        var encoderParams = new EncoderParameters(1);
                        encoderParams.Param[0] = new EncoderParameter(Encoder.Quality, quality);
                        resized.Save(outputMs, jpegEncoder, encoderParams);
                        return outputMs.ToArray();
                    }
                    // 如果获取编码器失败，返回原图
                    return imageData;
                }
            }
        }
    }
    catch (Exception ex)
    {
        logger.LogWarning(ex, "图片压缩失败，使用原图");
        return imageData;
    }
}

/// <summary>
/// 获取图片编码器
/// </summary>
private ImageCodecInfo GetEncoder(ImageFormat format)
{
    var codecs = ImageCodecInfo.GetImageEncoders();
    foreach (ImageCodecInfo codec in codecs)
    {
        if (codec.FormatID == format.Guid)
            return codec;
    }
    return null;
}
```

**调用方式**（在上传循环中）：
```csharp

{
```

### 问题3：UI端缺少重试机制，上传稳定性不足

#### 问题位置
- **文件**: `FileManagementService.cs`
- **行号**: 236-238
- **代码**:
```csharp
// 直接发送请求，无重试逻辑
var response = await _communicationService.SendCommandWithResponseAsync<FileUploadResponse>(
    FileCommands.FileUpload, request, ct);
```

#### 问题描述
1. 网络波动时直接失败，用户体验差
2. 缺少重试机制，成功率依赖网络稳定性

#### 修复方案
在 `FileManagementService.cs` 中添加重试逻辑：

```csharp
/// <summary>
/// 最大重试次数
/// </summary>
private const int MaxRetryCount = 3;

/// <summary>
/// 初始重试延迟（毫秒）
/// </summary>
private const int InitialRetryDelayMs = 1000;

/// <summary>
/// 带重试机制的文件上传
/// </summary>
public async Task<FileUploadResponse> UploadFileAsync(FileUploadRequest request, CancellationToken ct = default)
{
    // ... 参数验证 ...

    Exception lastException = null;
    for (int retryCount = 0; retryCount < MaxRetryCount; retryCount++)
    {
        try
        {
            // 检查连接状态
            if (!_communicationService.ConnectionManager.IsConnected)
            {
                return ResponseFactory.CreateSpecificErrorResponse<FileUploadResponse>("未连接到服务器，请检查网络连接后重试");
            }

            var response = await _communicationService.SendCommandWithResponseAsync<FileUploadResponse>(
                FileCommands.FileUpload, request, ct);

            // 检查响应
            if (response == null)
            {
                throw new Exception("服务器返回了空的响应数据");
            }

            if (!response.IsSuccess)
            {
                // 如果是业务错误（如文件格式不支持），不重试
                if (response.ErrorMessage?.Contains("格式不支持") == true)
                {
                    return response;
                }
                throw new Exception(response.ErrorMessage);
            }

            return response;
        }
        catch (OperationCanceledException)
        {
            throw;
        }
        catch (Exception ex) when (retryCount < MaxRetryCount - 1)
        {
            lastException = ex;
            var delayMs = InitialRetryDelayMs * (int)Math.Pow(2, retryCount); // 指数退避: 1s, 2s, 4s
            _log?.LogWarning("文件上传失败，{Delay}ms后重试 ({Retry}/{Max}): {Error}",
                delayMs, retryCount + 1, MaxRetryCount, ex.Message);
            
            await Task.Delay(delayMs, ct);
        }
    }

    // 所有重试都失败
    return ResponseFactory.CreateSpecificErrorResponse<FileUploadResponse>(
        $"文件上传失败，已重试{MaxRetryCount}次: {lastException?.Message}");
}
```

**同样的修复也需要应用到 `DownloadFileAsync` 和 `DeleteFileAsync` 方法。**

### 问题4：ImageCacheService (UI端) 缺少图片数据缓存

#### 问题位置
- **文件**: `ImageCacheService.cs` (UI端)
- **行号**: 全文

#### 问题描述
1. 当前只缓存图片元数据(FileStorageInfo)
2. 每次访问图片都需要从服务器下载完整数据
3. 重复查看同一图片时造成不必要的网络请求

#### 修复方案
在UI端 `ImageCacheService.cs` 中添加图片数据缓存功能：

```csharp
using Microsoft.Extensions.Caching.Memory;
using RUINORERP.Model;
using RUINORERP.PacketSpec.Models.FileManagement;
using RUINORERP.UI.Network.Services;
using System;
using System.Collections.Generic;
using System.Threading.Tasks;

namespace RUINORERP.UI.Network.Services
{
    /// <summary>
    /// 图片缓存服务（UI端）
    /// 缓存图片元数据和实际数据，减少服务器请求
    /// </summary>
    public class ImageCacheService : IDisposable
    {
        private readonly IMemoryCache _cache;
        private readonly FileManagementService _fileService;
        private bool _disposed = false;

        /// <summary>
        /// 缓存键前缀
        /// </summary>
        private const string ImageDataCachePrefix = "ImageData_";
        private const string ImageInfoCachePrefix = "ImageInfo_";

        /// <summary>
        /// 图片数据缓存过期时间（小时）
        /// </summary>
        private const int ImageDataCacheExpirationHours = 2;

        public ImageCacheService(
            IMemoryCache cache,
            FileManagementService fileService)
        {
            _cache = cache ?? throw new ArgumentNullException(nameof(cache));
            _fileService = fileService;
        }

        /// <summary>
        /// 获取图片数据（带缓存）
        /// </summary>
        /// <param name="fileId">文件ID</param>
        /// <returns>图片数据</returns>
        public async Task<byte[]> GetImageDataAsync(long fileId)
        {
            if (fileId <= 0)
                return null;

            string cacheKey = $"{ImageDataCachePrefix}{fileId}";

            // 尝试从缓存获取
            if (_cache.TryGetValue(cacheKey, out byte[] cachedData))
            {
                return cachedData;
            }

            // 缓存未命中，从服务器下载
            try
            {
                var request = new FileDownloadRequest
                {
                    FileStorageInfo = new tb_FS_FileStorageInfo { FileId = fileId }
                };

                var response = await _fileService.DownloadFileAsync(request);
                if (response?.IsSuccess == true && response.FileStorageInfos?.Count > 0)
                {
                    var fileData = response.FileStorageInfos[0].FileData;
                    
                    // 存入缓存
                    if (fileData != null)
                    {
                        var cacheOptions = new MemoryCacheEntryOptions()
                            .SetAbsoluteExpiration(TimeSpan.FromHours(ImageDataCacheExpirationHours))
                            .SetSize(fileData.Length);
                        
                        _cache.Set(cacheKey, fileData, cacheOptions);
                    }
                    
                    return fileData;
                }
            }
            catch
            {
                // 缓存失败时返回null，让调用方走正常下载流程
            }

            return null;
        }

        /// <summary>
        /// 清除指定图片的缓存
        /// </summary>
        /// <param name="fileId">文件ID</param>
        public void InvalidateImageCache(long fileId)
        {
            _cache.Remove($"{ImageDataCachePrefix}{fileId}");
            _cache.Remove($"{ImageInfoCachePrefix}{fileId}");
        }

        /// <summary>
        /// 清空所有图片缓存
        /// </summary>
        public void ClearAllCache()
        {
            // 注意：MemoryCache没有Clear方法，可以记录所有键后逐个删除
            // 或者依赖过期机制自动清理
        }

        public void Dispose()
        {
            if (_disposed)
                return;

            _disposed = true;
        }
    }
}
```

```csharp
        }

        if (_hashCalculator != null)
        {
            _hashCalculator.Dispose();
        }
        if (_imageCache != null)

        }
```

### 问题7：FileCleanupService物理删除路径错误

#### 问题位置
- **文件**: `FileCleanupService.cs`
- **行号**: 302-306
- **代码**:
```csharp
var filePath = FileStorageHelper.ResolveToAbsolutePath(fileInfo.StoragePath);
if (File.Exists(filePath))
{
    File.Delete(filePath);
}
```

#### 问题描述
1. `StoragePath` 存储的是目录路径，不是文件路径
2. 直接删除目录路径会失败或删除错误
3. 应拼接 `StorageFileName` 后再删除

#### 修复方案
```csharp
/// <summary>
/// 删除物理文件
/// </summary>
private async Task<bool> DeletePhysicalFileAsync(tb_FS_FileStorageInfo fileInfo)
{
    try
    {
        var db = _unitOfWorkManage.GetDbClient();

        // 【修复】正确拼接文件完整路径
        string filePath;
        if (!string.IsNullOrEmpty(fileInfo.StoragePath) && !string.IsNullOrEmpty(fileInfo.StorageFileName))
        {
            // StoragePath是目录，StorageFileName是文件名
            string relativePath = Path.Combine(fileInfo.StoragePath, fileInfo.StorageFileName);
            filePath = FileStorageHelper.ResolveToAbsolutePath(relativePath);
        }
        else if (!string.IsNullOrEmpty(fileInfo.StorageFileName))
        {
            // 只有文件名，尝试直接解析
            filePath = FileStorageHelper.ResolveToAbsolutePath(fileInfo.StorageFileName);
        }
        else
        {
            _logger?.LogWarning("文件路径信息不完整，无法删除: FileId={FileId}", fileInfo.FileId);
            return false;
        }

        if (File.Exists(filePath))
        {
            File.Delete(filePath);
            _logger?.LogDebug("物理文件删除成功: {FilePath}", filePath);
        }
        else
        {
            _logger?.LogWarning("物理文件不存在: {FilePath}", filePath);
        }

        // 1. 删除业务关联记录
        await db.Deleteable<tb_FS_BusinessRelation>()
            .Where(r => r.FileId == fileInfo.FileId)
            .ExecuteCommandAsync();

        // 2. 删除文件版本记录(如果有)
        await db.Deleteable<tb_FS_FileStorageVersion>()
            .Where(v => v.FileId == fileInfo.FileId)
            .ExecuteCommandAsync();

        // 3. 逻辑删除文件元数据
        fileInfo.isdeleted = true;
        fileInfo.Modified_at = DateTime.Now;
        await db.Updateable(fileInfo).ExecuteCommandAsync();
        
        return true;
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "物理删除文件失败, FileId: {FileId}", fileInfo.FileId);
        return false;
    }
}
```

### 问题10：产品图片未使用服务器端缓存

#### 问题描述
服务器端 `ImageCacheService` 缓存了图片元数据，但客户端每次仍从服务器下载完整数据

#### 修复方案
在 `FileCommandHandler.cs` 的 `DownloadByFileIdAsync` 方法中添加文件数据缓存：

```csharp
/// <summary>
/// 按文件ID下载文件（带缓存）
/// </summary>
private async Task<tb_FS_FileStorageInfo> DownloadByFileIdAsync(long fileId, CancellationToken cancellationToken)
{
    // 1. 先从元数据缓存获取
    var cachedImageInfo = _imageCacheService.GetImageInfo(fileId);
    if (cachedImageInfo != null)
    {
        _logger?.LogDebug("从缓存获取图片信息: FileId={FileId}", fileId);
        
        // 2. 如果只需要元数据，直接返回
        // 如果需要文件数据，从物理文件读取
        if (cachedImageInfo.FileData == null)
        {
            var filePath = GetPhysicalFilePath(cachedImageInfo);
            if (File.Exists(filePath))
            {
                cachedImageInfo.FileData = await File.ReadAllBytesAsync(filePath, cancellationToken);
            }
        }
        return cachedImageInfo;
    }

    // 3. 缓存未命中，从数据库查询
    var fileInfos = await _fileStorageInfoController.QueryByNavAsync(
        c => c.FileStatus == (int)FileStatus.Active && c.FileId == fileId && c.isdeleted == false);
    
    if (fileInfos != null && fileInfos.Count > 0)
    {
        var imageInfo = fileInfos[0] as tb_FS_FileStorageInfo;
        
        // 读取文件数据
        var filePath = GetPhysicalFilePath(imageInfo);
        if (File.Exists(filePath))
        {
            imageInfo.FileData = await File.ReadAllBytesAsync(filePath, cancellationToken);
        }
        
        // 添加到缓存
        _imageCacheService.AddImageInfo(imageInfo);
        _logger?.LogDebug("从数据库获取图片信息并缓存: FileId={FileId}", fileId);
        
        return imageInfo;
    }
    return null;
}

/// <summary>
/// 获取物理文件路径
/// </summary>
private string GetPhysicalFilePath(tb_FS_FileStorageInfo fileInfo)
{
    if (string.IsNullOrEmpty(fileInfo.StoragePath) || string.IsNullOrEmpty(fileInfo.StorageFileName))
        return null;
        
    string relativePath = Path.Combine(fileInfo.StoragePath, fileInfo.StorageFileName);
    return FileStorageHelper.ResolveToAbsolutePath(relativePath);
}
```
