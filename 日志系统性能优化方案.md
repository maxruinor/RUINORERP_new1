# 日志系统性能优化方案

## 一、当前性能问题分析

### 1. ThreadContext.Properties 设置开销过大 ⚠️⚠️⚠️

**问题**: 每次写日志都要从 `ApplicationContext.Current` 读取 10+ 个字段并设置到 ThreadContext

**影响**: 
- 单次调用 5-10 微秒
- 高频日志(如每秒1000条)会导致 5-10ms CPU 时间浪费
- 即使日志级别被禁用,只要 IsEnabled 通过就会执行

**现状代码**:
```csharp
private void SetContextProperties(string message, Exception exception)
{
    var appContext = ApplicationContext.Current;  // 每次都访问
    
    // 10+ 次赋值
    ThreadContext.Properties["UserName"] = appContext?.log?.UserName ?? string.Empty;
    ThreadContext.Properties["ModName"] = appContext?.log?.ModName ?? string.Empty;
    // ... 其他 8 个属性
}
```

---

### 2. 缓冲机制设计不合理 ⚠️⚠️

**问题**: 
- 上层对 Debug/Trace 缓冲 100ms
- 下层 AdoNetAppender 配置 bufferSize=0(立即写入)
- 结果: 缓冲意义不大,反而增加复杂度

**现状代码**:
```csharp
// Log4NetLogger.cs
if (logLevel == LogLevel.Debug || logLevel == LogLevel.Trace)
{
    BufferLog(logLevel, message, exception);  // 缓冲 100ms
}

// Log4net.config
<bufferSize value="0" />  // 立即写入数据库
```

---

### 3. FlushBufferCallback 重复工作 ⚠️

**问题**: 刷新缓冲时,每条日志都重新设置所有 ThreadContext 属性

**影响**: 100 条日志 → 1000+ 次属性赋值

---

### 4. 异常堆栈序列化开销 ⚠️

**问题**: GetFullExceptionStatic 可能生成大量字符串,造成 GC 压力

---

## 二、优化方案

### 方案 A: 最小改动方案(推荐) ⭐⭐⭐⭐⭐

**核心思路**: 缓存 ThreadContext 属性,只在变化时更新

#### 1. 添加属性缓存机制

```csharp
public class Log4NetLogger : ILogger
{
    // 新增: 线程局部的属性缓存
    private static readonly AsyncLocal<LogContextCache> _contextCache = new AsyncLocal<LogContextCache>();
    
    private class LogContextCache
    {
        public string UserName { get; set; }
        public string ModName { get; set; }
        public string ActionName { get; set; }
        public string Path { get; set; }
        public string IP { get; set; }
        public string MAC { get; set; }
        public string MachineName { get; set; }
        public string Operator { get; set; }
        public long LastUpdateTick { get; set; }  // 最后更新时间戳
        
        // 缓存有效期(毫秒),默认 1 秒
        public const long CacheValidityMs = 1000;
        
        public bool IsExpired => (Environment.TickCount64 - LastUpdateTick) > CacheValidityMs;
    }
    
    /// <summary>
    /// 智能设置上下文属性(带缓存)
    /// </summary>
    private void SetContextPropertiesCached(string message, Exception exception)
    {
        try
        {
            var cache = _contextCache.Value;
            
            // 如果缓存不存在或已过期,重新加载
            if (cache == null || cache.IsExpired)
            {
                cache = new LogContextCache();
                RefreshContextCache(cache);
                _contextCache.Value = cache;
            }
            
            // 从缓存读取属性(快速路径)
            ThreadContext.Properties["UserName"] = cache.UserName;
            ThreadContext.Properties["ModName"] = cache.ModName;
            ThreadContext.Properties["ActionName"] = cache.ActionName;
            ThreadContext.Properties["Path"] = cache.Path;
            ThreadContext.Properties["IP"] = cache.IP;
            ThreadContext.Properties["MAC"] = cache.MAC;
            ThreadContext.Properties["MachineName"] = cache.MachineName;
            ThreadContext.Properties["Operator"] = cache.Operator;
            
            // Message 和 Exception 每次都设置(因为它们每次都变)
            ThreadContext.Properties["Message"] = message ?? string.Empty;
            ThreadContext.Properties["Exception"] = exception != null ? GetFullException(exception) : string.Empty;
        }
        catch (Exception ex)
        {
            System.Diagnostics.Debug.WriteLine($"设置日志上下文属性失败: {ex.Message}");
        }
    }
    
    /// <summary>
    /// 刷新缓存(慢速路径,只在缓存过期时调用)
    /// </summary>
    private void RefreshContextCache(LogContextCache cache)
    {
        var appContext = ApplicationContext.Current;
        
        if (appContext?.log != null)
        {
            cache.UserName = appContext.log.UserName ?? string.Empty;
            cache.ModName = appContext.log.ModName ?? string.Empty;
            cache.ActionName = appContext.log.ActionName ?? string.Empty;
            cache.Path = appContext.log.Path ?? string.Empty;
            cache.IP = appContext.log.IP ?? string.Empty;
            cache.MAC = appContext.log.MAC ?? string.Empty;
            cache.MachineName = appContext.log.MachineName ?? Environment.MachineName;
        }
        else
        {
            cache.UserName = string.Empty;
            cache.ModName = string.Empty;
            cache.ActionName = string.Empty;
            cache.Path = string.Empty;
            cache.IP = string.Empty;
            cache.MAC = string.Empty;
            cache.MachineName = Environment.MachineName ?? string.Empty;
        }
        
        if (appContext?.CurUserInfo != null)
        {
            cache.Operator = appContext.CurUserInfo.客户端版本 ?? "已登录用户";
        }
        else
        {
            cache.Operator = "系统服务";
        }
        
        cache.LastUpdateTick = Environment.TickCount64;
    }
}
```

### 方案 B: 简化架构方案(中等改动) ⭐⭐⭐⭐

**核心思路**: 移除不必要的抽象层,直接使用 log4net

#### 1. 移除 Log4NetLogger 的缓冲机制

```csharp
// 删除所有缓冲相关代码:
// - _logBuffer
// - _flushTimer
// - BufferLog 方法
// - FlushBufferCallback 方法
// - LogEntry 结构体

// 简化 Log 方法:
public void Log<TState>(LogLevel logLevel, EventId eventId, TState state, Exception exception, Func<TState, Exception, string> formatter)
{
    if (!IsEnabled(logLevel))
        return;

    try
    {
        string message = formatter?.Invoke(state, exception) ?? string.Empty;
        SetContextPropertiesCached(message, exception);  // 使用方案A的缓存
        
        switch (logLevel)
        {
            case LogLevel.Debug:
            case LogLevel.Trace:
                _log.Debug(message, exception);
                break;
            case LogLevel.Information:
                _log.Info(message, exception);
                break;
            // ... 其他级别
        }
    }
    catch (Exception ex)
    {
        System.Diagnostics.Debug.WriteLine($"记录日志失败 [{logLevel}]: {ex.Message}");
    }
}
```

#### 2. 让 AdoNetAppender 批量写入

修改 `Log4net.config`:
```xml

```

**优势**:
- ✅ 代码更简洁,易于维护
- ✅ log4net 自身的批量写入更高效
- ✅ 减少 80% 的数据库 INSERT 调用

### 2. 异常堆栈截断

```csharp
private static string GetFullExceptionStatic(Exception exception)
{
    if (exception == null)
        return string.Empty;

    var sb = new System.Text.StringBuilder(2048);  // ✅ 预分配容量
    Exception current = exception;
    int depth = 0;
    int maxLength = 10000;  // ✅ 限制总长度
    
    while (current != null && depth < 10 && sb.Length < maxLength)
    {
        if (depth > 0)
        {
            sb.AppendLine("--- InnerException ---");
        }
        
        sb.AppendLine($"类型: {current.GetType().FullName}");
        sb.AppendLine($"消息: {current.Message}");
        
        // ✅ 截断堆栈跟踪
        string stackTrace = current.StackTrace;
        if (stackTrace != null && stackTrace.Length > 2000)
        {
            stackTrace = stackTrace.Substring(0, 2000) + "... [truncated]";
        }
        sb.AppendLine($"堆栈: {stackTrace}");
        
        current = current.InnerException;
        depth++;
    }
    
    if (sb.Length >= maxLength)
    {
        sb.Append("\n... [异常信息过长,已截断]");
    }
    
    return sb.ToString();
}
```
