### 🟡 P1 - 重要问题

#### 问题2: ConnectionManager状态更新时机仍存风险

**位置**: `ConnectionManager.cs` 第214-217行

**当前代码**:
```csharp
// 在尝试连接之前更新服务器信息
// 这样可以确保如果连接失败，重连机制会使用正确的地址
_serverAddress = serverAddress;
_serverPort = serverPort;

// 尝试连接
bool connected = await _socketClient.ConnectAsync(serverAddress, serverPort, cancellationToken);
```

**潜在风险**:
1. **场景A**: 如果连接失败,`_serverAddress` 和 `_serverPort` 已被更新为新值
2. **场景B**: 此时自动重连机制启动,会使用新地址重试
3. **场景C**: 但用户可能希望保持旧连接,或想切换到其他地址
4. **结果**: 重连行为与用户预期不符

**示例**:
```
用户操作:
1. 已连接到 192.168.1.100:5000
2. 修改为 192.168.1.200:5000
3. 点击登录 → ConnectAsync 被调用
4. _serverAddress 更新为 192.168.1.200
5. 连接失败(网络问题)
6. 自动重连启动,持续尝试连接 192.168.1.200
7. 用户想改回 192.168.1.100,但后台一直在重连 192.168.1.200
```

**修复建议**:

**方案1**: 仅在连接成功后更新状态
```csharp
// 尝试连接
bool connected = await _socketClient.ConnectAsync(serverAddress, serverPort, cancellationToken);

if (connected)
{
    // 仅在连接成功后更新服务器信息
    _serverAddress = serverAddress;
    _serverPort = serverPort;
    
    _logger?.LogDebug("成功连接到服务器 {ServerAddress}:{ServerPort}", serverAddress, serverPort);
    
    // 清除重连状态标志
    lock (_reconnectStateLock)
    {
        if (_isReconnecting)
        {
            _logger?.LogDebug("连接成功，清除IsReconnecting标志");
            _isReconnecting = false;
        }
    }
    
    OnConnectionStateChanged(true);
}
else
{
    _logger?.LogWarning("连接服务器失败 {ServerAddress}:{ServerPort}", serverAddress, serverPort);
}

return connected;
```

**方案2**: 添加"临时地址"字段,区分目标地址和已确认地址
```csharp
private string _pendingServerAddress = string.Empty;  // 待确认的地址
private int _pendingServerPort = 0;

public async Task<bool> ConnectAsync(string serverAddress, int serverPort, ...)
{
    // 设置临时地址
    _pendingServerAddress = serverAddress;
    _pendingServerPort = serverPort;
    
    bool connected = await _socketClient.ConnectAsync(serverAddress, serverPort, cancellationToken);
    
    if (connected)
    {
        // 连接成功,确认为正式地址
        _serverAddress = _pendingServerAddress;
        _serverPort = _pendingServerPort;
        _pendingServerAddress = string.Empty;
        _pendingServerPort = 0;
    }
    
    return connected;
}
```

**推荐**: 方案1更简单,符合最小修改原则

**优先级**: P1 (建议在下一版本修复)

#### 问题3: ReconnectAndWelcomeAsync 中重复调用 InitializeConnectionAndWelcomeFlowAsync

**位置**: `FrmLogin.cs` 第858行

**当前代码**:
```csharp
private async Task ReconnectAndWelcomeAsync()
{
    // ... 验证和断开连接 ...
    
    // 重置欢迎流程状态
    _welcomeCompletionTcs = new TaskCompletionSource<(bool, string)>();
    _welcomeCompleted = false;
    _welcomeAnnouncement = string.Empty;
    
    // 执行连接和欢迎流程
    await InitializeConnectionAndWelcomeFlowAsync();  // ❌ 可能导致递归调用
}
```

**问题分析**:
1. `InitializeConnectionAndWelcomeFlowAsync()` 在窗体加载时已被调用(第233行)
2. 用户在输入IP时触发 `txtServerIP_TextChanged` → `DebouncedReconnectAsync` → `ReconnectAndWelcomeAsync`
3. `ReconnectAndWelcomeAsync` 又调用 `InitializeConnectionAndWelcomeFlowAsync`
4. **潜在问题**: 
   - 如果窗体加载时的初始化还在进行中,用户修改IP会启动第二个初始化流程
   - 两个流程可能同时操作 `_welcomeCompletionTcs`,导致竞态条件

**示例场景**:
```
时间线:
T0: 窗体加载,启动 InitializeConnectionAndWelcomeFlowAsync (任务A)
T1: 任务A正在连接服务器...
T2: 用户修改IP,触发 TextChanged
T3: 1.5秒后,DebouncedReconnectAsync 执行
T4: ReconnectAndWelcomeAsync 创建新的 _welcomeCompletionTcs
T5: ReconnectAndWelcomeAsync 调用 InitializeConnectionAndWelcomeFlowAsync (任务B)
T6: 任务A和任务B同时运行,都尝试设置 _welcomeCompletionTcs
T7: 只有一个能成功,另一个的 TrySetResult 返回 false
```

**修复建议**:

**方案1**: 在 ReconnectAndWelcomeAsync 中直接调用 ConnectionManager,不重用 InitializeConnectionAndWelcomeFlowAsync
```csharp
private async Task ReconnectAndWelcomeAsync()
{
    try
    {
        MainForm.Instance?.PrintInfoLog("开始重新连接并执行欢迎流程...");

        // 验证服务器配置
        if (string.IsNullOrWhiteSpace(txtServerIP.Text) || 
            !int.TryParse(txtPort.Text, out int serverPort))
        {
            return;
        }

        string newServerIP = txtServerIP.Text.Trim();
        int newServerPort = serverPort;

        // 检查是否需要重连
        bool needsReconnect = true;
        if (connectionManager.IsConnected)
        {
            string currentServerIP = (connectionManager.CurrentServerAddress ?? "").Trim();
            int currentPort = connectionManager.CurrentServerPort;

            needsReconnect = !string.Equals(currentServerIP, newServerIP, 
                StringComparison.OrdinalIgnoreCase) || currentPort != newServerPort;
        }

        if (!needsReconnect)
        {
            MainForm.Instance?.logger?.LogDebug("当前已连接到目标服务器，无需重连");
            return;
        }

        // 断开现有连接
        if (connectionManager.IsConnected)
        {
            MainForm.Instance?.ShowStatusText("正在断开现有连接...");
            await connectionManager.DisconnectAsync();
            await Task.Delay(200);
        }

        // 更新原始服务器信息
        _originalServerIP = newServerIP;
        _originalServerPort = newServerPort.ToString();

        // 重置欢迎流程状态
        _welcomeCompletionTcs = new TaskCompletionSource<(bool, string)>();
        _welcomeCompleted = false;
        _welcomeAnnouncement = string.Empty;

        // 直接调用 ConnectionManager 连接,不重用 InitializeConnectionAndWelcomeFlowAsync
        MainForm.Instance?.ShowStatusText($"正在连接到服务器 {newServerIP}:{newServerPort}...");
        
        var connectTask = connectionManager.ConnectAsync(newServerIP, newServerPort);
        var completedTask = await Task.WhenAny(connectTask, Task.Delay(TimeSpan.FromSeconds(10)));

        bool connectResult = false;
        if (completedTask == connectTask)
        {
            connectResult = await connectTask;
        }
        else
        {
            MainForm.Instance?.logger?.LogWarning($"连接服务器 {newServerIP}:{newServerPort} 超时");
            MainForm.Instance?.ShowStatusText($"连接服务器 {newServerIP}:{newServerPort} 超时");
            return;
        }

        if (!connectResult)
        {
            MainForm.Instance?.ShowStatusText($"无法连接到服务器 {newServerIP}:{newServerPort}");
            return;
        }

        MainForm.Instance?.ShowStatusText($"服务器 {newServerIP}:{newServerPort} 连接成功");
        MainForm.Instance.PrintInfoLog("服务器连接成功,等待欢迎消息...");

        // 等待欢迎流程完成
        var welcomeTimeout = TimeSpan.FromSeconds(15);
        var welcomeTask = _welcomeCompletionTcs.Task;
        completedTask = await Task.WhenAny(welcomeTask, Task.Delay(welcomeTimeout));

        if (completedTask == welcomeTask)
        {
            var (success, announcement) = await welcomeTask;
            _welcomeCompleted = success;

            if (success && !string.IsNullOrEmpty(announcement))
            {
                if (this.InvokeRequired)
                {
                    this.Invoke(new Action(() => DisplayAnnouncement(announcement)));
                }
                else
                {
                    DisplayAnnouncement(announcement);
                }
                MainForm.Instance.ShowStatusText($"服务器连接成功 | 公告: {announcement}");
            }
            else if (success)
            {
                MainForm.Instance.ShowStatusText("服务器连接成功,欢迎消息验证通过");
            }
            else
            {
                MainForm.Instance?.logger?.LogWarning("欢迎流程验证失败");
                MainForm.Instance.ShowStatusText("服务器连接成功,但欢迎消息验证失败");
            }
        }
        else
        {
            MainForm.Instance?.logger?.LogWarning("欢迎流程验证超时,但连接已建立");
            MainForm.Instance.ShowStatusText("服务器连接成功,欢迎验证超时");
        }
    }
    catch (Exception ex)
    {
        MainForm.Instance?.logger?.LogError(ex, "重新连接和欢迎流程时发生异常");
        MainForm.Instance?.ShowStatusText($"重新连接时发生错误: {ex.Message}");
    }
}
```

**方案2**: 添加互斥锁,确保同一时间只有一个初始化流程
```csharp
private readonly SemaphoreSlim _initializationLock = new SemaphoreSlim(1, 1);

private async Task ReconnectAndWelcomeAsync()
{
    // 尝试获取锁,如果已有初始化在进行中,则等待
    if (!await _initializationLock.WaitAsync(0))
    {
        MainForm.Instance?.logger?.LogDebug("已有初始化流程在进行中,跳过本次重连");
        return;
    }

    try
    {
        // ... 原有逻辑 ...
        await InitializeConnectionAndWelcomeFlowAsync();
    }
    finally
    {
        _initializationLock.Release();
    }
}
```

**推荐**: 方案1更清晰,避免了方法复用带来的复杂性

**优先级**: P1 (建议修复)

### 🟢 P2 - 次要问题

#### 问题4: 缺少对 _debounceCancellationTokenSource 的空值保护

**位置**: `FrmLogin.cs` 多处

**当前代码**:
```csharp
// btnok_Click 第408-418行
if (_debounceCancellationTokenSource != null)
{
    try
    {
        _debounceCancellationTokenSource.Cancel();
        _debounceCancellationTokenSource.Dispose();
    }
    catch { }
    _debounceCancellationTokenSource = null;
    MainForm.Instance?.logger?.LogDebug("已取消防抖重连任务，开始登录流程");
}
```

**问题**:
- `catch { }` 捕获所有异常但不记录,可能隐藏真正的问题
- 如果 `Cancel()` 或 `Dispose()` 抛出异常,会导致 `_debounceCancellationTokenSource` 未被置为 null
- 下次调用时会再次尝试取消已释放的对象,可能抛出 `ObjectDisposedException`

**修复建议**:
```csharp
if (_debounceCancellationTokenSource != null)
{
    var cts = _debounceCancellationTokenSource;
    _debounceCancellationTokenSource = null;  // 先置为null
    
    try
    {
        if (!cts.IsCancellationRequested)
        {
            cts.Cancel();
        }
    }
    catch (ObjectDisposedException)
    {
        // 已释放,忽略
        MainForm.Instance?.logger?.LogDebug("防抖令牌已释放");
    }
    catch (Exception ex)
    {
        MainForm.Instance?.logger?.LogWarning(ex, "取消防抖任务时发生异常");
    }
    finally
    {
        try
        {
            cts.Dispose();
        }
        catch (ObjectDisposedException)
        {
            // 已释放,忽略
        }
    }
    
    MainForm.Instance?.logger?.LogDebug("已取消防抖重连任务，开始登录流程");
}
```

**优先级**: P2 (可优化)

#### 问题5: txtServerIP_TextChanged 中的异步 void 方法

**位置**: `FrmLogin.cs` 第714行

**当前代码**:
```csharp
private async void txtServerIP_TextChanged(object sender, EventArgs e)  // ❌ async void
{
    // ...
    _ = DebouncedReconnectAsync();
}
```

**问题**:
- `async void` 方法无法被调用者等待或捕获异常
- 如果 `DebouncedReconnectAsync()` 内部抛出未处理的异常,会导致应用程序崩溃
- 虽然使用了 `_ = ` 丢弃Task,但异常仍可能传播到UI线程

**修复建议**:

虽然TextChanged事件处理程序必须是 `void` 返回类型,但应添加顶层异常处理:

```csharp
private async void txtServerIP_TextChanged(object sender, EventArgs e)
{
    try
    {
        // 检测IP地址是否发生变更
        string currentIP = txtServerIP.Text.Trim();
        string currentPort = txtPort.Text.Trim();

        bool ipChanged = !string.Equals(currentIP, _originalServerIP, StringComparison.OrdinalIgnoreCase) ||
                          !string.Equals(currentPort, _originalServerPort, StringComparison.OrdinalIgnoreCase);

        _ipAddressChanged = ipChanged;

        if (ipChanged)
        {
            MainForm.Instance?.ShowStatusText("服务器地址已变更，准备重新连接...");
            MainForm.Instance.PrintInfoLog($"检测到服务器地址变更，准备重新连接: {_originalServerIP}:{_originalServerPort} -> {currentIP}:{currentPort}");

            // 使用防抖机制，避免频繁触发
            _ = DebouncedReconnectAsync();
        }
    }
    catch (Exception ex)
    {
        // 记录异常但不中断UI
        MainForm.Instance?.logger?.LogError(ex, "处理服务器地址变更时发生异常");
    }
}
```

**优先级**: P2 (建议添加)
