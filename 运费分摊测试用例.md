# 销售订单转销售出库运费分摊测试用例

## 测试目标
验证销售订单多次出库场景下的运费分摊逻辑是否正确，确保运费总和等于订单总运费。

## 测试用例

### 1. 单次出库场景
**测试数据**：
- 销售订单：总数量100个，总运费10元
- 出库数量：100个（全部出库）

**预期结果**：
- 销售出库单运费收入：10元
- 明细运费分摊：按数量比例分摊，总和为10元

### 2. 多次出库场景
**测试数据**：
- 销售订单：总数量100个，总运费10元
- 第一次出库：80个
- 第二次出库：20个

**预期结果**：
- 第一次出库单运费收入：8元（10 * 80/100）
- 第二次出库单运费收入：2元（10 * 20/100）
- 两次出库运费总和：10元

### 3. 部分出库场景
**测试数据**：
- 销售订单：总数量100个，总运费10元
- 出库数量：50个（部分出库）

**预期结果**：
- 销售出库单运费收入：5元（10 * 50/100）
- 明细运费分摊：按数量比例分摊，总和为5元

### 4. 零运费场景
**测试数据**：
- 销售订单：总数量100个，总运费0元
- 出库数量：50个

**预期结果**：
- 销售出库单运费收入：0元
- 明细运费分摊：0元

### 5. 订单总数量为零场景
**测试数据**：
- 销售订单：总数量0个，总运费10元
- 出库数量：0个

**预期结果**：
- 销售出库单运费收入：0元
- 明细运费分摊：0元

### 6. 多次出库后运费总和验证
**测试数据**：
- 销售订单：总数量100个，总运费10元
- 第一次出库：30个
- 第二次出库：40个
- 第三次出库：30个

**预期结果**：
- 第一次出库单运费收入：3元
- 第二次出库单运费收入：4元
- 第三次出库单运费收入：3元
- 三次出库运费总和：10元

## 测试步骤
1. 创建销售订单，设置总数量和总运费
2. 生成销售出库单，指定出库数量
3. 审核销售出库单
4. 重复步骤2-3，进行多次出库
5. 验证每次出库单的运费收入
6. 验证所有出库单运费总和是否等于订单总运费

## 验证方法
- 检查销售出库单主表的FreightIncome字段
- 检查销售出库单明细的AllocatedFreightIncome字段
- 计算所有出库单运费总和，与订单总运费比较

## 异常场景处理
- 订单总数量为零
- 运费为零
- 出库数量为零
- 多次出库后数量总和超过订单总数量（系统应自动调整）
