## 🔍 问题分析

### 死锁形成机制

```
时间线分析：

T0: ApprovalAsync 开启事务 (Transaction-A)
    └─ 锁定 tb_MaterialRequisition

T1: 更新库存表 tb_Inventory
    └─ 持有库存表锁

T2: BatchRecordTransactions 尝试开启新事务 (Transaction-B)
    ├─ 检测到 TranCount == 0（AsyncLocal 上下文丢失）
    └─ 调用 BeginTranAsync() → 开启新事务

T3: Transaction-B 尝试插入库存流水
    └─ 需要访问 tb_Inventory（外键/索引检查）
    └─ 等待 Transaction-A 释放锁

T4: Transaction-A 也在等待某些资源
    └─ 💀 循环等待 → 死锁

T5: 60秒后超时，强制回滚
```

### 根本原因

1. **嵌套事务**：`BatchRecordTransactions` 在已有事务的情况下又开启新事务
2. **AsyncLocal 上下文丢失**：在某些异步场景下，`TranCount` 判断不准确
3. **同步查询阻塞**：`.Single()` 同步查询可能导致线程阻塞，延长事务持有时间
4. **循环内数据库查询**：N+1 查询问题导致长时间持锁

## 🎯 最佳实践总结

### 1. 事务设计原则

```csharp
// ✅ 推荐的事务模式
public async Task<Result> BusinessOperation()
{
    // 1. 事务外：预加载数据、验证参数
    var data = await PreloadData();
    
    // 2. 开启事务（Controller 层）
    await unitOfWork.BeginTranAsync();
    
    try
    {
        // 3. 事务内：执行业务逻辑（Service 层）
        var result = await ProcessBusiness(data);
        
        // 4. 提交事务
        await unitOfWork.CommitTranAsync();
        return Success(result);
    }
    catch
    {
        // 5. 异常回滚
        await unitOfWork.RollbackTranAsync();
        throw;
    }
}
```

### 2. 避免嵌套事务

```csharp
// ✅ 子方法不应该管理事务
public async Task<bool> BatchRecordTransactions(...)
{
    // 检查是否在事务中
    bool inTransaction = _unitOfWorkManage.TranCount > 0;
    
    if (!inTransaction)
    {
        // ⚠️ 只记录警告，不开启新事务
        _logger.LogWarning("当前不在事务中");
    }
    
    // 直接执行数据库操作
    return await dbClient.Insertable(...).ExecuteCommandAsync();
}
```

### 3. 异步查询规范

```csharp
// ❌ 错误：同步查询
var entity = db.Queryable<T>().Where(...).Single();

// ✅ 正确：异步查询
var entity = await db.Queryable<T>().Where(...).SingleAsync();
```

### 4. 提前返回必须回滚

```csharp
// ✅ 正确：提前返回时回滚事务
if (!IsValid(data))
{
    await _unitOfWorkManage.RollbackTranAsync();
    return Error("数据无效");
}
```

## 📝 经验教训

### 1. 事务管理的黄金法则

1. **单一事务原则**：一个业务操作只应该有一个事务边界
2. **事务传递**：子方法应该使用调用方的事务，而不是创建新事务
3. **最短持锁原则**：尽快完成事务，减少锁持有时间
4. **固定访问顺序**：所有事务按相同顺序访问资源

### 2. 异步编程注意事项

1. **全程异步**：从 Controller 到 Repository 都应该使用异步方法
2. **避免 sync-over-async**：不要在异步方法中调用同步方法
3. **ConfigureAwait**：在库代码中使用 `ConfigureAwait(false)`

### 3. 死锁预防策略

1. **预加载数据**：在事务外预加载所有需要的数据
2. **批量操作**：减少数据库往返次数
3. **避免循环查询**：使用字典缓存，避免 N+1 查询
4. **合理隔离级别**：根据业务选择适当的隔离级别
