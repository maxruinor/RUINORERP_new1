# 领料单审核事务上下文丢失问题修复

## 问题描述

执行领料单审核时出现以下日志错误：

```
[Transaction-xxx] 检测到残留事务对象，先回滚
[CommitTranAsync] 提交请求但无事务上下文
[CommitTranAsync] 提交请求但无事务上下文
```

**影响**: 领料单审核失败，事务无法正常提交。

## 根本原因分析

### 问题代码结构（修复前）

```csharp
public async override Task<ReturnResults<T>> ApprovalAsync(T ObjectEntity)
{
    try
    {
        // ❌ 问题1: 在事务外执行数据库查询
        entity.tb_manufacturingorder = _unitOfWorkManage.GetDbClient()
            .Queryable<tb_ManufacturingOrder>()
            .Where(c => c.MOID == entity.MOID).Single();  // 同步查询
        
        // ❌ 问题2: 又在事务外执行第二次查询（死锁优化预加载）
        var inventoryList = await _unitOfWorkManage.GetDbClient()
            .Queryable<tb_Inventory>()
            .Where(...)
            .ToListAsync();
        
        // ❌ 问题3: 此时才开启事务
        await _unitOfWorkManage.BeginTranAsync();
        
        // ... 业务逻辑 ...
        
        await _unitOfWorkManage.CommitTranAsync();
    }
    catch (Exception ex)
    {
        await _unitOfWorkManage.RollbackTranAsync();
    }
}
```

### 问题分析

1. **第一次查询（第 134-139 行）**：
   - 使用 `.Single()` 同步方法
   - 在事务外执行
   - SqlSugar 可能为此查询创建临时事务上下文

2. **第二次查询（第 293-296 行）**：
   - 使用 `.ToListAsync()` 异步方法
   - 仍在事务外执行
   - 再次创建或复用事务上下文

3. **第三次开启事务（第 302 行）**：
   - 调用 `BeginTranAsync()` 时检测到已有残留事务
   - UnitOfWorkManage 自动回滚残留事务
   - 导致事务上下文被重置

4. **最终提交时（第 459 行）**：
   - 调用 `CommitTranAsync()` 时发现无事务上下文
   - 因为之前已被重置

## 修复方案

### 核心原则
**所有数据库操作必须在同一个事务上下文中执行**

### 修复后的代码结构

```csharp
public async override Task<ReturnResults<T>> ApprovalAsync(T ObjectEntity)
{
    try
    {
        if (entity.ApprovalResults.Value)
        {
            // ✅ 修复1: 先开启事务
            await _unitOfWorkManage.BeginTranAsync();
            
            // ✅ 修复2: 在事务内执行所有数据库查询
            entity.tb_manufacturingorder = await _unitOfWorkManage.GetDbClient()
                .Queryable<tb_ManufacturingOrder>()
                .Includes(...)
                .Where(c => c.MOID == entity.MOID)
                .SingleAsync();  // 使用异步方法
            
            // ✅ 修复3: 预加载也在事务内执行
            var inventoryList = await _unitOfWorkManage.GetDbClient()
                .Queryable<tb_Inventory>()
                .Where(...)
                .ToListAsync();
            
            // ... 业务逻辑 ...
            
            // ✅ 修复4: 提前返回时必须回滚事务
            if (validationFailed)
            {
                await _unitOfWorkManage.RollbackTranAsync();
                return rrs;
            }
            
            // ... 更多业务逻辑 ...
        }
        
        // ✅ 修复5: 统一在方法末尾提交事务
        await _unitOfWorkManage.CommitTranAsync();
        return rrs;
    }
    catch (Exception ex)
    {
        // ✅ 修复6: 异常时回滚事务
        await _unitOfWorkManage.RollbackTranAsync();
        throw;
    }
}
```

## 经验教训

### 1. 事务开启时机
**原则**: 在执行任何数据库操作之前开启事务

```csharp
// ❌ 错误
var data = db.Queryable<T>().First();  // 事务外查询
await unitOfWork.BeginTranAsync();     // 后开启事务

// ✅ 正确
await unitOfWork.BeginTranAsync();     // 先开启事务
var data = await db.Queryable<T>().FirstAsync();  // 事务内查询
```

### 2. 异步方法一致性
**原则**: 事务内的所有数据库操作都应使用异步方法

```csharp
// ❌ 错误
entity = db.Queryable<T>().Single();           // 同步
var list = await db.Queryable<T>().ToListAsync(); // 异步

// ✅ 正确
entity = await db.Queryable<T>().SingleAsync();     // 异步
var list = await db.Queryable<T>().ToListAsync();   // 异步
```

### 3. 提前返回处理
**原则**: 任何提前返回的路径都必须先回滚事务

```csharp
// ❌ 错误
if (validationFailed)
{
    return errorResult;  // 事务泄漏
}

// ✅ 正确
if (validationFailed)
{
    await unitOfWork.RollbackTranAsync();
    return errorResult;
}
```

### 4. 嵌套事务避免
**原则**: 避免在同一方法中多次调用 `BeginTranAsync()`

```csharp
// ❌ 错误
await unitOfWork.BeginTranAsync();
// ... 业务逻辑 ...
await unitOfWork.BeginTranAsync();  // 嵌套事务

// ✅ 正确
await unitOfWork.BeginTranAsync();
// ... 所有业务逻辑 ...
await unitOfWork.CommitTranAsync();
```
